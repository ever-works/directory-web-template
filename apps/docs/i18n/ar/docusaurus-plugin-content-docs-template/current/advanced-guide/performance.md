# تحسين الأداء

يغطي هذا الدليل تحسينات الأداء المضمنة في قالب Ever Works وتقنيات الحفاظ على أوقات التحميل السريعة مع نمو تطبيقك.

## تكوين Next.js

يتضمن القالب `next.config.ts` العديد من الإعدادات التي تركز على الأداء:

### الإخراج المستقل

```typescript
const nextConfig: NextConfig = {
  output: "standalone",
  // ...
};
```

يقوم وضع الإخراج `standalone` بإنشاء بنية قائمة بذاتها تتضمن فقط الملفات المطلوبة لتشغيل التطبيق. وهذا يقلل من حجم الحاوية ووقت بدء الإنتاج.

### تحسين استيراد الحزمة

```typescript
experimental: {
  optimizePackageImports: ["@heroui/react", "lucide-react"],
},
```

يتيح هذا الإعداد اهتزاز الشجرة للحزم ذات الملفات الثقيلة. بدلاً من استيراد المكتبة 0 أو 1 بأكملها، يتم تضمين المكونات المستخدمة فعليًا فقط في الحزمة.

### تحسين مشاهدة Webpack

```typescript
if (dev) {
  config.watchOptions = {
    ...config.watchOptions,
    ignored: ['**/node_modules/**', '**/.git/**', '**/.content/**']
  };
}
```

يتم استبعاد الدليل 0 (CMS المستند إلى Git مع أكثر من 220 ملف تخفيض السعر) من مراقب ملفات webpack قيد التطوير. وهذا يمنع عمليات إعادة البناء غير الضرورية عند تغيير ملفات المحتوى ويقلل بشكل كبير من استخدام وحدة المعالجة المركزية أثناء التطوير.

### التحذيرات المكبوتة

يتم منع تسجيل البنية التحتية المطول في بيئات CI وVercel:

```typescript
if (process.env.CI || process.env.VERCEL) {
  config.infrastructureLogging = { level: 'error' };
}
```

## تحسين الصورة

### الأنماط عن بعد

يقوم القالب بإنشاء أنماط الصور البعيدة المسموح بها ديناميكيًا باستخدام `generateImageRemotePatterns()` . وهذا يضمن تحسين الصور من شبكات CDN التي تم تكوينها والمصادر الخارجية من خلال مسار الصور المدمج في Next.js.

### التعامل مع SVG

```typescript
images: {
  dangerouslyAllowSVG: true,
  contentDispositionType: 'attachment',
  contentSecurityPolicy: "default-src 'self'; script-src 'none'; sandbox;",
  unoptimized: false,
},
```

يُسمح بصور SVG ولكن يتم وضع الحماية لها باستخدام سياسة أمان المحتوى الصارمة التي تعمل على تعطيل تنفيذ البرنامج النصي. يسمح هذا بشعارات وأيقونات SVG مع منع XSS من خلال حقن SVG.

### أفضل الممارسات للصور

| تقنية | التنفيذ | التأثير |
|---|---|---|
| استخدم 0  | مكون مدمج مع التحميل البطيء | WebP/AVIF تلقائي، أحجام مستجيبة |
| تعيين أبعاد صريحة | 1- و 2- الدعائم | يمنع التحول التراكمي للتخطيط (CLS) |
| استخدم `priority` لـ LCP | `<Image priority />` لصور البطل | التحميل المسبق لأكبر صورة رسم محتوى |
| استخدم الدعامة 5 | 6ـ | يمنع تنزيل الصور كبيرة الحجم |
| طمس العناصر النائبة | `placeholder="blur"` مع `blurDataURL` | يحسن سرعة التحميل المتصورة |

## استراتيجيات التخزين المؤقت

### رؤوس HTTP

يقوم القالب بتعيين الرؤوس المرتبطة بذاكرة التخزين المؤقت في `next.config.ts` :

```typescript
headers: [
  { key: "X-DNS-Prefetch-Control", value: "on" },
]
```

يتم تمكين الجلب المسبق لـ DNS عالميًا لتقليل زمن وصول بحث DNS للموارد الخارجية.

### الجيل الثابت

يستخدم القالب مهلة سخية لإنشاء صفحة ثابتة:

```typescript
staticPageGenerationTimeout: 180, // 3 minutes
```

يستوعب هذا الصفحات التي تجلب البيانات من واجهات برمجة التطبيقات الخارجية أو نظام إدارة المحتوى المستند إلى Git أثناء وقت الإنشاء.

### تكوين علامة ETag

```typescript
generateEtags: false,
```

يتم تعطيل علامات ETag على مستوى Next.js لأن وكيل CDN/العكسي (Vercel Edge Network أو Cloudflare) يتعامل مع التحقق من صحة ذاكرة التخزين المؤقت بشكل أكثر كفاءة.

### التخزين المؤقت على مستوى التطبيق

يقوم معالج الخلفية التحليلية بتسخين ذاكرة التخزين المؤقت مسبقًا على فترات منتظمة:

| نوع ذاكرة التخزين المؤقت | الفاصل الزمني للتحديث | البيانات |
|---|---|---|
| اتجاهات نمو المستخدم | 10 دقائق | نمو المستخدم الشهري لمدة 6، 12، 24 شهرًا |
| اتجاهات النشاط | 5 دقائق | بيانات النشاط لمدة 7، 14، 30 يومًا |
| أعلى العناصر الترتيب | 15 دقيقة | أعلى 10، 20، 50 عنصرًا |
| الأنشطة الأخيرة | 2 دقيقة | أحدث 10 و 20 إدخالات للنشاط |
| مقاييس الأداء | 30 ثانية | إحصائيات أداء الاستعلام |
| تنظيف ذاكرة التخزين المؤقت | 1 ساعة | انتهت صلاحية إزالة إدخال ذاكرة التخزين المؤقت |

## تحميل كسول

### التحميل البطيء على مستوى المكونات

استخدم `next/dynamic` للمكونات الثقيلة غير المطلوبة في العرض الأولي:

```typescript
import dynamic from 'next/dynamic';

const HeavyChart = dynamic(() => import('@/components/charts/HeavyChart'), {
  loading: () => <ChartSkeleton />,
  ssr: false, // disable SSR for client-only components
});
```

### تقسيم التعليمات البرمجية على مستوى الطريق

يقوم جهاز توجيه التطبيقات Next.js تلقائيًا بتقسيم التعليمات البرمجية حسب المسار. تحصل كل صفحة في `app/[locale]/` على حزمتها الخاصة، بحيث يقوم المستخدمون بتنزيل JavaScript المطلوب للصفحة الحالية فقط.

### الواردات الديناميكية في وظائف الخلفية

يستخدم القالب عمليات الاستيراد الديناميكية داخل عمليات رد الاتصال الخاصة بالمهمة لمنع حزمة الويب من سحب وحدات الخادم فقط إلى حزمة العميل:

```typescript
manager.scheduleJob('repository-sync', 'Repository Synchronization', async () => {
  const { syncManager } = await import('@/lib/services/sync-service');
  await syncManager.performSync();
}, 5 * 60 * 1000);
```

## تحسين حجم الحزمة

### تحليل الحزمة

قم بتشغيل ما يلي لفحص تكوين الحزمة:

```bash
ANALYZE=true pnpm build
```

إذا تم تكوين `@next/bundle-analyzer` ، فسينتج عن ذلك مخطط هيكلي تفاعلي يوضح الوحدات النمطية التي تساهم في حجم الحزمة.

### تقنيات التحسين الشائعة

| تقنية | مثال | التوفير |
|---|---|---|
| تحسين ملف البرميل | `optimizePackageImports` في التكوين | يمنع استيراد مكتبات الأيقونات/واجهة المستخدم بأكملها |
| وحدات الخادم فقط | `import 'server-only'` في ملفات lib | يمنع تجميع العميل العرضي |
| الواردات الديناميكية | `await import('@/lib/services/...')` | يؤجل التحميل لحين الحاجة |
| الباقات الخارجية | 4ـ | يُستثنى من تجميع حزمة الويب |

التكوين 5 مهم بشكل خاص:

```typescript
serverExternalPackages: ['postgres', 'bcryptjs', 'drizzle-orm'],
```

يتم استبعاد هذه الحزم من تجميع حزم الويب وتحميلها محليًا في وقت التشغيل، مما يقلل من وقت الإنشاء ويتجنب مشكلات التوافق مع الوحدات النمطية الأصلية.

## نصائح لتحسين المنارة

### الأهداف الأساسية لمؤشرات الويب الحيوية

| متري | الهدف | العوامل الرئيسية |
|---|---|---|
| **LCP** (أكبر طلاء محتوى) | <2.5 ثانية | تحسين الصورة، أولوية التحميل، وقت استجابة الخادم |
| **FID** (تأخير الإدخال الأول) | <100 مللي ثانية | تقسيم الكود، الحد الأدنى من حظر الخيط الرئيسي |
| **CLS** (إزاحة التخطيط التراكمي) | < 0.1 | أبعاد الصورة الصريحة، استراتيجية تحميل الخط |
| **TTFB** (الوقت حتى البايت الأول) | <800 مللي ثانية | التخزين المؤقت لـ CDN، ووظائف الحافة، وتحسين استعلام قاعدة البيانات |

### قائمة مرجعية عملية

1. **الصور**: استخدم 0 مع الدعائم الصريحة 1 و2 و3. قم بتمييز الصور الموجودة في الجزء العلوي بـ 4.
2. **الخطوط**: استخدم `next/font` لاستضافة الخطوط ذاتيًا مع `display: swap` وتحميل ملفات الخطوط المهمة مسبقًا.
3. **JavaScript**: قم بمراجعة وإضافة أي مكتبات كبيرة تستخدم الملفات البرميلية.
4. **CSS**: يستخدم القالب Tailwind CSS، والذي تم إزالته بالفعل في إصدارات الإنتاج. تجنب استيراد وحدات CSS غير المستخدمة.
5. **البرامج النصية لجهة خارجية**: قم بتأجيل البرامج النصية غير الهامة باستخدام `next/script` مع 9.
6. **مكونات الخادم**: الإعداد الافتراضي لمكونات خادم React (RSC) واستخدام `"use client"` فقط عندما يكون التفاعل مطلوبًا.

### تشغيل المنارة

يتضمن القالب تكوين 11. تشغيل اختبارات Lighthouse الآلية:

```bash
npx lhci autorun --config=lighthouse-test.json
```

أو استخدم لوحة Chrome DevTools Lighthouse لإجراء عمليات التدقيق اليدوية.

## أداء استعلام قاعدة البيانات

### تجمع الاتصال

استخدم تجمع الاتصالات لتجنب فتح اتصال قاعدة بيانات جديد لكل طلب. راجع [دليل القياس](/deployment/scaling) للحصول على تفاصيل التكوين.

### تحسين الاستعلام

- استخدم نمط المستودع ( `lib/repositories/` ) لمركزية الاستعلامات وتحسينها.
- يتضمن مستودع التحليلات طبقات ذاكرة تخزين مؤقت مدمجة مع TTL قابلة للتكوين.
- مراقبة الاستعلامات البطيئة عبر وظيفة خلفية مقاييس الأداء.

### استراتيجية الفهرسة

قم بمراجعة `lib/db/schema.ts` للفهارس الموجودة. إضافة فهارس لـ:
- الأعمدة المستخدمة في الجمل 2
- أعمدة المفاتيح الخارجية
- الأعمدة المستخدمة في عبارات 3
- فهارس مركبة لعمليات البحث متعددة الأعمدة

## مراقبة الأداء

### التكامل الحراسة

يدمج القالب Sentry لمراقبة الأداء في 4:

```typescript
Sentry.init({
  tracesSampleRate: process.env.NODE_ENV === 'production' ? 0.1 : 1.0,
});
```

يتم أخذ عينات من الآثار بنسبة 10% في الإنتاج و100% في التطوير. اضبط `tracesSampleRate` بناءً على حجم حركة المرور وحدود خطة Sentry.

### علامات الأداء المخصصة

استخدم واجهة برمجة تطبيقات أداء الويب للتوقيت المخصص:

```typescript
performance.mark('data-fetch-start');
const data = await fetchData();
performance.mark('data-fetch-end');
performance.measure('data-fetch', 'data-fetch-start', 'data-fetch-end');
```

## ملخص

| المنطقة | التحسين المدمج | خطوات إضافية |
|---|---|---|
| صور | وضع الحماية التلقائي لـ WebP/AVIF وSVG | أضف `priority` إلى صور LCP، استخدم `sizes` |
| جافا سكريبت | تحسين الحزمة وتقسيم التعليمات البرمجية | إضافة مكتبات إلى `optimizePackageImports` |
| التخزين المؤقت | تسخين ذاكرة التخزين المؤقت في الخلفية، الجلب المسبق لـ DNS | تكوين قواعد ذاكرة التخزين المؤقت لـ CDN |
| قاعدة بيانات | تجمع الاتصال، نمط المستودع | إضافة فهارس ومراقبة الاستعلامات البطيئة |
| بناء | مخرجات مستقلة، حزم خارجية | تمكين محلل الحزم |
| الرصد | آثار الحراسة، وظيفة مقاييس الأداء | إعداد تنبيهات للمقاييس المتدهورة |
