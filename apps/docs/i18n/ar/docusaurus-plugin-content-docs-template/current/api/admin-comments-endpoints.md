# نقاط نهاية واجهة برمجة تطبيقات تعليقات المشرف

توفر واجهة برمجة تطبيقات Admin Comments إمكانات الإشراف لإدارة تعليقات المستخدمين. يمكن للمسؤولين إدراج التعليقات وعرضها وتحديثها وحذفها بشكل مبدئي. تستخدم جميع نقاط النهاية وقت تشغيل Node.js وتتطلب توفر قاعدة البيانات. تستخدم عمليات التحقق من المصادقة `403 Forbidden` للمستخدمين غير الإداريين.

## ملخص الطريق

|الطريقة|المسار|مصادقة|الوصف|
|--------|------|------|-------------|
|`GET`|`/api/admin/comments`|المشرف|قائمة التعليقات (مرقّمة، قابلة للبحث)|
|`GET`|`/api/admin/comments/{id}`|المشرف|احصل على تعليق واحد مع معلومات المستخدم|
|`PUT`|`/api/admin/comments/{id}`|المشرف|تحديث محتوى التعليق|
|`DELETE`|`/api/admin/comments/{id}`|المشرف|حذف تعليق ناعم|

## المصادقة

تتحقق نقاط نهاية الإشراف على التعليق من حالة المسؤول وترجع `403 Forbidden` (وليس `401`) للمستخدمين غير الإداريين:

```typescript
const session = await auth();
if (!session?.user?.isAdmin) {
  return NextResponse.json(
    { success: false, error: "Forbidden" },
    { status: 403 }
  );
}
```

## متطلبات قاعدة البيانات

تتحقق نقاط نهاية التعليق من توفر قاعدة البيانات قبل معالجة الطلبات:

```typescript
const dbCheck = checkDatabaseAvailability();
if (dbCheck) return dbCheck;
```

إذا لم يتم تكوين قاعدة البيانات، فسيتم إرجاع استجابة مناسبة للخطأ قبل إجراء أي فحص للمصادقة.

## نقاط النهاية

### احصل على `/api/admin/comments`

إرجاع قائمة مقسمة إلى صفحات من التعليقات مع معلومات المستخدم المرتبطة بها. يدعم البحث عن النص الكامل عبر محتوى التعليقات وأسماء المستخدمين ورسائل البريد الإلكتروني للمستخدم. يتم إرجاع التعليقات غير المحذوفة فقط.

**معلمات الاستعلام:**

|المعلمة|اكتب|الافتراضي|الوصف|
|-----------|------|---------|-------------|
|`page`|عدد صحيح| `1` |رقم الصفحة لترقيم الصفحات|
|`limit`|عدد صحيح| `10` |التعليقات لكل صفحة (1--100)|
|`search`|سلسلة| `""` |البحث في المحتوى أو اسم المستخدم أو البريد الإلكتروني|

**سلوك البحث:**

تتم مطابقة استعلام البحث بشكل غير حساس لحالة الأحرف (باستخدام `ILIKE`) مقابل:
- محتوى التعليق
- اسم عرض المستخدم
- عنوان البريد الإلكتروني للمستخدم

يتم الهروب من الأحرف الخاصة `%` و`_` و`\` لمنع حقن نمط SQL.

**الرد (200):**

```json
{
  "success": true,
  "data": {
    "comments": [
      {
        "id": "comment_123abc",
        "content": "This is a great product! Highly recommended.",
        "rating": 5,
        "userId": "user_456def",
        "itemId": "item_789ghi",
        "createdAt": "2024-01-20T10:30:00.000Z",
        "updatedAt": "2024-01-20T10:30:00.000Z",
        "user": {
          "id": "user_456def",
          "name": "John Doe",
          "email": "john.doe@example.com",
          "image": "https://example.com/avatar.jpg"
        }
      }
    ],
    "pagination": {
      "total": 156,
      "page": 1,
      "limit": 10,
      "totalPages": 16
    }
  }
}
```

### احصل على `/api/admin/comments/{id}`

يسترد تعليقًا محددًا بواسطة المعرف الخاص به مع معلومات ملف تعريف المستخدم الكاملة. يتضمن صلة يسرى بجدول `clientProfiles` لبيانات المستخدم.

** معلمات المسار: **

|المعلمة|اكتب|الوصف|
|-----------|------|-------------|
|`id`|سلسلة|تعليق معرف فريد|

**الرد (200):**

```json
{
  "success": true,
  "data": {
    "id": "comment_123abc",
    "content": "This is a great product! Highly recommended.",
    "rating": 5,
    "userId": "user_456def",
    "itemId": "item_789ghi",
    "createdAt": "2024-01-20T10:30:00.000Z",
    "updatedAt": "2024-01-20T14:45:00.000Z",
    "user": {
      "id": "user_456def",
      "name": "John Doe",
      "email": "john.doe@example.com",
      "image": "https://example.com/avatar.jpg"
    }
  }
}
```

**الاحتياطي للمستخدم:** إذا لم يتم العثور على ملف تعريف المستخدم (مستخدم محذوف)، فسيتم إرجاع كائن نائب:

```json
{
  "user": {
    "id": "",
    "name": "Unknown User",
    "email": "",
    "image": null
  }
}
```

### ضع `/api/admin/comments/{id}`

يقوم بتحديث محتوى تعليق معين. يمكن تعديل الحقل `content` فقط. يجب أن يكون التعليق موجودًا وألا يتم حذفه بشكل بسيط.

** معلمات المسار: **

|المعلمة|اكتب|الوصف|
|-----------|------|-------------|
|`id`|سلسلة|تعليق معرف فريد|

**نص الطلب:**

```json
{
  "content": "This is an updated comment with more details."
}
```

|الميدان|اكتب|مطلوب|الوصف|
|-------|------|----------|-------------|
|`content`|سلسلة|نعم|نص التعليق الجديد (يجب ألا يكون فارغًا بعد الاقتطاع)|

**قواعد التحقق:**
- `content` مطلوب ويجب ألا يكون فارغًا أو به مسافة بيضاء فقط
- يجب أن يكون التعليق الهدف موجودًا ويجب ألا يحتوي على طابع زمني `deletedAt`

**الرد (200):**

```json
{
  "success": true,
  "data": {
    "id": "comment_123abc",
    "content": "This is an updated comment with more details.",
    "rating": 5,
    "userId": "user_456def",
    "itemId": "item_789ghi",
    "createdAt": "2024-01-20T10:30:00.000Z",
    "updatedAt": "2024-01-20T16:15:00.000Z",
    "user": { "id": "user_456def", "name": "John Doe", "email": "john.doe@example.com", "image": null }
  },
  "message": "Comment updated successfully"
}
```

### احذف `/api/admin/comments/{id}`

إجراء حذف بسيط للتعليق عن طريق تعيين الطابع الزمني `deletedAt`. يجب أن يكون التعليق موجودًا ولم يتم حذفه بالفعل. يتم استبعاد التعليقات المحذوفة بشكل أولي من جميع استعلامات القائمة.

** معلمات المسار: **

|المعلمة|اكتب|الوصف|
|-----------|------|-------------|
|`id`|سلسلة|تعليق معرف فريد|

**الرد (200):**

```json
{
  "success": true,
  "message": "Comment deleted successfully"
}
```

## نموذج بيانات التعليق

|الميدان|اكتب|لاغية|الوصف|
|-------|------|----------|-------------|
|`id`|سلسلة|لا|معرف التعليق الفريد|
|`content`|سلسلة|لا|محتوى نص التعليق|
|`rating`|عدد صحيح|نعم|قيمة التصنيف (1-5)|
|`userId`|سلسلة|لا|معرف مستخدم المؤلف|
|`itemId`|سلسلة|لا|معرف العنصر المرتبط|
|`createdAt`|التاريخ والوقت|نعم|الطابع الزمني للإنشاء|
|`updatedAt`|التاريخ والوقت|نعم|الطابع الزمني للتحديث الأخير|
|`deletedAt`|التاريخ والوقت|نعم|الطابع الزمني للحذف الناعم (خالي إذا كان نشطًا)|

## رموز الخطأ

|الحالة|خطأ|السبب|
|--------|-------|-------|
| `400` |المحتوى مطلوب|محتوى فارغ أو مفقود عند التحديث|
| `403` |ممنوع|مستخدم غير إداري يحاول الوصول|
| `404` |لم يتم العثور على التعليق|معرف غير صالح أو تم حذفه بالفعل|
| `500` |خطأ داخلي في الخادم|فشل قاعدة البيانات أو الخادم|

## ملاحظات التنفيذ

- تستخدم التعليقات **الحذف البسيط** - يتم تعيين الحقل `deletedAt` بدلاً من إزالة الصف. وهذا يحافظ على سلامة البيانات ويسمح بالاسترداد المحتمل.
- تتم تصفية جميع استعلامات القائمة باستخدام `isNull(comments.deletedAt)` لاستبعاد التعليقات المحذوفة.
- يتم جلب بيانات المستخدم عبر `LEFT JOIN` على `clientProfiles`، مما يضمن إمكانية استرجاع التعليقات من المستخدمين المحذوفين.
- تم تعيين `runtime` على `"nodejs"` لهذه المسارات (وليس Edge).
