# نقاط نهاية واجهة برمجة تطبيقات إشعارات المشرف

تدير Admin Notifications API الإشعارات داخل التطبيق للمستخدمين الإداريين. وهو يدعم إدراج الإشعارات ذات الأعداد غير المقروءة، وإنشاء إشعارات جديدة لمستخدمين محددين، ووضع علامة على الإشعارات كمقروءة (بشكل فردي أو مجمع). يتم تخزين الإشعارات في قاعدة البيانات ويتم تحديد نطاقها للمستخدمين الفرديين.

## ملخص الطريق

|الطريقة|المسار|مصادقة|الوصف|
|--------|------|------|-------------|
|`GET`|`/api/admin/notifications`|المشرف|قائمة الإخطارات للمسؤول الحالي|
|`POST`|`/api/admin/notifications`|مصادقة|إنشاء إشعار جديد|
|`PATCH`|`/api/admin/notifications/{id}/read`|مصادقة|وضع علامة على إشعار واحد كمقروء|
|`PATCH`|`/api/admin/notifications/mark-all-read`|مصادقة|وضع علامة على جميع الإخطارات كمقروءة|

## المصادقة

تستخدم نقاط نهاية الإشعارات مستويين من المصادقة:

**المسؤول فقط (قائمة GET):** يتطلب المصادقة ودور المسؤول.

```typescript
const session = await auth();
if (!session?.user?.id) {
  return NextResponse.json({ success: false, error: 'Unauthorized' }, { status: 401 });
}
if (!session.user.isAdmin) {
  return NextResponse.json({ success: false, error: 'Forbidden' }, { status: 403 });
}
```

**مستخدم تمت مصادقته (POST, PATCH):** يتطلب جلسة عمل صالحة ولكنه لا يتطلب دور المسؤول. يتم تحديد نطاق نقاط نهاية وضع علامة كمقروءة على إشعارات المستخدم الذي تمت مصادقته.

## نقاط النهاية

### احصل على `/api/admin/notifications`

يسترد أحدث 50 إشعارًا للمستخدم الإداري المعتمد، مرتبة حسب تاريخ الإنشاء (الأحدث أولاً). يُرجع أيضًا العدد الإجمالي للإشعارات غير المقروءة.

**الرد (200):**

```json
{
  "success": true,
  "data": {
    "notifications": [
      {
        "id": "notif_123abc",
        "userId": "user_456def",
        "type": "item_approved",
        "title": "Item Approved",
        "message": "Your item 'Awesome Tool' has been approved and is now live.",
        "data": "{\"itemId\": \"item_789ghi\", \"itemName\": \"Awesome Tool\"}",
        "isRead": false,
        "readAt": null,
        "createdAt": "2024-01-20T10:30:00.000Z",
        "updatedAt": "2024-01-20T10:30:00.000Z"
      }
    ],
    "unreadCount": 3
  }
}
```

** تفاصيل السلوك: **
- يتم إرجاع 50 إشعارًا كحد أقصى لكل طلب
- يتم ترتيب النتائج حسب `createdAt` تنازليًا (الأحدث أولاً)
- `unreadCount` يتم حسابه بشكل منفصل عن طريق حساب الإشعارات حيث `isRead = false`
- يتم تحديد نطاق الإخطارات لمعرف المستخدم المصادق عليه

### نشر `/api/admin/notifications`

إنشاء إشعار جديد لمستخدم محدد. يقبل الحقل `data` كائنًا سيتم تقييده بواسطة JSON قبل التخزين. لا تتطلب نقطة النهاية هذه امتيازات المسؤول - يمكن لأي مستخدم تمت مصادقته إنشاء إشعارات (عادةً ما يستدعيها النظام داخليًا).

**نص الطلب:**

```json
{
  "type": "item_approved",
  "title": "Item Approved",
  "message": "Your item 'Awesome Tool' has been approved and is now live.",
  "userId": "user_456def",
  "data": {
    "itemId": "item_789ghi",
    "itemName": "Awesome Tool",
    "action": "approved"
  }
}
```

|الميدان|اكتب|مطلوب|الوصف|
|-------|------|----------|-------------|
|`type`|سلسلة|نعم|معرف نوع الإشعار (على سبيل المثال، `"item_approved"`، `"comment_received"`)|
|`title`|سلسلة|نعم|عنوان الإخطار القصير|
|`message`|سلسلة|نعم|رسالة إعلام كاملة|
|`userId`|سلسلة|نعم|استهدف معرف المستخدم لتلقي الإشعار|
|`data`|كائن|لا|البيانات الوصفية الإضافية (مقيدة بصيغة JSON على التخزين)|

**الرد (200):**

```json
{
  "success": true,
  "notification": {
    "id": "notif_123abc",
    "userId": "user_456def",
    "type": "item_approved",
    "title": "Item Approved",
    "message": "Your item 'Awesome Tool' has been approved and is now live.",
    "data": "{\"itemId\": \"item_789ghi\", \"itemName\": \"Awesome Tool\", \"action\": \"approved\"}",
    "isRead": false,
    "readAt": null,
    "createdAt": "2024-01-20T10:30:00.000Z",
    "updatedAt": "2024-01-20T10:30:00.000Z"
  }
}
```

### التصحيح `/api/admin/notifications/{id}/read`

وضع علامة على إشعار محدد كمقروء. يضبط `isRead` على `true`، ويسجل الطابع الزمني الحالي في `readAt`، ويحدث `updatedAt`. يمكن لمالك الإشعار فقط وضع علامة على الإشعارات الخاصة به - حيث تقوم مرشحات الاستعلام حسب معرف الإشعار ومعرف المستخدم المصادق عليه.

** معلمات المسار: **

|المعلمة|اكتب|الوصف|
|-----------|------|-------------|
|`id`|سلسلة|معرف فريد للإخطار|

**الرد (200):**

```json
{
  "success": true,
  "notification": {
    "id": "notif_123abc",
    "userId": "user_456def",
    "type": "item_approved",
    "title": "Item Approved",
    "message": "Your item 'Awesome Tool' has been approved and is now live.",
    "isRead": true,
    "readAt": "2024-01-20T16:45:00.000Z",
    "createdAt": "2024-01-20T10:30:00.000Z",
    "updatedAt": "2024-01-20T16:45:00.000Z"
  }
}
```

### التصحيح `/api/admin/notifications/mark-all-read`

وضع علامة على جميع الإشعارات غير المقروءة للمستخدم المصادق عليه كمقروءة في عملية مجمعة واحدة. تحديثات `isRead` و`readAt` و`updatedAt` لكل إشعار مطابق. إرجاع عدد الإخطارات التي تم تحديثها.

**الرد (200):**

```json
{
  "success": true,
  "updatedCount": 5
}
```

** تفاصيل السلوك: **
- يقوم فقط بتحديث الإشعارات حيث `isRead = false` للمستخدم الحالي
- `updatedCount` قد يكون `0` إذا لم تكن هناك إشعارات غير مقروءة
- يتم تحديث كافة الإخطارات المطابقة في استعلام قاعدة بيانات واحدة

## نموذج بيانات الإخطار

|الميدان|اكتب|لاغية|الوصف|
|-------|------|----------|-------------|
|`id`|سلسلة|لا|معرف الإخطار الفريد|
|`userId`|سلسلة|لا|معرف المستخدم الذي يتلقى الإخطار|
|`type`|سلسلة|لا|نوع الإشعار (على سبيل المثال، `"item_approved"`، `"comment_received"`)|
|`title`|سلسلة|لا|عنوان العرض القصير|
|`message`|سلسلة|لا|رسالة إعلام كاملة|
|`data`|سلسلة|نعم|البيانات الوصفية الإضافية المقيدة بـ JSON|
|`isRead`|منطقية|لا|ما إذا كان قد تم قراءة الإخطار|
|`readAt`|التاريخ والوقت|نعم|الطابع الزمني عند وضع علامة "مقروءة".|
|`createdAt`|التاريخ والوقت|لا|الطابع الزمني للإنشاء|
|`updatedAt`|التاريخ والوقت|نعم|الطابع الزمني للتحديث الأخير|

## رموز الخطأ

|الحالة|خطأ|السبب|
|--------|-------|-------|
| `400` |الحقول المطلوبة مفقودة|POST يفتقد النوع أو العنوان أو الرسالة أو معرف المستخدم|
| `400` |معرف الإخطار مطلوب|تصحيح بمعلمة معرف فارغة|
| `401` |غير مصرح به|لا توجد جلسة نشطة|
| `403` |ممنوع|مستخدم غير إداري في نقطة نهاية قائمة GET|
| `404` |لم يتم العثور على الإخطار|معرف أو إشعار غير صالح يخص مستخدم آخر|
| `500` |خطأ داخلي في الخادم|فشل قاعدة البيانات أو الخادم|

## أنواع الإخطارات الشائعة

الحقل `type` عبارة عن سلسلة ذات شكل حر، ولكن التطبيق يستخدم عادةً هذه القيم:

|اكتب|الوصف|
|------|-------------|
|`item_approved`|تمت الموافقة على العنصر من قبل المشرف|
|`item_rejected`|تم رفض العنصر|
|`comment_received`|تم نشر تعليق جديد على عنصر|
|`submission_received`|تم استلام إرسال عنصر جديد|
