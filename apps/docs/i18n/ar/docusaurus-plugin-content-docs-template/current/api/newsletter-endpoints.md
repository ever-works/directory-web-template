# إجراءات خادم النشرة الإخبارية

يستخدم نظام الرسائل الإخبارية إجراءات خادم Next.js بدلاً من معالجات مسار واجهة برمجة التطبيقات التقليدية. تدير هذه الإجراءات اشتراكات البريد الإلكتروني بما في ذلك الاشتراك وإلغاء الاشتراك واسترداد الإحصائيات. يتم إرسال إشعارات البريد الإلكتروني لكل من أحداث الاشتراك وإلغاء الاشتراك باستخدام موفري البريد الإلكتروني القابلين للتكوين.

## نظرة عامة

|العمل|مصادقة|الوصف|
|---|---|---|
|`subscribeToNewsletter`|عام|اشترك بالبريد الإلكتروني في النشرة الإخبارية|
|`unsubscribeFromNewsletter`|عام|إلغاء الاشتراك في رسالة بريد إلكتروني من النشرة الإخبارية|
|`getNewsletterStatistics`|لا شيء|الحصول على إحصائيات الاشتراك|

هذه هي إجراءات الخادم المحددة بواسطة `'use server'` ويتم استدعاؤها من مكونات React عبر عمليات إرسال النماذج أو الاستدعاءات المباشرة، وليس عبر نقاط نهاية HTTP.

## إجراءات الخادم

### اشترك في النشرة الإخبارية

```typescript
subscribeToNewsletter(data: { email: string })
```

يشترك في عنوان بريد إلكتروني في النشرة الإخبارية. يتحقق من صحة البريد الإلكتروني باستخدام Zod، ويتحقق من الاشتراكات النشطة المكررة، وينشئ سجل قاعدة البيانات، ويرسل بريدًا إلكترونيًا ترحيبيًا. يتم تطبيع البريد الإلكتروني تلقائيًا إلى أحرف صغيرة وقصه.

**التحقق من صحة الإدخال (Zod):**

|الميدان|اكتب|مطلوب|القيود|
|---|---|---|---|
|`email`|سلسلة|نعم|يجب أن يكون تنسيق البريد الإلكتروني صالحًا|

**رد النجاح:**

```json
{
  "success": true
}
```

** ردود الأخطاء: **

```json
{
  "error": "Email is already subscribed to the newsletter",
  "email": "user@example.com"
}
```

|خطأ|الحالة|
|---|---|
|`"Please enter a valid email address"`|تنسيق البريد الإلكتروني غير صالح (التحقق من صحة Zod)|
|`"Email is already subscribed to the newsletter"`|الاشتراك النشط موجود بالفعل|
|`"Failed to create subscription. Please try again."`|فشل إدراج قاعدة البيانات|
|`"Failed to subscribe to newsletter. Please try again."`|خطأ غير متوقع|

**خطوات المعالجة:**

1. التحقق من صحة البريد الإلكتروني وتطبيعه (أحرف صغيرة، قطع)
2. تحقق من وجود اشتراك نشط موجود عبر `getNewsletterSubscriptionByEmail`
3. قم بإنشاء سجل اشتراك مع المصدر `"footer"` عبر `createNewsletterSubscription`
4. إرسال بريد إلكتروني ترحيبي باستخدام موفر البريد الإلكتروني الذي تم تكوينه (Resend أو Novu)

يتم اكتشاف حالات فشل إرسال البريد الإلكتروني بصمت ولا تمنع نجاح الاشتراك.

**المصدر:** `template/app/[locale]/newsletter/actions.ts`

### إلغاء الاشتراك من النشرة الإخبارية

```typescript
unsubscribeFromNewsletter(data: { email: string })
```

إلغاء الاشتراك في رسالة بريد إلكتروني من النشرة الإخبارية عن طريق تعيين `isActive` على `false`. يرسل رسالة تأكيد بالبريد الإلكتروني لإلغاء الاشتراك.

**رد النجاح:**

```json
{
  "success": true
}
```

** ردود الأخطاء: **

|خطأ|الحالة|
|---|---|
|`"Email is not subscribed to the newsletter"`|لم يتم العثور على اشتراك نشط|
|`"Failed to unsubscribe. Please try again."`|فشل تحديث قاعدة البيانات|

**المصدر:** `template/app/[locale]/newsletter/actions.ts`

### الحصول على إحصائيات النشرة الإخبارية

```typescript
getNewsletterStatistics()
```

إرجاع إحصائيات النشرة الإخبارية الإجمالية. لا توجد معلمات الإدخال المطلوبة.

**رد النجاح:**

```json
{
  "success": true,
  "data": {
    "totalActive": 1250,
    "recentSubscriptions": 45
  }
}
```

|الميدان|اكتب|الوصف|
|---|---|---|
|`totalActive`|عدد صحيح|عدد الاشتراكات النشطة حاليًا|
|`recentSubscriptions`|عدد صحيح|الاشتراكات التي تم إنشاؤها في آخر 30 يومًا|

إرجاع الأصفار لكلا الحقلين في حالة فشل الاستعلام، مما يضمن تدهورًا سلسًا.

**المصدر:** `template/app/[locale]/newsletter/actions.ts`

## استعلامات قاعدة البيانات

تتم إدارة بيانات الاشتراك في النشرة الإخبارية من خلال وظائف الاستعلام المخصصة في `lib/db/queries/newsletter.queries.ts`.

### عمليات الاشتراك

|وظيفة|الوصف|
|---|---|
|`createNewsletterSubscription(email, source)`|إنشاء سجل اشتراك جديد|
|`getNewsletterSubscriptionByEmail(email)`|يبحث عن الاشتراك عن طريق البريد الإلكتروني|
|`updateNewsletterSubscription(email, updates)`|تحديثات حقول الاشتراك|
|`unsubscribeFromNewsletter(email)`|يعين `isActive: false` ويسجل `unsubscribedAt`|
|`resubscribeToNewsletter(email)`|يضبط `isActive: true` ويمسح `unsubscribedAt`|
|`getNewsletterStats()`|إرجاع العدد النشط وعدد الاشتراكات لمدة 30 يومًا|

تعمل جميع عمليات البحث في البريد الإلكتروني على تطبيع الإدخال إلى أحرف صغيرة وتقليص المسافات البيضاء قبل الاستعلام.

**المصدر:** `template/lib/db/queries/newsletter.queries.ts`

## التكوين

يتم تعريف ثوابت تكوين النشرة الإخبارية في `lib/newsletter/config.ts`:

```
NEWSLETTER_CONFIG.DEFAULT_PROVIDER = "resend"
NEWSLETTER_CONFIG.DEFAULT_FROM = "onboarding@resend.dev"
NEWSLETTER_CONFIG.DEFAULT_COMPANY_NAME = "Ever Works"
```

### مصادر الاشتراك

|المصدر|الوصف|
|---|---|
|`footer`|الاشتراك من نموذج تذييل الموقع|
|`popup`|الاشتراك من مربع الحوار المنبثق|
|`signup`|الاشتراك أثناء تسجيل المستخدم|

### مخططات التحقق من الصحة

يتم تصدير مخططين Zod للتحقق من صحتهما:

- **`emailSchema`** - التحقق من صحة حقل بريد إلكتروني واحد وتطبيعه
- **`newsletterSubscriptionSchema`** - التحقق من صحة البريد الإلكتروني والمصدر (الإعداد الافتراضي هو `"footer"`)

### موفرو البريد الإلكتروني

يدعم النظام اثنين من موفري البريد الإلكتروني الذين تم تكوينهم عبر `config.yml` ومتغيرات البيئة:

|مزود|متغير البيئة|الوصف|
|---|---|---|
|إعادة الإرسال|`RESEND_API_KEY`|مزود البريد الإلكتروني الافتراضي|
|نوفو|`NOVU_API_KEY`|مزود بديل مع دعم القالب|

يتم تحديد الموفر بناءً على الحقل `mail.provider` في `config.yml`. يتم إنشاء تكوين البريد الإلكتروني ديناميكيًا من تكوين التطبيق باستخدام `createEmailConfig()`.

**المصدر:** `template/lib/newsletter/config.ts`

## تفاصيل التنفيذ الرئيسية

- **إجراءات الخادم:** هذه ليست نقاط نهاية REST API. يستخدمون المجمع `validatedAction` من `lib/auth/middleware` والذي يوفر التحقق من صحة مخطط Zod قبل تنفيذ الإجراء.
- **تسوية البريد الإلكتروني:** تتم تسوية جميع رسائل البريد الإلكتروني إلى أحرف صغيرة وتقليصها على مستوى الإجراء ومستوى استعلام قاعدة البيانات لإجراء عمليات بحث متسقة.
- ** حالات فشل البريد الإلكتروني الرائعة: ** يتم إرسال رسائل البريد الإلكتروني لتأكيد الترحيب وإلغاء الاشتراك عبر `sendEmailSafely()`، والتي تكتشف الأخطاء بصمت. البريد الإلكتروني الفاشل لا يمنع اكتمال عملية الاشتراك.
- **منع التكرار:** قبل إنشاء اشتراك، يتحقق النظام من وجود اشتراك نشط موجود باستخدام `validateExistingSubscription()`.
- **إلغاء الاشتراك البسيط:** مجموعات إلغاء الاشتراك `isActive: false` بدلاً من حذف السجل، والحفاظ على سجل الاشتراك.
