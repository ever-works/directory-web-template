# نقاط نهاية واجهة برمجة التطبيقات المساعدة

توفر نقاط النهاية المساعدة خدمات البنية التحتية بما في ذلك عمليات التحقق من الصحة ومعلومات الإصدار وتكوين الميزات والترميز الجغرافي والتحقق من reCAPTCHA واستخراج عنوان URL وبيانات الموقع والعمليات الداخلية.

## فحص الصحة (`/api/health`)

### صحة قاعدة البيانات

|الطريقة|المسار|الوصف|
|--------|------|-------------|
|`GET`|`/api/health/database`|التحقق من اتصال قاعدة البيانات|

إرجاع حالة اتصال قاعدة البيانات. يتم استخدامه من خلال أنظمة المراقبة وفحوصات صحة النشر.

**الرد (صحي):**

```json
{
  "status": "healthy",
  "database": "connected",
  "timestamp": "2025-01-15T10:30:00.000Z"
}
```

**الرد (غير صحي):**

```json
{
  "status": "unhealthy",
  "database": "disconnected",
  "error": "Connection refused",
  "timestamp": "2025-01-15T10:30:00.000Z"
}
```

**المصادقة:** عامة (لا يلزم المصادقة). يجب أن تكون نقطة النهاية هذه قابلة للوصول عن طريق موازنات التحميل وخدمات المراقبة.

## الإصدار (`/api/version`)

|الطريقة|المسار|الوصف|
|--------|------|-------------|
|`GET`|`/api/version`|الحصول على معلومات إصدار التطبيق|
|`GET`|`/api/version/sync`|الحصول على الإصدار وحالة المزامنة|

### استجابة الإصدار

إرجاع إصدار التطبيق ومعلومات البناء وبيئة التشغيل:

```json
{
  "version": "1.0.0",
  "environment": "production",
  "timestamp": "2025-01-15T10:30:00.000Z"
}
```

### الإصدار + حالة المزامنة

تعمل نقطة النهاية `/api/version/sync` على توسيع معلومات الإصدار مع حالة مزامنة مستودع المحتوى، وهي مفيدة لتصحيح أخطاء حداثة المحتوى.

**المصادقة:** عامة.

## تكوين الميزة (`/api/config`)

|الطريقة|المسار|الوصف|
|--------|------|-------------|
|`GET`|`/api/config/features`|احصل على إشارات الميزات الممكّنة|

إرجاع تكوين علامة الميزة الحالية للتطبيق من جانب العميل. يسمح هذا للواجهة الأمامية بعرض الميزات بشكل مشروط بناءً على التكوين من جانب الخادم.

**الرد:**

```json
{
  "features": {
    "payments": true,
    "sponsorAds": true,
    "surveys": false,
    "map": true,
    "newsletter": true
  }
}
```

**المصادقة:** عامة. أعلام الميزات ليست بيانات حساسة.

## استخراج URL (`/api/extract`)

|الطريقة|المسار|الوصف|
|--------|------|-------------|
|`POST`|`/api/extract`|استخراج البيانات الوصفية من عنوان URL|

جلب عنوان URL واستخراج البيانات الوصفية بما في ذلك العنوان والوصف والصورة والرمز المفضل. يتم استخدامه بواسطة نموذج إرسال العنصر لملء الحقول تلقائيًا من عنوان URL.

**الطلب:**

```json
{
  "url": "https://example.com/product"
}
```

**الرد:**

```json
{
  "success": true,
  "data": {
    "title": "Product Name",
    "description": "Product description from meta tags",
    "image": "https://example.com/og-image.png",
    "favicon": "https://example.com/favicon.ico",
    "siteName": "Example.com"
  }
}
```

**المصادقة:** مطلوب. يمنع إساءة استخدام جلب URL من جانب الخادم.

## الترميز الجغرافي (`/api/geocode`)

|الطريقة|المسار|الوصف|
|--------|------|-------------|
|`POST`|`/api/geocode`|ترميز جغرافي لعنوان للإحداثيات|

لتحويل عنوان نصي إلى إحداثيات جغرافية (خط العرض/خط الطول) باستخدام خدمة ترميز جغرافي خارجية.

**الطلب:**

```json
{
  "address": "1600 Amphitheatre Parkway, Mountain View, CA"
}
```

**الرد:**

```json
{
  "success": true,
  "data": {
    "lat": 37.4224764,
    "lng": -122.0842499,
    "formattedAddress": "1600 Amphitheatre Pkwy, Mountain View, CA 94043, USA"
  }
}
```

**المصادقة:** مطلوب.

## بيانات الموقع (`/api/location`)

نقاط النهاية للبحث عن الموقع والبيانات المرجعية.

|الطريقة|المسار|الوصف|
|--------|------|-------------|
|`GET`|`/api/location/countries`|قائمة جميع البلدان|
|`GET`|`/api/location/cities`|قائمة المدن (مع مرشح البلد)|
|`GET`|`/api/location/coordinates`|الحصول على إحداثيات الموقع|
|`GET`|`/api/location/search`|مواقع البحث عن طريق سلسلة الاستعلام|

### البلدان

إرجاع قائمة بالدول التي تحتوي على رموز ISO وأسماء وبيانات تعريفية اختيارية.

### المدن

يدعم التصفية حسب رمز البلد:

```
GET /api/location/cities?country=US
```

### البحث عن الموقع

البحث عن الموقع بالنص الكامل:

```
GET /api/location/search?q=San Francisco
```

**المصادقة:** عامة.

## التحقق من reCAPTCHA (`/api/verify-recaptcha`)

|الطريقة|المسار|الوصف|
|--------|------|-------------|
|`POST`|`/api/verify-recaptcha`|التحقق من رمز reCAPTCHA|

التحقق من جانب الخادم لرموز Google reCAPTCHA المميزة. يتم استخدامه في النماذج التي تتطلب حماية الروبوتات.

**الطلب:**

```json
{
  "token": "recaptcha-response-token"
}
```

**الرد:**

```json
{
  "success": true,
  "score": 0.9,
  "action": "submit"
}
```

**المصادقة:** عامة (reCAPTCHA تكون عادةً في النماذج العامة).

## البيانات المرجعية (`/api/reference`)

|الطريقة|المسار|الوصف|
|--------|------|-------------|
|`GET`|`/api/reference`|الحصول على البيانات المرجعية للقوائم المنسدلة للنموذج|

إرجاع البيانات المرجعية المستخدمة لملء القوائم المنسدلة وحقول التحديد في التطبيق، مثل نماذج التسعير وأنواع التراخيص وفئات النظام الأساسي.

**المصادقة:** عامة.

## العمليات الداخلية (`/api/internal`)

|الطريقة|المسار|الوصف|
|--------|------|-------------|
|`POST`|`/api/internal/db-init`|تهيئة مخطط قاعدة البيانات والبيانات الأولية|

### تهيئة قاعدة البيانات

تقوم نقطة النهاية `/api/internal/db-init` بتشغيل ترحيل قاعدة البيانات وإدراج البيانات الأولية الاختيارية. يتم استدعاء هذا عادةً مرة واحدة أثناء النشر الأولي أو عند إعادة تعيين بيئة التطوير.

**المصادقة:** يجب تأمين نقطة النهاية هذه عبر عناصر تحكم الوصول الخاصة بالبيئة أو سر مشترك. وليس المقصود للاستخدام المنتظم.

## الاعتبارات الأمنية

### نقاط النهاية العامة

نقاط النهاية المساعدة التالية عامة عن عمد:
- فحوصات السلامة (المطلوبة من خلال موازنات المراقبة/الأحمال)
- معلومات الإصدار (غير حساسة)
- أعلام الميزات (التكوين غير الحساس)
- بيانات الموقع (بيانات مرجعية)
- التحقق من reCAPTCHA (حماية النموذج العام)
- البيانات المرجعية (القيم المنسدلة)

### نقاط النهاية المحمية

تتطلب نقاط النهاية هذه المصادقة لمنع إساءة الاستخدام:
- استخراج عنوان URL (يمنع إساءة استخدام تزوير الطلب من جانب الخادم)
- الترميز الجغرافي (استدعاءات واجهة برمجة التطبيقات الخارجية ذات المعدل المحدود)
- تهيئة قاعدة البيانات (العملية المدمرة)
