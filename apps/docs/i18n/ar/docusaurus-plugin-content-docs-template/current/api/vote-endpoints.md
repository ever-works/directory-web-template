# نقاط نهاية التصويت

يوفر نظام التصويت نقاط نهاية للتصويت الإيجابي والتصويت السلبي للعناصر. تستخدم الأصوات نموذج صافي النتيجة حيث يمثل العدد الأصوات المؤيدة ناقص الأصوات السلبية. تقوم نقاط النهاية العامة بإرجاع عدد الأصوات بينما تسمح نقاط النهاية المعتمدة بالإدلاء بالأصوات وتحديثها وإزالتها. يتم منع المستخدمين المحظورين من التصويت.

## نظرة عامة

|نقطة النهاية|الطريقة|مصادقة|الوصف|
|---|---|---|---|
|`/api/items/[slug]/votes`|احصل على|عام|الحصول على عدد الأصوات وحالة تصويت المستخدم|
|`/api/items/[slug]/votes`|بريد|المستخدم|إرسال أو تحديث التصويت|
|`/api/items/[slug]/votes`|حذف|المستخدم|إزالة التصويت|
|`/api/items/[slug]/votes/count`|احصل على|عام|الحصول على عدد الأصوات الصافية فقط|
|`/api/items/[slug]/votes/status`|احصل على|المستخدم|الحصول على سجل التصويت الكامل للمستخدم|

## نقطة نهاية التصويت المجمعة

### الحصول على معلومات التصويت

```
GET /api/items/[slug]/votes
```

إرجاع صافي عدد الأصوات لعنصر ما وحالة تصويت المستخدم الحالي في حالة المصادقة عليه. لا يلزم إجراء مصادقة، لكن المستخدمين الذين تمت مصادقتهم يحصلون على حالة تصويتهم في الرد.

** معلمات المسار: **

|المعلمة|اكتب|الوصف|
|---|---|---|
|`slug`|سلسلة|سبيكة البند|

**رد النجاح (200):**

```json
{
  "success": true,
  "count": 15,
  "userVote": "up"
}
```

|الميدان|اكتب|الوصف|
|---|---|---|
|`success`|منطقية|دائما `true` على النجاح|
|`count`|عدد صحيح|صافي عدد الأصوات (الأصوات المؤيدة ناقص الأصوات السلبية)|
|`userVote`|سلسلة أو فارغة|`"up"`، `"down"`، أو `null` إذا لم تتم المصادقة أو لم يتم التصويت|

بالنسبة للمستخدمين غير المصادقين، `userVote` يكون دائمًا `null`. يمكن أن يكون `count` سلبيًا إذا كان عدد الأصوات السلبية أكثر من الأصوات المؤيدة.

**المصدر:** `template/app/api/items/[slug]/votes/route.ts`

### إرسال أو تحديث التصويت

```
POST /api/items/[slug]/votes
```

إجراء تصويت جديد أو استبدال تصويت موجود على عنصر ما. إذا كان لدى المستخدم تصويت بالفعل، فسيتم حذف التصويت السابق قبل إنشاء التصويت الجديد. وهذا يعني أن التغيير من التصويت الإيجابي إلى التصويت السلبي (أو العكس) هو عملية واحدة.

**المصادقة:** مطلوب

**نص الطلب:**

```json
{
  "type": "up"
}
```

|الميدان|اكتب|مطلوب|الوصف|
|---|---|---|---|
|`type`|سلسلة|نعم|`"up"` للتصويت الإيجابي، `"down"` للتصويت السلبي|

**رد النجاح (200):**

```json
{
  "success": true,
  "count": 16,
  "userVote": "up"
}
```

تقوم الاستجابة بإرجاع صافي عدد الأصوات المحدث بعد تطبيق التصويت.

** ردود الأخطاء: **

|الحالة|الحالة|
|---|---|
| 400 |نوع التصويت غير صالح (يجب أن يكون `"up"` أو `"down"`)|
| 401 |لم تتم المصادقة عليها|
| 403 |تم تعليق المستخدم أو حظره|
| 404 |لم يتم العثور على ملف تعريف العميل|

**المصدر:** `template/app/api/items/[slug]/votes/route.ts`

### إزالة التصويت

```
DELETE /api/items/[slug]/votes
```

إزالة تصويت المستخدم الحالي من أحد العناصر. في حالة عدم وجود تصويت، تكتمل العملية بنجاح دون خطأ (عاجز). بعد الإزالة، `userVote` هو `null`.

**المصادقة:** مطلوب

**رد النجاح (200):**

```json
{
  "success": true,
  "count": 14,
  "userVote": null
}
```

|الحالة|الحالة|
|---|---|
| 401 |لم تتم المصادقة عليها|
| 404 |لم يتم العثور على ملف تعريف العميل|

**المصدر:** `template/app/api/items/[slug]/votes/route.ts`

## نقطة نهاية عد الأصوات

### الحصول على عدد الأصوات

```
GET /api/items/[slug]/votes/count
```

إرجاع فقط صافي عدد الأصوات لعنصر ما. هذه نقطة نهاية عامة خفيفة الوزن تم تحسينها لاسترجاع عدد الأصوات بسرعة دون حالة التصويت الخاصة بالمستخدم.

**رد النجاح (200):**

```json
{
  "success": true,
  "count": 15
}
```

يمكن أن يكون العدد موجبًا أو سالبًا أو صفرًا اعتمادًا على رصيد الأصوات المؤيدة والأصوات المعارضة.

**المصدر:** `template/app/api/items/[slug]/votes/count/route.ts`

## نقطة نهاية حالة التصويت

### احصل على حالة تصويت المستخدم

```
GET /api/items/[slug]/votes/status
```

إرجاع سجل التصويت الكامل للمستخدم الذي تمت مصادقته على عنصر معين. يُرجع `null` إذا لم يقم المستخدم بالتصويت على العنصر.

**المصادقة:** مطلوب

**استجابة النجاح (200)--قام المستخدم بالتصويت:**

```json
{
  "id": "vote_123abc",
  "userId": "client_456def",
  "itemId": "item_123abc",
  "voteType": "UPVOTE",
  "createdAt": "2024-01-20T10:30:00.000Z",
  "updatedAt": "2024-01-20T10:30:00.000Z"
}
```

**إجابة النجاح (200) -- لا يوجد تصويت:**

```json
null
```

لاحظ أن نقطة النهاية هذه تُرجع قاعدة البيانات الأولية `voteType` القيم (`"UPVOTE"` أو `"DOWNVOTE"`) بدلاً من التنسيق المبسط `"up"` / `"down"` الذي تستخدمه نقطة النهاية المدمجة.

|الحالة|الحالة|
|---|---|
| 401 |لم تتم المصادقة عليها|
| 404 |لم يتم العثور على ملف تعريف العميل|

**المصدر:** `template/app/api/items/[slug]/votes/status/route.ts`

## تفاصيل التنفيذ الرئيسية

- **النتيجة الصافية:** يتم حساب عدد الأصوات على أنها أصوات مؤيدة مطروحًا منها أصوات معارضة. يشير العدد السلبي إلى عدد أكبر من الأصوات السلبية مقارنة بالأصوات المؤيدة.
- **استبدال التصويت:** عندما يقوم المستخدم بتغيير نوع التصويت الخاص به، يتم حذف التصويت الحالي وإنشاء صوت جديد. لا يوجد تحديث في المكان.
- **منع المستخدم المحظور:** يمنع فحص `isUserBlocked()` على نقطة نهاية POST المستخدمين المعلقين أو المحظورين من التصويت. يتم فرض فحص الكتلة فقط عند إنشاء التصويت، وليس عند إزالة التصويت.
- **VoteType Enum:** تقوم قاعدة البيانات بتخزين الأصوات كـ `VoteType.UPVOTE` و`VoteType.DOWNVOTE`. تترجم واجهة برمجة التطبيقات هذه إلى `"up"` و`"down"` للمستهلكين الخارجيين.
- **حذف غير فعال:** يؤدي حذف التصويت غير الموجود إلى إرجاع 200 استجابة بالعدد الحالي و`userVote: null`.
