# مرجع المخطط

يتم تعريف كافة جداول قاعدة البيانات في `lib/db/schema.ts`. يقوم هذا المستند بفهرسة كل جدول وأعمدته الرئيسية وعلاقاته والغرض منه.

## المستخدمون والمصادقة

### المستخدمين

جدول المستخدم الأساسي، الذي يستخدمه NextAuth.js للمصادقة.

|العمود|اكتب|ملاحظات|
|--------|------|-------|
|`id`|النص (PK)|UUID، تم إنشاؤه تلقائيًا|
|`email`|نص|فريدة من نوعها|
|`image`|نص|عنوان URL لصورة الملف الشخصي|
|`emailVerified`|الطابع الزمني|تاريخ التحقق من البريد الإلكتروني|
|`passwordHash`|نص|تجزئة Bcrypt لمصادقة بيانات الاعتماد|
|`createdAt`|الطابع الزمني|ضبط تلقائي|
|`updatedAt`|الطابع الزمني|ضبط تلقائي|
|`deletedAt`|الطابع الزمني|حذف ناعم|

**الفهارس**: `users_created_at_idx`

### الحسابات

روابط حساب OAuth وبيانات الاعتماد، باتباع مخطط محول NextAuth.js.

|العمود|اكتب|ملاحظات|
|--------|------|-------|
|`userId`|النص (فك)|المراجع `users.id` (حذف متتالي)|
|`type`|نص|نوع الحساب (Oauth، بيانات الاعتماد، وما إلى ذلك)|
|`provider`|نص|اسم الموفر (google، github، بيانات الاعتماد)|
|`providerAccountId`|نص|معرف الحساب الخاص بالموفر|
|`email`|نص|البريد الإلكتروني للحساب|
|`passwordHash`|نص|لمصادقة بيانات اعتماد العميل|
|`refresh_token`|نص|رمز تحديث OAuth|
|`access_token`|نص|رمز وصول OAuth|
|`expires_at`|عدد صحيح|انتهاء صلاحية الرمز المميز|

**المفتاح الأساسي**: مركب على (`provider`، `providerAccountId`)
**الفهارس**: `accounts_email_idx`، `accounts_provider_idx`

### جلسات

جلسات المستخدم النشطة.

|العمود|اكتب|ملاحظات|
|--------|------|-------|
|`sessionToken`|النص (PK)|معرف الجلسة|
|`userId`|النص (فك)|المراجع `users.id`|
|`expires`|الطابع الزمني|انتهاء الجلسة|

### VerificationTokens

رموز التحقق من البريد الإلكتروني.

|العمود|اكتب|ملاحظات|
|--------|------|-------|
|`identifier`|نص|معرف المستخدم|
|`email`|نص|عنوان البريد الإلكتروني|
|`token`|نص|رمز التحقق|
|`expires`|الطابع الزمني|انتهاء صلاحية الرمز المميز|

**المفتاح الأساسي**: مركب على (`identifier`، `token`)

### المصادقون

تخزين بيانات اعتماد WebAuthn/FIDO2.

|العمود|اكتب|ملاحظات|
|--------|------|-------|
|`credentialID`|نص|معرف بيانات الاعتماد الفريد|
|`userId`|النص (فك)|المراجع `users.id`|
|`providerAccountId`|نص|مرجع حساب المزود|
|`credentialPublicKey`|نص|المفتاح العام للتحقق|
|`counter`|عدد صحيح|عداد المصادقة|

### this.passwordResetTokens

الرموز المميزة لإعادة تعيين كلمة المرور لتدفق كلمة المرور المنسية.

|العمود|اكتب|ملاحظات|
|--------|------|-------|
|`id`|النص (PK)|UUID|
|`email`|نص|البريد الإلكتروني المستهدف|
|`token`|نص|رمز إعادة الضبط الفريد|
|`expires`|الطابع الزمني|انتهاء صلاحية الرمز المميز|

### سجل النشاط

يتتبع أنشطة المستخدم والعميل لأغراض التدقيق.

|العمود|اكتب|ملاحظات|
|--------|------|-------|
|`id`|المسلسل (PK)|الزيادة التلقائية|
|`userId`|النص (فك)|المراجع `users.id` (قيمة فارغة)|
|`clientId`|النص (فك)|المراجع `clientProfiles.id` (قيمة فارغة)|
|`action`|نص|نوع النشاط (SIGN_UP، SIGN_IN، إلخ.)|
|`timestamp`|الطابع الزمني|عندما حدث النشاط|
|`ipAddress`|فارتشار(45)|عنوان IP للعميل|

**الفهارس**: `activity_logs_user_idx`، `activity_logs_timestamp_idx`، `activity_logs_action_idx`

|العمود|اكتب|ملاحظات|

## جداول أخرى

### إعلانات الراعي

إعلانات العناصر الدعائية مع دورة حياة الدفع الكاملة.

الأعمدة الرئيسية: `userId`، `itemSlug`، `status` (في انتظار الدفع، معلق، مرفوض، نشط، منتهية الصلاحية، ملغى)، `interval` (أسبوعي، شهري)، `amount`، `paymentProvider`، `subscriptionId`.

### الشركات / العناصرالشركات

سجلات الشركة وارتباطات شركة العناصر لقوائم الدليل.

### استطلاعات الرأي / استطلاعات الرأي

منشئ الاستطلاعات مع تعريفات الأسئلة المستندة إلى JSON وتخزين الردود.

### TwentyCrmConfig / IntegrationMappings

جداول تكامل CRM لوظيفة مزامنة Twenty CRM. يفرض جدول التكوين نمطًا مفردًا (يُسمح بصف واحد فقط).

### الاشتراكات في النشرة الإخبارية

تتبع الاشتراك في النشرة الإخبارية عبر البريد الإلكتروني مع الطوابع الزمنية للاشتراك/إلغاء الاشتراك.

### SeedStatus

تتبع حالة البذر في قاعدة بيانات الجدول الفردي (البذر، مكتمل، فاشل) لمنع عمليات البذور المتزامنة.

## اكتب الصادرات

يقوم ملف المخطط بتصدير أنواع TypeScript لكل جدول باستخدام استنتاج Drizzle:

```typescript
export type User = typeof users.$inferSelect;
export type NewUser = typeof users.$inferInsert;
export type Subscription = typeof subscriptions.$inferSelect;
export type NewSubscription = typeof subscriptions.$inferInsert;
// ... and so on for all tables
```

يتم استخدام هذه الأنواع في جميع أنحاء التطبيق لعمليات قاعدة البيانات الآمنة.
