# قائمة تحقق جاهزية الإنتاج

قائمة تحقق شاملة لضمان جاهزية Ever Works لبيئة الإنتاج.

## قائمة التحقق قبل النشر

### 1. تكوين البيئة

#### متغيرات البيئة المطلوبة

- [ ] **قاعدة البيانات**
  - تم تكوين `DATABASE_URL` مع PostgreSQL للإنتاج
  - تم تفعيل تجمع اتصالات قاعدة البيانات
  - تم تفعيل وضع SSL للإنتاج

- [ ] **المصادقة**
  - تم تعيين `NEXTAUTH_URL` لنطاق الإنتاج
  - تم إنشاء `NEXTAUTH_SECRET` (32 حرفاً على الأقل)
  - تم تكوين موفري OAuth (Google وGitHub وغيرها)
  - بيانات اعتماد Supabase Auth (إذا كانت مستخدمة)

- [ ] **موفرو الدفع**
  - مفاتيح Stripe للإنتاج (`STRIPE_SECRET_KEY` و`NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY`)
  - مفاتيح LemonSqueezy للإنتاج (إذا كانت مستخدمة)
  - تم تكوين أسرار الـ Webhook
  - تم تعطيل وضع الاختبار

- [ ] **خدمات البريد الإلكتروني**
  - تم تكوين مفتاح API لـ Resend
  - تم تعيين بيانات اعتماد Novu (إذا كانت مستخدمة)
  - تم اختبار قوالب البريد الإلكتروني
  - تم التحقق من نطاق المرسل

- [ ] **التحليلات والمراقبة**
  - مفتاح PostHog للإنتاج
  - تم تكوين DSN لـ Sentry
  - تم تعريف موفر تتبع الاستثناءات
  - تم تفعيل Vercel Analytics (إذا كان على Vercel)

- [ ] **تكامل CRM**
  - بيانات اعتماد Twenty CRM (إذا كانت مستخدمة)
  - تم تكوين نقاط نهاية الـ Webhook

- [ ] **الأمان**
  - `NODE_ENV=production`
  - تم تكوين تحديد المعدل
  - تمت مراجعة إعدادات CORS
  - تم تكوين رؤوس CSP

### 2. قاعدة البيانات

- [ ] **المخطط والترحيل**
  - تم تطبيق جميع عمليات الترحيل
  - مخطط قاعدة البيانات يتطابق مع الكود
  - تم إنشاء فهارس الأداء
  - تم التحقق من قيود المفاتيح الخارجية

- [ ] **سلامة البيانات**
  - تم تحميل بيانات البذر (إذا لزم الأمر)
  - تم حذف بيانات الاختبار
  - تم تطبيق قواعد التحقق من البيانات

- [ ] **النسخ الاحتياطي والاستعادة**
  - تم تكوين النسخ الاحتياطي التلقائي
  - تم اختبار استعادة النسخ الاحتياطي
  - تم تفعيل الاستعادة إلى نقطة زمنية
  - تم تحديد سياسة الاحتفاظ بالنسخ الاحتياطية

- [ ] **الأداء**
  - تم تكوين تجمع الاتصالات
  - تم تحسين أداء الاستعلام
  - تم تفعيل تسجيل الاستعلامات البطيئة
  - تم تفعيل مراقبة قاعدة البيانات

### 3. الأمان

- [ ] **المصادقة والتفويض**
  - تم التحقق من تجزئة كلمة المرور (bcrypt)
  - إدارة الجلسات آمنة
  - توقيع رموز JWT صحيح
  - تم اختبار التحكم في الوصول المستند إلى الأدوار

- [ ] **حماية البيانات**
  - تم تشفير بيانات PII في حالة السكون
  - تم تكوين تنظيف البيانات الحساسة
  - تم فرض استخدام HTTPS
  - تم تفعيل ملفات تعريف الارتباط الآمنة

- [ ] **أمان API**
  - تم تفعيل تحديد المعدل
  - تم فرض مصادقة API
  - التحقق من صحة المدخلات على جميع نقاط النهاية
  - تم التحقق من الحماية من حقن SQL

- [ ] **التبعيات**
  - تم تحديث جميع التبعيات
  - تم فحص الثغرات الأمنية (`npm audit`)
  - لا توجد ثغرات حرجة
  - تم إيداع ملف القفل للتبعيات

### 4. الأداء

- [ ] **تحسين الواجهة الأمامية**
  - تم تحسين الصور (مكون Next.js Image)
  - تم تنفيذ تقسيم الكود
  - التحميل الكسول للمكونات الثقيلة
  - تم تحليل حجم الحزمة

- [ ] **التخزين المؤقت**
  - تم تخزين الأصول الثابتة مؤقتاً
  - تم تخزين استجابات API مؤقتاً (عند الاقتضاء)
  - تم تكوين CDN
  - تم تطبيق استراتيجية إبطال التخزين المؤقت

- [ ] **Core Web Vitals**
  - LCP أقل من 2.5 ثانية
  - FID أقل من 100 ميلي ثانية
  - CLS أقل من 0.1
  - تم تفعيل مراقبة الأداء

- [ ] **استعلامات قاعدة البيانات**
  - تم التخلص من استعلامات N+1
  - تم إنشاء الفهارس المناسبة
  - تم تفعيل تخزين الاستعلامات مؤقتاً
  - تم تحسين تجمع الاتصالات

### 5. المراقبة والتسجيل

- [ ] **تتبع الأخطاء**
  - تم تكوين Sentry/PostHog
  - تم إعداد تنبيهات الأخطاء
  - تم رفع خرائط المصدر
  - تم تكوين تجميع الأخطاء

- [ ] **مراقبة التطبيق**
  - نقطة نهاية فحص الصحة (`/api/health`)
  - تم تكوين مراقبة وقت التشغيل
  - تم تتبع مقاييس الأداء
  - تم تعريف المقاييس المخصصة

- [ ] **التسجيل**
  - تم تنفيذ التسجيل المنظم
  - تم تكوين مستويات التسجيل
  - تم تكوين تجميع السجلات
  - تم تحديد سياسة الاحتفاظ بالسجلات

- [ ] **التنبيهات**
  - تنبيهات الأخطاء الحرجة
  - تنبيهات تدهور الأداء
  - تنبيهات وقت التشغيل
  - تنبيهات فشل الدفع

### 6. المحتوى والبيانات

- [ ] **CMS المستند إلى Git**
  - تم تكوين مستودع `.content`
  - مزامنة المحتوى تعمل بشكل صحيح
  - تم حفظ بيانات اعتماد Git بأمان
  - تم إعداد استراتيجية نسخ احتياطي للمحتوى

- [ ] **الأصول الإعلامية**
  - تم تحسين الصور
  - تم تكوين CDN للوسائط
  - تم تكوين حدود الرفع
  - تم تفعيل مراقبة حصة التخزين

- [ ] **التدويل**
  - اكتملت جميع الترجمات
  - تم اختبار دعم RTL (العربية)
  - اكتشاف اللغة يعمل بشكل صحيح
  - تم التحقق من تنسيق التاريخ/الأرقام

### 7. وثائق API

- [ ] **نظام التوثيق**
  - تم إنشاء مواصفة OpenAPI (`yarn generate-docs`)
  - واجهة Scalar UI متاحة على `/api/reference`
  - تم توثيق جميع نقاط النهاية
  - تم اختبار الأمثلة

- [ ] **معايير API**
  - اتفاقيات تسمية متسقة
  - رموز حالة HTTP صحيحة
  - استجابات خطأ موحدة
  - تم توثيق تحديد المعدل

### 8. نظام الدفع

- [ ] **تكوين Stripe**
  - تم تفعيل وضع الإنتاج
  - تم تكوين Webhooks واختبارها
  - تم تفعيل بوابة العملاء
  - تم تكوين إعدادات الفاتورة

- [ ] **تكوين LemonSqueezy** (إذا كان مستخدماً)
  - تم تعيين بيانات اعتماد الإنتاج
  - تم تكوين Webhooks
  - تم التحقق من الامتثال الضريبي

- [ ] **إدارة الاشتراكات**
  - تم اختبار إنشاء الخطط
  - تم اختبار عمليات الترقية/التخفيض
  - تم اختبار عملية الإلغاء
  - تم توثيق عملية الاسترداد

### 9. نظام البريد الإلكتروني

- [ ] **رسائل البريد الإلكتروني التعاملية**
  - تم اختبار رسالة الترحيب
  - تم اختبار إعادة تعيين كلمة المرور
  - تم اختبار التحقق من البريد الإلكتروني
  - تم اختبار رسائل الاشتراك

- [ ] **قوالب البريد الإلكتروني**
  - تمت مراجعة جميع القوالب
  - العلامة التجارية متسقة
  - متوافقة مع الأجهزة المحمولة
  - روابط إلغاء الاشتراك تعمل بشكل صحيح

- [ ] **القابلية للتسليم**
  - تم تكوين سجلات SPF
  - تم تكوين DKIM
  - تم تحديد سياسة DMARC
  - تم تفعيل مراقبة سمعة المرسل

### 10. الاختبار

- [ ] **اختبار الوظائف**
  - تدفق تسجيل المستخدم
  - تدفق تسجيل الدخول/الخروج
  - تدفق إعادة تعيين كلمة المرور
  - تدفق إضافة عنصر
  - تدفق الدفع
  - وظائف المسؤول

- [ ] **اختبار عبر المتصفحات**
  - تم اختبار Chrome
  - تم اختبار Firefox
  - تم اختبار Safari
  - تم اختبار Edge
  - تم اختبار متصفحات الأجهزة المحمولة

- [ ] **اختبار الاستجابة**
  - الأجهزة المحمولة (320 بكسل – 480 بكسل)
  - الأجهزة اللوحية (768 بكسل – 1024 بكسل)
  - سطح المكتب (1280 بكسل فأكثر)
  - الشاشات الكبيرة (1920 بكسل فأكثر)

- [ ] **اختبار التحميل**
  - تم محاكاة حركة المرور المتوقعة
  - أداء قاعدة البيانات تحت الحمل مقبول
  - أوقات استجابة API مقبولة
  - لا توجد تسريبات للذاكرة

### 11. الامتثال والقانون

- [ ] **الخصوصية**
  - تم نشر سياسة الخصوصية
  - تم تنفيذ موافقة ملفات تعريف الارتباط
  - الامتثال للائحة GDPR (لمستخدمي الاتحاد الأوروبي)
  - وظيفة تصدير البيانات متاحة

- [ ] **شروط الاستخدام**
  - تم نشر شروط الاستخدام
  - تم تنفيذ تدفق قبول المستخدم
  - تم تفعيل تتبع إصدار الشروط

- [ ] **إمكانية الوصول**
  - الامتثال لـ WCAG 2.1 AA
  - التنقل بلوحة المفاتيح يعمل بشكل صحيح
  - تم الاختبار بقارئ الشاشة
  - تم إضافة نص بديل للصور

### 12. DevOps والبنية التحتية

- [ ] **النشر**
  - تم تكوين خط أنابيب CI/CD
  - الاختبارات الآلية تعمل في خط الأنابيب
  - تم إعداد خطة التراجع عن النشر
  - تم تنفيذ النشر بدون توقف

- [ ] **قابلية التوسع**
  - تم تكوين التوسع التلقائي
  - تم تكوين موازن التحميل
  - نسخ للقراءة فقط من قاعدة البيانات (إذا لزم الأمر)
  - شبكة CDN للأصول الثابتة

- [ ] **التعافي من الكوارث**
  - تم اختبار استعادة النسخ الاحتياطي
  - تم توثيق خطة تجاوز الفشل
  - تم إعداد خطة الاستجابة للحوادث
  - تم تحديد الخدمة أثناء التوقف

- [ ] **التوثيق**
  - تم تحديث دليل النشر
  - تم إنشاء الـ Runbook
  - تم تحديث مخططات البنية
  - اكتمل تدريب الفريق

## أوامر التحقق

قم بتشغيل هذه الأوامر للتحقق من جاهزية الإنتاج:

### التدقيق الأمني

```bash
# التحقق من الثغرات الأمنية
npm audit --production

# إصلاح الثغرات
npm audit fix

# التحقق من التبعيات القديمة
npm outdated
```

### التحقق من البناء

```bash
# بناء الإنتاج
npm run build

# فحص مخرجات البناء
ls -lh .next/

# تحليل حجم الحزمة
npm run analyze
```

### التحقق من قاعدة البيانات

```bash
# التحقق من حالة الترحيل
npx drizzle-kit check

# إنشاء ترحيل إذا لزم الأمر
npx drizzle-kit generate

# تطبيق عمليات الترحيل
npx drizzle-kit push
```

### وثائق API

```bash
# إنشاء مواصفة OpenAPI
yarn generate-docs

# التحقق من صحة الوثائق
yarn docs:validate

# التحقق من حداثة الوثائق
git diff --exit-code public/openapi.json
```

### متغيرات البيئة

```bash
# التحقق من تعيين جميع المتغيرات المطلوبة
node scripts/check-env.js

# اختبار تكوين البيئة
npm run test:env
```

## سير عمل النشر

### قبل النشر

1. **مراجعة الكود**
   - تمت مراجعة وموافقة جميع طلبات السحب
   - لا توجد تعارضات في الدمج
   - اجتاز خط أنابيب CI/CD

2. **الاختبار**
   - اجتازت جميع الاختبارات
   - اكتمل اختبار QA اليدوي
   - تم اختبار بيئة Staging

3. **التوثيق**
   - تم تحديث سجل التغييرات
   - تم إعادة إنشاء وثائق API
   - تم تجهيز ملاحظات النشر

### خطوات النشر

1. **النسخ الاحتياطي**

   ```bash
   # نسخ احتياطي لقاعدة البيانات
   pg_dump $DATABASE_URL > backup-$(date +%Y%m%d).sql
   ```

2. **النشر**

   ```bash
   # النشر إلى الإنتاج
   git push production main

   # أو باستخدام Vercel
   vercel --prod
   ```

3. **التحقق**

   ```bash
   # التحقق من نقطة نهاية الصحة
   curl https://your-domain.com/api/health

   # التحقق من سجلات الأخطاء
   tail -f logs/error.log
   ```

4. **المراقبة**
   - مراقبة معدل الأخطاء في Sentry
   - مراقبة الأداء في PostHog
   - التحقق من مراقبة وقت التشغيل

### بعد النشر

1. **اختبارات الدخان**
   - الصفحة الرئيسية تتحمل بشكل صحيح
   - يستطيع المستخدم تسجيل الدخول
   - تدفق الدفع يعمل بشكل صحيح
   - لوحة المشرف متاحة

2. **المراقبة**
   - معدل الأخطاء طبيعي
   - أوقات الاستجابة مقبولة
   - لا توجد تسريبات للذاكرة
   - أداء قاعدة البيانات مستقر

3. **التواصل**
   - إبلاغ الفريق بالنشر
   - تحديث صفحة الحالة
   - الإعلان عن الميزات الجديدة (إذا انطبق)

## خطة التراجع

إذا اكتُشفت مشاكل بعد النشر:

### التراجع السريع

```bash
# الرجوع إلى النشر السابق
git revert HEAD
git push production main

# أو باستخدام Vercel
vercel rollback
```

### تراجع قاعدة البيانات

```bash
# الاستعادة من النسخ الاحتياطي
psql $DATABASE_URL < backup-YYYYMMDD.sql

# أو استخدام الاستعادة إلى نقطة زمنية
# (إذا كان موفر الاستضافة يدعم ذلك)
```

### التواصل

1. إبلاغ الفريق فوراً
2. تحديث صفحة الحالة
3. التواصل مع المستخدمين المتضررين
4. توثيق الحادثة لمراجعة ما بعد الحادث

## مقاييس النجاح

تتبع هذه المقاييس لضمان صحة بيئة الإنتاج:

### الأداء

- **وقت الاستجابة**: أقل من 200 ميلي ثانية (p95)
- **وقت التشغيل**: أكثر من 99.9%
- **معدل الأخطاء**: أقل من 0.1%
- **Core Web Vitals**: جميعها خضراء

### الأعمال

- **تسجيلات المستخدمين**: التتبع يعمل بشكل صحيح
- **معدل نجاح الدفع**: أكثر من 95%
- **قابلية تسليم البريد الإلكتروني**: أكثر من 98%
- **توافر API**: أكثر من 99.9%

### الأمان

- **محاولات تسجيل الدخول الفاشلة**: تتبع نشط
- **ضربات حد معدل API**: أقل من 1%
- **ثغرات الأمان**: 0 حرجة
- **شهادة SSL**: صالحة ومع تجديد تلقائي

## ملخص قائمة التحقق

استخدم هذا الملخص الموجز لتتبع التقدم الإجمالي:

- [ ] **البيئة**: جميع المتغيرات مكونة
- [ ] **قاعدة البيانات**: تم تطبيق عمليات الترحيل وتكوين النسخ الاحتياطية
- [ ] **الأمان**: المصادقة والتشفير وتحديد المعدل
- [ ] **الأداء**: محسّن ومخزن مؤقتاً ومراقب
- [ ] **المراقبة**: تتبع الأخطاء والتسجيل والتنبيهات
- [ ] **المحتوى**: تم تكوين CMS وتحسين الوسائط وإكمال i18n
- [ ] **API**: تم إنشاء الوثائق واتباع المعايير
- [ ] **المدفوعات**: تم تكوين Stripe/LS واختبار Webhooks
