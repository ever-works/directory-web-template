# دليل الاختبار المتجاوب

يتناول هذا الدليل أفضل الممارسات لاختبار التصميم المتجاوب عبر أجهزة وأحجام شاشات مختلفة.

## أجهزة الاختبار

## قائمة تحقق الاختبار

### 1. التنقل

- [ ] **الجوال**: زر القائمة (hamburger) مرئي وفعَّال
- [ ] **سطح المكتب**: شريط التنقل الأفقي يظهر بشكل صحيح
- [ ] **جميع الأجهزة**: جميع روابط التنقل قابلة للوصول
- [ ] **مناطق اللمس**: 44x44px كحد أدنى على الجوال
- [ ] **التنقل بلوحة المفاتيح**: ترتيب Tab منطقي

### 2. المحتوى

- [ ] **قابلية قراءة النص**: لا حاجة للتكبير للقراءة
- [ ] **الصور**: متجاوبة ومقيَّسة بشكل صحيح عند كل نقطة توقف
- [ ] **لا تمرير أفقي**: يتلاءم المحتوى مع نافذة العرض
- [ ] **طول السطر**: عرض قراءة مثالي (45-75 حرف)
- [ ] **أحجام الخطوط**: مناسبة لكل حجم جهاز

### 3. التفاعلات

- [ ] **مناطق اللمس**: 44x44px كحد أدنى للجوال
- [ ] **الهوامش**: مساحة كافية بين العناصر القابلة للنقر
- [ ] **حالات التحويم**: فقط على الأجهزة الداعمة له
- [ ] **حالات التركيز**: مؤشرات تركيز لوحة المفاتيح مرئية
- [ ] **النماذج**: سهلة الملء على الجوال

### 4. الأداء

- [ ] **وقت التحميل**: أقل من 3 ثوان عند اتصال 3G
- [ ] **الصور**: مُحسَّنة لكل حجم شاشة
- [ ] **الرسوم المتحركة**: أداء سلس 60 FPS
- [ ] **Core Web Vitals**: تلبية حدود Google
- [ ] **حجم الحزمة**: JavaScript وCSS مُحسَّنان

### 5. التخطيط

- [ ] **الشبكة**: يتكيف بشكل صحيح عند نقاط التوقف
- [ ] **Flexbox/Grid**: لا أخطاء في التخطيط
- [ ] **الحشو**: padding وmargin متسقان
- [ ] **المحاذاة**: محاذاة صحيحة بجميع الأحجام
- [ ] **الفيضان**: لا مشاكل تجاوز المحتوى

## أدوات الاختبار

### أدوات المطور في المتصفح

#### Chrome DevTools
1. افتح DevTools (F12 أو Cmd+Option+I)
2. انقر أيقونة شريط الأجهزة (Cmd+Shift+M)
3. اختر جهازًا أو أدخل أبعادًا مخصصة
4. اختبر سرعات شبكة مختلفة

#### Firefox Developer Tools
1. افتح DevTools (F12)
2. انقر "وضع التصميم المتجاوب" (Cmd+Option+M)
3. اختر إعدادات مسبقة للأجهزة
4. اختبر أحداث اللمس

### خدمات الاختبار عبر الإنترنت

- **[BrowserStack](https://www.browserstack.com/)** - الاختبار على أجهزة حقيقية
- **[LambdaTest](https://www.lambdatest.com/)** - اختبار عبر المتصفحات
- **[Sauce Labs](https://saucelabs.com/)** - منصة الاختبار الآلي

### اختبار الأداء

- **[Lighthouse](https://developers.google.com/web/tools/lighthouse)** - تدقيق الأداء
- **[WebPageTest](https://www.webpagetest.org/)** - تحليل الأداء التفصيلي
- **[PageSpeed Insights](https://pagespeed.web.dev/)** - أداة أداء Google

## المقاييس المستهدفة

### درجات Lighthouse

| المقياس         | الهدف | الحرج |
|-----------------|-------|-------|
| الأداء          | > 90  | > 50  |
| إمكانية الوصول  | > 95  | > 80  |
| أفضل الممارسات  | > 95  | > 80  |
| SEO             | > 95  | > 80  |

### Core Web Vitals

| المقياس | جيد | يحتاج تحسين | سيئ |
|---------|-----|-------------|-----|
| **LCP** (أكبر رسم للمحتوى) | < 2.5s | 2.5s - 4.0s | > 4.0s |
| **FID** (تأخر أول إدخال) | < 100ms | 100ms - 300ms | > 300ms |
| **CLS** (تراكم تحول التخطيط) | < 0.1 | 0.1 - 0.25 | > 0.25 |

### مقاييس الجوال

- **أول رسم للمحتوى**: < 1.8s
- **وقت التفاعل**: < 3.8s
- **حجم منطقة اللمس**: >= 48x48px
- **تأخر اللمس**: < 300ms

## سير عمل الاختبار

### 1. الاختبار المحلي

```bash
# تشغيل خادم التطوير
npm run dev

# فتح في المتصفح
http://localhost:3000
```

### 2. اختبار الأجهزة

1. **استخدام الوضع المتجاوب في DevTools**
2. **الاختبار على أجهزة فعلية** عند الإمكان
3. **استخدام التصحيح عن بُعد** للأجهزة الجوالة
4. **اختبار الاتجاهات المختلفة** (عمودي/أفقي)

### 3. الاختبار الآلي

```bash
# تشغيل تدقيق Lighthouse
npm run lighthouse

# تشغيل اختبارات إمكانية الوصول
npm run test:a11y

# تشغيل اختبارات الانحدار البصري
npm run test:visual
```

## المشاكل الشائعة وحلولها

### المشكلة: التمرير الأفقي على الجوال

**الحل**: تحقق من العناصر ذات العرض الثابت

```css
/* ❌ سيئ */
.container {
  width: 1200px;
}

/* ✅ جيد */
.container {
  max-width: 1200px;
  width: 100%;
  padding: 0 1rem;
}
```

### المشكلة: نص صغير جدًا على الجوال

**الحل**: استخدم أحجام خطوط متجاوبة

```css
/* ❌ سيئ */
body {
  font-size: 12px;
}

/* ✅ جيد */
body {
  font-size: 16px; /* حجم أساسي للجوال */
}

@media (min-width: 768px) {
  body {
    font-size: 18px;
  }
}
```

### المشكلة: مناطق لمس صغيرة جدًا

**الحل**: تأكد من حجم أدنى 44x44px

```css
/* ✅ جيد */
.button {
  min-height: 44px;
  min-width: 44px;
  padding: 12px 24px;
}
```

### المشكلة: صور غير متجاوبة

**الحل**: استخدم تقنيات الصور المتجاوبة

```jsx
// ✅ جيد - مكوِّن Image في Next.js
import Image from 'next/image';

<Image
  src="/image.jpg"
  alt="Description"
  width={800}
  height={600}
  sizes="(max-width: 768px) 100vw, (max-width: 1200px) 50vw, 33vw"
  priority
/>
```

## أفضل الممارسات

### 1. نهج الجوال أولًا
- صمِّم للجوال أولًا ثم حسِّن للشاشات الأكبر
- استخدم استعلامات وسائط `min-width` بدلًا من `max-width`

### 2. التحسين التدريجي
- تأكد من عمل الوظائف الأساسية بدون JavaScript
- أضف تحسينات للمتصفحات الداعمة

### 3. تصميم ملائم للمس
- مناطق لمس بحجم أدنى 44x44px
- مساحة كافية بين العناصر التفاعلية
- تجنب التفاعلات المعتمدة فقط على التحويم

### 4. تحسين الأداء
- تحميل كسول للصور والمكوِّنات
- تقليل حجم حزمة JavaScript
- استخدام تقسيم الكود
- تحسين الخطوط والموارد

### 5. إمكانية الوصول
- اختبر مع قارئات الشاشة
- تأكد من عمل التنقل بلوحة المفاتيح
- حافظ على تباين كافٍ للألوان
- قدِّم بدائل نصية للصور
