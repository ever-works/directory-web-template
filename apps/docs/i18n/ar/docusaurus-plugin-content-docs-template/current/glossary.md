# مسرد المصطلحات

المصطلحات والمفاهيم الرئيسية المستخدمة في جميع أنحاء توثيق Directory Web Template.

## مفاهيم النطاق الأساسية

### الدليل

مجموعة من القوائم المنظمة (العناصر) حول موضوع أو مجال محدد. الدليل هو الكيان الأعلى مستوى. أمثلة: "دليل أدوات SaaS"، "دليل موارد المطورين"، أو "دليل الأعمال المحلية".

### العنصر

مدخل أو قائمة منفردة ضمن دليل. يمثل العنصر كياناً واحداً يتم تصنيفه (أداة، أو عمل، أو مورد، أو خدمة). تحتوي العناصر على حقول مهيكلة (الاسم، الوصف، الرابط، الشعار)، وتنتمي إلى فئات، ويمكن وسمها.

### الفئة

تصنيف هيكلي يُستخدم لتنظيم العناصر. تُشكّل الفئات بنية شجرية (علاقات الأصل/الفرع) وتوفر آلية التنقل والتصفية الأساسية.

### الوسم

ملصق مسطح وغير هيكلي يُرفق بالعناصر للتصنيف المتقاطع. تُستخدم الوسوم للتصفية الثانوية والاكتشاف. يمكن أن يحتوي عنصر على وسوم متعددة مثل "open-source" و"freemium" و"API-available".

### المجموعة

تجميع محكم للعناصر، مستقل عن الفئات أو الوسوم. المجموعات هي مجموعات محددة من قبل المستخدم أو مُعدَّة بشكل تحريري، مثل "أفضل 10 اختيارات" أو "جديد هذا الشهر".

### التصنيف

نظام التصنيف الشامل للدليل، الذي يشمل الفئات والوسوم وأي هياكل تنظيمية أخرى.

### Slug

معرّف صديق للروابط وقابل للقراءة من قبل الإنسان، مشتق من اسم كيان ما. يُستخدم Slug في الروابط بدلاً من المعرفات الرقمية. على سبيل المثال، "Visual Studio Code" يصبح `visual-studio-code`.

## أنماط المعمارية

### المستودع

فئة في طبقة الوصول إلى البيانات تُغلّف استعلامات قاعدة البيانات والتعديلات عليها لكيان معين. تجرّد المستودعات Drizzle ORM وتوفر واجهة نظيفة للخدمات. موقعها في `lib/repositories/`.

### الخدمة

فئة في طبقة منطق الأعمال تنسّق العمليات عبر المستودعات وواجهات API الخارجية والخدمات الأخرى. تحتوي الخدمات على منطق التطبيق الأساسي ويتم استدعاؤها من قبل معالجات مسارات API. موقعها في `lib/services/`.

### Webhook

استدعاء خلفي HTTP يُشغَّل بواسطة حدث. يستخدم القالب webhooks لإشعارات موفري الدفع (Stripe وLemonSqueezy وPolar) وتحديثات حالة النشر. تتحقق نقاط نهاية webhook من صحة الطلبات الواردة باستخدام التواقيع أو الأسرار المشتركة.

## إدارة المحتوى

### نظام إدارة المحتوى المستند إلى Git

نهج إدارة المحتوى المستخدم في القالب. يُخزَّن بيانات الدليل (العناصر، الفئات، البيانات الوصفية) كملفات مهيكلة (YAML وMarkdown) في مستودع Git. يستنسخ القالب هذا المستودع في وقت البناء ويقرأ المحتوى من نظام الملفات المحلي. يتم إجراء التغييرات عبر commits وpull requests.

### Community PR

طلب سحب يُرسله عضو مجتمع لإضافة أو تحديث عناصر في مستودع CMS المستند إلى Git الخاص بدليل. تخضع Community PRs لعملية مراجعة قبل الدمج.

## قاعدة البيانات

### Drizzle ORM

نظام ORM خفيف الوزن وذو أولوية TypeScript يستخدمه القالب. يوفر Drizzle منشئ استعلامات شبيه بـ SQL مع أمان كامل للأنواع. تُكتب تعريفات المخطط كرمز TypeScript، ويتم توليد عمليات الترحيل كملفات SQL عادية عبر Drizzle Kit.

### الترحيل

تغيير مُعادل للإصدار في مخطط قاعدة البيانات. يتم توليد عمليات الترحيل باستخدام `pnpm db:generate` وتطبيقها باستخدام `pnpm db:migrate`. تُخزَّن ملفات الترحيل في `lib/db/migrations/`.

## المصادقة

### NextAuth.js

مكتبة المصادقة (v5) التي يستخدمها القالب. توفر دعم OAuth لموفرين متعددين (Google وGitHub وFacebook وTwitter وMicrosoft) مع إدارة الجلسات ورموز JWT.

### Supabase Auth

واجهة خلفية بديلة للمصادقة مدعومة من القالب. يوفر Supabase Auth مصادقة البريد الإلكتروني/كلمة المرور والروابط السحرية والمصادقة الاجتماعية OAuth من خلال خدمة Supabase المُدارة.

## المدفوعات

### الاشتراك

ترتيب دفع متكرر يُدار من خلال أحد موفري الدفع المدعومين (Stripe أو LemonSqueezy أو Polar). يتعامل القالب مع إنشاء الاشتراكات وإدارتها ومعالجة webhooks.

## النشر

### Vercel

منصة النشر الأساسية للقالب. يوفر Vercel نشراً دون تهيئة لتطبيقات Next.js، بما في ذلك عمليات نشر المعاينة التلقائية والدوال الحدية وتوزيع CDN.

### Docker

طريقة نشر بديلة. يمكن احتواء القالب ونشره في أي بيئة استضافة متوافقة مع Docker.
