# التدويل (i18n)

يستخدم Ever Works Template مكتبة `next-intl` للتدويل، وتدعم 21 لغةً مع توجيه قائم على عنوان URL، والكشف التلقائي من المتصفح، ورسائل احتياطية، وتخطيطات من اليمين إلى اليسار، وعلامات hreflang لتحسين محركات البحث. توثّق هذه الصفحة بنية i18n وإعداداتها وأنماط الاستخدام.

## البنية المعمارية

| الملف | الغرض |
|------|---------|
| `i18n/routing.ts` | إعداد التوجيه (اللغات المدعومة، استراتيجية البادئة) |
| `i18n/request.ts` | الكشف عن اللغة وتحميل الرسائل لكل طلب |
| `i18n/navigation.ts` | Link وredirect وusePathname وuseRouter مع دعم اللغة |
| `lib/constants.ts` | قائمة اللغات، اللغة الافتراضية، تعريفات RTL |
| `lib/seo/hreflang.ts` | توليد علامات hreflang لمحركات البحث |
| `components/language-switcher.tsx` | مبدّل اللغة في الواجهة |
| `messages/*.json` | ملفات الترجمة (واحد لكل لغة) |
| `app/[locale]/layout.tsx` | التخطيط الجذري مع NextIntlClientProvider |

## اللغات المدعومة

يُعرِّف القالب 21 لغةً في `lib/constants.ts`:

```typescript
export const DEFAULT_LOCALE = 'en';
export const LOCALES = [
  'en', 'fr', 'es', 'de', 'zh', 'ar', 'he',
  'ru', 'uk', 'pt', 'it', 'ja', 'ko', 'nl',
  'pl', 'tr', 'vi', 'th', 'hi', 'id', 'bg'
] as const;

export type Locale = (typeof LOCALES)[number];

/** اللغات التي تستخدم اتجاه النص من اليمين إلى اليسار */
export const RTL_LOCALES: readonly Locale[] = ['ar', 'he'] as const;
```

| الرمز | اللغة | الاتجاه |
|------|----------|-----------|
| `en` | الإنجليزية | LTR |
| `fr` | الفرنسية | LTR |
| `es` | الإسبانية | LTR |
| `de` | الألمانية | LTR |
| `zh` | الصينية المبسطة | LTR |
| `ar` | العربية | **RTL** |
| `he` | العبرية | **RTL** |
| `it` | الإيطالية | LTR |
| `pt` | البرتغالية | LTR |
| `nl` | الهولندية | LTR |
| `pl` | البولندية | LTR |
| `ru` | الروسية | LTR |

## إعداد التوجيه

يُعرِّف وحدة التوجيه في `i18n/routing.ts` تعيين اللغات إلى عناوين URL:

```typescript
import { defineRouting } from "next-intl/routing";
import { DEFAULT_LOCALE, LOCALES } from "@/lib/constants";

export const routing = defineRouting({
  locales: LOCALES,
  defaultLocale: DEFAULT_LOCALE,
  localeDetection: true,
  localePrefix: "as-needed",
});
```

### استراتيجية البادئة

تُبقي استراتيجية `"as-needed"` عناوين URL نظيفةً للغة الافتراضية:

| عنوان URL | اللغة المكتشفة | الشرح |
|-----|-----------------|-------------|
| `/about` | `en` (افتراضي) | لا حاجة لبادئة للغة الافتراضية |
| `/fr/about` | `fr` | البادئة مطلوبة للغات الأخرى |
| `/ar/about` | `ar` | البادئة مطلوبة للغات الأخرى |

### الكشف عن اللغة

عند تفعيل `localeDetection: true`، يتحقق النظام تلقائياً من:

1. ملف تعريف ارتباط اللغة (يُعيّنه مكوّن LanguageSwitcher)
2. رأس HTTP `Accept-Language` من المتصفح
3. الرجوع إلى `DEFAULT_LOCALE` (`'en'`) إن لم يُعثر على تطابق

## تحميل الرسائل والاحتياطية

تُنفَّذ وحدة `i18n/request.ts` في كل طلب خادم:

```typescript
export default getRequestConfig(async ({ requestLocale }) => {
  let locale = await requestLocale;

  if (!locale || !routing.locales.includes(locale as any)) {
    locale = routing.defaultLocale;
  }

  const userMessages = (await import(`../messages/${locale}.json`)).default;
  const defaultMessages = (await import(`../messages/en.json`)).default;
  const messages = deepmerge(defaultMessages, userMessages);

  return { locale, messages };
});
```

تضمن استراتيجية `deepmerge` أن المفاتيح غير المترجمة تعرض النص الإنجليزي.

## ملفات الترجمة

تُخزَّن رسائل الترجمة كملفات JSON في دليل `messages/`:

```
messages/
  en.json    # الإنجليزية (المصدر، جميع المفاتيح موجودة)
  fr.json    # الفرنسية
  ar.json    # العربية
  ... (21 ملفاً إجمالاً)
```

### استخدام الترجمات في المكوّنات

تستخدم **مكوّنات جانب العميل** خطاف `useTranslations`:

```tsx
import { useTranslations } from "next-intl";

function LoginButton() {
  const t = useTranslations("auth");
  return <button>{t("SIGN_IN")}</button>;
}
```

تستخدم **مكوّنات الخادم** `getTranslations` غير المتزامنة:

```tsx
import { getTranslations } from "next-intl/server";

export default async function Page() {
  const t = await getTranslations("common");
  return <h1>{t("DISCOVER")}</h1>;
}
```

## إضافة لغة جديدة

1. أضف رمز اللغة إلى مصفوفة `LOCALES` في `lib/constants.ts`
2. إن كانت اللغة RTL — أضفها إلى `RTL_LOCALES`
3. أنشئ `messages/<الرمز>.json` بالمفاتيح المترجمة
4. أضف ملف SVG للعلم في `public/flags/<الرمز>.svg`
5. أضف إدخال اللغة إلى `languageMap` في `components/language-switcher.tsx`
6. أضف تعيين hreflang في `lib/seo/hreflang.ts`

## أفضل الممارسات

1. **استخدام `Link` من `@/i18n/navigation`** بدلاً من `next/link`
2. **إضافة المفاتيح الجديدة أولاً إلى `en.json`** إذ هي لغة المصدر
3. **عدم ترميز النصوص الإنجليزية بشكل صريح** في المكوّنات
4. **الاعتماد على احتياطي `deepmerge`** — الملفات المترجمة جزئياً تعمل بشكل صحيح
5. **اختبار تخطيطات RTL** بالتبديل إلى العربية أو العبرية
