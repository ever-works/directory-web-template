# خارطة الطريق

تستعرض هذه الصفحة اتجاه التطوير الحالي لـ Directory Web Template وكيف يمكن للمجتمع المشاركة في تشكيل مستقبله.

## رؤية المنتج

يهدف Directory Web Template إلى أن يكون الحل الأكثر شمولاً مفتوح المصدر لبناء مواقع الدليل الاحترافية. تشمل الرؤية طويلة المدى:

- **مواقع دليل جاهزة للإنتاج** تكون جميلة وعالية الأداء وقابلة للتخصيص بالكامل
- **إدارة محتوى سهلة** من خلال نظام إدارة المحتوى المستند إلى Git مع توليد المحتوى الاختياري بالذكاء الاصطناعي عبر [منصة Ever Works](https://docs.ever.works)
- **مدفوعات ومصادقة قابلة للتوسيع** مع دعم مزودين متعددين من البداية
- **تدويل من الدرجة الأولى** مع دعم RTL الكامل وتغطية لغوية متنامية

## مجالات التطوير النشط

### الأداء ومؤشرات Core Web Vitals

- تحسين أكبر عنصر محتوى مرئي (LCP) لصفحات قوائم العناصر والتفاصيل
- تقليل حجم حزمة JavaScript من خلال تقسيم أفضل للكود وتقليص الأجزاء غير المستخدمة
- تحسين خط أنابيب تحسين الصور للقطات الشاشة وشعارات عناصر الدليل
- تطبيق التصيير المسبق الجزئي لتحميل أسرع للصفحات الأولية

### تحسينات الميزات

- إضافة المزيد من إمكانات التصفية والبحث (البحث متعدد الأوجه، المرشحات المتقدمة)
- تطبيق ميزات المحتوى الذي ينشئه المستخدمون (المراجعات والتقييمات والتعليقات)
- إضافة المزيد من تكاملات مزودي الدفع وميزات إدارة الاشتراكات
- توسيع نظام التعقيب بمزيد من الثيمات المدمجة وتخصيص أسهل

### تجربة المطورين

- تحسين إعداد بيئة التطوير المحلية بتوثيق أفضل ورسائل خطأ أوضح
- إضافة تغطية اختبارات E2E أكثر شمولاً مع Playwright
- إنشاء قوالب بداية لأنواع الدلائل الشائعة (SaaS، الأعمال المحلية، الموارد)
- تحسين أمان نوع TypeScript عبر قاعدة الكود

### التدويل

- إضافة المزيد من ترجمات اللغات المدمجة
- تحسين دعم تخطيط RTL للعربية والعبرية
- دعم تهيئة اللغة لكل دليل على حدة
- إضافة سير عمل للترجمة الآلية

### التوثيق

- توسيع وثائق مرجع API بمزيد من الأمثلة
- إضافة دروس فيديو للمهام الشائعة
- إنشاء سجلات قرارات الهندسة المعمارية (ADR) لقرارات التصميم الكبرى
- بناء أدلة تفاعلية وبيئات تجريبية

## كيفية اقتراح الميزات

### GitHub Issues

الطريقة الأساسية لاقتراح الميزات هي من خلال GitHub Issues على [github.com/ever-works/directory-web-template/issues](https://github.com/ever-works/directory-web-template/issues).

عند إنشاء طلب ميزة:

1. **تحقق من الـ issues الموجودة** أولاً لتجنب التكرار.
2. **صِف المشكلة** التي تحاول حلها، وليس فقط الحل الذي تريده.
3. **قدِّم السياق** حول حالة الاستخدام ونوع الدليل والنطاق.
4. **أدرج أمثلة** (نماذج محاكاة، مخططات API، أمثلة تهيئة).

### GitHub Discussions

للأفكار الأوسع التي تحتاج مدخلات من المجتمع: [github.com/ever-works/directory-web-template/discussions](https://github.com/ever-works/directory-web-template/discussions)

### Discord

انضم إلى [Discord الخاص بـ Ever Works](https://discord.gg/ever) للمحادثات الفورية حول الميزات واتجاه المشروع.

## كيف تُحدَّد الأولويات

| العامل                    | الوزن  | الوصف                                                               |
| ------------------------- | ------ | ------------------------------------------------------------------- |
| **طلب المستخدمين**        | عالٍ   | عدد الطلبات والتصويتات المؤيدة واهتمام المجتمع                     |
| **التوافق الاستراتيجي**   | عالٍ   | مدى توافق الميزة مع رؤية المنتج                                     |
| **جهد التنفيذ**           | متوسط  | التعقيد والاستثمار الزمني وعبء الصيانة                             |
| **مخاطر التغييرات الجذرية** | متوسط | احتمالية الإخلال بالمستخدمين الحاليين                            |
| **توافر المساهمين**       | متوسط  | ما إذا كان المشرفون أو أعضاء المجتمع قادرين على تولي المهمة        |

### مستويات الأولوية

- **P0 (حرج):** ثغرات الأمان أو أخطاء فقدان البيانات أو المشكلات الإعاقة. تُعالج فوراً.
- **P1 (عالٍ):** الميزات أو الإصلاحات التي يُعمل عليها بنشاط للإصدار التالي.
- **P2 (متوسط):** الميزات المعتمدة المخططة لكن لم تُجدوَل بعد.
- **P3 (منخفض):** تحسينات يُستحسن وجودها. مرشحات جيدة لمساهمات المجتمع.

## المساهمة في خارطة الطريق

1. **إرسال طلبات ميزات مكتوبة بشكل جيد** مع بيانات واضحة للمشاكل وحالات الاستخدام.
2. **المساهمة بالكود.** طلبات السحب هي أسرع طريق من الفكرة إلى الواقع. انظر [دليل المساهمة](/contributing).
3. **المشاركة في النقاشات.** تقديم ملاحظات على المقترحات ومشاركة تجربتك.
4. **الإبلاغ عن الأخطاء.** تساعد تقارير الأخطاء الموثوقة على ترتيب أولويات الإصلاحات وتحسين الاستقرار.

## وتيرة الإصدار

تصدر الإصدارات عندما تكون مجموعة ذات معنى من الميزات والإصلاحات جاهزة:

- **إصدارات Patch** (إصلاحات الأخطاء) تُنشر حسب الحاجة، وغالباً ما تكون أسبوعية خلال التطوير النشط.
- **إصدارات Minor** (ميزات جديدة) تُنشر تقريباً شهرياً.
- **إصدارات Major** (تغييرات جذرية) نادرة وتأتي مصحوبة بأدلة الترحيل.

انظر صفحة [سجل التغييرات](/changelog) للتفاصيل.
