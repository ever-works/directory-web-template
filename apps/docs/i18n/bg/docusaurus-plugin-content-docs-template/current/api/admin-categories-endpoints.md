# Крайни точки на API за администраторски категории

API за администраторски категории предоставя пълни CRUD операции за управление на категории съдържание, включително пренареждане и базирана на Git синхронизация с отдалечено хранилище на данни. Всички крайни точки изискват администраторско удостоверяване чрез базирано на сесия удостоверяване.

## Резюме на маршрута

|Метод|Пътека|авт|Описание|
|--------|------|------|-------------|
|`GET`|`/api/admin/categories`|Админ|Списък на категории (страничен)|
|`POST`|`/api/admin/categories`|Админ|Създайте нова категория|
|`GET`|`/api/admin/categories/all`|Админ|Вземете всички категории (от кеша на съдържанието)|
|`GET`|`/api/admin/categories/{id}`|Админ|Вземете една категория по ID|
|`PUT`|`/api/admin/categories/{id}`|Админ|Актуализирайте категория|
|`DELETE`|`/api/admin/categories/{id}`|Админ|Плавно или трудно изтриване на категория|
|`PUT`|`/api/admin/categories/reorder`|Админ|Пренареждане на категориите по ID масив|
|`GET`|`/api/admin/categories/git`|Админ|Вземете статус и категории на Git repo|
|`POST`|`/api/admin/categories/git`|Админ|Създайте категория чрез Git commit|

## Удостоверяване

Всички крайни точки за управление на категории проверяват за активна сесия с администраторски привилегии:

```typescript
const session = await auth();
if (!session?.user?.isAdmin) {
  return NextResponse.json(
    { success: false, error: "Unauthorized. Admin access required." },
    { status: 401 }
  );
}
```

## Крайни точки

### ВЗЕМЕТЕ `/api/admin/categories`

Връща пагиниран списък от категории с незадължително филтриране и сортиране.

**Параметри на заявката:**

|Параметър|Тип|По подразбиране|Описание|
|-----------|------|---------|-------------|
|`page`|цяло число| `1` |Номер на страницата (минимум: 1)|
|`limit`|цяло число| `10` |Елементи на страница (1--100)|
|`includeInactive`|низ|`"false"`|Включете неактивни категории|
|`sortBy`|низ|`"name"`|Поле за сортиране: `"name"` или `"id"`|
|`sortOrder`|низ|`"asc"`|Посока на сортиране: `"asc"` или `"desc"`|

**Отговор (200):**

```json
{
  "success": true,
  "categories": [
    {
      "id": "productivity",
      "name": "Productivity",
      "isActive": true,
      "itemCount": 15,
      "createdAt": "2024-01-15T10:30:00.000Z",
      "updatedAt": "2024-01-15T10:30:00.000Z"
    }
  ],
  "total": 25,
  "page": 1,
  "limit": 10,
  "totalPages": 3
}
```

### ПУБЛИКУВАНЕ `/api/admin/categories`

Създава нова категория. Полето `id` не е задължително и ще бъде автоматично генерирано от името, ако не е предоставено. Анулира кешовете на съдържанието при успех.

**Тяло на заявката:**

```json
{
  "id": "productivity",
  "name": "Productivity"
}
```

|Поле|Тип|Задължително|Описание|
|-------|------|----------|-------------|
|`id`|низ|не|Удобен за URL охлюв (`^[a-z0-9-]+$`). Генерира се автоматично, ако е пропуснато.|
|`name`|низ|да|Показвано име (2--100 знака)|

**Отговор (201):**

```json
{
  "success": true,
  "category": {
    "id": "productivity",
    "name": "Productivity",
    "isActive": true,
    "itemCount": 0,
    "createdAt": "2024-01-20T15:30:00.000Z",
    "updatedAt": "2024-01-20T15:30:00.000Z"
  },
  "message": "Category created successfully"
}
```

### ВЗЕМЕТЕ `/api/admin/categories/all`

Връща всички категории от кеша на съдържанието за даден локал. Полезно за администраторски падащи менюта и селектори.

**Параметри на заявката:**

|Параметър|Тип|По подразбиране|Описание|
|-----------|------|---------|-------------|
|`locale`|низ|`"en"`|Локален код за извличане на съдържание|

**Отговор (200):**

```json
{
  "success": true,
  "data": [
    { "id": "productivity", "name": "Productivity", "isActive": true, "itemCount": 15 }
  ]
}
```

### ВЗЕМЕТЕ `/api/admin/categories/{id}`

Извлича една категория по нейния уникален идентификатор.

**Отговор (200):**

```json
{
  "success": true,
  "data": {
    "id": "productivity",
    "name": "Productivity",
    "isActive": true,
    "itemCount": 15,
    "createdAt": "2024-01-15T10:30:00.000Z",
    "updatedAt": "2024-01-15T10:30:00.000Z"
  }
}
```

### ПОСТАВЕТЕ `/api/admin/categories/{id}`

Актуализира името на съществуваща категория. Анулира кешовете на съдържанието при успех.

**Тяло на заявката:**

```json
{ "name": "Productivity Tools" }
```

**Отговор (200):**

```json
{
  "success": true,
  "data": { "id": "productivity", "name": "Productivity Tools", "isActive": true },
  "message": "Category updated successfully"
}
```

### ИЗТРИВАНЕ `/api/admin/categories/{id}`

Изтрива категория. По подразбиране извършва меко изтриване (деактивиране). Използвайте параметъра на заявката `hard=true` за постоянно изтриване. Анулира кешовете на съдържанието при успех.

**Параметри на заявката:**

|Параметър|Тип|По подразбиране|Описание|
|-----------|------|---------|-------------|
|`hard`|низ|`"false"`|Задайте на `"true"` за постоянно изтриване|

**Отговор (200):**

```json
{
  "success": true,
  "message": "Category deactivated successfully"
}
```

### ПОСТАВЕТЕ `/api/admin/categories/reorder`

Пренарежда категориите въз основа на масив от идентификатори на категории. Позицията на всеки идентификатор в масива определя новия му ред на показване.

**Тяло на заявката:**

```json
{ "categoryIds": ["productivity", "design", "development", "marketing"] }
```

**Правила за валидиране:**
- `categoryIds` трябва да е непразен масив
- Всички стойности трябва да са низове

**Отговор (200):**

```json
{
  "success": true,
  "message": "Categories reordered successfully"
}
```

### ВЗЕМЕТЕ `/api/admin/categories/git`

Извлича състоянието и категориите на хранилището на Git от конфигурираното хранилище на данни на GitHub. Изисква променливи на средата `DATA_REPOSITORY` и `GITHUB_TOKEN`.

**Отговор (200):**

```json
{
  "success": true,
  "status": {
    "repository": "ever-co/awesome-time-tracking-data",
    "branch": "main",
    "lastCommit": "abc123def456",
    "lastCommitDate": "2024-01-20T10:30:00.000Z",
    "isUpToDate": true
  },
  "categories": [],
  "message": "Git repository status retrieved successfully"
}
```

### ПУБЛИКУВАНЕ `/api/admin/categories/git`

Създава нова категория и я ангажира директно в хранилището на данни GitHub. Изисква променливи на средата `DATA_REPOSITORY` и `GH_TOKEN`.

**Тяло на заявката:**

```json
{ "id": "productivity", "name": "Productivity" }
```

Както `id`, така и `name` са необходими за създаване, базирано на Git.

**Отговор (200):**

```json
{
  "success": true,
  "category": { "id": "productivity", "name": "Productivity" },
  "message": "Category created and committed to Git repository"
}
```

## Кодове за грешки

|Статус|Грешка|причина|
|--------|-------|-------|
| `400` |Невалидни параметри за пагиниране|Страница < 1 или ограничение извън 1--100|
| `400` |Името на категорията е задължително|Липсва `name` в заявката за създаване|
| `400` |categoryIds трябва да е масив|Невалиден полезен товар за пренареждане|
| `401` |Неразрешено. Изисква се администраторски достъп.|Липсваща или неадминистративна сесия|
| `404` |Категорията не е намерена|Невалиден идентификатор на категория|
| `409` |Категория с това име вече съществува|Дублирано име при създаване/актуализация|
| `500` |DATA_REPOSITORY не е конфигуриран|Липсва env var за Git крайни точки|
| `500` |Токенът GitHub не е конфигуриран|Липсва `GITHUB_TOKEN` или `GH_TOKEN`|

## Кеш невалидност

Всички операции по писане (създаване, актуализиране, изтриване, пренареждане) се обаждат на `invalidateContentCaches()`, за да се гарантира, че промените са незабавно видими в приложението.
