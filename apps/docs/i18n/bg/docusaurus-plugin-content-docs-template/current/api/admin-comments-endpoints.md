# Крайни точки на API за коментари на администратора

API за администраторски коментари предоставя възможности за модериране за управление на потребителски коментари. Администраторите могат да изброяват, преглеждат, актуализират и безпроблемно изтриват коментари. Всички крайни точки използват Node.js runtime и изискват наличност на база данни. Проверките за удостоверяване използват `403 Forbidden` за потребители, които не са администратори.

## Резюме на маршрута

|Метод|Пътека|авт|Описание|
|--------|------|------|-------------|
|`GET`|`/api/admin/comments`|Админ|Списък с коментари (страничен, с възможност за търсене)|
|`GET`|`/api/admin/comments/{id}`|Админ|Вземете единичен коментар с потребителска информация|
|`PUT`|`/api/admin/comments/{id}`|Админ|Актуализирайте съдържанието на коментара|
|`DELETE`|`/api/admin/comments/{id}`|Админ|Плавно изтриване на коментар|

## Удостоверяване

Крайните точки за модериране на коментари проверяват статуса на администратор и връщат `403 Forbidden` (не `401`) за потребители без администратор:

```typescript
const session = await auth();
if (!session?.user?.isAdmin) {
  return NextResponse.json(
    { success: false, error: "Forbidden" },
    { status: 403 }
  );
}
```

## Изискване за база данни

Крайните точки на коментарите проверяват наличността на базата данни, преди да обработят заявките:

```typescript
const dbCheck = checkDatabaseAvailability();
if (dbCheck) return dbCheck;
```

Ако базата данни не е конфигурирана, се връща подходящ отговор за грешка преди всяка проверка за удостоверяване.

## Крайни точки

### ВЗЕМЕТЕ `/api/admin/comments`

Връща списък с коментари със страничен списък със свързана потребителска информация. Поддържа пълнотекстово търсене в съдържание на коментари, потребителски имена и потребителски имейли. Връщат се само неизтрити коментари.

**Параметри на заявката:**

|Параметър|Тип|По подразбиране|Описание|
|-----------|------|---------|-------------|
|`page`|цяло число| `1` |Номер на страница за пагинация|
|`limit`|цяло число| `10` |Коментари на страница (1--100)|
|`search`|низ| `""` |Търсете в съдържание, потребителско име или имейл|

**Поведение при търсене:**

Заявката за търсене се съпоставя без значение за главни и малки букви (с помощта на `ILIKE`) срещу:
- Съдържание на коментара
- Екранно име на потребителя
- Имейл адрес на потребителя

Специалните символи `%`, `_` и `\` се екранират, за да се предотврати инжектирането на SQL модел.

**Отговор (200):**

```json
{
  "success": true,
  "data": {
    "comments": [
      {
        "id": "comment_123abc",
        "content": "This is a great product! Highly recommended.",
        "rating": 5,
        "userId": "user_456def",
        "itemId": "item_789ghi",
        "createdAt": "2024-01-20T10:30:00.000Z",
        "updatedAt": "2024-01-20T10:30:00.000Z",
        "user": {
          "id": "user_456def",
          "name": "John Doe",
          "email": "john.doe@example.com",
          "image": "https://example.com/avatar.jpg"
        }
      }
    ],
    "pagination": {
      "total": 156,
      "page": 1,
      "limit": 10,
      "totalPages": 16
    }
  }
}
```

### ВЗЕМЕТЕ `/api/admin/comments/{id}`

Извлича конкретен коментар по неговия идентификатор с пълна информация за потребителския профил. Включва ляво съединение към таблицата `clientProfiles` за потребителски данни.

**Параметри на пътя:**

|Параметър|Тип|Описание|
|-----------|------|-------------|
|`id`|низ|Уникален идентификатор на коментара|

**Отговор (200):**

```json
{
  "success": true,
  "data": {
    "id": "comment_123abc",
    "content": "This is a great product! Highly recommended.",
    "rating": 5,
    "userId": "user_456def",
    "itemId": "item_789ghi",
    "createdAt": "2024-01-20T10:30:00.000Z",
    "updatedAt": "2024-01-20T14:45:00.000Z",
    "user": {
      "id": "user_456def",
      "name": "John Doe",
      "email": "john.doe@example.com",
      "image": "https://example.com/avatar.jpg"
    }
  }
}
```

**Резервен потребител:** Ако потребителският профил не бъде намерен (изтрит потребител), се връща обект на контейнер:

```json
{
  "user": {
    "id": "",
    "name": "Unknown User",
    "email": "",
    "image": null
  }
}
```

### ПОСТАВЕТЕ `/api/admin/comments/{id}`

Актуализира съдържанието на конкретен коментар. Може да се променя само полето `content`. Коментарът трябва да съществува и да не се изтрива безпроблемно.

**Параметри на пътя:**

|Параметър|Тип|Описание|
|-----------|------|-------------|
|`id`|низ|Уникален идентификатор на коментара|

**Тяло на заявката:**

```json
{
  "content": "This is an updated comment with more details."
}
```

|Поле|Тип|Задължително|Описание|
|-------|------|----------|-------------|
|`content`|низ|да|Нов текст на коментар (не трябва да е празен след изрязване)|

**Правила за валидиране:**
- `content` е задължителен и не трябва да е празен или само с интервали
- Целевият коментар трябва да съществува и не трябва да има клеймо за време `deletedAt`

**Отговор (200):**

```json
{
  "success": true,
  "data": {
    "id": "comment_123abc",
    "content": "This is an updated comment with more details.",
    "rating": 5,
    "userId": "user_456def",
    "itemId": "item_789ghi",
    "createdAt": "2024-01-20T10:30:00.000Z",
    "updatedAt": "2024-01-20T16:15:00.000Z",
    "user": { "id": "user_456def", "name": "John Doe", "email": "john.doe@example.com", "image": null }
  },
  "message": "Comment updated successfully"
}
```

### ИЗТРИВАНЕ `/api/admin/comments/{id}`

Извършва меко изтриване на коментар чрез задаване на клеймото за време `deletedAt`. Коментарът трябва да съществува и вече да не е изтрит. Плажно изтритите коментари се изключват от всички заявки в списъка.

**Параметри на пътя:**

|Параметър|Тип|Описание|
|-----------|------|-------------|
|`id`|низ|Уникален идентификатор на коментара|

**Отговор (200):**

```json
{
  "success": true,
  "message": "Comment deleted successfully"
}
```

## Модел на данни за коментар

|Поле|Тип|Nullable|Описание|
|-------|------|----------|-------------|
|`id`|низ|не|Уникален идентификатор на коментар|
|`content`|низ|не|Коментирайте текстовото съдържание|
|`rating`|цяло число|да|Рейтингова стойност (1--5)|
|`userId`|низ|не|ID на потребителя на автора|
|`itemId`|низ|не|Идент. № на асоцииран артикул|
|`createdAt`|дата и час|да|Дата на създаване|
|`updatedAt`|дата и час|да|Дата на последна актуализация|
|`deletedAt`|дата и час|да|Времево клеймо за меко изтриване (нула, ако е активно)|

## Кодове за грешки

|Статус|Грешка|причина|
|--------|-------|-------|
| `400` |Изисква се съдържание|Празно или липсващо съдържание при актуализация|
| `403` |Забранено|Потребител, който не е администратор, се опитва да получи достъп|
| `404` |Коментарът не е намерен|Невалиден идентификатор или вече е изтрит плавно|
| `500` |Вътрешна грешка на сървъра|Повреда в база данни или сървър|

## Бележки за изпълнение

- Коментарите използват **меко изтриване** -- полето `deletedAt` е зададено, вместо да премахва реда. Това запазва целостта на данните и позволява потенциално възстановяване.
- Всички заявки за списък се филтрират с `isNull(comments.deletedAt)`, за да се изключат изтритите коментари.
- Потребителските данни се извличат чрез `LEFT JOIN` на `clientProfiles`, което гарантира, че коментарите от изтритите потребители все още могат да бъдат извлечени.
- `runtime` е настроен на `"nodejs"` за тези маршрути (не Edge).
