# Крайни точки на API за администратор

Административният API съдържа приблизително 60 манипулатори на маршрути в 19 групи ресурси. Всички администраторски крайни точки са защитени от `withAdminAuth` мидълуер, който проверява както удостоверяването, така и присвояването на ролята на администратор чрез заявка към база данни.

## Удостоверяване

Всяка администраторска крайна точка изисква:

1. Валидна JWT сесия (проверена чрез `auth()`)
2. Роля на администратор в таблицата `user_roles` (проверена чрез `isAdmin()` от `lib/db/roles.ts`)

Неавтентифицираните заявки получават отговор `401`. Удостоверените, но неадминистративни заявки получават отговор `403`.

## Групи ресурси

### Категории (`/api/admin/categories`)

Управлявайте категории съдържание с постоянство, базирано на Git.

|Метод|Пътека|Описание|
|--------|------|-------------|
|`GET`|`/api/admin/categories`|Избройте категориите със страници|
|`POST`|`/api/admin/categories`|Създайте нова категория|
|`GET`|`/api/admin/categories/all`|Вземете всички категории (без пагинация)|
|`POST`|`/api/admin/categories/git`|Синхронизиране на категории с Git хранилище|
|`POST`|`/api/admin/categories/reorder`|Пренаредете позициите на категориите|
|`GET`|`/api/admin/categories/[id]`|Вземете категория по ID|
|`PUT`|`/api/admin/categories/[id]`|Актуализиране на категория|
|`DELETE`|`/api/admin/categories/[id]`|Изтриване на категория|

## Често срещани модели

### Масови операции

Няколко ресурса поддържат групови операции чрез POST с масив от идентификатори:

```json
POST /api/admin/items/bulk
{
  "action": "approve",
  "ids": ["item-1", "item-2", "item-3"]
}
```

### Статистически крайни точки

Повечето групи ресурси включват `/stats` крайна точка, връщаща обобщени преброявания:

```json
GET /api/admin/items/stats
{
  "success": true,
  "data": {
    "total": 1250,
    "published": 980,
    "pending": 120,
    "rejected": 50,
    "draft": 100
  }
}
```

### История на одита

Елементите поддържат проследяване на историята на одита чрез крайната точка `/[id]/history`, записвайки кой е направил промени и кога.
