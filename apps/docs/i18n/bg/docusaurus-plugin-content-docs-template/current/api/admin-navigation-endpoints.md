# Крайни точки за навигация на администратора и индекс на местоположение

Тези администраторски крайни точки управляват персонализирани връзки за навигация на сайта и индекса за географско местоположение. Крайните точки за навигация позволяват конфигуриране на персонализирани връзки към горен и долен колонтитул, съхранени в `config.yml`. Крайните точки на индекса на местоположение управляват пространствения индекс, използван за географски анализи и функции на картата.

## Преглед

|Крайна точка|Метод|авт|Описание|
|---|---|---|---|
|`/api/admin/navigation`|ВЗЕМЕТЕ|Админ|Вземете персонализирана конфигурация за навигация|
|`/api/admin/navigation`|КРЕПКА|Админ|Актуализирайте персонализирани елементи за навигация|
|`/api/admin/location-index`|ВЗЕМЕТЕ|Админ|Вземете статистически данни за индекса на местоположението|
|`/api/admin/location-index`|ПУБЛИКУВАНЕ|Админ|Създайте отново или изчистете индекса на местоположението|

## Крайни точки на навигация

### Вземете конфигурация за навигация

```
GET /api/admin/navigation
```

Извлича елементите за навигация `custom_header` и `custom_footer` от файла `config.yml` на сайта. Връща празни масиви, ако не е конфигурирана персонализирана навигация.

**Удостоверяване:** Изисква се администратор (чрез `getCachedApiSession`)

**Успешен отговор (200):**

```json
{
  "custom_header": [
    {
      "label": "About",
      "path": "/about"
    },
    {
      "label": "Documentation",
      "path": "/pages/docs"
    }
  ],
  "custom_footer": [
    {
      "label": "GitHub",
      "path": "https://github.com/example"
    },
    {
      "label": "footer.PRIVACY_POLICY",
      "path": "/pages/privacy-policy"
    }
  ]
}
```

Всеки навигационен елемент има две полета:

|Поле|Тип|Описание|
|---|---|---|
|`label`|низ|Показване на текст (обикновен текст или i18n ключ за превод като `"footer.PRIVACY_POLICY"`)|
|`path`|низ|URL път (вътрешен маршрут, започващ с `/` или външен URL с `http://`/`https://`)|

|Статус|Състояние|
|---|---|
| 401 |Не е удостоверен като администратор|
| 500 |Неуспешно четене на конфигурацията|

**Източник:** `template/app/api/admin/navigation/route.ts`

### Актуализиране на конфигурацията за навигация

```
PATCH /api/admin/navigation
```

Актуализира персонализираните елементи за навигация в горен или долен колонтитул в `config.yml`. Валидира формата на пътя на всеки елемент, за да предотврати XSS атаки чрез опасни URL схеми.

**Удостоверяване:** Изисква се администратор

**Тяло на заявката:**

```json
{
  "type": "header",
  "items": [
    {
      "label": "About",
      "path": "/about"
    },
    {
      "label": "Blog",
      "path": "https://blog.example.com"
    }
  ]
}
```

|Поле|Тип|Задължително|Описание|
|---|---|---|---|
|`type`|низ|да|`"header"` или `"footer"`|
|`items`|масив|да|Масив от елементи за навигация|
|`items[].label`|низ|да|Непразен етикет на дисплея|
|`items[].path`|низ|да|Валиден URL път|

**Потвърждение на пътя:**

Функцията `isValidNavigationPath()` налага строги правила за формат на пътя:

|Формат на пътя|Разрешено|Пример|
|---|---|---|
|Вътрешни маршрути|да|`/about`, `/pages/docs`|
|HTTPS URL адреси|да|`https://example.com`|
|HTTP URL адреси|да|`http://example.com`|
|Свързани с протокол URL адреси|не|`//evil.com`|
|URL адреси на JavaScript|не|`javascript:alert(1)`|
|URL адреси на данни|не|`data:text/html,...`|
|Други схеми|не|`vbscript:`, `file:`|

**Успешен отговор (200):**

```json
{
  "success": true,
  "type": "header",
  "items": [
    {
      "label": "About",
      "path": "/about"
    }
  ]
}
```

**Отговори за грешка:**

|Статус|Състояние|
|---|---|
| 400 |`type` не е `"header"` или `"footer"`|
| 400 |`items` не е масив|
| 400 |В елемента липсват низови полета `label` или `path`|
| 400 |Невалиден формат на пътя (предотвратяване на XSS)|
| 401 |Не е удостоверен като администратор|
| 500 |Неуспешно записване на конфигурация|

Подайте празен масив `items`, за да изчистите цялата персонализирана навигация за посочения тип.

**Източник:** `template/app/api/admin/navigation/route.ts`

## Крайни точки на индекса на местоположението

### Вземете статистически данни за индекса на местоположението

```
GET /api/admin/location-index
```

Връща статистически данни за индекса на географско местоположение, включително общо индексирани елементи, брой градове и държави и метаданни за повторно изграждане. Използва услугата индекс на местоположението за извличане на данни.

**Удостоверяване:** Изисква се администратор (чрез `checkAdminAuth()`)

**Кеширане:** Деактивирано -- използва `force-dynamic`, `revalidate: 0` и `force-no-store`.

**Успешен отговор (200):**

```json
{
  "success": true,
  "data": {
    "totalIndexed": 450,
    "citiesCount": 85,
    "countriesCount": 25,
    "remoteCount": 30,
    "lastIndexedAt": "2024-01-20T10:30:00.000Z",
    "lastRebuildAt": "2024-01-15T08:00:00.000Z"
  }
}
```

|Статус|Състояние|
|---|---|
| 401 |Не е удостоверен като администратор|
| 500 |Вътрешна грешка на сървъра|

**Източник:** `template/app/api/admin/location-index/route.ts`

### Управление на индекса на местоположението

```
POST /api/admin/location-index
```

Извършва действия за управление на индекса на местоположението. Поддържа повторно изграждане на индекса от нулата или изчистване на всички записи.

**Удостоверяване:** Изисква се администратор

**Тяло на заявката:**

```json
{
  "action": "rebuild"
}
```

|Поле|Тип|Задължително|Описание|
|---|---|---|---|
|`action`|низ|да|`"rebuild"` или `"clear"`|

**Действия:**

|Действие|Описание|
|---|---|
|`rebuild`|Извлича всички елементи от хранилището и повторно индексира техните данни за местоположение. Връща статистика за възстановяване.|
|`clear`|Премахва всички записи от индекса на местоположението. Връща броя на изчистените записи.|

**Успешен отговор (200) -- Възстановяване:**

```json
{
  "success": true,
  "data": {
    "indexed": 420,
    "skipped": 80,
    "errors": 0
  }
}
```

**Успешен отговор (200) -- Ясно:**

```json
{
  "success": true,
  "data": {
    "cleared": 450
  }
}
```

**Отговори за грешка:**

|Статус|Състояние|
|---|---|
| 400 |Невалидно действие (не `"rebuild"` или `"clear"`)|
| 401 |Не е удостоверен като администратор|
| 500 |Вътрешна грешка на сървъра|

**Източник:** `template/app/api/admin/location-index/route.ts`

## Ключови подробности за внедряването

- **XSS Prevention:** Проверката на пътя за навигация отхвърля всички URL схеми с изключение на `/`, `http://` и `https://`. Това блокира `javascript:`, `data:`, `vbscript:` и свързани с протокол URL адреси (`//evil.com`), които могат да се използват за междусайтови скриптове.
- **Съхранение на конфигурацията:** Елементите за навигация се съхраняват в `config.yml` под ключове `custom_header` и `custom_footer`, поддържани чрез `configManager.updateNestedKey()`.
- **i18n етикети:** навигационните етикети могат да бъдат или обикновен текст, или ключове за превод (напр. `"footer.PRIVACY_POLICY"`). Предният интерфейс е отговорен за разрешаването на ключове за превод.
- **Възстановяване на индекс на местоположение:** Операцията по повторно изграждане зарежда всички елементи от `ItemRepository` и ги предава на услугата за индекс на местоположение. Това може да бъде ресурсоемка операция за големи масиви от данни.
- **Премахване на кеша:** Крайните точки на индекса на местоположение изрично деактивират цялото кеширане, за да се гарантира, че таблото за управление на администратора винаги показва текущи данни.
