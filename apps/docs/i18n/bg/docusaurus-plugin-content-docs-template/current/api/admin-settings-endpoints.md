# Крайни точки на администраторските настройки

API за администраторски настройки предоставя крайни точки за четене и промяна на конфигурацията на сайта, съхранена в `config.yml`. Това включва общи настройки и статус на доставчик на карти. Всички крайни точки изискват администраторско удостоверяване.

## Преглед

|Крайна точка|Метод|авт|Описание|
|---|---|---|---|
|`/api/admin/settings`|ВЗЕМЕТЕ|Админ|Вземете всички настройки|
|`/api/admin/settings`|КРЕПКА|Админ|Актуализирайте конкретна настройка|
|`/api/admin/settings/map-status`|ВЗЕМЕТЕ|Админ|Получете състояние на конфигурация на доставчика на карта|

## Вземете настройки

```
GET /api/admin/settings
```

Извлича пълния `settings` раздел от `config.yml` файла на сайта.

**Удостоверяване:** Изисква се администратор (чрез `getCachedApiSession`)

**Успешен отговор (200):**

```json
{
  "settings": {
    "theme": "light",
    "itemsPerPage": 20,
    "enableComments": true,
    "enableVoting": true,
    "enableNewsletter": true,
    "mapProvider": "mapbox",
    "defaultLanguage": "en"
  }
}
```

Точната форма на обекта `settings` зависи от конфигурацията на сайта `config.yml`. Крайната точка връща всичко, което се съхранява под ключа `settings`.

|Статус|Състояние|
|---|---|
| 401 |Не е удостоверен като администратор|
| 500 |Неуспешно четене на конфигурацията|

**Източник:** `template/app/api/admin/settings/route.ts`

## Актуализиране на настройка

```
PATCH /api/admin/settings
```

Актуализира една стойност на настройка в секцията `settings` на `config.yml`. Ключът автоматично се включва в пространството от имена `settings` (напр. предоставяне на ключ `"theme"` актуализации `settings.theme` в конфигурационния файл).

**Удостоверяване:** Изисква се администратор

**Тяло на заявката:**

```json
{
  "key": "itemsPerPage",
  "value": 30
}
```

|Поле|Тип|Задължително|Описание|
|---|---|---|---|
|`key`|низ|да|Ключът за настройка за актуализиране (спрямо `settings.`)|
|`value`|всякакви|да|Новата стойност за настройката|

**Успешен отговор (200):**

```json
{
  "success": true,
  "key": "itemsPerPage",
  "value": 30
}
```

Актуализацията се поддържа чрез `configManager.updateNestedKey()`, което променя основния `config.yml` файл. Ключът автоматично се префиксира с `settings.`, преди да бъде предаден на конфигурационния мениджър.

**Отговори за грешка:**

|Статус|Състояние|
|---|---|
| 400 |Липсва поле `key` в основния текст на заявката|
| 401 |Не е удостоверен като администратор|
| 500 |Неуспешно записване на конфигурация|

**Източник:** `template/app/api/admin/settings/route.ts`

## Състояние на доставчика на карта

### Получете статус на картата

```
GET /api/admin/settings/map-status
```

Връща състоянието на конфигурацията на поддържаните доставчици на карти, без да разкрива действителните API ключове. Това позволява на таблото за управление на администратора да показва кои доставчици на карти са налични за използване.

**Удостоверяване:** Изисква се администратор

**Успешен отговор (200):**

```json
{
  "status": {
    "mapbox": {
      "isConfigured": true,
      "isPreviewAvailable": true,
      "name": "Mapbox"
    },
    "google": {
      "isConfigured": false,
      "isPreviewAvailable": false,
      "name": "Google Maps"
    }
  }
}
```

|Поле|Тип|Описание|
|---|---|---|
|`mapbox.isConfigured`|булево|Дали `NEXT_PUBLIC_MAPBOX_ACCESS_TOKEN` е зададено|
|`mapbox.isPreviewAvailable`|булево|Същото като `isConfigured` -- визуализацията изисква токена|
|`google.isConfigured`|булево|Дали `NEXT_PUBLIC_GOOGLE_MAPS_API_KEY` е зададено|
|`google.isPreviewAvailable`|булево|Същото като `isConfigured`|

Крайната точка проверява за наличието на променливи на средата:

- `NEXT_PUBLIC_MAPBOX_ACCESS_TOKEN` за Mapbox
- `NEXT_PUBLIC_GOOGLE_MAPS_API_KEY` за Google Maps

В отговора не са изложени действителни ключови стойности.

|Статус|Състояние|
|---|---|
| 401 |Не е удостоверен като администратор|
| 500 |Вътрешна грешка на сървъра|

**Източник:** `template/app/api/admin/settings/map-status/route.ts`

## Конфигурационна архитектура

Системата за настройки е изградена върху `configManager` singleton от `lib/config-manager`:

- **Съхранение:** Настройките се съхраняват в YAML конфигурационен файл (`config.yml`)
- **Достъп:** Методът `configManager.getConfig()` чете пълната конфигурация
- **Актуализации:** Методът `configManager.updateNestedKey()` модифицира конкретни вложени ключове
- **Кеширане:** Сесиите се кешират чрез `getCachedApiSession()` за производителност

Всички актуализации на настройките са обхванати от пространството от имена `settings` в конфигурационния файл. Това предотвратява случайна промяна на конфигурационни ключове от най-високо ниво чрез API за настройки.
