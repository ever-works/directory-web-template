# Крайни точки на API за администраторски маркери

API за административни етикети предоставя пълни CRUD операции за управление на тагове за съдържание. Етикетите се използват за класифициране и филтриране на елементи в директорията. API поддържа пагиниран списък, създаване с активни/неактивни състояния, актуализации, изтриване и извличане в зависимост от локала от кеша на съдържанието. Всички операции по запис обезсилват кешовете на съдържанието за незабавна видимост.

## Резюме на маршрута

|Метод|Пътека|авт|Описание|
|--------|------|------|-------------|
|`GET`|`/api/admin/tags`|Админ|Списък с етикети (страничен)|
|`POST`|`/api/admin/tags`|Админ|Създайте нов етикет|
|`GET`|`/api/admin/tags/all`|Админ|Вземете всички тагове (от кеша на съдържанието)|
|`GET`|`/api/admin/tags/{id}`|Админ|Вземете един етикет по ID|
|`PUT`|`/api/admin/tags/{id}`|Админ|Актуализиране на етикет|
|`DELETE`|`/api/admin/tags/{id}`|Админ|Постоянно изтриване на етикет|

## Удостоверяване

Всички крайни точки за управление на маркери изискват администраторски привилегии:

```typescript
const session = await auth();
if (!session?.user?.isAdmin) {
  return NextResponse.json(
    { success: false, error: 'Unauthorized' },
    { status: 401 }
  );
}
```

## Крайни точки

### ВЗЕМЕТЕ `/api/admin/tags`

Връща списък със страници с всички тагове в системата. Параметрите за страниране се валидират с помощта на споделената помощна програма `validatePaginationParams`.

**Параметри на заявката:**

|Параметър|Тип|По подразбиране|Описание|
|-----------|------|---------|-------------|
|`page`|цяло число| `1` |Номер на страницата (минимум: 1)|
|`limit`|цяло число| `10` |Елементи на страница (1--100)|

**Отговор (200):**

```json
{
  "success": true,
  "data": {
    "tags": [
      {
        "id": "productivity",
        "name": "Productivity",
        "isActive": true,
        "itemCount": 156,
        "created_at": "2024-01-20T10:30:00.000Z",
        "updated_at": "2024-01-20T10:30:00.000Z"
      },
      {
        "id": "design",
        "name": "Design",
        "isActive": true,
        "itemCount": 89,
        "created_at": "2024-01-19T15:20:00.000Z",
        "updated_at": "2024-01-19T15:20:00.000Z"
      }
    ],
    "total": 45,
    "page": 1,
    "limit": 10,
    "totalPages": 5
  }
}
```

### ПУБЛИКУВАНЕ `/api/admin/tags`

Създава нов етикет с посочен ID, име и активен статус по избор. Анулира кешовете на съдържанието при успех.

**Тяло на заявката:**

```json
{
  "id": "artificial-intelligence",
  "name": "Artificial Intelligence",
  "isActive": true
}
```

|Поле|Тип|Задължително|Описание|
|-------|------|----------|-------------|
|`id`|низ|да|Удобен за URL идентификатор на охлюв|
|`name`|низ|да|Четимо от човека име на етикет (2--50 знака)|
|`isActive`|булево|не|Дали маркерът е активен (по подразбиране: `true`)|

**Правила за валидиране:**
- Необходими са както `id`, така и `name`
- Името на етикета трябва да е между 2 и 50 знака
- Идентификаторът на етикета трябва да бъде уникален във всички съществуващи тагове
- Името на етикета трябва да е уникално във всички съществуващи тагове

**Отговор (201):**

```json
{
  "success": true,
  "tag": {
    "id": "artificial-intelligence",
    "name": "Artificial Intelligence",
    "isActive": true,
    "itemCount": 0,
    "created_at": "2024-01-20T10:30:00.000Z",
    "updated_at": "2024-01-20T10:30:00.000Z"
  }
}
```

### ВЗЕМЕТЕ `/api/admin/tags/all`

Връща всички тагове от кеша на съдържанието за даден локал. Тази крайна точка чете от слоя с кеширано съдържание, а не от базата данни, което я прави подходяща за попълване на селектори на тагове в потребителския интерфейс на администратора.

**Параметри на заявката:**

|Параметър|Тип|По подразбиране|Описание|
|-----------|------|---------|-------------|
|`locale`|низ|`"en"`|Локален код за извличане на съдържание|

**Отговор (200):**

```json
{
  "success": true,
  "data": [
    {
      "id": "productivity",
      "name": "Productivity",
      "isActive": true,
      "itemCount": 156
    }
  ]
}
```

### ВЗЕМЕТЕ `/api/admin/tags/{id}`

Извлича един етикет по неговия уникален идентификатор с пълни подробности, включително статистически данни за употребата.

**Параметри на пътя:**

|Параметър|Тип|Описание|
|-----------|------|-------------|
|`id`|низ|Уникален идентификатор на етикета|

**Отговор (200):**

```json
{
  "success": true,
  "data": {
    "id": "productivity",
    "name": "Productivity",
    "isActive": true,
    "itemCount": 156,
    "created_at": "2024-01-20T10:30:00.000Z",
    "updated_at": "2024-01-20T10:30:00.000Z"
  }
}
```

### ПОСТАВЕТЕ `/api/admin/tags/{id}`

Актуализира името и/или активния статус на маркер. Идентификационният номер на етикета не може да бъде променен след създаването. Анулира кешовете на съдържанието при успех.

**Параметри на пътя:**

|Параметър|Тип|Описание|
|-----------|------|-------------|
|`id`|низ|Уникален идентификатор на етикета|

**Тяло на заявката:**

```json
{
  "name": "Productivity & Efficiency",
  "isActive": true
}
```

|Поле|Тип|Задължително|Описание|
|-------|------|----------|-------------|
|`name`|низ|да|Актуализирано показвано име на маркер|
|`isActive`|булево|не|Актуализиран активен статус|

**Отговор (200):**

```json
{
  "success": true,
  "data": {
    "id": "productivity",
    "name": "Productivity & Efficiency",
    "isActive": true,
    "itemCount": 156,
    "created_at": "2024-01-20T10:30:00.000Z",
    "updated_at": "2024-01-20T16:45:00.000Z"
  },
  "message": "Tag updated successfully"
}
```

### ИЗТРИВАНЕ `/api/admin/tags/{id}`

Изтрива завинаги етикет от системата. Това също така премахва етикета от всички свързани елементи. Анулира кешовете на съдържанието при успех.

**Параметри на пътя:**

|Параметър|Тип|Описание|
|-----------|------|-------------|
|`id`|низ|Уникален идентификатор на етикета|

**Отговор (200):**

```json
{
  "success": true,
  "message": "Tag deleted successfully"
}
```

:::внимание
Изтриването на етикет е постоянно и не може да бъде отменено. Всички асоциации на артикулен етикет за изтрития етикет ще бъдат премахнати. Вместо това помислете за дезактивиране на маркера (задаване на `isActive` на `false` чрез PUT), ако искате да запазите целостта на данните.
:::

## Модел на данни за етикети

|Поле|Тип|Nullable|Описание|
|-------|------|----------|-------------|
|`id`|низ|не|Подходящ за URL уникален идентификатор|
|`name`|низ|не|Екранно име, разбираемо от човека|
|`isActive`|булево|не|Дали етикетът може да бъде присвоен на елементи|
|`itemCount`|цяло число|не|Брой елементи, използващи този етикет|
|`created_at`|дата и час|не|Дата на създаване|
|`updated_at`|дата и час|не|Дата на последна актуализация|

## Кодове за грешки

|Статус|Грешка|причина|
|--------|-------|-------|
| `400` |Необходими са идентификатор и име на етикет|Липсват задължителни полета при създаване|
| `400` |Изисква се име на етикет|Липсва име при актуализация|
| `400` |Името на етикета трябва да е между 2 и 50 знака|Неуспешно валидиране на дължината на името|
| `400` |Невалиден параметър за страница/лимит|Параметърът за пагинация е извън диапазона|
| `401` |Неразрешено|Липсваща или неадминистративна сесия|
| `404` |Етикетът не е намерен|Няма етикет с дадения идентификатор|
| `409` |Таг с ID вече съществува|Дублиране на ID при създаване|
| `409` |Таг с име вече съществува|Дублирано име при създаване/актуализация|
| `500` |Неуспешно извличане/създаване/актуализиране/изтриване на маркер|Грешка в сървъра или базата данни|

## Кеш невалидност

Всички операции за писане (създаване, актуализиране, изтриване) извикват `invalidateContentCaches()`, за да се гарантира, че промените в таговете се отразяват незабавно в публичното съдържание:

```typescript
await invalidateContentCaches();
```

Това изчиства както кеша на съдържанието в паметта, така и всички кешове на ниво CDN, които може да са активни.

## Източници на данни

API на маркера използва два различни източника на данни в зависимост от крайната точка:

|Крайна точка|Източник на данни|Случай на употреба|
|----------|------------|----------|
|`GET /api/admin/tags`|`tagRepository` (база данни)|Административно управление с пагинация|
|`POST /api/admin/tags`|`tagRepository` (база данни)|Създаване на нови етикети|
|`GET /api/admin/tags/all`|`getCachedItems()` (кеш на съдържанието)|Падащи селектори, бързи търсения|
|`GET /api/admin/tags/{id}`|`tagRepository` (база данни)|Подробен изглед на етикета|
|`PUT /api/admin/tags/{id}`|`tagRepository` (база данни)|Актуализиране на свойствата на етикета|
|`DELETE /api/admin/tags/{id}`|`tagRepository` (база данни)|Премахване на етикети|
