# Отчита крайни точки

Системата за отчети позволява на удостоверените потребители да маркират неподходящо съдържание и предоставя на администраторите инструменти за преглед, модериране и разрешаване на доклади. Отчетите поддържат типове съдържание, включително елементи и коментари, с вградена защита срещу дублиране.

## Преглед

|Крайна точка|Метод|авт|Описание|
|---|---|---|---|
|`/api/reports`|ПУБЛИКУВАНЕ|Потребител|Изпратете отчет за съдържанието|
|`/api/admin/reports`|ВЗЕМЕТЕ|Админ|Списък на отчетите с филтриране|
|`/api/admin/reports/stats`|ВЗЕМЕТЕ|Админ|Вземете статистически данни за отчета|
|`/api/admin/reports/[id]`|ВЗЕМЕТЕ|Админ|Вземете един отчет|
|`/api/admin/reports/[id]`|ПОСТАВЕТЕ|Админ|Актуализирайте състоянието и резолюцията на отчета|

## Публични крайни точки

### Изпратете отчет

```
POST /api/reports
```

Удостоверените потребители могат да докладват елементи или коментари за неподходящо съдържание. Всеки потребител може да докладва едно и също съдържание само веднъж (предотвратяване на дублиране чрез `hasUserReportedContent` проверка). Блокираните потребители (спряни или забранени) не могат да изпращат отчети.

**Удостоверяване:** Изисква се (базирано на сесия)

**Тяло на заявката:**

```json
{
  "contentType": "item",
  "contentId": "awesome-productivity-tool",
  "reason": "spam",
  "details": "This tool is promoting malicious software"
}
```

|Поле|Тип|Задължително|Описание|
|---|---|---|---|
|`contentType`|низ|да|Тип съдържание: `"item"` или `"comment"`|
|`contentId`|низ|да|Идентификационен номер или охлюв на докладваното съдържание|
|`reason`|низ|да|Едно от: `"spam"`, `"harassment"`, `"inappropriate"`, `"other"`|
|`details`|низ|не|Допълнителен контекст за доклада|

**Успешен отговор (200):**

```json
{
  "success": true,
  "message": "Report submitted successfully",
  "report": {
    "id": "rpt_abc123",
    "contentType": "item",
    "contentId": "awesome-productivity-tool",
    "reason": "spam",
    "status": "pending",
    "createdAt": "2024-01-20T10:30:00.000Z"
  }
}
```

**Отговори за грешка:**

|Статус|Състояние|
|---|---|
| 400 |Невалиден тип съдържание, липсващ идентификатор на съдържание или невалидна причина|
| 401 |Потребителят не е удостоверен|
| 403 |Изисква се клиентски профил или потребителят е спрян/баннат|
| 404 |Клиентският профил не е намерен|
| 409 |Потребителят вече е съобщил за това съдържание|
| 500 |Вътрешна грешка на сървъра|

**Източник:** `template/app/api/reports/route.ts`

## Административни крайни точки

Всички администраторски крайни точки изискват `session.user.isAdmin` да е вярно.

### Списък на отчетите

```
GET /api/admin/reports
```

Връща списък със страници с отчети за съдържание с информация за репортера. Поддържа филтриране по статус, тип съдържание и причина, плюс търсене на текст в идентификатор на съдържание, подробности и име/имейл на репортера.

**Параметри на заявката:**

|Параметър|Тип|По подразбиране|Описание|
|---|---|---|---|
|`page`|цяло число| 1 |Номер на страница (минимум 1)|
|`limit`|цяло число| 10 |Резултати на страница (1-100)|
|`search`|низ| - |Търсете в ID на съдържанието, подробности, име/имейл на репортер|
|`status`|низ| - |Филтър: `"pending"`, `"reviewed"`, `"resolved"`, `"dismissed"`|
|`contentType`|низ| - |Филтър: `"item"`, `"comment"`|
|`reason`|низ| - |Филтър: `"spam"`, `"harassment"`, `"inappropriate"`, `"other"`|

**Успешен отговор (200):**

```json
{
  "success": true,
  "data": {
    "reports": [
      {
        "id": "rpt_abc123",
        "contentType": "item",
        "contentId": "some-item-slug",
        "reason": "spam",
        "status": "pending",
        "details": "Suspicious content",
        "reportedBy": "client_456",
        "createdAt": "2024-01-20T10:30:00.000Z"
      }
    ],
    "pagination": {
      "total": 42,
      "page": 1,
      "limit": 10,
      "totalPages": 5
    }
  }
}
```

**Източник:** `template/app/api/admin/reports/route.ts`

### Вземете статистически данни за отчета

```
GET /api/admin/reports/stats
```

Връща обобщена статистика за отчетите, включително преброяване по състояние, тип съдържание и причина.

**Успешен отговор (200):**

```json
{
  "success": true,
  "data": {
    "total": 156,
    "pendingCount": 23,
    "resolvedCount": 120,
    "byStatus": {
      "pending": 23,
      "reviewed": 10,
      "resolved": 120,
      "dismissed": 3
    },
    "byContentType": {
      "item": 100,
      "comment": 56
    },
    "byReason": {
      "spam": 80,
      "inappropriate": 45,
      "harassment": 20,
      "other": 11
    }
  }
}
```

**Източник:** `template/app/api/admin/reports/stats/route.ts`

### Вземете отчет по ID

```
GET /api/admin/reports/[id]
```

Извлича един отчет с пълни подробности, включително информация за репортера и рецензента.

**Параметри на пътя:**

|Параметър|Тип|Описание|
|---|---|---|
|`id`|низ|ID на отчета|

**Успешен отговор (200):**

```json
{
  "success": true,
  "data": {
    "id": "rpt_abc123",
    "contentType": "item",
    "contentId": "some-item-slug",
    "reason": "spam",
    "status": "reviewed",
    "details": "Suspicious content",
    "reportedBy": "client_456",
    "reviewedBy": "admin_789",
    "reviewNote": "Confirmed as spam",
    "resolution": "content_removed",
    "createdAt": "2024-01-20T10:30:00.000Z",
    "updatedAt": "2024-01-21T09:00:00.000Z"
  }
}
```

|Статус|Състояние|
|---|---|
| 403 |Не е администратор|
| 404 |Докладът не е намерен|

**Източник:** `template/app/api/admin/reports/[id]/route.ts`

### Доклад за актуализация

```
PUT /api/admin/reports/[id]
```

Актуализира състоянието на отчета, резолюцията и бележката за преглед. Когато е зададена резолюция, системата автоматично изпълнява съответното действие за модериране (премахване на съдържание, предупреждение за потребителя, спиране или забрана).

**Тяло на заявката:**

```json
{
  "status": "resolved",
  "resolution": "content_removed",
  "reviewNote": "Confirmed spam content, removed from listing"
}
```

|Поле|Тип|Задължително|Описание|
|---|---|---|---|
|`status`|низ|не|`"pending"`, `"reviewed"`, `"resolved"`, `"dismissed"`|
|`resolution`|низ|не|`"content_removed"`, `"user_warned"`, `"user_suspended"`, `"user_banned"`, `"no_action"`|
|`reviewNote`|низ|не|Бележки на администратора относно прегледа|

**Действия за модериране чрез резолюция:**

Въз основа на стойността на резолюцията се задействат следните автоматизирани действия:

|Резолюция|Действие|
|---|---|
|`content_removed`|Извиква `removeContent()`, за да премахне докладвания елемент или коментар|
|`user_warned`|Извиква `warnUser()`, за да издаде предупреждение на собственика на съдържанието|
|`user_suspended`|Извиква `suspendUser()`, за да спре акаунта на собственика на съдържанието|
|`user_banned`|Извиква `banUser()` за постоянно забраняване на собственика на съдържанието|
|`no_action`|Не се предприемат действия за модериране|

**Успешен отговор (200):**

```json
{
  "success": true,
  "message": "Report updated successfully",
  "data": {
    "id": "rpt_abc123",
    "status": "resolved",
    "resolution": "content_removed",
    "reviewNote": "Confirmed spam content"
  },
  "moderationResult": {
    "success": true,
    "message": "Content removed successfully"
  }
}
```

|Статус|Състояние|
|---|---|
| 400 |Невалиден статус или стойност за резолюция; собственик на съдържание не е намерен за действия на ниво потребител|
| 403 |Не е администратор|
| 404 |Докладът не е намерен|

**Източник:** `template/app/api/admin/reports/[id]/route.ts`

## Модел на данни

Отчетите използват следните преброявания, дефинирани в `lib/db/schema`:

- **ReportContentType:** `"item"`, `"comment"`
- **ReportReason:** `"spam"`, `"harassment"`, `"inappropriate"`, `"other"`
- **Състояние на отчета:** `"pending"`, `"reviewed"`, `"resolved"`, `"dismissed"`
- **Резолюция на отчета:** `"content_removed"`, `"user_warned"`, `"user_suspended"`, `"user_banned"`, `"no_action"`

## Интеграция с модериране

Когато даден отчет бъде разрешен с резолюция на ниво потребител (`user_warned`, `user_suspended`, `user_banned`), системата:

1. Търси собственика на съдържанието чрез `getContentOwner()`
2. Изпълнява подходящата функция за модериране от `lib/services/moderation.service`
3. Използва `reviewNote` като причина за модераторското действие
4. Записва ID на администратора като рецензент

Ако действието по модериране е неуспешно, актуализацията на отчета продължава да е успешна, но грешката се регистрира. Полето `moderationResult` в отговора показва дали действието е успешно.
