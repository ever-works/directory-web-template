# Потребителски крайни точки

Потребителският API предоставя крайни точки за управление на удостоверени потребителски предпочитания, подробности за абонамента, хронология на плащанията и настройки за местоположение на профила. Всички крайни точки изискват удостоверяване, базирано на сесия.

## Преглед

|Крайна точка|Метод|авт|Описание|
|---|---|---|---|
|`/api/user/currency`|ВЗЕМЕТЕ|Обществен|Откриване на потребителска валута от заглавки|
|`/api/user/currency`|ПОСТАВЕТЕ|Потребител|Актуализиране на предпочитанията за валута|
|`/api/user/payments`|ВЗЕМЕТЕ|Потребител|Вземете хронология на плащанията от Stripe|
|`/api/user/plan-status`|ВЗЕМЕТЕ|Потребител|Получете статус на плана с информация за изтичане|
|`/api/user/subscription`|ВЗЕМЕТЕ|Потребител|Получете подробности за абонамента|
|`/api/user/profile/location`|ВЗЕМЕТЕ|Потребител|Вземете запазени настройки за местоположение|
|`/api/user/profile/location`|КРЕПКА|Потребител|Актуализиране на настройките за местоположение|

## Откриване на валута и предпочитания

### Откриване на валута

```
GET /api/user/currency
```

Открива валутата на потребителя въз основа на HTTP заглавки от CDN/прокси доставчици. Тази крайна точка използва грациозна деградация -- тя винаги връща 200 OK с валиден код на валутата, връщайки се към USD, ако откриването е неуспешно. Не се изисква удостоверяване.

**Параметри на заявката:**

|Параметър|Тип|По подразбиране|Описание|
|---|---|---|---|
|`provider`|низ|`"smart"`|Доставчик на откриване: `"cloudflare"`, `"vercel"`, `"cloudfront"`, `"fastly"`, `"generic"`, `"auto"`, `"smart"`|

**Успешен отговор (200):**

```json
{
  "currency": "EUR",
  "country": "FR",
  "detected": true
}
```

|Поле|Тип|Описание|
|---|---|---|
|`currency`|низ|Код на валутата ISO 4217 (3 знака), по подразбиране `"USD"`|
|`country`|низ или нула|ISO 3166-1 алфа-2 код на държавата, нула, ако откриването е неуспешно|
|`detected`|булево|Дали откриването е успешно или стойността е резервна|

Когато откриването е неуспешно, отговорът все още връща 200 с `"USD"` и `detected: false`.

**Източник:** `template/app/api/user/currency/route.ts`

### Актуализиране на предпочитанията за валута

```
PUT /api/user/currency
```

Актуализира предпочитаната валута и държава на удостоверения потребител. Валидирано чрез Zod със списъка `SUPPORTED_CURRENCIES` от `lib/config/billing`.

**Удостоверяване:** Изисква се

**Тяло на заявката:**

```json
{
  "currency": "EUR",
  "country": "FR"
}
```

|Поле|Тип|Задължително|Описание|
|---|---|---|---|
|`currency`|низ|да|Код на валутата ISO 4217 (точно 3 знака, главни букви)|
|`country`|низ или нула|не|ISO 3166-1 алфа-2 код на държавата (точно 2 знака)|

**Успешен отговор (200):**

```json
{
  "currency": "EUR",
  "country": "FR"
}
```

|Статус|Състояние|
|---|---|
| 400 |Невалиден JSON, неподдържан код на валута или невалиден формат на държавата|
| 401 |Потребителят не е удостоверен|
| 500 |Неуспешно поддържане на актуализацията|

**Източник:** `template/app/api/user/currency/route.ts`

## История на плащанията

### Вземете хронология на плащанията

```
GET /api/user/payments
```

Извлича пълната история на плащанията на удостоверения потребител от Stripe. Извлича фактури и абонаменти, обогатява ги с метаданни за плана и връща сортиран списък със записи за плащания.

**Удостоверяване:** Изисква се

**Успешен отговор (200):**

```json
[
  {
    "id": "in_1234567890abcdef",
    "date": "2024-01-15T10:30:00.000Z",
    "amount": 29.99,
    "currency": "USD",
    "plan": "Premium Plan",
    "planId": "pro",
    "status": "Paid",
    "billingInterval": "monthly",
    "paymentProvider": "stripe",
    "subscriptionId": "sub_1234567890abcdef",
    "description": "Premium Plan - monthly billing",
    "invoiceUrl": "https://invoice.stripe.com/i/acct_123/test_abc",
    "invoicePdf": "https://pay.stripe.com/invoice/acct_123/test_abc/pdf",
    "invoiceNumber": "INV-2024-001",
    "period_end": "2024-02-15T10:30:00.000Z",
    "period_start": "2024-01-15T10:30:00.000Z"
  }
]
```

Ключови подробности за обработката:

- Филтрира само за `"paid"` и `"open"` фактури
- Преобразува суми от центове в основни валутни единици (дели на 100)
- Сортира по дата, първо най-новите
- Съпоставя състоянието на четими от човека стойности: `"Paid"`, `"Pending"`, `"Draft"`, `"Unknown"`
- Връща празен масив `[]`, ако не съществува клиент на Stripe

**Източник:** `template/app/api/user/payments/route.ts`

## Състояние на плана

### Получете статус на плана

```
GET /api/user/plan-status
```

Връща изчерпателна информация за състоянието на плана, включително подробности за изтичане. Използва се от интерфейса за показване на предупреждения за планове и функции на портала зад проверките на плана.

**Удостоверяване:** Изисква се

**Успешен отговор (200):**

```json
{
  "success": true,
  "data": {
    "planId": "premium",
    "effectivePlan": "premium",
    "isExpired": false,
    "expiresAt": "2024-12-31T23:59:59.000Z",
    "daysUntilExpiration": 45,
    "isInWarningPeriod": false,
    "canAccessPlanFeatures": true,
    "warningMessage": null,
    "status": "active"
  }
}
```

|Поле|Тип|Описание|
|---|---|---|
|`planId`|низ|Абонаментен план на потребителя: `"free"`, `"standard"`, `"premium"`|
|`effectivePlan`|низ|Планът, до който потребителят действително има достъп (може да се различава, ако е изтекъл)|
|`isExpired`|булево|Дали абонаментът е изтекъл|
|`expiresAt`|низ или нула|Срок на годност в ISO формат|
|`daysUntilExpiration`|цяло число или нула|Дни до изтичане (отрицателни, ако вече е изтекъл)|
|`isInWarningPeriod`|булево|Вярно, ако абонаментът изтича в рамките на 7 дни|
|`canAccessPlanFeatures`|булево|Дали потребителят има достъп до функциите на своя план|
|`warningMessage`|низ или нула|Предупредително съобщение към потребителя, ако е приложимо|
|`status`|низ или нула|Сурово състояние на абонамента|

Използва `subscriptionService.getUserPlanWithExpiration()` от `lib/services/subscription.service`.

**Източник:** `template/app/api/user/plan-status/route.ts`

## Подробности за абонамента

### Получете статус на абонамент

```
GET /api/user/subscription
```

Извлича подробна информация за абонамента от Stripe, включително текущия активен абонамент и пълната хронология на абонамента.

**Удостоверяване:** Изисква се

**Успешен отговор (200) -- Активен абонамент:**

```json
{
  "hasActiveSubscription": true,
  "currentSubscription": {
    "id": "sub_1234567890abcdef",
    "planId": "price_1234567890abcdef",
    "planName": "Premium Plan",
    "status": "active",
    "startDate": "2024-01-15T10:30:00.000Z",
    "endDate": "2024-02-15T10:30:00.000Z",
    "nextBillingDate": "2024-02-15T10:30:00.000Z",
    "paymentProvider": "stripe",
    "subscriptionId": "sub_1234567890abcdef",
    "amount": 29.99,
    "currency": "USD",
    "billingInterval": "monthly"
  },
  "subscriptionHistory": [
    {
      "id": "sub_1234567890abcdef",
      "planId": "price_1234567890abcdef",
      "planName": "Premium Plan",
      "status": "active",
      "startDate": "2024-01-15T10:30:00.000Z",
      "endDate": "2024-02-15T10:30:00.000Z",
      "amount": 29.99,
      "currency": "USD",
      "billingInterval": "monthly"
    }
  ]
}
```

Активните абонаменти се идентифицират с `status === "active"` или `status === "trialing"`. Записите в историята може да включват `cancelledAt` и `cancelReason` за анулирани абонаменти.

**Източник:** `template/app/api/user/subscription/route.ts`

## Местоположение на профила

### Вземете настройки за местоположение

```
GET /api/user/profile/location
```

Връща запазеното местоположение по подразбиране и предпочитание за поверителност на удостоверения потребител.

**Удостоверяване:** Изисква се (клиентски профил)

**Успешен отговор (200):**

```json
{
  "defaultLatitude": 48.8566,
  "defaultLongitude": 2.3522,
  "defaultCity": "Paris",
  "defaultCountry": "FR",
  "locationPrivacy": "city"
}
```

**Източник:** `template/app/api/user/profile/location/route.ts`

### Актуализиране на настройките за местоположение

```
PATCH /api/user/profile/location
```

Актуализира местоположението по подразбиране и предпочитанията за поверителност на удостоверения потребител. Валидирано с помощта на `updateLocationSchema` от `lib/validations/user-location`.

**Тяло на заявката:**

```json
{
  "defaultLatitude": 48.8566,
  "defaultLongitude": 2.3522,
  "defaultCity": "Paris",
  "defaultCountry": "FR",
  "locationPrivacy": "city"
}
```

|Поле|Тип|Задължително|Описание|
|---|---|---|---|
|`defaultLatitude`|число или нула|не|Географска ширина|
|`defaultLongitude`|число или нула|не|Координата на дължината|
|`defaultCity`|низ или нула|не|Име на града|
|`defaultCountry`|низ или нула|не|Код на държавата|
|`locationPrivacy`|низ|не|Ниво на поверителност: `"private"`, `"city"`, `"exact"`|

Както географската ширина, така и дължината трябва да бъдат предоставени заедно.

**Източник:** `template/app/api/user/profile/location/route.ts`
