# Крайни точки на API за помощни програми

Крайните точки на помощните програми предоставят инфраструктурни услуги, включително проверки на състоянието, информация за версията, конфигурация на функции, геокодиране, проверка на reCAPTCHA, извличане на URL адреси, данни за местоположение и вътрешни операции.

## Проверка на състоянието (`/api/health`)

### Здраве на базата данни

|Метод|Пътека|Описание|
|--------|------|-------------|
|`GET`|`/api/health/database`|Проверете свързаността на базата данни|

Връща състоянието на връзката с базата данни. Използва се от системи за наблюдение и проверки на състоянието на внедряване.

**Отговор (здравословен):**

```json
{
  "status": "healthy",
  "database": "connected",
  "timestamp": "2025-01-15T10:30:00.000Z"
}
```

**Отговор (нездравословен):**

```json
{
  "status": "unhealthy",
  "database": "disconnected",
  "error": "Connection refused",
  "timestamp": "2025-01-15T10:30:00.000Z"
}
```

**Удостоверяване:** Публично (не се изисква удостоверяване). Тази крайна точка трябва да бъде достъпна от балансиращите на натоварването и услугите за наблюдение.

## Версия (`/api/version`)

|Метод|Пътека|Описание|
|--------|------|-------------|
|`GET`|`/api/version`|Получете информация за версията на приложението|
|`GET`|`/api/version/sync`|Получете версия и състояние на синхронизиране|

### Отговор на версията

Връща версията на приложението, информация за компилация и среда за изпълнение:

```json
{
  "version": "1.0.0",
  "environment": "production",
  "timestamp": "2025-01-15T10:30:00.000Z"
}
```

### Версия + състояние на синхронизиране

Крайната точка `/api/version/sync` разширява информацията за версията със състоянието на синхронизиране на хранилището на съдържанието, полезно за отстраняване на грешки в актуалността на съдържанието.

**Удостоверяване:** Публично.

## Конфигурация на функции (`/api/config`)

|Метод|Пътека|Описание|
|--------|------|-------------|
|`GET`|`/api/config/features`|Вземете активирани флагове за функции|

Връща текущата конфигурация на флага на функцията за приложението от страна на клиента. Това позволява на интерфейса да изобразява условно функции въз основа на конфигурация от страна на сървъра.

**Отговор:**

```json
{
  "features": {
    "payments": true,
    "sponsorAds": true,
    "surveys": false,
    "map": true,
    "newsletter": true
  }
}
```

**Удостоверяване:** Публично. Флаговете на функциите не са чувствителни данни.

## Извличане на URL (`/api/extract`)

|Метод|Пътека|Описание|
|--------|------|-------------|
|`POST`|`/api/extract`|Извличане на метаданни от URL|

Извлича URL и извлича метаданни, включително заглавие, описание, изображение и фавикон. Използва се от формуляра за подаване на артикул за автоматично попълване на полета от URL адрес.

**Заявка:**

```json
{
  "url": "https://example.com/product"
}
```

**Отговор:**

```json
{
  "success": true,
  "data": {
    "title": "Product Name",
    "description": "Product description from meta tags",
    "image": "https://example.com/og-image.png",
    "favicon": "https://example.com/favicon.ico",
    "siteName": "Example.com"
  }
}
```

**Удостоверяване:** Изисква се. Предотвратява злоупотреба с извличане на URL адреси от страна на сървъра.

## Геокодиране (`/api/geocode`)

|Метод|Пътека|Описание|
|--------|------|-------------|
|`POST`|`/api/geocode`|Геокодиране на адрес към координати|

Преобразува текстов адрес в географски координати (ширина/дължина) с помощта на външна услуга за геокодиране.

**Заявка:**

```json
{
  "address": "1600 Amphitheatre Parkway, Mountain View, CA"
}
```

**Отговор:**

```json
{
  "success": true,
  "data": {
    "lat": 37.4224764,
    "lng": -122.0842499,
    "formattedAddress": "1600 Amphitheatre Pkwy, Mountain View, CA 94043, USA"
  }
}
```

**Удостоверяване:** Изисква се.

## Данни за местоположение (`/api/location`)

Крайни точки за търсене на местоположение и справочни данни.

|Метод|Пътека|Описание|
|--------|------|-------------|
|`GET`|`/api/location/countries`|Избройте всички държави|
|`GET`|`/api/location/cities`|Списък на градове (с филтър за държави)|
|`GET`|`/api/location/coordinates`|Вземете координати за местоположение|
|`GET`|`/api/location/search`|Търсете местоположения по низ на заявка|

### Държави

Връща списък с държави с ISO кодове, имена и незадължителни метаданни.

### градове

Поддържа филтриране по код на държава:

```
GET /api/location/cities?country=US
```

### Търсене на местоположение

Пълнотекстово търсене на местоположение:

```
GET /api/location/search?q=San Francisco
```

**Удостоверяване:** Публично.

## reCAPTCHA Проверка (`/api/verify-recaptcha`)

|Метод|Пътека|Описание|
|--------|------|-------------|
|`POST`|`/api/verify-recaptcha`|Проверете reCAPTCHA токен|

Проверка от страна на сървъра на Google reCAPTCHA токени. Използва се от формуляри, които изискват защита от ботове.

**Заявка:**

```json
{
  "token": "recaptcha-response-token"
}
```

**Отговор:**

```json
{
  "success": true,
  "score": 0.9,
  "action": "submit"
}
```

**Удостоверяване:** Публично (reCAPTCHA обикновено е на публични формуляри).

## Референтни данни (`/api/reference`)

|Метод|Пътека|Описание|
|--------|------|-------------|
|`GET`|`/api/reference`|Вземете справочни данни за падащи менюта на формуляри|

Връща референтни данни, използвани за попълване на падащи менюта и полета за избор в приложението, като модели на ценообразуване, типове лицензи и категории на платформата.

**Удостоверяване:** Публично.

## Вътрешни операции (`/api/internal`)

|Метод|Пътека|Описание|
|--------|------|-------------|
|`POST`|`/api/internal/db-init`|Инициализирайте схемата на базата данни и началните данни|

### Инициализация на база данни

Крайната точка `/api/internal/db-init` задейства миграция на база данни и незадължително вмъкване на начални данни. Това обикновено се извиква веднъж по време на първоначалното внедряване или при нулиране на среда за разработка.

**Удостоверяване:** Тази крайна точка трябва да бъде защитена чрез специфични за средата контроли за достъп или споделена тайна. Не е предназначен за редовна употреба.

## Съображения за сигурност

### Публични крайни точки

Следните крайни точки на полезност са умишлено публични:
- Проверки на здравето (необходими за мониторинг/балансиране на натоварването)
- Информация за версията (нечувствителна)
- Флагове за функции (нечувствителна конфигурация)
- Данни за местоположение (референтни данни)
- reCAPTCHA проверка (защита на публичен формуляр)
- Референтни данни (падащи стойности)

### Защитени крайни точки

Тези крайни точки изискват удостоверяване за предотвратяване на злоупотреби:
- Извличане на URL (предотвратява злоупотреба с фалшифициране на заявка от страна на сървъра)
- Геокодиране (ограничени по скорост външни извиквания на API)
- Инициализация на база данни (разрушителна операция)
