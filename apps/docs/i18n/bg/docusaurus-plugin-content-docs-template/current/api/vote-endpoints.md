# Крайни точки за гласуване

Системата за гласуване предоставя крайни точки за гласуване за и против елементи. Гласовете използват модел на нетен резултат, където броят представлява гласовете за минус гласовете против. Публичните крайни точки връщат броя на гласовете, докато автентифицираните крайни точки позволяват подаване, актуализиране и премахване на гласове. Блокираните потребители не могат да гласуват.

## Преглед

|Крайна точка|Метод|авт|Описание|
|---|---|---|---|
|`/api/items/[slug]/votes`|ВЗЕМЕТЕ|Обществен|Получете преброяване на гласовете и статус на гласуване на потребителите|
|`/api/items/[slug]/votes`|ПУБЛИКУВАНЕ|Потребител|Пуснете или актуализирайте гласуване|
|`/api/items/[slug]/votes`|ИЗТРИВАНЕ|Потребител|Премахване на глас|
|`/api/items/[slug]/votes/count`|ВЗЕМЕТЕ|Обществен|Получете само нетен брой гласове|
|`/api/items/[slug]/votes/status`|ВЗЕМЕТЕ|Потребител|Вземете пълен запис на гласуване за потребителя|

## Крайна точка на комбинирано гласуване

### Вземете информация за гласуване

```
GET /api/items/[slug]/votes
```

Връща нетния брой гласове за даден елемент и статуса на гласуване на текущия потребител, ако е удостоверен. Не се изисква удостоверяване, но удостоверените потребители получават статуса си на глас в отговора.

**Параметри на пътя:**

|Параметър|Тип|Описание|
|---|---|---|
|`slug`|низ|елемент плужек|

**Успешен отговор (200):**

```json
{
  "success": true,
  "count": 15,
  "userVote": "up"
}
```

|Поле|Тип|Описание|
|---|---|---|
|`success`|булево|Винаги `true` за успех|
|`count`|цяло число|Нетен брой гласове (гласове за минус гласове против)|
|`userVote`|низ или нула|`"up"`, `"down"` или `null`, ако не е удостоверено или няма гласуване|

За неупълномощени потребители `userVote` винаги е `null`. `count` може да бъде отрицателен, ако има повече гласове против, отколкото гласове за.

**Източник:** `template/app/api/items/[slug]/votes/route.ts`

### Гласувайте или актуализирайте

```
POST /api/items/[slug]/votes
```

Пуска ново гласуване или заменя съществуващо гласуване за даден елемент. Ако потребителят вече има глас, предишният глас се изтрива, преди да се създаде новият. Това означава, че промяната от гласуване „за“ към гласуване „против“ (или обратното) е една операция.

**Удостоверяване:** Изисква се

**Тяло на заявката:**

```json
{
  "type": "up"
}
```

|Поле|Тип|Задължително|Описание|
|---|---|---|---|
|`type`|низ|да|`"up"` за гласуване за, `"down"` за гласуване против|

**Успешен отговор (200):**

```json
{
  "success": true,
  "count": 16,
  "userVote": "up"
}
```

Отговорът връща актуализирания нетен брой гласове след прилагането на гласуването.

**Отговори за грешка:**

|Статус|Състояние|
|---|---|
| 400 |Невалиден тип гласуване (трябва да е `"up"` или `"down"`)|
| 401 |Не е удостоверено|
| 403 |Потребителят е спрян или забранен|
| 404 |Клиентският профил не е намерен|

**Източник:** `template/app/api/items/[slug]/votes/route.ts`

### Премахване на глас

```
DELETE /api/items/[slug]/votes
```

Премахва гласа на текущия потребител от даден елемент. Ако не съществува глас, операцията завършва успешно без грешка (идемпотент). След премахването `userVote` е `null`.

**Удостоверяване:** Изисква се

**Успешен отговор (200):**

```json
{
  "success": true,
  "count": 14,
  "userVote": null
}
```

|Статус|Състояние|
|---|---|
| 401 |Не е удостоверено|
| 404 |Клиентският профил не е намерен|

**Източник:** `template/app/api/items/[slug]/votes/route.ts`

## Крайна точка за преброяване на гласове

### Вземете преброяване на гласовете

```
GET /api/items/[slug]/votes/count
```

Връща само нетния брой гласове за даден елемент. Това е лека публична крайна точка, оптимизирана за бързо извличане на броя на гласовете без специфичен за потребителя статус на гласуване.

**Успешен отговор (200):**

```json
{
  "success": true,
  "count": 15
}
```

Броят може да бъде положителен, отрицателен или нула в зависимост от баланса на гласовете за и против.

**Източник:** `template/app/api/items/[slug]/votes/count/route.ts`

## Крайна точка на състоянието на гласуване

### Получете статус на потребителско гласуване

```
GET /api/items/[slug]/votes/status
```

Връща пълния запис на гласуване за удостоверения потребител за конкретен елемент. Връща `null`, ако потребителят не е гласувал за елемента.

**Удостоверяване:** Изисква се

**Успешен отговор (200) -- потребителят е гласувал:**

```json
{
  "id": "vote_123abc",
  "userId": "client_456def",
  "itemId": "item_123abc",
  "voteType": "UPVOTE",
  "createdAt": "2024-01-20T10:30:00.000Z",
  "updatedAt": "2024-01-20T10:30:00.000Z"
}
```

**Успешен отговор (200) -- Без гласуване:**

```json
null
```

Имайте предвид, че тази крайна точка връща необработените стойности на базата данни `voteType` (`"UPVOTE"` или `"DOWNVOTE"`), а не опростения формат `"up"` / `"down"`, използван от комбинираната крайна точка.

|Статус|Състояние|
|---|---|
| 401 |Не е удостоверено|
| 404 |Клиентският профил не е намерен|

**Източник:** `template/app/api/items/[slug]/votes/status/route.ts`

## Ключови подробности за внедряването

- **Нетен резултат:** Броят на гласовете се изчислява като гласове за минус гласове против. Отрицателният брой показва повече гласове против, отколкото гласове за.
- **Замяна на гласуване:** Когато потребител промени типа на своя вот, съществуващият вот се изтрива и се създава нов. Няма актуализация на място.
- **Превенция на блокирани потребители:** Проверката `isUserBlocked()` на крайната точка на POST не позволява на спрени или забранени потребители да гласуват. Блоковата проверка се прилага само при създаване на глас, но не и при премахване на глас.
- **VoteType Enum:** Базата данни съхранява гласовете като `VoteType.UPVOTE` и `VoteType.DOWNVOTE`. API ги превежда в `"up"` и `"down"` за външни потребители.
- **Идемпотентно изтриване:** Изтриването на глас, който не съществува, все още връща отговор 200 с текущия брой и `userVote: null`.
