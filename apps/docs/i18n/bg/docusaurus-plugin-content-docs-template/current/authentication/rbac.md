# Контрол на Достъпа Базиран на Роли

## Преглед

Шаблонът имплементира RBAC с роли, съхранени в базата данни.

## Роли по Подразбиране

| Роля | Описание |
|------|----------|
| admin | Пълен системен достъп |
| moderator | Достъп за модерация на съдържание |
| user | Стандартен удостоверен достъп |
| guest | Ограничен публичен достъп |

## Назначаване на Роли

Ролите се назначават в базата данни. Администраторите могат да управляват роли чрез административното табло на /admin/users.

## Проверка на Разрешения

```typescript
// В API маршрути
const session = await auth();
if (!session?.user?.role || session.user.role !== 'admin') {
  return NextResponse.json({ error: 'Forbidden' }, { status: 403 });
}
```

## Защита на Маршрути

Използвайте middleware за защита на маршрути въз основа на роли. Auth middleware проверява сесията и ролята преди да разреши достъп.
