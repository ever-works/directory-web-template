# Управление на Сесии

## Стратегия за Сесии

Шаблонът поддържа две стратегии за сесии:
1. JWT (по подразбиране) — Без съхранение на състояние, съхранява се в бисквитки
2. База данни — Съхранява се в база данни, поддържа отмяна

## Конфигурация на Сесиите

```typescript
// auth.config.ts
export const authConfig = {
  session: {
    strategy: 'jwt',
    maxAge: 30 * 24 * 60 * 60, // 30 дни
  }
}
```

## Сигурност

- HttpOnly бисквитките предотвратяват XSS
- SameSite=Lax предотвратява CSRF
- Автоматично обновяване на сесии
- Флаг Secure в продукция

## Излизане от системата

Сесията се изчиства при излизане. Всички активни сесии могат да бъдат анулирани чрез промяна на AUTH_SECRET.
