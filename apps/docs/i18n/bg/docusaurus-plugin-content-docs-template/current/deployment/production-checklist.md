# Контролен Списък за Продукция

Изчерпателен контролен списък за осигуряване готовността на Ever Works за производствена среда.

## Контролен Списък Преди Внедряване

### 1. Конфигурация на Средата

#### Задължителни Променливи на Средата

- [ ] **База Данни**
  - `DATABASE_URL` конфигуриран с производствен PostgreSQL
  - Включен connection pooling на базата данни
  - Включен SSL режим за производство

- [ ] **Удостоверяване**
  - `NEXTAUTH_URL` зададен за производствения домейн
  - `NEXTAUTH_SECRET` генериран (минимум 32 символа)
  - OAuth доставчиците конфигурирани (Google, GitHub и др.)
  - Идентификационни данни Supabase Auth (ако се използва)

- [ ] **Доставчици на Плащания**
  - Производствени ключове на Stripe (`STRIPE_SECRET_KEY`, `NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY`)
  - Производствени ключове на LemonSqueezy (ако се използва)
  - Конфигурирани webhook секрети
  - Тестовият режим деактивиран

- [ ] **Имейл Услуги**
  - Конфигуриран API ключ на Resend
  - Зададени идентификационни данни Novu (ако се използва)
  - Тествани имейл шаблони
  - Верифициран домейн на подателя

- [ ] **Анализи и Мониторинг**
  - Производствен ключ на PostHog
  - Конфигуриран DSN на Sentry
  - Дефиниран доставчик за проследяване на изключения
  - Активиран Vercel Analytics (ако е на Vercel)

- [ ] **CRM Интеграция**
  - Идентификационни данни Twenty CRM (ако се използва)
  - Конфигурирани webhook крайни точки

- [ ] **Сигурност**
  - `NODE_ENV=production`
  - Конфигурирано ограничаване на честотата
  - Прегледани CORS настройки
  - Конфигурирани CSP заглавия

### 2. База Данни

- [ ] **Схема и Миграции**
  - Всички миграции приложени
  - Схемата на базата данни съответства на кода
  - Създадени индекси за производителност
  - Валидирани ограничения на чужди ключове

- [ ] **Целостност на Данните**
  - Заредени seed данни (ако е необходимо)
  - Изтрити тестови данни
  - Приложени правила за валидация на данни

- [ ] **Резервно Копиране и Възстановяване**
  - Конфигурирани автоматични резервни копия
  - Тествано възстановяване от резервно копие
  - Активирано point-in-time възстановяване
  - Дефинирана политика за съхранение на резервни копия

- [ ] **Производителност**
  - Конфигуриран connection pooling
  - Оптимизирана производителност на заявките
  - Активирано логване на бавни заявки
  - Активен мониторинг на базата данни

### 3. Сигурност

- [ ] **Удостоверяване и Оторизация**
  - Проверено хеширане на паролата (bcrypt)
  - Сигурно управление на сесиите
  - Правилно подписани JWT токени
  - Тестван контрол на достъпа базиран на роли

- [ ] **Защита на Данните**
  - PII данните криптирани в покой
  - Конфигурирано почистване на чувствителни данни
  - Наложен HTTPS
  - Активирани защитени бисквитки

- [ ] **API Сигурност**
  - Активно ограничаване на честотата
  - Задължително API удостоверяване
  - Валидация на входа на всички крайни точки
  - Проверена защита от SQL инжекция

- [ ] **Зависимости**
  - Всички зависимости актуализирани
  - Проверени уязвимости в сигурността (`npm audit`)
  - Без критични уязвимости
  - Комитнат lock файл на зависимостите

### 4. Производителност

- [ ] **Оптимизация на Фронтенда**
  - Оптимизирани изображения (компонент Image на Next.js)
  - Имплементирано разделяне на кода
  - Lazy loading за тежки компоненти
  - Анализиран размер на пакета

- [ ] **Кеширане**
  - Статичните активи в кеша
  - API отговорите кеширани (където е подходящо)
  - Конфигуриран CDN
  - Приложена стратегия за инвалидиране на кеша

- [ ] **Core Web Vitals**
  - LCP < 2,5с
  - FID < 100мс
  - CLS < 0,1
  - Активен мониторинг на производителността

- [ ] **Заявки към Базата Данни**
  - Елиминирани N+1 заявки
  - Създадени подходящи индекси
  - Активирано кеширане на заявките
  - Оптимизиран connection pooling

### 5. Мониторинг и Логване

- [ ] **Проследяване на Грешки**
  - Конфигурирани Sentry/PostHog
  - Конфигурирани предупреждения за грешки
  - Качени source maps
  - Конфигурирано групиране на грешки

- [ ] **Мониторинг на Приложението**
  - Крайна точка за проверка на здравето (`/api/health`)
  - Конфигуриран мониторинг на времето на работа
  - Проследявани показатели за производителност
  - Дефинирани персонализирани показатели

- [ ] **Логване**
  - Имплементирано структурирано логване
  - Конфигурирани нива на логване
  - Конфигурирана агрегация на логове
  - Дефинирана политика за съхранение на логове

- [ ] **Предупреждения**
  - Предупреждения за критични грешки
  - Предупреждения за деградация на производителността
  - Предупреждения за времето на работа
  - Предупреждения за неуспешни плащания

### 6. Съдържание и Данни

- [ ] **Git-базиран CMS**
  - Конфигурирано хранилище `.content`
  - Работеща синхронизация на съдържанието
  - Защитени Git идентификационни данни
  - Стратегия за резервно копиране на съдържанието

- [ ] **Медийни Активи**
  - Оптимизирани изображения
  - Конфигуриран CDN за медия
  - Конфигурирани ограничения за качване
  - Мониторинг на квотата за съхранение

- [ ] **Интернационализация**
  - Всички преводи завършени
  - Тествана RTL поддръжка (арабски)
  - Работещо разпознаване на locale
  - Проверено форматиране на дата/число

### 7. API Документация

- [ ] **Система за Документация**
  - Генерирана OpenAPI спецификация (`yarn generate-docs`)
  - Scalar UI достъпен на `/api/reference`
  - Всички крайни точки документирани
  - Тествани примери

- [ ] **API Стандарти**
  - Последователни конвенции за именуване
  - Правилни HTTP статус кодове
  - Стандартизирани отговори за грешки
  - Документирано ограничаване на честотата

### 8. Платежна Система

- [ ] **Конфигурация на Stripe**
  - Активиран производствен режим
  - Конфигурирани и тествани webhooks
  - Активиран клиентски портал
  - Конфигурирани настройки за фактури

- [ ] **Конфигурация на LemonSqueezy** (ако се използва)
  - Зададени производствени идентификационни данни
  - Конфигурирани webhooks
  - Проверено данъчно съответствие

- [ ] **Управление на Абонаменти**
  - Тествано създаване на планове
  - Тествани процеси за надграждане/понижаване
  - Тестван процес на отказ
  - Документиран процес на възстановяване

### 9. Имейл Система

- [ ] **Транзакционни Имейли**
  - Тестван приветствен имейл
  - Тествано нулиране на парола
  - Тествана имейл верификация
  - Тествани имейли за абонамент

- [ ] **Имейл Шаблони**
  - Всички шаблони прегледани
  - Последователен брандинг
  - Адаптивен за мобилни устройства
  - Работещи линкове за отписване

- [ ] **Доставяемост**

  - Мониторинг репутацията на подателя

## Команди за Проверка

Изпълнете тези команди за проверка на готовността за производство:

### Одит на Сигурността

```bash
# Проверка за уязвимости в сигурността
npm audit --production

# Поправяне на уязвимости
npm audit fix

# Проверка за остарели зависимости
npm outdated
```

### Проверка на Изграждането

```bash
# Производствено изграждане
npm run build

# Проверка на изхода от изграждането
ls -lh .next/

# Анализ на размера на пакета
npm run analyze
```

### Проверка на Базата Данни

```bash
# Проверка на статуса на миграциите
npx drizzle-kit check

# Генериране на миграция при необходимост
npx drizzle-kit generate

# Прилагане на миграции
npx drizzle-kit push
```

### API Документация

```bash
# Генериране на OpenAPI спецификация
yarn generate-docs

# Валидиране на документацията
yarn docs:validate

# Проверка дали документацията е актуална
git diff --exit-code public/openapi.json
```

### Променливи на Средата

```bash
# Проверка дали всички задължителни променливи са зададени
node scripts/check-env.js

# Тестване на конфигурацията на средата
npm run test:env
```

## Работен Процес на Внедряване

### Преди Внедряване

1. **Преглед на Кода**
   - Всички PR прегледани и одобрени
   - Без конфликти при сливане
   - CI/CD конвейерът е преминал

2. **Тестване**
   - Всички тестове са преминали
   - Завършено ръчно QA тестване
   - Тествана staging среда

3. **Документация**
   - Актуализиран changelog
   - Регенерирана API документация
   - Подготвени бележки за внедряване

### Стъпки на Внедряване

1. **Резервно Копиране**

   ```bash
   # Резервно копие на базата данни
   pg_dump $DATABASE_URL > backup-$(date +%Y%m%d).sql
   ```

2. **Внедряване**

   ```bash
   # Внедряване в производство
   git push production main

   # Или с Vercel
   vercel --prod
   ```

3. **Проверка**

   ```bash
   # Проверка на крайната точка за здраве
   curl https://your-domain.com/api/health

   # Проверка на логовете за грешки
   tail -f logs/error.log
   ```

4. **Мониторинг**
   - Мониторинг на честотата на грешките в Sentry
   - Мониторинг на производителността в PostHog
   - Проверка на мониторинга за времето на работа

### След Внедряване

1. **Smoke тестове**
   - Началната страница се зарежда
   - Потребителят може да влезе
   - Платежният поток работи
   - Административният панел е достъпен

2. **Мониторинг**
   - Нормална честота на грешките
   - Приемливи времена за отговор
   - Без изтичане на памет
   - Стабилна производителност на базата данни

3. **Комуникация**
   - Уведомяване на екипа за внедряването
   - Актуализиране на страницата за статус
   - Обявяване на нови функции (ако е приложимо)

## План за Откат

Ако след внедряването се открият проблеми:

### Бърз Откат

```bash
# Връщане към предишното внедряване
git revert HEAD
git push production main

# Или с Vercel
vercel rollback
```

### Откат на Базата Данни

```bash
# Възстановяване от резервно копие
psql $DATABASE_URL < backup-YYYYMMDD.sql

# Или използване на point-in-time възстановяване
# (ако се поддържа от вашия хостинг доставчик)
```

### Комуникация

1. Незабавно уведомяване на екипа
2. Актуализиране на страницата за статус
3. Комуникация с засегнатите потребители
4. Документиране на инцидента за post-mortem

## Показатели за Успех

Следете тези показатели за осигуряване здравето на производствената среда:

### Производителност

- **Време за Отговор**: < 200мс (p95)
- **Наличност**: > 99,9%
- **Честота на Грешки**: < 0,1%
- **Core Web Vitals**: Всички в зелено

### Бизнес

- **Регистрации на Потребители**: Работещо проследяване
- **Успеваемост на Плащанията**: > 95%
- **Доставяемост на Имейли**: > 98%
- **Наличност на API**: > 99,9%

### Сигурност

- **Неуспешни Опити за Вход**: Проследявани
- **API Rate Limit Удари**: < 1%
- **Уязвимости в Сигурността**: 0 критични
- **SSL Сертификат**: Валиден и с автоподновяване

## Резюме на Контролния Списък

Използвайте това кратко резюме за проследяване на общия прогрес:

- [ ] **Среда**: Всички променливи конфигурирани

- [ ] **Сигурност**: Удостоверяване, криптиране, ограничаване на честотата

- [ ] **Мониторинг**: Проследяване на грешки, логване, предупреждения
- [ ] **Съдържание**: CMS конфигуриран, медията оптимизирана, i18n завършено
- [ ] **API**: Документацията генерирана, стандартите следвани
- [ ] **Плащания**: Stripe/LS конфигуриран, webhooks тествани
