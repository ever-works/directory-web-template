# Конфигурация на Логове Sentry

Този документ обяснява как да конфигурирате и използвате Sentry Logs в хранилището Template и хранилището Ever Works.

## Преглед

Sentry Logs предоставя централизирано управление на логовете, позволявайки да улавяте, препращате и анализирате логовете на приложението в Logs Explorer на Sentry. Всички логове се препращат автоматично към Sentry при активиране, предоставяйки унифициран изглед на поведението на приложението в различни среди.

## Функционалности

- ✅ Автоматично препращане на логове към Sentry
- ✅ Поддръжка за всички нива на логване (debug, info, warn, error)
- ✅ Контекстно логване с автоматично маркиране
- ✅ Конфигурация специфична за средата
- ✅ Структурирано логване с поддръжка на метаданни
- ✅ Интеграция със съществуващата помощна програма logger

## Конфигурация

### Променливи на Средата

Добавете тези променливи към файла `.env.local` за локална разработка:

```env
# Конфигурация на Sentry (задължително за логове)
NEXT_PUBLIC_SENTRY_DSN=https://your-dsn@sentry.io/your-project-id
SENTRY_ORG=your-org-name
SENTRY_PROJECT=your-project-name
SENTRY_AUTH_TOKEN=your-auth-token

# Активиране на Sentry при разработка (незадължително, по подразбиране само в производство)
SENTRY_ENABLE_DEV=true

# Режим на отстраняване на грешки Sentry (незадължително)
SENTRY_DEBUG=false

# Конфигурация на логове Sentry
SENTRY_LOGS_ENABLED=true  # Активиране/деактивиране на Sentry Logs (по подразбиране: true)
SENTRY_LOGS_LEVEL=info    # Минимално ниво на логване за улавяне (по подразбиране: info)
```

### Конфигурация Специфична за Средата

#### Локална Разработка

```env
SENTRY_ENABLE_DEV=true
SENTRY_LOGS_ENABLED=true
SENTRY_LOGS_LEVEL=debug  # Улавяне на всички логове при разработка
```

#### Разработка/Staging

```env
SENTRY_ENABLE_DEV=true
SENTRY_LOGS_ENABLED=true
SENTRY_LOGS_LEVEL=info  # Улавяне на info, warn и error логове
```

#### Производство

```env
SENTRY_ENABLE_DEV=false  # Не е необходимо в производство
SENTRY_LOGS_ENABLED=true
SENTRY_LOGS_LEVEL=warn  # Улавяне само на предупреждения и грешки в производство
```

## Употреба

### Основно Логване

Logger-ът автоматично препраща логове към Sentry при активиране:

```typescript
import { logger } from '@/lib/logger';

// Info лог
logger.info('User logged in', { userId: '12345' });

// Предупредителен лог
logger.warn('Rate limit approaching', { current: 90, limit: 100 });

// Лог за грешка
logger.error('Payment failed', { orderId: '67890', error: errorObject });

// Debug лог (само при разработка)
logger.debug('API request', { method: 'GET', url: '/api/users' });
```

### Контекстно Логване

Създайте logger с конкретен контекст за по-добра организация:

```typescript
import { Logger } from '@/lib/logger';

const paymentLogger = Logger.create('PaymentService');

paymentLogger.info('Processing payment', { amount: 100, currency: 'USD' });
paymentLogger.error('Payment failed', error);
```

### Нива на Логване

Logger-ът поддържа четири нива на логване, автоматично съпоставени с нивата на тежест на Sentry:

| Ниво Logger | Ниво Sentry | Описание |
|-------------|-------------|-------------|
| `DEBUG` | `debug` | Подробна информация за отстраняване на грешки (само разработка) |
| `INFO` | `info` | Общи информационни съобщения |
| `WARN` | `warning` | Предупредителни съобщения за потенциални проблеми |
| `ERROR` | `error` | Съобщения за грешки при повреди |

## Как Работи

### Инициализация

Sentry Logs е активиран както при инструментацията на клиента, така и при инструментацията на сървъра:

1. **От страна на сървъра** (`instrumentation.ts`): Инициализира Sentry за Node.js runtime
2. **От страна на клиента** (`instrumentation-client.ts`): Инициализира Sentry за браузърния runtime

И двете конфигурации включват:
```typescript
_experiments: {
  enableLogs: SENTRY_LOGS_ENABLED,
}
```

### Препращане на Логове

Помощната програма logger (`lib/logger.ts`) автоматично:
1. Проверява дали Sentry Logs е активиран
2. Форматира записите на логовете с контекст и метаданни
3. Препраща логовете към Sentry с помощта на `Sentry.captureMessage()` с подходящи тагове и нива
4. Изящно превключва към fallback режим ако Sentry е недостъпен

### Структура на Логовете

Всеки запис в лога изпратен към Sentry включва:
- **Съобщение**: Съобщението на лога с незадължителен контекстен префикс
- **Ниво**: Ниво на тежест (debug, info, warning, error)
- **Тагове**:
  - `logLevel`: Оригиналното ниво на лога
  - `logType`: Винаги `application_log`
  - `context`: Незадължителен идентификатор на контекст
- **Допълнителни Данни**:
  - `data`: Всякакви допълнителни предоставени данни
  - `timestamp`: ISO timestamp

## Преглед на Логове в Sentry

### Logs Explorer

1. Навигирайте до вашия проект Sentry
2. Отидете на **Logs** → **Logs Explorer**
3. Използвайте филтри за намиране на конкретни логове:
   - Филтриране по таг `logLevel` (debug, info, warn, error)
   - Филтриране по таг `context` за преглед на логове от конкретни модули
   - Филтриране по `logType:application_log` за преглед само на логове на приложението

### Заявки за Логове

Примерни заявки в Logs Explorer на Sentry:

```
# Всички error логове
logLevel:error

# Логове от конкретен контекст
context:PaymentService

# Всички логове на приложението
logType:application_log

# Грешки от конкретен времеви диапазон
logLevel:error timestamp:>2024-01-01
```

## Интеграция с Пакета за Мониторинг

Ако използвате пакета `@ever-works/monitoring`, уверете се, че е конфигуриран да работи с Sentry Logs:

1. Пакетът за мониторинг трябва да инициализира Sentry с активирани логове
2. Помощната програма logger в този шаблон автоматично ще препраща логовете към Sentry
3. И двете системи работят заедно за осигуряване на цялостен мониторинг

## Отстраняване на Неизправности

### Логовете Не Се Появяват в Sentry

1. **Проверете конфигурацията на DSN**
   ```bash
   echo $NEXT_PUBLIC_SENTRY_DSN
   ```
   Уверете се, че DSN е правилно зададен и достъпен.

2. **Проверете дали логовете са активирани**
   ```bash
   echo $SENTRY_LOGS_ENABLED
   ```
   Трябва да бъде `true` за препращане на логовете.

3. **Проверете инициализацията на Sentry**
   - Проверете дали `SENTRY_ENABLED` е true
   - Проверете конзолата на браузъра за грешки при инициализацията на Sentry
   - Уверете се, че `_experiments.enableLogs` е зададено на `true`

4. **Проверете филтрирането по ниво на логване**
   - Уверете се, че вашето ниво на логване отговаря на прага `SENTRY_LOGS_LEVEL`
   - Debug логовете се улавят само ако нивото е зададено на `debug`

### Съображения за Производителност

- Логовете се изпращат асинхронно и няма да блокират приложението
- В производство обмислете задаване на `SENTRY_LOGS_LEVEL=warn` за намаляване на обема на логовете
- Sentry автоматично обработва ограничаването на честотата и пакетирането

### Деактивиране на Логовете

За деактивиране на Sentry Logs без деактивиране на Sentry изцяло:

```env
SENTRY_LOGS_ENABLED=false
```

Logger-ът ще продължи да работи нормално, но логовете няма да бъдат препращани към Sentry.

## Най-Добри Практики

1. **Използвайте Подходящи Нива на Логване**
   - Използвайте `debug` за подробна информация при разработка
   - Използвайте `info` за общия поток на приложението
   - Използвайте `warn` за потенциални проблеми, които не нарушават функционалността
   - Използвайте `error` за реални грешки и изключения

2. **Включвайте Контекст**
   - Използвайте контекстни logger-и за по-добра организация
   - Включвайте подходящи метаданни в данните на лога

3. **Избягвайте Чувствителни Данни**
   - Никога не логвайте пароли, токени или лични данни
   - Почиствайте данните преди логване

4. **Конфигурация на Производство**
   - Задайте `SENTRY_LOGS_LEVEL=warn` в производство
   - Мониторинг на използването на квотата на Sentry
   - Редовно преглеждайте логовете за модели

## Контролен Списък за Валидация

- [ ] DSN на Sentry е правилно конфигуриран
- [ ] Зададено е `SENTRY_LOGS_ENABLED=true`
- [ ] Логовете се появяват в Logs Explorer на Sentry
- [ ] Нивата на логовете са правилно съпоставени (info, warn, error, debug)
- [ ] Таговете за контекст са видими в Sentry
- [ ] Логовете работят както локално, така и в средите за внедряване
- [ ] QA може да вижда и филтрира логовете в Logs Explorer на Sentry
