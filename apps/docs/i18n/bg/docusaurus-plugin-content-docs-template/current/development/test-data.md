# Тестови Данни и Фикстури

E2E тестовете в шаблона Ever Works разчитат на стабилни тестови данни, управлявани чрез TypeScript константи, фикстури за удостоверяване и seed скриптове.

## `TEST_DATA` Константи

Основните тестови данни са дефинирани в `apps/web-e2e/fixtures/test-data.ts`:

```typescript
export const TEST_DATA = {
  admin: {
    email: 'admin@example.com',
    password: 'Admin123!',
    name: 'Admin User',
  },
  client: {
    email: 'client@example.com',
    password: 'Client123!',
    name: 'Client User',
  },
  company: {
    name: 'Test Company',
    slug: 'test-company',
    description: 'A test company for E2E testing',
  },
  // ... etc.
};
```

## `PUBLIC_ROUTES` Масив

Публично достъпните маршрути се тестват без удостоверяване:

```typescript
export const PUBLIC_ROUTES = [
  '/',
  '/en',
  '/en/about',
  '/en/contact',
  '/en/companies',
  '/en/blog',
  // ... etc.
];
```

## Фикстури за Удостоверяване

Playwright използва запазени сесийни файлове за обход на интерфейса за вход в теамовете, изискващи удостоверяване.

### `admin.json`

Разполага се в `apps/web-e2e/fixtures/admin.json` след изпълнение на глобалното настройване:

```json
{
  "cookies": [
    {
      "name": "authjs.session-token",
      "value": "...",
      "domain": "localhost",
      "path": "/",
      "httpOnly": true,
      "secure": false
    }
  ],
  "origins": []
}
```

### `client.json`

Аналогично на `admin.json`, но за потребителска сесия с клиентски роля.

## Попълване на Базата Данни за Тестове

Преди E2E тестовете трябва да се изпълни seed скриптът, за да се гарантира наличието на тестовите данни:

```bash
cd apps/web
pnpm run db:seed
```

Seed скриптът `scripts/seed.ts` създава:

- **Администраторски потребители**: Акаунти с пълен достъп за тестване на административни функции
- **Клиентски потребители**: Стандартни акаунти за тестване на потребителски работни процеси
- **Примерни компании**: Обяви в директорията с пълни профили
- **Категории**: Структура на таксономията
- **Примерен контент**: Публикации в блога, коментари, рецензии

## Демо Режим

Когато `DEMO_MODE=true` е зададен, приложението попълва инициалните данни, ако базата данни е празна:

```bash
DEMO_MODE=true pnpm run dev
```

Полезно за прегледи и демонстрации без ръчно изпълнение на seed.

## Стратегии за Последователност на Данните

| Среда       | Стратегия                                    | Команда              |
|-------------|----------------------------------------------|----------------------|
| Разработка  | Ръчен seed + локална база данни              | `pnpm db:seed`       |
| Тестване    | Автоматичен seed в CI преди тестовете        | Конфигурира CI/CD    |
| Продукция   | Само миграции — без произволен seed          | `pnpm db:migrate`    |

## Добри Практики

1. **Никога не зависете от реда на тестовете** — Всеки тест трябва да може да се изпълни самостоятелно
2. **Използвайте фикстури за удостоверяване** — Ускорява тестовете спрямо вход в UI при всеки тест
3. **Почистете след себе си** — Тестовете, създаващи данни, трябва да почистват след изпълнение
4. **Константи вместо инлайн стринг** — Управлявайте тестовите данни централно в `TEST_DATA`
5. **Изолирайте тестовата среда** — Използвайте отделна база данни или схема за тестовете

## Промен­ли на Средата за Тестване

```bash
# Задайте в apps/web-e2e/.env или apps/web/.env.test
DATABASE_URL=file:./test.db           # Изолирана тестова база данни
BASE_URL=http://localhost:3000         # Базов URL за тестовете
TEST_ADMIN_EMAIL=admin@example.com
TEST_ADMIN_PASSWORD=Admin123!
```

## Свързани Файлове

- `apps/web-e2e/fixtures/test-data.ts` — Централни константи за тестови данни
- `apps/web-e2e/global-setup.ts` — Настройване на сесия преди всички тестове
- `apps/web-e2e/global-teardown.ts` — Почистване след всички тестове
- `apps/web/scripts/seed.ts` — Seed скрипт за базата данни
- `apps/web/scripts/clean-database.js` — Скрипт за нулиране на базата данни
