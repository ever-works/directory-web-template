# Ръководство за Отзивчиво Тестване

Това ръководство обхваща добрите практики за тестване на отзивчив дизайн на различни устройства и размери на екрана.

## Контролен Списък за Тестване

### 1. Навигация

- [ ] **Мобилни**: Бутонът-хамбургер е видим и функционален
- [ ] **Настолни**: Хоризонталната навигационна лента се показва правилно
- [ ] **Всички устройства**: Всички навигационни връзки са достъпни
- [ ] **Зони за докосване**: Минимум 44x44px на мобилни
- [ ] **Навигация с клавиатура**: Редът на табулация е логичен

### 2. Съдържание

- [ ] **Четимост на текста**: Не е необходимо мащабиране за четене
- [ ] **Изображения**: Отзивчиви и правилно мащабирани на всяка точка на прекъсване
- [ ] **Без хоризонтален скрол**: Съдържанието се вмества в прозореца за преглед
- [ ] **Дължина на реда**: Оптимална ширина за четене (45-75 символа)
- [ ] **Размери на шрифта**: Подходящи за всеки размер устройство

### 3. Взаимодействия

- [ ] **Зони за докосване**: Минимум 44x44px за мобилни
- [ ] **Отстоп**: Достатъчно пространство между кликабилните елементи
- [ ] **Hover състояния**: Само на устройства, поддържащи hover
- [ ] **Focus състояния**: Видими индикатори за фокус на клавиатурата
- [ ] **Формуляри**: Удобни за попълване на мобилни

### 4. Производителност

- [ ] **Скорост на зареждане**: < 3 секунди при 3G връзка
- [ ] **Изображения**: Оптимизирани за всеки размер на екрана
- [ ] **Анимации**: Плавна производителност 60 FPS
- [ ] **Core Web Vitals**: Отговарят на праговете на Google
- [ ] **Размер на пакета**: Оптимизирани JavaScript и CSS

### 5. Оформление

- [ ] **Мрежа**: Коректно адаптиране в точките на прекъсване
- [ ] **Flexbox/Grid**: Без нарушения на оформлението
- [ ] **Отстояния**: Последователни отстъпи и полета
- [ ] **Подравняване**: Правилно подравняване при всички размери
- [ ] **Overflow**: Без проблеми с излизане на съдържание извън границите

## Инструменти за Тестване

### DevTools на Браузъра

#### Chrome DevTools
1. Отворете DevTools (F12 или Cmd+Option+I)
2. Натиснете иконата за панел на устройствата (Cmd+Shift+M)
3. Изберете устройство или въведете персонализирани размери
4. Тествайте различни скорости на мрежата

#### Firefox Developer Tools
1. Отворете DevTools (F12)
2. Натиснете „Режим на отзивчив дизайн" (Cmd+Option+M)
3. Изберете предварителни настройки за устройства
4. Тествайте докосвателни събития

### Онлайн Услуги за Тестване

- **[BrowserStack](https://www.browserstack.com/)** - Тестване на реални устройства
- **[LambdaTest](https://www.lambdatest.com/)** - Кросс-браузърно тестване
- **[Sauce Labs](https://saucelabs.com/)** - Платформа за автоматизирано тестване

### Тестване на Производителността

- **[Lighthouse](https://developers.google.com/web/tools/lighthouse)** - Одит на производителността
- **[WebPageTest](https://www.webpagetest.org/)** - Детаен анализ на производителността
- **[PageSpeed Insights](https://pagespeed.web.dev/)** - Инструмент за производителност на Google

## Целеви Метрики

### Оценки Lighthouse

| Метрика              | Цел   | Критична |
|----------------------|-------|----------|
| Производителност     | > 90  | > 50     |
| Достъпност           | > 95  | > 80     |
| Добри Практики       | > 95  | > 80     |
| SEO                  | > 95  | > 80     |

### Core Web Vitals

| Метрика | Добро | Нужно Подобрение | Лошо |
|---------|-------|------------------|------|
| **LCP** (Largest Contentful Paint) | < 2.5s | 2.5s - 4.0s | > 4.0s |
| **FID** (First Input Delay) | < 100ms | 100ms - 300ms | > 300ms |
| **CLS** (Cumulative Layout Shift) | < 0.1 | 0.1 - 0.25 | > 0.25 |

### Мобилни Метрики

- **First Contentful Paint**: < 1.8s
- **Time to Interactive**: < 3.8s
- **Размер на зоната за докосване**: >= 48x48px
- **Закъснение при докосване**: < 300ms

## Работен Процес за Тестване

### 1. Локално Тестване

```bash
# Стартиране на development сървъра
npm run dev

# Отваряне в браузъра
http://localhost:3000
```

### 2. Тестване на Устройства

1. **Използвайте отзивчив режим на DevTools**
2. **Тествайте на физически устройства** когато е възможно
3. **Използвайте отдалечено отстраняване на грешки** за мобилни устройства
4. **Тествайте различни ориентации** (портрет/пейзаж)

### 3. Автоматизирано Тестване

```bash
# Изпълнение на Lighthouse одит
npm run lighthouse

# Изпълнение на тестове за достъпност
npm run test:a11y

# Изпълнение на тестове за визуална регресия
npm run test:visual
```

## Чести Проблеми и Решения

### Проблем: Хоризонтален Скрол на Мобилни

**Решение**: Проверете за елементи с фиксирана ширина

```css
/* ❌ Лошо */
.container {
  width: 1200px;
}

/* ✅ Добро */
.container {
  max-width: 1200px;
  width: 100%;
  padding: 0 1rem;
}
```

### Проблем: Прекалено Малък Текст на Мобилни

**Решение**: Използвайте отзивчиви размери на шрифтовете

```css
/* ❌ Лошо */
body {
  font-size: 12px;
}

/* ✅ Добро */
body {
  font-size: 16px; /* Основен размер за мобилни */
}

@media (min-width: 768px) {
  body {
    font-size: 18px;
  }
}
```

### Проблем: Прекалено Малки Зони за Докосване

**Решение**: Осигурете минимален размер 44x44px

```css
/* ✅ Добро */
.button {
  min-height: 44px;
  min-width: 44px;
  padding: 12px 24px;
}
```

### Проблем: Неотзивчиви Изображения

**Решение**: Използвайте техники за отзивчиви изображения

```jsx
// ✅ Добро - компонент Image на Next.js
import Image from 'next/image';

<Image
  src="/image.jpg"
  alt="Description"
  width={800}
  height={600}
  sizes="(max-width: 768px) 100vw, (max-width: 1200px) 50vw, 33vw"
  priority
/>
```

## Добри Практики

### 1. Подход Mobile-First
- Разработвайте първо за мобилни, след това подобрявайте за по-големи екрани
- Използвайте медийни заявки `min-width` вместо `max-width`

### 2. Прогресивно Подобрение
- Уверете се, че основната функционалност работи без JavaScript
- Добавяйте подобрения за поддържащите браузъри

### 3. Дизайн, Подходящ за Докосване
- Минимални зони за докосване 44x44px
- Достатъчни отстояния между интерактивните елементи
- Избягвайте взаимодействия само чрез hover

### 4. Оптимизация на Производителността
- Мързеливо зареждане на изображения и компоненти
- Минимизирайте размера на JavaScript пакета
- Използвайте разделяне на кода
- Оптимизирайте шрифтовете и ресурсите

### 5. Достъпност
- Тествайте с четци на екрана
- Уверете се, че навигацията с клавиатура работи
- Поддържайте достатъчен контраст на цветовете
- Предоставяйте текстови алтернативи за изображения
