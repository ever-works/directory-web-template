# Интернационализация (i18n)

Ever Works Template използва `next-intl` за интернационализация, поддържайки 21 езикови стандарта с маршрутизация базирана на URL, автоматично разпознаване от браузъра, резервни съобщения, оформления отдясно наляво и SEO hreflang тагове. Тази страница документира i18n архитектурата, конфигурацията и шаблоните за използване.

## Архитектура

| Файл | Предназначение |
|------|---------|
| `i18n/routing.ts` | Конфигурация на маршрутизацията (поддържани езици, стратегия за префикси) |
| `i18n/request.ts` | Разпознаване на език и зареждане на съобщения за всяка заявка |
| `i18n/navigation.ts` | Link, redirect, usePathname, useRouter с поддръжка на езика |
| `lib/constants.ts` | Списък с езици, език по подразбиране, RTL дефиниции |
| `lib/seo/hreflang.ts` | Генериране на SEO hreflang тагове за търсачки |
| `components/language-switcher.tsx` | Превключвател на езика в интерфейса |
| `messages/*.json` | Файлове с преводи (по един на език) |
| `app/[locale]/layout.tsx` | Основно оформление с NextIntlClientProvider |

## Поддържани езици

Шаблонът дефинира 21 езикови стандарта в `lib/constants.ts`:

```typescript
export const DEFAULT_LOCALE = 'en';
export const LOCALES = [
  'en', 'fr', 'es', 'de', 'zh', 'ar', 'he',
  'ru', 'uk', 'pt', 'it', 'ja', 'ko', 'nl',
  'pl', 'tr', 'vi', 'th', 'hi', 'id', 'bg'
] as const;

export type Locale = (typeof LOCALES)[number];

/** Езикови стандарти, използващи посока на текста отдясно наляво */
export const RTL_LOCALES: readonly Locale[] = ['ar', 'he'] as const;
```

|------|----------|-----------|

| `fr` | Френски | LTR |
| `es` | Испански | LTR |
| `de` | Немски | LTR |
| `zh` | Китайски опростен | LTR |
| `ar` | Арабски | **RTL** |
| `he` | Иврит | **RTL** |
| `bg` | Български | LTR |
| `ru` | Руски | LTR |
| `nl` | Нидерландски | LTR |

## Конфигурация на маршрутизацията

Модулът за маршрутизация в `i18n/routing.ts` дефинира съответствието на езиците към URL адреси:

```typescript
import { defineRouting } from "next-intl/routing";
import { DEFAULT_LOCALE, LOCALES } from "@/lib/constants";

export const routing = defineRouting({
  locales: LOCALES,
  defaultLocale: DEFAULT_LOCALE,
  localeDetection: true,
  localePrefix: "as-needed",
});
```

### Стратегия за префикси

Стратегията `"as-needed"` запазва чисти URL адреси за езика по подразбиране:

| URL | Разпознат език | Обяснение |
|-----|-----------------|-------------|
| `/about` | `en` (по подразбиране) | Префиксът не е нужен за езика по подразбиране |
| `/fr/about` | `fr` | Префиксът е задължителен за другите езици |
| `/ar/about` | `ar` | Префиксът е задължителен за другите езици |

### Разпознаване на езика

При `localeDetection: true` системата автоматично проверява:

1. Cookie на езика (задавана от компонента LanguageSwitcher)
2. HTTP заглавие `Accept-Language` от браузъра
3. Връща към `DEFAULT_LOCALE` (`'en'`) ако не е намерено съвпадение

## Зареждане на съобщения и резервен вариант

Модулът `i18n/request.ts` се изпълнява при всяка сървърна заявка:

```typescript
export default getRequestConfig(async ({ requestLocale }) => {
  let locale = await requestLocale;

  if (!locale || !routing.locales.includes(locale as any)) {
    locale = routing.defaultLocale;
  }

  const userMessages = (await import(`../messages/${locale}.json`)).default;
  const defaultMessages = (await import(`../messages/en.json`)).default;
  const messages = deepmerge(defaultMessages, userMessages);

  return { locale, messages };
});
```

Стратегията `deepmerge` гарантира, че непреведените ключове показват английски текст.

## Файлове с преводи

Съобщенията за преводи се съхраняват като JSON файлове в директорията `messages/`:

```
messages/
  en.json    # Английски (източник, всички ключове присъстват)
  fr.json    # Френски
  bg.json    # Български
  ... (21 файла общо)
```

### Използване на преводи в компоненти

**Клиентски компоненти** използват `useTranslations` hook:

```tsx
import { useTranslations } from "next-intl";

function LoginButton() {
  const t = useTranslations("auth");
  return <button>{t("SIGN_IN")}</button>;
}
```

**Сървърни компоненти** използват асинхронния `getTranslations`:

```tsx
import { getTranslations } from "next-intl/server";

export default async function Page() {
  const t = await getTranslations("common");
  return <h1>{t("DISCOVER")}</h1>;
}
```

## Добавяне на нов език

1. Добавете кода на езика в масива `LOCALES` в `lib/constants.ts`
2. Ако езикът е RTL — добавете го в `RTL_LOCALES`
3. Създайте `messages/<код>.json` с преведени ключове
4. Добавете SVG флаг в `public/flags/<код>.svg`
5. Добавете запис на езика в `languageMap` в `components/language-switcher.tsx`
6. Добавете hreflang съпоставяне в `lib/seo/hreflang.ts`

## Добри практики

1. **Използвайте `Link` от `@/i18n/navigation`** вместо `next/link`
2. **Добавяйте нови ключове първо в `en.json`** тъй като това е езикът-източник
3. **Никога не хардкодвайте английски низове** в компоненти
4. **Разчитайте на резервния `deepmerge`** — частично преведените файлове работят правилно
5. **Тествайте RTL оформленията** като превключвате на арабски или иврит
