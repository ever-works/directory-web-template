# Определяне на Локал и Маршрутизиране

Шаблонът използва `next-intl` за разпознаване на езика с автоматично съпоставяне на езика на браузъра, URL-базирано маршрутизиране, запазване в cookie и система за резервни съобщения.

## Процес на разпознаване

При постъпване на заявка езикът се разпознава по следния начин:

1. **URL префикс** — Ако URL-ът съдържа езиков префикс (напр. `/fr/about`), този език се използва директно
2. **Cookie** — Ако няма URL префикс, системата проверява cookie на езика, зададена от компонента LanguageSwitcher
3. **Заглавие Accept-Language** — Ако няма cookie, се чете заглавието за езикови предпочитания на браузъра
4. **Резервен вариант** — Ако не е намерено съвпадение, се използва езикът по подразбиране (`en`)

## Изходни файлове

| Файл | Роля при разпознаването |
|------|-------------------|
| `i18n/routing.ts` | Дефинира поддържаните езици, стратегия за префикси |
| `i18n/request.ts` | Валидира разпознатия език, зарежда съобщения |
| `i18n/navigation.ts` | Предоставя Link, router, redirect с поддръжка на езика |
| `lib/constants.ts` | Единствен източник на истина за масива LOCALES и RTL_LOCALES |
| `components/language-switcher.tsx` | Задава cookie на езика чрез router.replace |
| `app/[locale]/layout.tsx` | Валидира езика, отхвърля невалидни чрез notFound() |

## Конфигурация на маршрутизацията

```typescript
import { defineRouting } from "next-intl/routing";
import { DEFAULT_LOCALE, LOCALES } from "@/lib/constants";

export const routing = defineRouting({
  locales: LOCALES,
  defaultLocale: DEFAULT_LOCALE,
  localeDetection: true,
  localePrefix: "as-needed",
});
```

### Стратегия за префикси

| Заявка | Разпознат език | Показван URL |
|---------|-----------------|-----------|
| `/about` | `en` | `/about` (без префикс за езика по подразбиране) |
| `/fr/about` | `fr` | `/fr/about` (префикс за другите езици) |
| `/en/about` | `en` | Пренасочен към `/about` |

## Логика за резервни съобщения

- Английските съобщения служат като базов слой с всички ключове
- Съобщенията за конкретен език презаписват само дефинираните ключове
- Липсващите ключове в езиковия файл запазват английската стойност
- Вложените обекти се обединяват рекурсивно

## Запазване в cookie

Когато потребителят избере език чрез LanguageSwitcher, `next-intl` задава cookie:

```typescript
const changeLanguage = useCallback(
  (locale: string) => {
    if (locale === currentLocale || isPending) return;

    startTransition(() => {
      router.replace(pathname, { locale });
    });
    setIsOpen(false);
  },
  [currentLocale, isPending, router, pathname]
);
```

## Разпознаване на Accept-Language

```
Accept-Language: fr-FR,fr;q=0.9,en-US;q=0.8,en;q=0.7
```

Системата търси съвпадение с поддържания масив `LOCALES`. Печели първият съвпаднал език.

## Отстраняване на проблеми с езика

| Проблем | Вероятна причина | Решение |
|---------|-------------|----------|
| Ключове за превод вместо текст | Липсва ключ в езиковия файл | Добавете ключа в `messages/en.json` (резервен) |
| Показва се грешен език | Cookie презаписва URL | Изчистете cookies или използвайте режим инкогнито |
| 404 при URL на езика | Езикът не е в масива LOCALES | Добавете код в `lib/constants.ts` |
| RTL оформлението не се прилага | Езикът не е в RTL_LOCALES | Добавете в `RTL_LOCALES` в `lib/constants.ts` |

## Добри практики

1. **Винаги използвайте `Link` от `@/i18n/navigation`** вместо `next/link`
2. **Добавяйте всички нови ключове първо в `en.json`** тъй като това е резервният език
3. **Тествайте разпознаването** като настройвате езиковите предпочитания на браузъра
4. **Разчитайте на резервния `deepmerge`** — частично преведените файлове са очаквани и обработвани
