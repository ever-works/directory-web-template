# Team-Schulungsleitfaden

Ein umfassendes Schulungsprogramm für das Onboarding und die Weiterbildung von Entwicklungsteams auf der Ever Works-Plattform.

## 🎯 Schulungsziele

Nach Abschluss dieses Schulungsprogramms werden Teammitglieder in der Lage sein:

- ✅ Ihre Entwicklungsumgebung einzurichten und zu konfigurieren
- ✅ Die Ever Works-Architektur und den Tech-Stack zu verstehen
- ✅ Etablierte Codierungsstandards und -konventionen einzuhalten
- ✅ APIs mit dem automatisierten Dokumentationssystem zu dokumentieren
- ✅ Funktionen gemäß Best Practices zu implementieren
- ✅ Häufige Probleme zu debuggen und zu beheben
- ✅ Selbstständig zum Codebase beizutragen

## 📚 Schulungsmodule

### 1. [Onboarding](/team-training/onboarding)

**Dauer**: 1–2 Tage  
**Zielgruppe**: Neue Teammitglieder

Erfahren Sie, wie Sie Ihre Entwicklungsumgebung einrichten, die Projektstruktur verstehen und Ihren ersten Beitrag leisten.

**Behandelte Themen**:
- Einrichtung und Konfiguration der Umgebung
- Überblick über die Projektstruktur
- Erste Bereitstellung
- Entwicklungsworkflow
- Git-Konventionen

### 2. [API-Dokumentation](/team-training/api-documentation)

**Dauer**: 2–3 Tage  
**Zielgruppe**: Alle Entwickler, die an API-Endpunkten arbeiten

Meistern Sie das automatisierte API-Dokumentationssystem mit Swagger-Annotationen und der Scalar UI.

**Behandelte Themen**:
- Swagger-Annotationssyntax
- Standardisierte Tag-Konventionen
- Dokumentationsgenerierungs-Workflow
- Qualitätsstandards
- Häufige Probleme beheben

### 3. [Best Practices](/team-training/best-practices)

**Dauer**: 1–2 Tage  
**Zielgruppe**: Alle Teammitglieder

Lernen Sie die Codierungsstandards, Muster und Best Practices, die in Ever Works verwendet werden.

**Behandelte Themen**:
- TypeScript-Best-Practices
- React-Muster und -Optimierung
- Datenbank-Best-Practices
- Sicherheitsrichtlinien
- Performance-Optimierung

### 4. [Praktische Übungen](/team-training/exercises)

**Dauer**: 3–5 Tage  
**Zielgruppe**: Alle Teammitglieder

Üben Sie das Gelernte anhand praxisnaher Übungen und Herausforderungen.

**Behandelte Themen**:
- Implementierung einer einfachen GET-Route
- POST-Route mit Validierung
- Vollständige Funktionsimplementierung
- Code-Review-Übungen

## 📅 Schulungszeitplan

### Neue Teammitglieder (2 Wochen)

**Woche 1: Grundlagen**
- Tag 1–2: [Onboarding](/team-training/onboarding)
- Tag 3–4: [API-Dokumentation](/team-training/api-documentation)
- Tag 5: [Best Practices](/team-training/best-practices)

**Woche 2: Praxis**
- Tag 1–3: [Praktische Übungen](/team-training/exercises)
- Tag 4–5: Erste echte Aufgabe mit Mentoring

### Bestehende Teammitglieder (3–5 Tage)

**Auffrischungsschulung**
- Tag 1: [Best Practices](/team-training/best-practices) überprüfen
- Tag 2: [API-Dokumentation](/team-training/api-documentation) aktualisierungen
- Tag 3: Übung mit [Übungen](/team-training/exercises)

## 📋 Voraussetzungen

### Erforderliche Kenntnisse

- **JavaScript/TypeScript**: Mittel bis fortgeschritten
- **React**: Fundiertes Verständnis von Hooks, Komponenten, State-Management
- **Next.js**: Grundlegende Kenntnisse (App Router bevorzugt)
- **Git**: Grundlagen der Versionskontrolle
- **REST-APIs**: HTTP-Methoden, Statuscodes, JSON

### Empfohlene Kenntnisse

- **PostgreSQL**: Grundlegende SQL-Abfragen
- **Tailwind CSS**: Utility-First-CSS
- **Drizzle ORM**: Typsichere Datenbankabfragen
- **OAuth**: Authentifizierungsabläufe
- **Stripe/Payment-APIs**: Zahlungsabwicklung

### Zu installierende Werkzeuge

Stellen Sie vor Beginn der Schulung sicher, dass Sie haben:

- Node.js 20.19.0+ (nvm verwenden)
- pnpm ([installieren](https://pnpm.io/installation)) – der Monorepo-Paketmanager
- PostgreSQL 14+
- Git
- VS Code (oder bevorzugte IDE)

**Empfohlene VS Code-Erweiterungen**:
- ESLint
- Prettier
- Tailwind CSS IntelliSense
- Swagger Viewer
- REST Client
- Error Lens

## 🎓 Zertifizierung

Nach Abschluss des Schulungsprogramms sollten Teammitglieder in der Lage sein:

- [ ] Entwicklungsumgebung selbstständig einzurichten
- [ ] API-Endpunkte gemäß Standards zu erstellen und zu dokumentieren
- [ ] Funktionen mit etablierten Mustern zu implementieren
- [ ] Sauberen, wartbaren und performanten Code zu schreiben
- [ ] Probleme mit verfügbaren Werkzeugen zu debuggen
- [ ] Code-Reviews konsistent zu bestehen
