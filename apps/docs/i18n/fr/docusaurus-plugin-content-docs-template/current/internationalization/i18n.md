# Vue d'ensemble de l'internationalisation

Le Template Ever Works utilise `next-intl` pour l'internationalisation, supportant 21 locales avec routage basé sur les URL, détection automatique du navigateur, repli des messages, mises en page de droite à gauche et balises SEO hreflang.

## Architecture

| Fichier | Objectif |
|---------|---------|
| `i18n/routing.ts` | Configuration des routes (locales supportées, stratégie de préfixe) |
| `i18n/request.ts` | Résolution de locale par requête et chargement des messages |
| `i18n/navigation.ts` | Link, redirect, usePathname, useRouter avec prise en charge des locales |
| `lib/constants.ts` | Liste des locales, locale par défaut, définitions de locales RTL |
| `lib/seo/hreflang.ts` | Génération de balises hreflang SEO pour les moteurs de recherche |
| `components/language-switcher.tsx` | Sélecteur de langue côté utilisateur |
| `messages/*.json` | Fichiers de traduction (un par locale) |
| `app/[locale]/layout.tsx` | Layout racine avec NextIntlClientProvider |

## Locales supportées

Le template définit 21 locales dans `lib/constants.ts` :

```typescript
export const DEFAULT_LOCALE = 'en';
export const LOCALES = [
  'en', 'fr', 'es', 'de', 'zh', 'ar', 'he',
  'ru', 'uk', 'pt', 'it', 'ja', 'ko', 'nl',
  'pl', 'tr', 'vi', 'th', 'hi', 'id', 'bg'
] as const;

export type Locale = (typeof LOCALES)[number];

/** Locales utilisant la direction de texte de droite à gauche */
export const RTL_LOCALES: readonly Locale[] = ['ar', 'he'] as const;
```

| Code | Langue | Direction |
|------|--------|-----------|
| `en` | Anglais | LTR |
| `fr` | Français | LTR |
| `es` | Espagnol | LTR |
| `de` | Allemand | LTR |
| `zh` | Chinois simplifié | LTR |
| `ar` | Arabe | **RTL** |
| `he` | Hébreu | **RTL** |
| `ru` | Russe | LTR |
| `pt` | Portugais | LTR |
| `it` | Italien | LTR |
| `nl` | Néerlandais | LTR |
| `pl` | Polonais | LTR |
| `bg` | Bulgare | LTR |

## Configuration du routage

```typescript
export const routing = defineRouting({
  locales: LOCALES,
  defaultLocale: DEFAULT_LOCALE,
  localeDetection: true,
  localePrefix: "as-needed",
});
```

La stratégie `"as-needed"` signifie :
- La locale par défaut (`en`) n'a pas de préfixe URL : `/about`
- Toutes les autres locales ont un préfixe : `/fr/about`, `/de/about`

## Utilisation dans les composants

```tsx
import { useTranslations } from 'next-intl';

export default function HomePage() {
  const t = useTranslations('HomePage');
  return <h1>{t('title')}</h1>;
}
```

Pour la navigation avec prise en charge des locales :

```tsx
import { Link } from '@/i18n/navigation';

// Génère automatiquement des URL avec le bon préfixe de locale
<Link href="/about">À propos</Link>
```
