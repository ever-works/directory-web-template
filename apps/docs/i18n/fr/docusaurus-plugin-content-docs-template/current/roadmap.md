# Feuille de route et direction future

Cette page présente la direction de développement actuelle du Modèle de site Ever Works et comment la communauté peut participer à façonner son avenir.

## Vision produit

Le Modèle de site Ever Works vise à être la solution open-source la plus complète pour la création de sites d'annuaires professionnels. La vision à long terme englobe :

- **Sites d'annuaires de qualité production** – beaux, performants et entièrement personnalisables
- **Gestion de contenu facile** – via le CMS basé sur Git avec génération de contenu optionnelle par IA via la [Plateforme Ever Works](https://docs.ever.works)
- **Paiement et authentification extensibles** – prenant en charge plusieurs fournisseurs dès le départ
- **Internationalisation de premier ordre** – avec support RTL complet et couverture linguistique croissante

## Domaines en développement actif

### Performance et Core Web Vitals

- Optimisation du Largest Contentful Paint (LCP) pour les pages de liste et de détail d'éléments
- Réduction de la taille du bundle JavaScript via un meilleur découpage du code
- Amélioration du pipeline d'optimisation des images pour les captures d'écran et logos
- Implémentation du pré-rendu partiel pour des chargements de page initiaux plus rapides

### Améliorations des fonctionnalités

- Ajout de plus de capacités de filtrage et de recherche (recherche facettée, filtres avancés)
- Implémentation de fonctionnalités de contenu généré par les utilisateurs (avis, notes, commentaires)
- Ajout de plus d'intégrations de fournisseurs de paiement
- Expansion du système de thèmes avec plus de thèmes intégrés

### Expérience développeur

- Amélioration de la configuration du développement local avec de meilleures docs et messages d'erreur
- Ajout d'une couverture de tests E2E plus complète avec Playwright
- Création de modèles de démarrage pour les types d'annuaires courants
- Amélioration de la sécurité des types TypeScript dans toute la base de code

### Internationalisation

- Ajout de plus de traductions de langues intégrées
- Amélioration du support de mise en page RTL pour l'arabe et l'hébreu
- Prise en charge de la configuration linguistique par annuaire
- Ajout de flux de traduction automatisés

## Comment proposer des fonctionnalités

### GitHub Issues

Soumettez vos propositions sur [GitHub Issues](https://github.com/ever-works/directory-web-template/issues) en utilisant le modèle "Feature Request".

### Discord

Rejoignez notre [communauté Discord](https://discord.gg/ever) pour discuter des idées de fonctionnalités avec les autres membres de la communauté.
