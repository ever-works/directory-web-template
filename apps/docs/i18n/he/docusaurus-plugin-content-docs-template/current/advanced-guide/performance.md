# מיטוב ביצועים

מדריך זה מכסה את אופטימיזציות הביצועים המובנות בתבנית Ever Works וטכניקות לשמירה על זמני טעינה מהירים ככל שהיישום שלך גדל.

## תצורת Next.js `next.config.ts` של התבנית כולל מספר הגדרות ממוקדות ביצועים:

### פלט עצמאי

```typescript
const nextConfig: NextConfig = {
  output: "standalone",
  // ...
};
```

מצב הפלט `standalone` יוצר מבנה עצמאי הכולל רק את הקבצים הדרושים להפעלת היישום. זה מקטין את גודל המכולה ואת זמן האתחול בייצור.

### אופטימיזציה של ייבוא ​​חבילות

```typescript
experimental: {
  optimizePackageImports: ["@heroui/react", "lucide-react"],
},
```

הגדרה זו מאפשרת ניעור עצים עבור חבילות כבדות בתיק חבית. במקום לייבא את כל ספריית `@heroui/react` או `lucide-react` , רק הרכיבים שבהם נעשה שימוש בפועל כלולים בחבילה.

### Webpack Watch Optimization

```typescript
if (dev) {
  config.watchOptions = {
    ...config.watchOptions,
    ignored: ['**/node_modules/**', '**/.git/**', '**/.content/**']
  };
}
```

ספריית `.content/` (CMS מבוסס-Git עם 220+ קבצי סימון) אינה נכללת בצופה הקבצים של webpack בפיתוח. זה מונע בנייה מחדש מיותרת כאשר קבצי תוכן משתנים ומפחית באופן משמעותי את השימוש במעבד במהלך הפיתוח.

### אזהרות כבויות

רישום תשתית מדוכא בסביבות CI ו-Vercel:

```typescript
if (process.env.CI || process.env.VERCEL) {
  config.infrastructureLogging = { level: 'error' };
}
```

## מיטוב תמונה

### תבניות מרוחקות

התבנית יוצרת באופן דינמי דפוסי תמונה מרוחקים מותרים באמצעות `generateImageRemotePatterns()` . זה מבטיח שתמונות מ-CDN מוגדרות וממקורות חיצוניים עוברות אופטימיזציה דרך צינור התמונות המובנה של Next.js.

### טיפול ב-SVG

```typescript
images: {
  dangerouslyAllowSVG: true,
  contentDispositionType: 'attachment',
  contentSecurityPolicy: "default-src 'self'; script-src 'none'; sandbox;",
  unoptimized: false,
},
```

תמונות SVG מותרות אך בארגז חול עם מדיניות אבטחת תוכן קפדנית המשביתה את ביצוע הסקריפט. זה מאפשר לוגו ואייקונים של SVG תוך מניעת XSS באמצעות הזרקת SVG.

### שיטות עבודה מומלצות לתמונות

| טכניקה | יישום | השפעה |
|---|---|---|
| השתמש ב- `next/image` | רכיב מובנה עם טעינה עצלנית | WebP/AVIF אוטומטי, גדלים רספונסיביים |
| הגדר מידות מפורשות | אביזרי `width` ו `height` | מונע שינוי פריסה מצטבר (CLS) |
| השתמש ב- `priority` עבור LCP | `<Image priority />` לתמונות גיבורים | טען מראש תמונת הצבע המכיל תוכן הגדול ביותר |
| השתמש ב- `sizes` prop | `sizes="(max-width: 768px) 100vw, 50vw"` | מונע הורדת תמונות גדולות מדי |
| טשטוש מצייני מיקום | `placeholder="blur"` עם `blurDataURL` | משפר את מהירות הטעינה הנתפסת |

## אסטרטגיות שמירה במטמון

### כותרות HTTP

התבנית מגדירה כותרות הקשורות למטמון ב- `next.config.ts` :

```typescript
headers: [
  { key: "X-DNS-Prefetch-Control", value: "on" },
]
```

שליפה מוקדמת של DNS מופעלת באופן גלובלי כדי להפחית את זמן האחזור של חיפוש DNS עבור משאבים חיצוניים.

### דור סטטי

התבנית משתמשת בפסק זמן נדיב ליצירת דפים סטטיים:

```typescript
staticPageGenerationTimeout: 180, // 3 minutes
```

זה מתאים לדפים שמביאים נתונים מממשקי API חיצוניים או מ-CMS מבוסס Git במהלך זמן הבנייה.

### תצורת ETag

```typescript
generateEtags: false,
```

תגי ET מושבתים ברמת Next.js מכיוון שה-CDN/פרוקסי הפוך (Vercel Edge Network או Cloudflare) מטפל באימות מטמון בצורה יעילה יותר.

### מטמון ברמת יישום

מעבד הרקע האנליטי מחמם מראש מטמונים במרווחי זמן קבועים:

| סוג מטמון | מרווח רענון | נתונים |
|---|---|---|
| מגמות צמיחת משתמשים | 10 דקות | גידול משתמשים חודשי במשך 6, 12, 24 חודשים |
| מגמות פעילות | 5 דקות | נתוני פעילות עבור חלונות 7, 14, 30 יום |
| דירוג פריטים מובילים | 15 דקות | 10, 20, 50 פריטים המובילים |
| פעילות אחרונה | 2 דקות | 10 ו-20 רשומות אחרונות לפעילות |
| מדדי ביצועים | 30 שניות | סטטיסטיקות ביצועי שאילתות |
| ניקוי מטמון | 1 שעה | הסרת ערך מטמון שפג תוקפו |

## טעינה עצלנית

### טעינה עצלנית ברמת הרכיב

השתמש ב- `next/dynamic` עבור רכיבים כבדים שאינם נחוצים בעיבוד הראשוני:

```typescript
import dynamic from 'next/dynamic';

const HeavyChart = dynamic(() => import('@/components/charts/HeavyChart'), {
  loading: () => <ChartSkeleton />,
  ssr: false, // disable SSR for client-only components
});
```

### פיצול קוד ברמת המסלול

נתב האפליקציה Next.js מתחלק אוטומטית לפי מסלול. כל עמוד ב- `app/[locale]/` מקבל חבילה משלו, כך שמשתמשים מורידים רק את ה-JavaScript הדרוש עבור הדף הנוכחי.

### יבוא דינמי בעבודות רקע

התבנית משתמשת בייבוא דינמי בתוך התקשרויות חוזרות לעבודה כדי למנוע מה-webpack למשוך מודולים לשרת בלבד לחבילת הלקוח:

```typescript
manager.scheduleJob('repository-sync', 'Repository Synchronization', async () => {
  const { syncManager } = await import('@/lib/services/sync-service');
  await syncManager.performSync();
}, 5 * 60 * 1000);
```

## אופטימיזציה של גודל החבילה

### ניתוח החבילה

הפעל את הפעולות הבאות כדי לבדוק את הרכב החבילה:

```bash
ANALYZE=true pnpm build
```

אם `@next/bundle-analyzer` מוגדר, הדבר מייצר מפת עץ אינטראקטיבית המציגה אילו מודולים תורמים לגודל החבילה.

### טכניקות אופטימיזציה נפוצות

| טכניקה | דוגמה | חיסכון |
|---|---|---|
| אופטימיזציה של קבצי חבית | `optimizePackageImports` בתצורה | מונע ייבוא ​​של ספריות אייקונים/ממשק משתמש שלמות |
| מודולים לשרת בלבד | `import 'server-only'` בקבצי lib | מונע צרור מקרי של לקוחות |
| יבוא דינמי | `await import('@/lib/services/...')` | דוחה טעינה עד הצורך |
| חבילות חיצוניות | `serverExternalPackages: ['postgres', 'bcryptjs', 'drizzle-orm']` | לא כולל חבילת חבילות אינטרנט |

התצורה `serverExternalPackages` חשובה במיוחד:

```typescript
serverExternalPackages: ['postgres', 'bcryptjs', 'drizzle-orm'],
```

חבילות אלו אינן נכללות בחבילת webpack ונטענות באופן מקורי בזמן ריצה, מצמצמות את זמן הבנייה ומונעות בעיות תאימות עם מודולים מקוריים.

## טיפים לאופטימיזציה של מגדלור

### יעדי ליבה של חיוני אינטרנט

| מדד | יעד | גורמי מפתח |
|---|---|---|
| **LCP** (הצבע התוכן הגדול ביותר) | < 2.5 שניות | אופטימיזציה של תמונה, טעינת עדיפות, זמן תגובה של שרת |
| **FID** (השהיית קלט ראשון) | < 100ms | פיצול קוד, חסימת חוט ראשי מינימלית |
| **CLS** (שינוי פריסה מצטבר) | < 0.1 | ממדי תמונה מפורשים, אסטרטגיית טעינת גופנים |
| **TTFB** (זמן עד בייט ראשון) | < 800ms | מטמון CDN, פונקציות קצה, אופטימיזציה של שאילתות מסד נתונים |

### רשימת בדיקה מעשית

1. **תמונות**: השתמשו ב- `next/image` עם אביזרים מפורשים `width` , `height` ו- `sizes` . סמן תמונות מעל הקפל עם `priority` .
2. **גופנים**: השתמש ב- `next/font` כדי לארח גופנים עם `display: swap` ולטעון מראש קובצי גופן קריטיים.
3. **JavaScript**: סקור את `optimizePackageImports` והוסף את כל הספריות הגדולות המשתמשות בקבצי Barrel.
4. **CSS**: התבנית משתמשת ב-Tailwind CSS, שכבר מטוהרת בבניית ייצור. הימנע מיבוא מודולי CSS שאינם בשימוש.
5. **סקריפטים של צד שלישי**: דחה סקריפטים לא קריטיים באמצעות `next/script` עם `strategy="lazyOnload"` .
6. **רכיבי שרת**: ברירת המחדל לרכיבי שרת תגובה (RSC) והשתמשו ב- `"use client"` רק כאשר נדרשת אינטראקטיביות.

### מגדלור פועל

התבנית כוללת תצורה `lighthouse-test.json` . הפעל בדיקות Lighthouse אוטומטיות:

```bash
npx lhci autorun --config=lighthouse-test.json
```

לחלופין, השתמש בחלונית Chrome DevTools Lighthouse לביקורות ידניות.

## ביצועי שאילתות מסד נתונים

### איחוד חיבורים

השתמש באיגוד חיבורים כדי להימנע מפתיחת חיבור מסד נתונים חדש לכל בקשה. עיין ב[מדריך קנה המידה](/פריסה/קנה מידה) לפרטי תצורה.

### מיטוב שאילתות

- השתמש בדפוס המאגר ( `lib/repositories/` ) כדי לרכז ולייעל שאילתות.
- מאגר הניתוח כולל שכבות מטמון מובנות עם TTL שניתן להגדרה.
- עקוב אחר שאילתות איטיות באמצעות עבודת הרקע של מדדי ביצועים.

### אסטרטגיית אינדקס

סקור `lib/db/schema.ts` עבור אינדקסים קיימים. הוסף אינדקסים עבור:
- עמודות המשמשות בסעיפים `WHERE` - עמודות מפתח זר
- עמודות המשמשות בסעיפים `ORDER BY` - אינדקסים מורכבים עבור חיפושים מרובי עמודות

## ניטור ביצועים

### אינטגרציה של Sentry

התבנית משלבת את Sentry לניטור ביצועים ב- `instrumentation.ts` :

```typescript
Sentry.init({
  tracesSampleRate: process.env.NODE_ENV === 'production' ? 0.1 : 1.0,
});
```

עקבות נדגמים ב-10% בייצור ו-100% בפיתוח. התאם את `tracesSampleRate` בהתבסס על נפח התנועה ומגבלות תוכנית Sentry.

### סמני ביצועים מותאמים אישית

השתמש ב-Web Performance API לתזמון מותאם אישית:

```typescript
performance.mark('data-fetch-start');
const data = await fetchData();
performance.mark('data-fetch-end');
performance.measure('data-fetch', 'data-fetch-start', 'data-fetch-end');
```

| לבנות | פלט עצמאי, חבילות חיצוניות | הפעל מנתח חבילות |
| ניטור | עקבות זקיף, משרת מדדי ביצועים | הגדר התראות עבור מדדים מושפלים |
