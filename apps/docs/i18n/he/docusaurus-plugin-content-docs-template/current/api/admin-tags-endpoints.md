# Admin Tags API נקודות קצה

ממשק ה-API של Admin Tags מספק פעולות CRUD מלאות לניהול תגי תוכן. תגיות משמשות לסיווג ולסינון של פריטים בספרייה. ה-API תומך ברישום מעומד, יצירה עם מצבים פעילים/לא פעילים, עדכונים, מחיקה ואחזור מודעת למיקום ממטמון התוכן. כל פעולות הכתיבה מבטלות תוקף של מטמון תוכן לחשיפה מיידית.

## סיכום מסלול

|שיטה|נתיב|Auth|תיאור|
|--------|------|------|-------------|
|`GET`|`/api/admin/tags`|מנהל מערכת|רשימת תגיות (בעמודים)|
|`POST`|`/api/admin/tags`|מנהל מערכת|צור תג חדש|
|`GET`|`/api/admin/tags/all`|מנהל מערכת|קבל את כל התגים (ממטמון התוכן)|
|`GET`|`/api/admin/tags/{id}`|מנהל מערכת|קבל תג בודד לפי תעודת זהות|
|`PUT`|`/api/admin/tags/{id}`|מנהל מערכת|עדכן תג|
|`DELETE`|`/api/admin/tags/{id}`|מנהל מערכת|מחק תג לצמיתות|

## אימות

כל נקודות הקצה לניהול תגים דורשות הרשאות מנהל:

```typescript
const session = await auth();
if (!session?.user?.isAdmin) {
  return NextResponse.json(
    { success: false, error: 'Unauthorized' },
    { status: 401 }
  );
}
```

## נקודות קצה

### קבל `/api/admin/tags`

מחזירה רשימה מעומדת של כל התגים במערכת. פרמטרי עידוד מאומתים באמצעות כלי השירות המשותף `validatePaginationParams`.

**פרמטרי שאילתה:**

|פרמטר|הקלד|ברירת מחדל|תיאור|
|-----------|------|---------|-------------|
|`page`|מספר שלם| `1` |מספר עמוד (מינימום: 1)|
|`limit`|מספר שלם| `10` |פריטים בעמוד (1--100)|

**תגובה (200):**

```json
{
  "success": true,
  "data": {
    "tags": [
      {
        "id": "productivity",
        "name": "Productivity",
        "isActive": true,
        "itemCount": 156,
        "created_at": "2024-01-20T10:30:00.000Z",
        "updated_at": "2024-01-20T10:30:00.000Z"
      },
      {
        "id": "design",
        "name": "Design",
        "isActive": true,
        "itemCount": 89,
        "created_at": "2024-01-19T15:20:00.000Z",
        "updated_at": "2024-01-19T15:20:00.000Z"
      }
    ],
    "total": 45,
    "page": 1,
    "limit": 10,
    "totalPages": 5
  }
}
```

### פרסם `/api/admin/tags`

יוצר תג חדש עם מזהה, שם ומצב פעיל אופציונלי. מבטל מטמון תוכן עם הצלחה.

**גוף הבקשה:**

```json
{
  "id": "artificial-intelligence",
  "name": "Artificial Intelligence",
  "isActive": true
}
```

|שדה|הקלד|חובה|תיאור|
|-------|------|----------|-------------|
|`id`|מחרוזת|כן|מזהה שבלול ידידותי לכתובות אתרים|
|`name`|מחרוזת|כן|שם תג קריא לאדם (2--50 תווים)|
|`isActive`|בוליאני|לא|האם התג פעיל (ברירת מחדל: `true`)|

**כללי אימות:**
- נדרשים גם `id` וגם `name`
- שם התג חייב להיות בין 2 ל-50 תווים
- מזהה התג חייב להיות ייחודי בכל התגים הקיימים
- שם התג חייב להיות ייחודי בכל התגים הקיימים

**תגובה (201):**

```json
{
  "success": true,
  "tag": {
    "id": "artificial-intelligence",
    "name": "Artificial Intelligence",
    "isActive": true,
    "itemCount": 0,
    "created_at": "2024-01-20T10:30:00.000Z",
    "updated_at": "2024-01-20T10:30:00.000Z"
  }
}
```

### קבל `/api/admin/tags/all`

מחזירה את כל התגים ממטמון התוכן עבור אזור נתון. נקודת קצה זו קוראת משכבת ​​התוכן המאוחסן במטמון ולא ממסד הנתונים, מה שהופך אותו למתאים לאכלוס בוררי תגים בממשק המשתמש של הניהול.

**פרמטרי שאילתה:**

|פרמטר|הקלד|ברירת מחדל|תיאור|
|-----------|------|---------|-------------|
|`locale`|מחרוזת|`"en"`|קוד מקומי לאחזור תוכן|

**תגובה (200):**

```json
{
  "success": true,
  "data": [
    {
      "id": "productivity",
      "name": "Productivity",
      "isActive": true,
      "itemCount": 156
    }
  ]
}
```

### קבל `/api/admin/tags/{id}`

מאחזר תג בודד לפי המזהה הייחודי שלו עם פרטים מלאים כולל סטטיסטיקת שימוש.

**פרמטרי נתיב:**

|פרמטר|הקלד|תיאור|
|-----------|------|-------------|
|`id`|מחרוזת|תג מזהה ייחודי|

**תגובה (200):**

```json
{
  "success": true,
  "data": {
    "id": "productivity",
    "name": "Productivity",
    "isActive": true,
    "itemCount": 156,
    "created_at": "2024-01-20T10:30:00.000Z",
    "updated_at": "2024-01-20T10:30:00.000Z"
  }
}
```

### PUT `/api/admin/tags/{id}`

עדכון שם ו/או סטטוס פעיל של תג. לא ניתן לשנות את מזהה התג לאחר היצירה. מבטל מטמון תוכן עם הצלחה.

**פרמטרי נתיב:**

|פרמטר|הקלד|תיאור|
|-----------|------|-------------|
|`id`|מחרוזת|תג מזהה ייחודי|

**גוף הבקשה:**

```json
{
  "name": "Productivity & Efficiency",
  "isActive": true
}
```

|שדה|הקלד|חובה|תיאור|
|-------|------|----------|-------------|
|`name`|מחרוזת|כן|שם תצוגת תג מעודכן|
|`isActive`|בוליאני|לא|סטטוס פעיל מעודכן|

**תגובה (200):**

```json
{
  "success": true,
  "data": {
    "id": "productivity",
    "name": "Productivity & Efficiency",
    "isActive": true,
    "itemCount": 156,
    "created_at": "2024-01-20T10:30:00.000Z",
    "updated_at": "2024-01-20T16:45:00.000Z"
  },
  "message": "Tag updated successfully"
}
```

### מחק `/api/admin/tags/{id}`

מוחק לצמיתות תג מהמערכת. זה גם מסיר את התג מכל הפריטים המשויכים. מבטל מטמון תוכן עם הצלחה.

**פרמטרי נתיב:**

|פרמטר|הקלד|תיאור|
|-----------|------|-------------|
|`id`|מחרוזת|תג מזהה ייחודי|

**תגובה (200):**

```json
{
  "success": true,
  "message": "Tag deleted successfully"
}
```

:::זהירות
מחיקת התג היא לצמיתות ולא ניתן לבטל אותה. כל שיוך הפריט-תג עבור התג שנמחק יוסרו. שקול לבטל את התג (הגדרת `isActive` ל-`false` באמצעות PUT) במקום זאת אם ברצונך לשמור על שלמות הנתונים.
:::

## מודל נתוני תג

|שדה|הקלד|ניתן לבטל|תיאור|
|-------|------|----------|-------------|
|`id`|מחרוזת|לא|מזהה ייחודי ידידותי ל-URL|
|`name`|מחרוזת|לא|שם תצוגה קריא לאדם|
|`isActive`|בוליאני|לא|האם ניתן להקצות את התג לפריטים|
|`itemCount`|מספר שלם|לא|מספר הפריטים המשתמשים בתג זה|
|`created_at`|תאריך שעה|לא|חותמת זמן ליצירה|
|`updated_at`|תאריך שעה|לא|חותמת זמן של עדכון אחרון|

## קודי שגיאה

|סטטוס|שגיאה|סיבה|
|--------|-------|-------|
| `400` |תג מזהה ושם נדרשים|חסרים שדות חובה ביצירה|
| `400` |נדרש שם תג|חסר שם בעדכון|
| `400` |שם התג חייב להיות בין 2 ל-50 תווים|כשל באימות אורך השם|
| `400` |פרמטר דף/מגבלה לא חוקי|פרמטר עימוד מחוץ לטווח|
| `401` |לא מורשה|הפעלה חסרה או שאינה מנהלת|
| `404` |התג לא נמצא|אין תג עם המזהה הנתון|
| `409` |תג עם מזהה כבר קיים|זיהוי כפול ביצירה|
| `409` |תג עם שם כבר קיים|שם כפול ביצירה/עדכון|
| `500` |אחזור/יצירה/עדכון/מחיקת תג נכשל|שגיאת שרת או מסד נתונים|

## אי תוקף מטמון

כל פעולות הכתיבה (יצירה, עדכון, מחיקה) התקשרו ל-`invalidateContentCaches()` כדי להבטיח ששינויי תג ישתקפו מיד בתוכן הפונה לציבור:

```typescript
await invalidateContentCaches();
```

פעולה זו מנקה גם את מטמון התוכן בזיכרון וכל מטמון ברמת CDN שעשויים להיות פעילים.

## מקורות נתונים

ממשק API של התגים משתמש בשני מקורות נתונים שונים בהתאם לנקודת הקצה:

|נקודת קצה|מקור נתונים|Use Case|
|----------|------------|----------|
|`GET /api/admin/tags`|`tagRepository` (מסד נתונים)|ניהול אדמין עם עימוד|
|`POST /api/admin/tags`|`tagRepository` (מסד נתונים)|יצירת תגים חדשים|
|`GET /api/admin/tags/all`|`getCachedItems()` (מטמון תוכן)|בוררים נפתחים, חיפושים מהירים|
|`GET /api/admin/tags/{id}`|`tagRepository` (מסד נתונים)|תצוגת תג מפורטת|
|`PUT /api/admin/tags/{id}`|`tagRepository` (מסד נתונים)|עדכון מאפייני התג|
|`DELETE /api/admin/tags/{id}`|`tagRepository` (מסד נתונים)|הסרת תגים|
