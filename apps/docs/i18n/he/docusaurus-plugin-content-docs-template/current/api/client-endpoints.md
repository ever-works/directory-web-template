# נקודות קצה של ממשק API

נקודות קצה של API הפונות ללקוח משרתות משתמשי קצה מאומתים (שאינם מנהלי מערכת). מסלולים אלה מטפלים בלוח המחוונים של הלקוח, הגשת פריטים, ניהול מועדפים ואינטראקציות עם פריטים ציבוריים כגון הערות, הצבעות וצפיות.

## לוח מחוונים ופריטים ללקוח (`/api/client`)

כל המסלולים `/api/client/*` דורשים הפעלה מאומתת עם `clientProfileId` חוקי.

### לוח מחוונים

|שיטה|נתיב|תיאור|
|--------|------|-------------|
|`GET`|`/api/client/dashboard/stats`|נתונים סטטיסטיים של לוח המחוונים של לקוחות (ספירת פריטים, צפיות, מעורבות)|

### פריטי לקוח

|שיטה|נתיב|תיאור|
|--------|------|-------------|
|`GET`|`/api/client/items`|רשימת פריטים שנשלחו על ידי הלקוח הנוכחי|
|`POST`|`/api/client/items`|שלח פריט חדש לבדיקה|
|`GET`|`/api/client/items/stats`|נתונים סטטיסטיים של פריט לקוח (פורסם, בהמתנה, נדחה)|
|`GET`|`/api/client/items/coordinates`|קבל קואורדינטות לפריטים של הלקוח|
|`GET`|`/api/client/items/[id]`|קבל פרטי פריט|
|`PUT`|`/api/client/items/[id]`|עדכן פריט משלו|
|`DELETE`|`/api/client/items/[id]`|מחק פריט משלו (מחיקה רכה)|
|`POST`|`/api/client/items/[id]/restore`|שחזר פריט שנמחק ברכות|

### סטטיסטיקות גיאוגרפיות

|שיטה|נתיב|תיאור|
|--------|------|-------------|
|`GET`|`/api/client/geo-stats`|סטטיסטיקה גיאוגרפית עבור פריטי לקוח|

## אינטראקציות עם פריטים ציבוריים (`/api/items`)

נקודות קצה אלו מטפלות בתכונות פריטים הפונות לציבור. חלקם דורשים אימות (למשל, הצבעה), בעוד שאחרים פומביים לחלוטין (למשל, צפייה).

### הערות

|שיטה|נתיב|תיאור|Auth|
|--------|------|-------------|------|
|`GET`|`/api/items/[slug]/comments`|רשום הערות לפריט|ציבורי|
|`POST`|`/api/items/[slug]/comments`|הוסף תגובה|חובה|
|`GET`|`/api/items/[slug]/comments/[commentId]`|קבל פרטי תגובה|ציבורי|
|`PUT`|`/api/items/[slug]/comments/[commentId]`|עדכן תגובה משלך|חובה|
|`DELETE`|`/api/items/[slug]/comments/[commentId]`|מחק תגובה משלך|חובה|

### דירוגי הערות

|שיטה|נתיב|תיאור|Auth|
|--------|------|-------------|------|
|`GET`|`/api/items/[slug]/comments/rating`|קבל סיכום דירוג|ציבורי|
|`POST`|`/api/items/[slug]/comments/rating`|שלח דירוג|חובה|
|`GET`|`/api/items/[slug]/comments/rating/[commentId]`|קבל דירוג לתגובה|ציבורי|

### הצבעות

|שיטה|נתיב|תיאור|Auth|
|--------|------|-------------|------|
|`GET`|`/api/items/[slug]/votes/count`|קבל ספירת קולות|ציבורי|
|`GET`|`/api/items/[slug]/votes/status`|קבל את סטטוס ההצבעה של המשתמש הנוכחי|חובה|
|`POST`|`/api/items/[slug]/votes`|הצבע על פריט (הצבעה למעלה/למטה)|חובה|

### צפיות

## מועדפים (`/api/favorites`)

|--------|------|-------------|

## דפוסי עימוד

נקודות קצה של רשימות הפונות ללקוח תומכות בפרמטרי העימוד הסטנדרטיים:

```
GET /api/client/items?page=1&limit=10&sort=createdAt&order=desc
GET /api/items/[slug]/comments?page=1&limit=20
GET /api/favorites?page=1&limit=50
```

התגובות כוללות מטא נתונים של עימוד:

```json
{
  "success": true,
  "data": [...],
  "pagination": {
    "page": 1,
    "limit": 10,
    "total": 45,
    "totalPages": 5
  }
}
```
