# נקודות קצה של Utility API

נקודות קצה של שירות מספקות שירותי תשתית כולל בדיקות תקינות, מידע על גרסאות, תצורת תכונה, קידוד גיאוגרפי, אימות reCAPTCHA, חילוץ URL, נתוני מיקום ופעולות פנימיות.

## בדיקת בריאות (`/api/health`)

### בריאות מסד הנתונים

|שיטה|נתיב|תיאור|
|--------|------|-------------|
|`GET`|`/api/health/database`|בדוק את קישוריות מסד הנתונים|

מחזירה את מצב החיבור למסד הנתונים. משמש על ידי מערכות ניטור ובדיקות תקינות פריסה.

**תגובה (בריאה):**

```json
{
  "status": "healthy",
  "database": "connected",
  "timestamp": "2025-01-15T10:30:00.000Z"
}
```

**תגובה (לא בריאה):**

```json
{
  "status": "unhealthy",
  "database": "disconnected",
  "error": "Connection refused",
  "timestamp": "2025-01-15T10:30:00.000Z"
}
```

**אימות:** ציבורי (אין צורך באימות). נקודת קצה זו צריכה להיות נגישה על ידי מאזני עומסים ושירותי ניטור.

## גרסה (`/api/version`)

|שיטה|נתיב|תיאור|
|--------|------|-------------|
|`GET`|`/api/version`|קבל מידע על גרסת האפליקציה|
|`GET`|`/api/version/sync`|קבל גרסה וסטטוס סנכרון|

### תגובת גרסה

מחזירה את גרסת היישום, מידע הבנייה וסביבת זמן הריצה:

```json
{
  "version": "1.0.0",
  "environment": "production",
  "timestamp": "2025-01-15T10:30:00.000Z"
}
```

### גרסה + סטטוס סנכרון

נקודת הקצה `/api/version/sync` מרחיבה את פרטי הגרסה עם סטטוס הסנכרון של מאגר התוכן, שימושית לאיתור שגיאות ברעננות התוכן.

**אימות:** ציבורי.

## תצורת תכונה (`/api/config`)

|שיטה|נתיב|תיאור|
|--------|------|-------------|
|`GET`|`/api/config/features`|קבל דגלי תכונה מופעלים|

מחזירה את תצורת דגל התכונה הנוכחית עבור היישום בצד הלקוח. זה מאפשר ל-frontend לעבד תכונות באופן מותנה בהתבסס על תצורת צד השרת.

**תגובה:**

```json
{
  "features": {
    "payments": true,
    "sponsorAds": true,
    "surveys": false,
    "map": true,
    "newsletter": true
  }
}
```

**אימות:** ציבורי. דגלי תכונה אינם נתונים רגישים.

## חילוץ כתובת אתר (`/api/extract`)

|שיטה|נתיב|תיאור|
|--------|------|-------------|
|`POST`|`/api/extract`|חלץ מטא נתונים מכתובת אתר|

שואב כתובת אתר ומחלץ מטא נתונים כולל כותרת, תיאור, תמונה וסמל. בשימוש בטופס הגשת הפריט כדי לאכלס אוטומטית שדות מכתובת אתר.

**בקשה:**

```json
{
  "url": "https://example.com/product"
}
```

**תגובה:**

```json
{
  "success": true,
  "data": {
    "title": "Product Name",
    "description": "Product description from meta tags",
    "image": "https://example.com/og-image.png",
    "favicon": "https://example.com/favicon.ico",
    "siteName": "Example.com"
  }
}
```

**אימות:** נדרש. מונע שימוש לרעה באחזור כתובת URL בצד השרת.

## קידוד גיאוגרפי (`/api/geocode`)

|שיטה|נתיב|תיאור|
|--------|------|-------------|
|`POST`|`/api/geocode`|קוד גיאוגרפי של כתובת לקואורדינטות|

ממירה כתובת טקסט לקואורדינטות גיאוגרפיות (קו רוחב/קו אורך) באמצעות שירות קידוד גיאוגרפי חיצוני.

**בקשה:**

```json
{
  "address": "1600 Amphitheatre Parkway, Mountain View, CA"
}
```

**תגובה:**

```json
{
  "success": true,
  "data": {
    "lat": 37.4224764,
    "lng": -122.0842499,
    "formattedAddress": "1600 Amphitheatre Pkwy, Mountain View, CA 94043, USA"
  }
}
```

**אימות:** נדרש.

## נתוני מיקום (`/api/location`)

נקודות קצה לחיפוש מיקום ונתוני הפניה.

|שיטה|נתיב|תיאור|
|--------|------|-------------|
|`GET`|`/api/location/countries`|רשום את כל המדינות|
|`GET`|`/api/location/cities`|רשימת ערים (עם מסנן מדינה)|
|`GET`|`/api/location/coordinates`|קבל קואורדינטות עבור מיקום|
|`GET`|`/api/location/search`|חפש מיקומים לפי מחרוזת שאילתה|

### מדינות

מחזירה רשימה של מדינות עם קודי ISO, שמות ומטא נתונים אופציונליים.

### ערים

תומך בסינון לפי קוד מדינה:

```
GET /api/location/cities?country=US
```

### חיפוש מיקום

חיפוש מיקום בטקסט מלא:

```
GET /api/location/search?q=San Francisco
```

**אימות:** ציבורי.

## אימות reCAPTCHA (`/api/verify-recaptcha`)

|שיטה|נתיב|תיאור|
|--------|------|-------------|
|`POST`|`/api/verify-recaptcha`|אמת אסימון reCAPTCHA|

אימות בצד השרת של אסימוני reCAPTCHA של Google. משמש על ידי טפסים הדורשים הגנת בוט.

**בקשה:**

```json
{
  "token": "recaptcha-response-token"
}
```

**תגובה:**

```json
{
  "success": true,
  "score": 0.9,
  "action": "submit"
}
```

**אימות:** ציבורי (reCAPTCHA נמצא בדרך כלל בטפסים ציבוריים).

## נתוני התייחסות (`/api/reference`)

|שיטה|נתיב|תיאור|
|--------|------|-------------|
|`GET`|`/api/reference`|קבל נתוני עזר לתפריטים נפתחים של טפסים|

מחזירה נתוני התייחסות המשמשים לאכלוס תפריטים נפתחים ושדות בחירה באפליקציה, כגון דגמי תמחור, סוגי רישיונות וקטגוריות פלטפורמה.

**אימות:** ציבורי.

## תפעול פנימי (`/api/internal`)

|שיטה|נתיב|תיאור|
|--------|------|-------------|
|`POST`|`/api/internal/db-init`|אתחול סכימת מסד נתונים ונתוני בסיס|

### אתחול מסד הנתונים

נקודת הקצה `/api/internal/db-init` מפעילה העברת מסד נתונים והכנסת נתוני זרעים אופציונליים. זה נקרא בדרך כלל פעם אחת במהלך הפריסה הראשונית או בעת איפוס סביבת פיתוח.

**אימות:** נקודת קצה זו צריכה להיות מאובטחת באמצעות בקרות גישה ספציפיות לסביבה או סוד משותף. זה לא מיועד לשימוש קבוע.

## שיקולי אבטחה

### נקודות קצה ציבוריות

נקודות הקצה של השירות הבאות פומביות בכוונה:
- בדיקות בריאות (דרושות על ידי ניטור/איזוני עומסים)
- פרטי גרסה (לא רגישים)
- דגלי תכונה (תצורה לא רגישה)
- נתוני מיקום (נתוני התייחסות)
- אימות reCAPTCHA (הגנה על טופס ציבורי)
- נתוני התייחסות (ערכים נפתחים)

### נקודות קצה מוגנות

נקודות קצה אלה דורשות אימות כדי למנוע שימוש לרעה:
- חילוץ כתובת אתר (מונע שימוש לרעה בזיוף בקשות בצד השרת)
- קידוד גיאוגרפי (שיחות API חיצוניות מוגבלות בתעריף)
- אתחול מסד הנתונים (פעולה הרסנית)
