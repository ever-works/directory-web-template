# נקודות קצה להצביע

מערכת ההצבעה מספקת נקודות קצה להצבעה בעד והורדת סעיפים. ההצבעות משתמשות במודל של ניקוד נטו שבו הספירה מייצגת את ההצבעות למעלה מינוס ההצבעות למטה. נקודות קצה ציבוריות מחזירות ספירת קולות בעוד שנקודות קצה מאומתות מאפשרות להצביע, לעדכן ולהסיר הצבעות. משתמשים חסומים מנועים מלהצביע.

## סקירה כללית

|נקודת קצה|שיטה|Auth|תיאור|
|---|---|---|---|
|`/api/items/[slug]/votes`|קבל|ציבורי|קבל ספירת קולות וסטטוס הצבעות של משתמשים|
|`/api/items/[slug]/votes`|פוסט|משתמש|הצע או עדכן הצבעה|
|`/api/items/[slug]/votes`|מחק|משתמש|הסר הצבעה|
|`/api/items/[slug]/votes/count`|קבל|ציבורי|קבל ספירת קולות נטו בלבד|
|`/api/items/[slug]/votes/status`|קבל|משתמש|קבל שיא הצבעה מלא למשתמש|

## נקודת קצה של הצבעה משולבת

### קבל מידע על הצבעה

```
GET /api/items/[slug]/votes
```

מחזירה את ספירת ההצבעות נטו עבור פריט ואת סטטוס ההצבעות של המשתמש הנוכחי אם מאומת. אין צורך באימות, אך משתמשים מאומתים מקבלים את סטטוס ההצבעה שלהם בתגובה.

**פרמטרי נתיב:**

|פרמטר|הקלד|תיאור|
|---|---|---|
|`slug`|מחרוזת|שבלול פריט|

**תגובת הצלחה (200):**

```json
{
  "success": true,
  "count": 15,
  "userVote": "up"
}
```

|שדה|הקלד|תיאור|
|---|---|---|
|`success`|בוליאני|תמיד `true` על הצלחה|
|`count`|מספר שלם|ספירת קולות נטו (הצבעות למעלה מינוס הצבעות כלפי מטה)|
|`userVote`|מחרוזת או ריק|`"up"`, `"down"`, או `null` אם לא מאומת או ללא הצבעה|

עבור משתמשים לא מאומתים, `userVote` הוא תמיד `null`. ה-`count` יכול להיות שלילי אם יש יותר הצבעות כלפי מטה מאשר הצבעות כלפי מעלה.

**מקור:** `template/app/api/items/[slug]/votes/route.ts`

### הצבעה או עדכן

```
POST /api/items/[slug]/votes
```

מטיל הצבעה חדשה או מחליף הצבעה קיימת על פריט. אם למשתמש כבר יש הצבעה, ההצבעה הקודמת נמחקת לפני יצירת ההצבעה החדשה. המשמעות היא ששינוי מהצבעה למעלה להצבעה כלפי מטה (או להיפך) היא פעולה אחת.

**אימות:** נדרש

**גוף הבקשה:**

```json
{
  "type": "up"
}
```

|שדה|הקלד|חובה|תיאור|
|---|---|---|---|
|`type`|מחרוזת|כן|`"up"` להצבעה למעלה, `"down"` להצבעה כלפי מטה|

**תגובת הצלחה (200):**

```json
{
  "success": true,
  "count": 16,
  "userVote": "up"
}
```

התגובה מחזירה את ספירת ההצבעות נטו המעודכנת לאחר החלת ההצבעה.

**תגובות שגיאה:**

|סטטוס|מצב|
|---|---|
| 400 |סוג הצבעה לא חוקי (חייב להיות `"up"` או `"down"`)|
| 401 |לא מאומת|
| 403 |המשתמש מושעה או נאסר|
| 404 |פרופיל הלקוח לא נמצא|

**מקור:** `template/app/api/items/[slug]/votes/route.ts`

### הסר הצבעה

```
DELETE /api/items/[slug]/votes
```

מסיר את ההצבעה של המשתמש הנוכחי מפריט. אם לא קיימת הצבעה, הפעולה מסתיימת בהצלחה ללא שגיאה (אימפוטנטית). לאחר ההסרה, `userVote` הוא `null`.

**אימות:** נדרש

**תגובת הצלחה (200):**

```json
{
  "success": true,
  "count": 14,
  "userVote": null
}
```

|סטטוס|מצב|
|---|---|
| 401 |לא מאומת|
| 404 |פרופיל הלקוח לא נמצא|

**מקור:** `template/app/api/items/[slug]/votes/route.ts`

## נקודת קצה של ספירת קולות

### קבל ספירת קולות

```
GET /api/items/[slug]/votes/count
```

מחזיר רק את ספירת הקולות נטו עבור פריט. זוהי נקודת קצה ציבורית קלת משקל המותאמת לאחזור מהיר של ספירת קולות ללא סטטוס הצבעה ספציפי למשתמש.

**תגובת הצלחה (200):**

```json
{
  "success": true,
  "count": 15
}
```

הספירה יכולה להיות חיובית, שלילית או אפס בהתאם למאזן ההצבעות כלפי מעלה ומטה.

**מקור:** `template/app/api/items/[slug]/votes/count/route.ts`

## נקודת קצה סטטוס הצבעה

### קבל סטטוס הצבעת משתמש

```
GET /api/items/[slug]/votes/status
```

מחזירה את רשומת ההצבעה המלאה עבור המשתמש המאומת בפריט ספציפי. מחזירה `null` אם המשתמש לא הצביע על הפריט.

**אימות:** נדרש

**תגובת הצלחה (200) -- המשתמש הצביע:**

```json
{
  "id": "vote_123abc",
  "userId": "client_456def",
  "itemId": "item_123abc",
  "voteType": "UPVOTE",
  "createdAt": "2024-01-20T10:30:00.000Z",
  "updatedAt": "2024-01-20T10:30:00.000Z"
}
```

**תגובת הצלחה (200) -- אין הצבעה:**

```json
null
```

שים לב שנקודת קצה זו מחזירה את ערכי מסד הנתונים הגולמיים `voteType` (`"UPVOTE"` או `"DOWNVOTE"`) במקום את הפורמט הפשוט `"up"` / `"down"` המשמש את נקודת הקצה המשולבת.

|סטטוס|מצב|
|---|---|
| 401 |לא מאומת|
| 404 |פרופיל הלקוח לא נמצא|

**מקור:** `template/app/api/items/[slug]/votes/status/route.ts`

## פרטי יישום מרכזיים

- **ציון נקי:** ספירת ההצבעות מחושבת כהצבעות כלפי מעלה פחות הצבעות כלפי מטה. ספירה שלילית מציינת יותר הצבעות כלפי מטה מאשר הצבעות כלפי מעלה.
- **החלפת הצבעה:** כאשר משתמש משנה את סוג ההצבעה שלו, ההצבעה הקיימת נמחקת ונוצרת חדשה. אין עדכון במקום.
- **מניעת משתמשים חסומים:** הסימון `isUserBlocked()` בנקודת הקצה של POST מונע ממשתמשים מושעים או אסורים להצביע. בדיקת החסימה נאכפת רק על יצירת הצבעה, לא על הסרת ההצבעה.
- **VoteType Enum:** מסד הנתונים מאחסן הצבעות כ-`VoteType.UPVOTE` ו-`VoteType.DOWNVOTE`. ה-API מתרגם את אלה ל-`"up"` ול-`"down"` עבור צרכנים חיצוניים.
- **Idempotent Delete:** מחיקת הצבעה שאינה קיימת עדיין מחזירה 200 תגובות עם הספירה הנוכחית ו-`userVote: null`.
