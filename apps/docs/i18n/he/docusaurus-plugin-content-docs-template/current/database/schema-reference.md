# הפניה לסכימה

כל טבלאות מסד הנתונים מוגדרות ב-`lib/db/schema.ts`. מסמך זה מקטלג כל טבלה, עמודות המפתח שלה, מערכות היחסים והמטרה שלה.

## משתמשים ואימות

### משתמשים

טבלת משתמש ליבה, המשמשת את NextAuth.js לאימות.

|עמודה|הקלד|הערות|
|--------|------|-------|
|`id`|טקסט (PK)|UUID, נוצר אוטומטית|
|`email`|טקסט|ייחודי|
|`image`|טקסט|כתובת האתר של תמונת הפרופיל|
|`emailVerified`|חותמת זמן|תאריך אימות דוא"ל|
|`passwordHash`|טקסט|Bcrypt hash עבור אישור אישורים|
|`createdAt`|חותמת זמן|הגדרה אוטומטית|
|`updatedAt`|חותמת זמן|הגדרה אוטומטית|
|`deletedAt`|חותמת זמן|מחיקה רכה|

**אינדקסים**: `users_created_at_idx`

### חשבונות

קישורים לחשבונות OAuth ואישורים, בעקבות סכימת המתאם NextAuth.js.

|עמודה|הקלד|הערות|
|--------|------|-------|
|`userId`|טקסט (FK)|הפניות `users.id` (מחיקה מדורגת)|
|`type`|טקסט|סוג חשבון (OAuth, אישורים וכו')|
|`provider`|טקסט|שם הספק (google, github, אישורים)|
|`providerAccountId`|טקסט|מזהה חשבון ספציפי לספק|
|`email`|טקסט|דוא"ל חשבון|
|`passwordHash`|טקסט|עבור אישורי לקוח|
|`refresh_token`|טקסט|אסימון רענון OAuth|
|`access_token`|טקסט|אסימון גישה ל-OAuth|
|`expires_at`|מספר שלם|תפוגה של אסימון|

**מפתח ראשי**: מורכב ב-(`provider`, `providerAccountId`)
**אינדקסים**: `accounts_email_idx`, `accounts_provider_idx`

## טבלאות אחרות

### מודעות חסות

פרסומות ממומנות של פריטים עם מחזור חיי תשלום מלא.

עמודות מפתח: `userId`, `itemSlug`, `status` (ממתין_תשלום, בהמתנה, נדחה, פעיל, פג תוקפו, בוטל), `interval` (שבועי, חודשי), @@@TOK004@@K@, @5@TOK004@@K@, @5@ `subscriptionId`.

### חברות / פריטיםחברות

רישומי חברה ואגודות פריטים-חברות לרישומי ספריות.

### סקרים / תגובות סקר

בונה סקרים עם הגדרות שאלות מבוססות JSON ואחסון תגובות.

### twentyCrmConfig / integrationMappings

טבלאות אינטגרציה של CRM עבור פונקציונליות סנכרון CRM של Twenty. טבלת התצורה אוכפת דפוס יחיד (רק שורה אחת מותרת).

### מנויים לניוזלטר

מעקב אחר הרשמה לניוזלטר בדוא"ל עם חותמות זמן של הרשמה/ביטול רישום.

### סטטוס seed

סטטוס זריעה של מסד נתונים למעקב אחר טבלת יחיד (זריעה, הושלמה, נכשלה) כדי למנוע פעולות זרע במקביל.

## סוג ייצוא

קובץ הסכימה מייצא סוגי TypeScript עבור כל טבלה באמצעות ההסקה של Drrizzle:

```typescript
export type User = typeof users.$inferSelect;
export type NewUser = typeof users.$inferInsert;
export type Subscription = typeof subscriptions.$inferSelect;
export type NewSubscription = typeof subscriptions.$inferInsert;
// ... and so on for all tables
```

סוגים אלה משמשים בכל היישום עבור פעולות מסד נתונים בטוחות.
