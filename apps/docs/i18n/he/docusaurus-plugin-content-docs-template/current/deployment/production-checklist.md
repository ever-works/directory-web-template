# רשימת בדיקה לסביבת ייצור

רשימת בדיקה מקיפה להבטחת מוכנות Ever Works לסביבת ייצור.

## רשימת הבדיקה לפני פריסה

### 1. הגדרת סביבה

#### משתני סביבה נדרשים

- [ ] **בסיס נתונים**
  - `DATABASE_URL` מוגדר עם PostgreSQL לייצור
  - מאגר חיבורים לבסיס הנתונים מופעל
  - מצב SSL לסביבת ייצור מופעל

- [ ] **אימות**
  - `NEXTAUTH_URL` מוגדר לדומיין ייצור
  - `NEXTAUTH_SECRET` נוצר (לפחות 32 תווים)
  - ספקי OAuth מוגדרים (Google, GitHub וכו')
  - פרטי כניסה של Supabase Auth (אם בשימוש)

- [ ] **ספקי תשלום**
  - מפתחות Stripe לייצור (`STRIPE_SECRET_KEY`, `NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY`)
  - מפתחות LemonSqueezy לייצור (אם בשימוש)
  - סודות Webhook מוגדרים
  - מצב בדיקה מושבת

- [ ] **שירותי דוא"ל**
  - מפתח API של Resend מוגדר
  - פרטי כניסה של Novu מוגדרים (אם בשימוש)
  - תבניות דוא"ל נבדקו
  - דומיין שולח מאומת

- [ ] **אנליטיקה ומעקב**
  - מפתח PostHog לייצור
  - DSN של Sentry מוגדר
  - ספק מעקב שגיאות מוגדר
  - Vercel Analytics מופעל (אם ב-Vercel)

- [ ] **אינטגרציית CRM**
  - פרטי כניסה של Twenty CRM (אם בשימוש)
  - נקודות קצה של Webhook מוגדרות

- [ ] **אבטחה**
  - `NODE_ENV=production`
  - הגבלת קצב מוגדרת
  - הגדרות CORS נבדקו
  - כותרות CSP מוגדרות

### 2. בסיס נתונים

- [ ] **סכמה ומיגרציות**
  - כל המיגרציות יושמו
  - סכמת בסיס הנתונים תואמת לקוד
  - אינדקסים לביצועים נוצרו
  - אילוצי מפתח זר אומתו

- [ ] **שלמות נתונים**
  - נתוני זרע נטענו (אם נדרש)
  - נתוני בדיקה נמחקו
  - כללי אימות נתונים יושמו

- [ ] **גיבוי ושחזור**
  - גיבוי אוטומטי מוגדר
  - שחזור מגיבוי נבדק
  - שחזור לנקודת זמן מופעל
  - מדיניות שמירת גיבויים מוגדרת

- [ ] **ביצועים**
  - מאגר חיבורים מוגדר
  - ביצועי שאילתות מאופטמים
  - רישום שאילתות איטיות מופעל
  - ניטור בסיס נתונים פעיל

### 3. אבטחה

- [ ] **אימות והרשאות**
  - גיבוב סיסמאות אומת (bcrypt)
  - ניהול סשן מאובטח
  - אסימוני JWT חתומים כראוי
  - בקרת גישה מבוססת תפקידים נבדקה

- [ ] **הגנת נתונים**
  - נתוני PII מוצפנים במנוחה
  - ניקוי נתונים רגישים מוגדר
  - HTTPS אוכף
  - עוגיות מאובטחות מופעלות

- [ ] **אבטחת API**
  - הגבלת קצב פעילה
  - אימות API נאכף
  - אימות קלט בכל נקודות הקצה
  - הגנה מפני SQL injection אומתה

- [ ] **תלויות**
  - כל התלויות עודכנו
  - פגיעויות אבטחה נבדקו (`npm audit`)
  - אין פגיעויות קריטיות
  - קובץ נעילת תלויות הוגש

### 4. ביצועים

- [ ] **אופטימיזציית צד לקוח**
  - תמונות מאופטמות (רכיב Next.js Image)
  - פיצול קוד מיושם
  - טעינה עצלה לרכיבים כבדים
  - גודל חבילה נותח

- [ ] **מטמון**
  - נכסים סטטיים במטמון
  - תגובות API במטמון (במקום המתאים)
  - CDN מוגדר
  - אסטרטגיית ביטול מטמון מיושמת

- [ ] **Core Web Vitals**
  - LCP פחות מ-2.5 שניות
  - FID פחות מ-100 אלפיות שנייה
  - CLS פחות מ-0.1
  - ניטור ביצועים מופעל

- [ ] **שאילתות בסיס נתונים**
  - שאילתות N+1 הוסרו
  - אינדקסים מתאימים נוצרו
  - מטמון שאילתות מופעל
  - מאגר חיבורים מאופטם

### 5. ניטור ורישום

- [ ] **מעקב שגיאות**

  - התראות שגיאה הוגדרו

## פקודות אימות

הפעל פקודות אלו לאימות מוכנות לייצור:

### ביקורת אבטחה

```bash
# בדיקת פגיעויות אבטחה
npm audit --production

# תיקון פגיעויות
npm audit fix

# בדיקת תלויות מיושנות
npm outdated
```

### אימות בנייה

```bash
# בנייה לייצור
npm run build

# בדיקת פלט הבנייה
ls -lh .next/

# ניתוח גודל חבילה
npm run analyze
```

### אימות בסיס נתונים

```bash
# בדיקת מצב מיגרציות
npx drizzle-kit check

# יצירת מיגרציה במידת הצורך
npx drizzle-kit generate

# יישום מיגרציות
npx drizzle-kit push
```

### תיעוד API

```bash
# יצירת מפרט OpenAPI
yarn generate-docs

# אימות תיעוד
yarn docs:validate

# בדיקה שהתיעוד עדכני
git diff --exit-code public/openapi.json
```

### משתני סביבה

```bash
# בדיקה שכל המשתנים הנדרשים מוגדרים
node scripts/check-env.js

# בדיקת הגדרת סביבה
npm run test:env
```

## תהליך עבודה לפריסה

### לפני פריסה

1. **סקירת קוד**
   - כל PR נסקרו ואושרו
   - אין קונפליקטים במיזוג
   - צינור CI/CD עבר

2. **בדיקות**
   - כל הבדיקות עברו
   - בדיקת QA ידנית הושלמה
   - סביבת Staging נבדקה

3. **תיעוד**
   - Changelog עודכן
   - תיעוד API נוצר מחדש
   - הערות פריסה הוכנו

### שלבי פריסה

1. **גיבוי**

   ```bash
   # גיבוי בסיס נתונים
   pg_dump $DATABASE_URL > backup-$(date +%Y%m%d).sql
   ```

2. **פריסה**

   ```bash
   # פריסה לייצור
   git push production main

   # או עם Vercel
   vercel --prod
   ```

3. **אימות**

   ```bash
   # בדיקת נקודת קצה לבריאות
   curl https://your-domain.com/api/health

   # בדיקת לוגי שגיאות
   tail -f logs/error.log
   ```

4. **ניטור**
   - מעקב אחר שיעור שגיאות ב-Sentry
   - מעקב ביצועים ב-PostHog
   - בדיקת ניטור זמן פעולה

### אחרי פריסה

1. **בדיקות עשן**
   - דף הבית נטען כראוי
   - משתמש יכול להתחבר
   - תהליך תשלום עובד
   - לוח מחוונים מנהל נגיש

2. **ניטור**
   - שיעור שגיאות תקין
   - זמני תגובה מקובלים
   - אין דליפות זיכרון
   - ביצועי בסיס נתונים יציבים

3. **תקשורת**
   - הודעה לצוות על הפריסה
   - עדכון דף סטטוס
   - הכרזה על תכונות חדשות (אם רלוונטי)

## תוכנית Rollback

אם בעיות מתגלות לאחר פריסה:

### Rollback מהיר

```bash
# חזרה לפריסה הקודמת
git revert HEAD
git push production main

# או עם Vercel
vercel rollback
```

### Rollback בסיס נתונים

```bash
# שחזור מגיבוי
psql $DATABASE_URL < backup-YYYYMMDD.sql

# או שימוש בשחזור נקודת זמן
# (אם נתמך על ידי ספק האחסון)
```

### תקשורת

1. הודעה מיידית לצוות
2. עדכון דף סטטוס
3. תקשורת עם משתמשים שנפגעו
4. תיעוד האירוע לניתוח post-mortem

## סיכום רשימת הבדיקה

השתמש בסיכום קצר זה למעקב אחרי התקדמות כוללת:

- [ ] **סביבה**: כל המשתנים מוגדרים
- [ ] **בסיס נתונים**: מיגרציות יושמו, גיבויים מוגדרים
- [ ] **אבטחה**: אימות, הצפנה, הגבלת קצב
- [ ] **ביצועים**: מאופטם, במטמון, מנוטר
- [ ] **ניטור**: מעקב שגיאות, רישום, התראות
- [ ] **תוכן**: CMS מוגדר, מדיה מאופטמת, i18n הושלם
- [ ] **API**: תיעוד נוצר, תקנים מיושמים
- [ ] **תשלומים**: Stripe/LS מוגדרים, Webhooks נבדקו
