# מילון מונחים

מונחים ומושגים מרכזיים המשמשים לאורך כל תיעוד Directory Web Template.

## מושגי ליבה של הדומיין

### ספרייה

אוסף של רשימות מאורגנות (פריטים) סביב נושא או נישה ספציפיים. הספרייה היא הישות ברמה הגבוהה ביותר. דוגמאות: "SaaS Tools Directory", "Developer Resources Directory" או "Local Business Directory".

### פריט

רשומה או רשימה בודדת בתוך ספרייה. פריט מייצג ישות אחת שמקוטלגת (כלי, עסק, משאב או שירות). לפריטים יש שדות מובנים (שם, תיאור, URL, לוגו), שייכים לקטגוריות ויכולים להיות מסומנים בתגיות.

### קטגוריה

סיווג היררכי המשמש לארגון פריטים. קטגוריות יוצרות מבנה עץ (יחסי הורה/ילד) ומספקות את מנגנון הניווט והסינון הראשי.

### תגית

תווית שטוחה, לא-היררכית, המצורפת לפריטים לסיווג צולב. תגיות משמשות לסינון משני וגילוי. פריט יכול להכיל תגיות מרובות כגון "open-source", "freemium" או "API-available".

### אוסף

קיבוץ נבחר של פריטים, ללא תלות בקטגוריות או בתגיות. אוספים הם קבוצות מוגדרות על ידי המשתמש או עורכות, כגון "10 הבחירות הטובות ביותר" או "חדש החודש".

### טקסונומיה

מערכת הסיווג הכוללת לספרייה, הכוללת קטגוריות, תגיות וכל מבנה ארגוני אחר.

### Slug

מזהה ידידותי ל-URL, קריא לאדם, הנגזר משם הישות. Slugs משמשים ב-URLs במקום מזהים מספריים. לדוגמה, "Visual Studio Code" הופך ל-`visual-studio-code`.

## דפוסי ארכיטקטורה

### Repository (מאגר נתונים)

מחלקת שכבת גישה לנתונים שמכמסת שאילתות ומוטציות של בסיס נתונים עבור ישות ספציפית. מאגרי הנתונים מנטרלים את Drizzle ORM ומספקים ממשק נקי לשירותים. ממוקם ב-`lib/repositories/`.

### Service (שירות)

מחלקת שכבת לוגיקת עסקים המתזמרת פעולות על פני מאגרי נתונים, APIs חיצוניים ושירותים אחרים. שירותים מכילים את לוגיקת היישום המרכזית ומופעלים על ידי מטפלי מסלולי API. ממוקם ב-`lib/services/`.

### Webhook

קריאה חוזרת HTTP המופעלת על ידי אירוע. ה-Template משתמש ב-webhooks להתראות מספקי תשלום (Stripe, LemonSqueezy, Polar) ועדכוני סטטוס פריסה. נקודות קצה של webhook מאמתות בקשות נכנסות באמצעות חתימות או סודות משותפים.

## ניהול תוכן

### CMS מבוסס Git

גישת ניהול התוכן המשמשת את ה-Template. נתוני הספרייה (פריטים, קטגוריות, מטא-נתונים) נשמרים כקבצים מובנים (YAML, Markdown) במאגר Git. ה-Template משכפל מאגר זה בזמן הבנייה וקורא תוכן ממערכת הקבצים המקומית. שינויים מבוצעים דרך commits ו-pull requests.

### Community PR

בקשת משיכה המוגשת על ידי חבר קהילה להוספה או עדכון פריטים במאגר ה-CMS המבוסס Git של ספרייה. Community PRs עוברים תהליך סקירה לפני מיזוג.

## בסיס נתונים

### Drizzle ORM

ה-ORM הקל ו-TypeScript-first המשמש את ה-Template. Drizzle מספק בונה שאילתות דמוי SQL עם בטיחות טיפוס מלאה. הגדרות סכמה נכתבות כקוד TypeScript, ומיגרציות נוצרות כקבצי SQL פשוטים דרך Drizzle Kit.

### מיגרציה

שינוי מגורסא בסכמת בסיס הנתונים. מיגרציות נוצרות עם `pnpm db:generate` ומיושמות עם `pnpm db:migrate`. קבצי המיגרציה מאוחסנים ב-`lib/db/migrations/`.

## אימות

### NextAuth.js

ספריית האימות (v5) המשמשת את ה-Template. היא מספקת תמיכת OAuth עבור מספר ספקים (Google, GitHub, Facebook, Twitter, Microsoft) עם ניהול הפעלה ואסימוני JWT.

### Supabase Auth

גיבוי אימות חלופי הנתמך על ידי ה-Template. Supabase Auth מספק אימות דוא"ל/סיסמה, קישורים קסומים ו-OAuth חברתי דרך השירות המנוהל של Supabase.

## תשלומים

### מנוי

הסדר תשלום חוזר המנוהל דרך אחד מספקי התשלום הנתמכים (Stripe, LemonSqueezy או Polar). ה-Template מטפל ביצירת מנויים, ניהולם ועיבוד webhooks.

## פריסה

### Vercel

פלטפורמת הפריסה הראשית עבור ה-Template. Vercel מספק פריסה ללא תצורה לאפליקציות Next.js, כולל פריסות תצוגה מקדימה אוטומטיות, edge functions והפצת CDN.

### Docker

שיטת פריסה חלופית. ה-Template יכול להיות מכולות ולהיפרס בכל סביבת אירוח תואמת Docker.
