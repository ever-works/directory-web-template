# Конечные точки аналитики администратора

API аналитики администратора предоставляет географические аналитические данные для панели администратора, включая статистику охвата, распределение и данные для визуализации на карте. Все конечные точки требуют аутентификации администратора.

## Обзор

| Конечная точка | Метод | Аутентификация | Описание |
|---|---|---|---|
| `/api/admin/geo-analytics` | GET | Администратор | Получить географические аналитические данные |

## Получить географическую аналитику

```
GET /api/admin/geo-analytics
```

Возвращает подробную географическую аналитику, включая статистику охвата, распределение по странам/городам/зонам обслуживания, координаты для маркеров карты и данные тепловой карты.

**Аутентификация:** Требуется администратор (через `checkAdminAuth()`)

**Кэширование:** Отключено — используются `force-dynamic`, `revalidate: 0` и `force-no-store` для обеспечения актуальных данных.

**Успешный ответ (200):**

### Поля ответа

#### Объект stats

| Поле | Тип | Описание |
|---|---|---|
| `totalIndexed` | integer | Общее число записей в индексе местоположений |
| `totalItems` | integer | Общее число элементов в репозитории |
| `itemsWithLocation` | integer | Элементы с данными о местоположении или помеченные как удалённые |
| `itemsRemote` | integer | Элементы, помеченные как удалённые/распределённые |
| `coveragePercent` | number | Процент элементов с данными о местоположении |
| `indexHealth.synced` | boolean | Совпадает ли число записей в индексе с ожидаемым |
| `citiesCount` | integer | Количество уникальных городов в индексе |
| `countriesCount` | integer | Количество уникальных стран в индексе |
| `lastIndexedAt` | string или null | Временная метка последнего обновления индекса |
| `lastRebuildAt` | string или null | Временная метка последней полной пересборки |

#### Объект distributions

| Поле | Описание |
|---|---|
| `byCountry` | Массив стран с количеством, сортировка по убыванию |
| `byCity` | Топ-20 городов с количеством, сортировка по убыванию |
| `byServiceArea` | Зоны обслуживания с количеством, сортировка по убыванию |

#### Массив locations

Каждый объект местоположения предоставляет данные для маркеров карты. Удалённые элементы с координатами `(0, 0)` отфильтровываются.

#### Данные тепловой карты

Массив пар широта/долгота только для неудалённых записей.

### Источники данных

Конечная точка агрегирует данные из трёх параллельных запросов:

1. **Служба индекса местоположений** (`getLocationIndexService().getIndexStats()`) — статистика индекса
2. **Записи индекса местоположений** (`getAllLocationEntries()`) — все проиндексированные местоположения
3. **Репозиторий элементов** (`itemRepository.findAll()`) — исходные данные для расчёта охвата

### Расчёт охвата

```
coveragePercent = round((itemsWithLocation / totalItems) * 100, 1)
```

Элемент считается «имеющим местоположение», если у него есть координаты широты или он помечен как удалённый (`is_remote: true`).
