# Конечные точки административной аналитики

API административной аналитики предоставляет данные географического анализа для панели администратора, включая статистику покрытия, разбивку распределения и данные визуализации карты. Все конечные точки требуют аутентификации администратора.

## Обзор

|Конечная точка|Метод|Авторизация|Описание|
|---|---|---|---|
|`/api/admin/geo-analytics`|ПОЛУЧИТЬ|Админ|Получите данные географической аналитики|

## Получите географическую аналитику

```
GET /api/admin/geo-analytics
```

Возвращает комплексную аналитику географического распределения, включая статистику покрытия, распределение стран/городов/зон обслуживания, координаты местоположения для маркеров на карте и данные тепловой карты. Эта конечная точка объединяет данные как из индекса местоположения, так и из хранилища элементов.

**Аутентификация:** Требуется администратор (через `checkAdminAuth()`)

**Кэширование:** отключено: используются `force-dynamic`, `revalidate: 0` и `force-no-store` для обеспечения актуальности данных для панели администратора.

### Поля ответа

#### Объект статистики

|Поле|Тип|Описание|
|---|---|---|
|`totalIndexed`|целое число|Всего записей в указателе местоположений|
|`totalItems`|целое число|Всего элементов в репозитории|
|`itemsWithLocation`|целое число|Элементы, имеющие данные о местоположении или помеченные как удаленные|
|`itemsRemote`|целое число|Элементы, помеченные как удаленные/распределенные|
|`coveragePercent`|номер|Процент элементов с данными о местоположении (округлено до 1 десятичного знака)|
|`indexHealth.synced`|логическое значение|Соответствует ли количество индексов ожидаемому количеству|
|`indexHealth.indexCount`|целое число|Неудаленные записи в индексе|
|`indexHealth.expectedCount`|целое число|Ожидаемые неудаленные записи на основе исходных данных|
|`citiesCount`|целое число|Количество отдельных городов в индексе|
|`countriesCount`|целое число|Количество отдельных стран в индексе|
|`remoteCount`|целое число|Количество удаленных записей в индексе|
|`lastIndexedAt`|строка или ноль|Временная метка ISO последнего обновления индекса|
|`lastRebuildAt`|строка или ноль|Временная метка ISO последней полной перестройки|

#### Объект распределения

|Поле|Описание|
|---|---|
|`byCountry`|Массив названий стран с количеством, отсортированный по убыванию количества.|
|`byCity`|Топ-20 городов с количеством городов, отсортированных по убыванию количества.|
|`byServiceArea`|Области обслуживания с количеством, отсортированные по убыванию количества|

#### Массив местоположений

Каждый объект местоположения предоставляет данные для маркеров карты. Удаленные элементы с координатами `(0, 0)` отфильтровываются, чтобы избежать искажения отображения карты.

#### Данные тепловой карты

Массив пар широты и долготы только для неудаленных записей, подходящий для визуализации тепловых карт плотности.

### Источники данных

Конечная точка агрегирует данные из трех параллельных запросов:

1. **Служба индексирования местоположения** (`getLocationIndexService().getIndexStats()`) — предоставляет статистику индексирования.
2. **Записи указателя местоположений** (`getAllLocationEntries()`) – предоставляет все индексированные местоположения для расчета распределения.
3. **Репозиторий предметов** (`itemRepository.findAll()`) – предоставляет исходные данные об элементах для расчета покрытия.

### Расчет покрытия

Процент покрытия рассчитывается как:

```
coveragePercent = round((itemsWithLocation / totalItems) * 100, 1)
```

Объект считается «имеющим местоположение», если он имеет координаты широты или помечен как удаленный (`is_remote: true`).

### Индекс здоровья

Состояние индекса сравнивает количество неудаленных записей в индексе местоположения с ожидаемым количеством, полученным на основе исходных данных:

```
expectedCount = itemsWithLocation - itemsRemote
indexCount = totalIndexed - remoteCount
synced = (indexCount === expectedCount)
```

Если `synced` имеет значение false, администраторам следует рассмотреть возможность перестроения индекса местоположения через конечную точку `/api/admin/location-index`.

|Статус|Состояние|
|---|---|
| 401 |Не авторизован как администратор|
| 500 |Внутренняя ошибка сервера|
