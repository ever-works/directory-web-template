# Конечные точки API категорий администратора

API категорий администратора предоставляет полные операции CRUD для управления категориями контента, включая изменение порядка и синхронизацию с удалённым репозиторием данных через Git. Все конечные точки требуют аутентификации администратора.

## Сводка маршрутов

| Метод | Путь | Аутентификация | Описание |
|--------|------|------|-------------|
| `GET` | `/api/admin/categories` | Администратор | Список категорий (с пагинацией) |
| `POST` | `/api/admin/categories` | Администратор | Создать новую категорию |
| `GET` | `/api/admin/categories/all` | Администратор | Получить все категории (из кэша контента) |
| `GET` | `/api/admin/categories/{id}` | Администратор | Получить категорию по ID |
| `PUT` | `/api/admin/categories/{id}` | Администратор | Обновить категорию |
| `DELETE` | `/api/admin/categories/{id}` | Администратор | Мягкое или жёсткое удаление категории |
| `PUT` | `/api/admin/categories/reorder` | Администратор | Изменить порядок категорий по массиву ID |
| `GET` | `/api/admin/categories/git` | Администратор | Получить статус Git-репозитория и категории |
| `POST` | `/api/admin/categories/git` | Администратор | Создать категорию через коммит Git |

## Аутентификация

Все конечные точки управления категориями проверяют наличие активной сессии с правами администратора:

```typescript
const session = await auth();
if (!session?.user?.isAdmin) {
  return NextResponse.json(
    { success: false, error: "Unauthorized. Admin access required." },
    { status: 401 }
  );
}
```

## Конечные точки

### GET `/api/admin/categories`

Возвращает список категорий с пагинацией, фильтрацией и сортировкой.

**Параметры запроса:**

| Параметр | Тип | По умолчанию | Описание |
|-----------|------|---------|-------------|
| `page` | integer | `1` | Номер страницы (минимум: 1) |
| `limit` | integer | `10` | Элементов на странице (1–100) |
| `includeInactive` | string | `"false"` | Включить неактивные категории |
| `sortBy` | string | `"name"` | Поле сортировки: `"name"` или `"id"` |
| `sortOrder` | string | `"asc"` | Направление: `"asc"` или `"desc"` |

**Ответ (200):**

```json
{
  "success": true,
  "categories": [
    {
      "id": "productivity",
      "name": "Productivity",
      "isActive": true,
      "itemCount": 15,
      "createdAt": "2024-01-15T10:30:00.000Z",
      "updatedAt": "2024-01-15T10:30:00.000Z"
    }
  ],
  "total": 25,
  "page": 1,
  "limit": 10,
  "totalPages": 3
}
```

### POST `/api/admin/categories`

Создаёт новую категорию. Поле `id` необязательно и будет автоматически сгенерировано из имени, если не указано. После успешного создания кэши контента сбрасываются.

**Тело запроса:**

```json
{
  "id": "productivity",
  "name": "Productivity"
}
```

| Поле | Тип | Обязательный | Описание |
|-------|------|----------|-------------|
| `id` | string | Нет | URL-безопасный слаг (`^[a-z0-9-]+$`). Генерируется автоматически. |
| `name` | string | Да | Отображаемое имя (2–100 символов) |

**Ответ (201):**

```json
{
  "success": true,
  "category": { "id": "productivity", "name": "Productivity", "isActive": true },
  "message": "Category created successfully"
}
```

### GET `/api/admin/categories/all`

Возвращает все категории из кэша контента для указанной локали.

**Параметры запроса:**

| Параметр | Тип | По умолчанию | Описание |
|-----------|------|---------|-------------|
| `locale` | string | `"en"` | Код локали для получения контента |

### GET `/api/admin/categories/{id}`

Возвращает одну категорию по уникальному идентификатору.

### PUT `/api/admin/categories/{id}`

Обновляет имя существующей категории. Сбрасывает кэш контента после успешного обновления.

**Тело запроса:**

```json
{ "name": "Productivity Tools" }
```

### DELETE `/api/admin/categories/{id}`

Удаляет категорию. По умолчанию выполняется мягкое удаление (деактивация). Используйте параметр `hard=true` для постоянного удаления.

**Параметры запроса:**

| Параметр | Тип | По умолчанию | Описание |
|-----------|------|---------|-------------|
| `hard` | string | `"false"` | Установите `"true"` для постоянного удаления |

### PUT `/api/admin/categories/reorder`

Изменяет порядок категорий на основе массива ID. Позиция ID в массиве определяет новый порядок отображения.

**Тело запроса:**

```json
{ "categoryIds": ["productivity", "design", "development"] }
```

### GET `/api/admin/categories/git`

Получает статус Git-репозитория и категории из настроенного GitHub-репозитория данных.

### POST `/api/admin/categories/git`

Создаёт новую категорию и фиксирует её в GitHub-репозитории данных.

## Коды ошибок

| Статус | Причина |
|--------|-------|
| `400` | Неверные параметры пагинации |
| `400` | Отсутствует имя категории |
| `401` | Отсутствует сессия администратора |
| `404` | Категория не найдена |
| `409` | Категория с таким именем уже существует |
| `500` | Не настроен `DATA_REPOSITORY` или GitHub-токен |

## Сброс кэша

Все операции записи вызывают `invalidateContentCaches()` для немедленного отражения изменений.
