# Конечные точки API категорий администратора

API категорий администратора предоставляет полные операции CRUD для управления категориями контента, включая изменение порядка и синхронизацию на основе Git с удаленным репозиторием данных. Все конечные точки требуют аутентификации администратора посредством аутентификации на основе сеанса.

## Сводка маршрута

|Метод|Путь|Авторизация|Описание|
|--------|------|------|-------------|
|`GET`|`/api/admin/categories`|Админ|Список категорий (постраничный)|
|`POST`|`/api/admin/categories`|Админ|Создать новую категорию|
|`GET`|`/api/admin/categories/all`|Админ|Получить все категории (из кеша контента)|
|`GET`|`/api/admin/categories/{id}`|Админ|Получить одну категорию по идентификатору|
|`PUT`|`/api/admin/categories/{id}`|Админ|Обновить категорию|
|`DELETE`|`/api/admin/categories/{id}`|Админ|Мягкое или жесткое удаление категории|
|`PUT`|`/api/admin/categories/reorder`|Админ|Изменение порядка категорий по массиву идентификаторов|
|`GET`|`/api/admin/categories/git`|Админ|Получить статус и категории репозитория Git|
|`POST`|`/api/admin/categories/git`|Админ|Создать категорию с помощью фиксации Git|

## Аутентификация

Все конечные точки управления категориями проверяют наличие активного сеанса с правами администратора:

```typescript
const session = await auth();
if (!session?.user?.isAdmin) {
  return NextResponse.json(
    { success: false, error: "Unauthorized. Admin access required." },
    { status: 401 }
  );
}
```

## Конечные точки

### ПОЛУЧИТЬ `/api/admin/categories`

Возвращает разбитый на страницы список категорий с дополнительной фильтрацией и сортировкой.

**Параметры запроса:**

|Параметр|Тип|По умолчанию|Описание|
|-----------|------|---------|-------------|
|`page`|целое число| `1` |Номер страницы (минимум: 1)|
|`limit`|целое число| `10` |Элементов на странице (1–100)|
|`includeInactive`|строка|`"false"`|Включить неактивные категории|
|`sortBy`|строка|`"name"`|Поле сортировки: `"name"` или `"id"`|
|`sortOrder`|строка|`"asc"`|Направление сортировки: `"asc"` или `"desc"`|

**Ответ (200):**

```json
{
  "success": true,
  "categories": [
    {
      "id": "productivity",
      "name": "Productivity",
      "isActive": true,
      "itemCount": 15,
      "createdAt": "2024-01-15T10:30:00.000Z",
      "updatedAt": "2024-01-15T10:30:00.000Z"
    }
  ],
  "total": 25,
  "page": 1,
  "limit": 10,
  "totalPages": 3
}
```

### ПОСТ `/api/admin/categories`

Создает новую категорию. Поле `id` является необязательным и будет автоматически сгенерировано на основе имени, если оно не указано. В случае успеха делает недействительными кэши контента.

**Тело запроса:**

```json
{
  "id": "productivity",
  "name": "Productivity"
}
```

|Поле|Тип|Требуется|Описание|
|-------|------|----------|-------------|
|`id`|строка|Нет|Удобный URL-адрес (`^[a-z0-9-]+$`). Генерируется автоматически, если опущено.|
|`name`|строка|Да|Отображаемое имя (2–100 символов)|

**Ответ (201):**

```json
{
  "success": true,
  "category": {
    "id": "productivity",
    "name": "Productivity",
    "isActive": true,
    "itemCount": 0,
    "createdAt": "2024-01-20T15:30:00.000Z",
    "updatedAt": "2024-01-20T15:30:00.000Z"
  },
  "message": "Category created successfully"
}
```

### ПОЛУЧИТЬ `/api/admin/categories/all`

Возвращает все категории из кэша контента для заданной локали. Полезно для раскрывающихся списков администратора и селекторов.

**Параметры запроса:**

|Параметр|Тип|По умолчанию|Описание|
|-----------|------|---------|-------------|
|`locale`|строка|`"en"`|Код локали для получения контента|

**Ответ (200):**

```json
{
  "success": true,
  "data": [
    { "id": "productivity", "name": "Productivity", "isActive": true, "itemCount": 15 }
  ]
}
```

### ПОЛУЧИТЬ `/api/admin/categories/{id}`

Извлекает одну категорию по ее уникальному идентификатору.

**Ответ (200):**

```json
{
  "success": true,
  "data": {
    "id": "productivity",
    "name": "Productivity",
    "isActive": true,
    "itemCount": 15,
    "createdAt": "2024-01-15T10:30:00.000Z",
    "updatedAt": "2024-01-15T10:30:00.000Z"
  }
}
```

### ПОСТАВЬТЕ `/api/admin/categories/{id}`

Обновляет имя существующей категории. В случае успеха делает недействительными кэши контента.

**Тело запроса:**

```json
{ "name": "Productivity Tools" }
```

**Ответ (200):**

```json
{
  "success": true,
  "data": { "id": "productivity", "name": "Productivity Tools", "isActive": true },
  "message": "Category updated successfully"
}
```

### УДАЛИТЬ `/api/admin/categories/{id}`

Удаляет категорию. По умолчанию выполняет мягкое удаление (деактивацию). Используйте параметр запроса `hard=true` для окончательного удаления. В случае успеха делает недействительными кэши контента.

**Параметры запроса:**

|Параметр|Тип|По умолчанию|Описание|
|-----------|------|---------|-------------|
|`hard`|строка|`"false"`|Установите `"true"` для окончательного удаления.|

**Ответ (200):**

```json
{
  "success": true,
  "message": "Category deactivated successfully"
}
```

### ПОСТАВЬТЕ `/api/admin/categories/reorder`

Изменяет порядок категорий на основе массива идентификаторов категорий. Положение каждого идентификатора в массиве определяет его новый порядок отображения.

**Тело запроса:**

```json
{ "categoryIds": ["productivity", "design", "development", "marketing"] }
```

**Правила проверки:**
- `categoryIds` должен быть непустым массивом
- Все значения должны быть строками

**Ответ (200):**

```json
{
  "success": true,
  "message": "Categories reordered successfully"
}
```

### ПОЛУЧИТЬ `/api/admin/categories/git`

Извлекает состояние и категории репозитория Git из настроенного репозитория данных GitHub. Требуются переменные среды `DATA_REPOSITORY` и `GITHUB_TOKEN`.

**Ответ (200):**

```json
{
  "success": true,
  "status": {
    "repository": "ever-co/awesome-time-tracking-data",
    "branch": "main",
    "lastCommit": "abc123def456",
    "lastCommitDate": "2024-01-20T10:30:00.000Z",
    "isUpToDate": true
  },
  "categories": [],
  "message": "Git repository status retrieved successfully"
}
```

### ПОСТ `/api/admin/categories/git`

Создает новую категорию и фиксирует ее непосредственно в репозитории данных GitHub. Требуются переменные среды `DATA_REPOSITORY` и `GH_TOKEN`.

**Тело запроса:**

```json
{ "id": "productivity", "name": "Productivity" }
```

Оба `id` и `name` необходимы для создания на основе Git.

**Ответ (200):**

```json
{
  "success": true,
  "category": { "id": "productivity", "name": "Productivity" },
  "message": "Category created and committed to Git repository"
}
```

## Коды ошибок

|Статус|Ошибка|Причина|
|--------|-------|-------|
| `400` |Неверные параметры пагинации|Страница < 1 или предел за пределами 1–100|
| `400` |Укажите название категории.|В запросе на создание отсутствует `name`.|
| `400` |Идентификаторы категорий должны быть массивом.|Неверные полезные данные повторного заказа|
| `401` |Несанкционированный. Требуется доступ администратора.|Сеанс отсутствует или не является администратором|
| `404` |Категория не найдена|Неверный идентификатор категории|
| `409` |Категория с таким названием уже существует|Повторяющееся имя при создании/обновлении|
| `500` |DATA_REPOSITORY не настроено|Отсутствует переменная env для конечных точек Git.|
| `500` |Токен GitHub не настроен|Отсутствует `GITHUB_TOKEN` или `GH_TOKEN`|

## Инвалидация кэша

Все операции записи (создание, обновление, удаление, изменение порядка) вызывают `invalidateContentCaches()`, чтобы изменения сразу же были видны во всем приложении.
