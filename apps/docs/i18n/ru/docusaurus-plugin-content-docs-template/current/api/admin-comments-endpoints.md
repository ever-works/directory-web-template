# Конечные точки API комментариев администратора

API комментариев администратора предоставляет возможности модерации для управления комментариями пользователей. Администраторы могут перечислять, просматривать, обновлять и удалять комментарии. Все конечные точки используют среду выполнения Node.js и требуют доступности базы данных. При проверке подлинности используется `403 Forbidden` для пользователей, не являющихся администраторами.

## Сводка маршрута

|Метод|Путь|Авторизация|Описание|
|--------|------|------|-------------|
|`GET`|`/api/admin/comments`|Админ|Список комментариев (постраничный, с возможностью поиска)|
|`GET`|`/api/admin/comments/{id}`|Админ|Получите один комментарий с информацией о пользователе|
|`PUT`|`/api/admin/comments/{id}`|Админ|Обновить содержание комментариев|
|`DELETE`|`/api/admin/comments/{id}`|Админ|Мягкое удаление комментария|

## Аутентификация

Конечные точки модерации комментариев проверяют статус администратора и возвращают `403 Forbidden` (не `401`) для пользователей без прав администратора:

```typescript
const session = await auth();
if (!session?.user?.isAdmin) {
  return NextResponse.json(
    { success: false, error: "Forbidden" },
    { status: 403 }
  );
}
```

## Требование к базе данных

Конечные точки комментариев проверяют доступность базы данных перед обработкой запросов:

```typescript
const dbCheck = checkDatabaseAvailability();
if (dbCheck) return dbCheck;
```

Если база данных не настроена, перед любой проверкой подлинности возвращается соответствующий ответ об ошибке.

## Конечные точки

### ПОЛУЧИТЬ `/api/admin/comments`

Возвращает постраничный список комментариев с соответствующей информацией о пользователе. Поддерживает полнотекстовый поиск по содержимому комментариев, именам пользователей и электронным адресам пользователей. Возвращаются только неудаленные комментарии.

**Параметры запроса:**

|Параметр|Тип|По умолчанию|Описание|
|-----------|------|---------|-------------|
|`page`|целое число| `1` |Номер страницы для нумерации страниц|
|`limit`|целое число| `10` |Комментариев на странице (1–100)|
|`search`|строка| `""` |Поиск по содержимому, имени пользователя или электронной почте|

**Поведение при поиске:**

Поисковый запрос сопоставляется без учета регистра (с использованием `ILIKE`) по отношению к:
- Содержание комментария
- Отображаемое имя пользователя
- Адрес электронной почты пользователя

Специальные символы `%`, `_` и `\` экранируются, чтобы предотвратить внедрение шаблонов SQL.

**Ответ (200):**

```json
{
  "success": true,
  "data": {
    "comments": [
      {
        "id": "comment_123abc",
        "content": "This is a great product! Highly recommended.",
        "rating": 5,
        "userId": "user_456def",
        "itemId": "item_789ghi",
        "createdAt": "2024-01-20T10:30:00.000Z",
        "updatedAt": "2024-01-20T10:30:00.000Z",
        "user": {
          "id": "user_456def",
          "name": "John Doe",
          "email": "john.doe@example.com",
          "image": "https://example.com/avatar.jpg"
        }
      }
    ],
    "pagination": {
      "total": 156,
      "page": 1,
      "limit": 10,
      "totalPages": 16
    }
  }
}
```

### ПОЛУЧИТЬ `/api/admin/comments/{id}`

Извлекает конкретный комментарий по его идентификатору с полной информацией профиля пользователя. Включает левое соединение с таблицей `clientProfiles` для пользовательских данных.

**Параметры пути:**

|Параметр|Тип|Описание|
|-----------|------|-------------|
|`id`|строка|Уникальный идентификатор комментария|

**Ответ (200):**

```json
{
  "success": true,
  "data": {
    "id": "comment_123abc",
    "content": "This is a great product! Highly recommended.",
    "rating": 5,
    "userId": "user_456def",
    "itemId": "item_789ghi",
    "createdAt": "2024-01-20T10:30:00.000Z",
    "updatedAt": "2024-01-20T14:45:00.000Z",
    "user": {
      "id": "user_456def",
      "name": "John Doe",
      "email": "john.doe@example.com",
      "image": "https://example.com/avatar.jpg"
    }
  }
}
```

**Резервный пользователь:** Если профиль пользователя не найден (пользователь удален), возвращается объект-заполнитель:

```json
{
  "user": {
    "id": "",
    "name": "Unknown User",
    "email": "",
    "image": null
  }
}
```

### ПОСТАВЬТЕ `/api/admin/comments/{id}`

Обновляет содержимое определенного комментария. Изменить можно только поле `content`. The comment must exist and not be soft-deleted.

**Параметры пути:**

|Параметр|Тип|Описание|
|-----------|------|-------------|
|`id`|строка|Уникальный идентификатор комментария|

**Тело запроса:**

```json
{
  "content": "This is an updated comment with more details."
}
```

|Поле|Тип|Требуется|Описание|
|-------|------|----------|-------------|
|`content`|строка|Да|Текст нового комментария (не должен быть пустым после обрезки)|

**Правила проверки:**
- `content` является обязательным и не может быть пустым или содержать только пробелы.
- Целевой комментарий должен существовать и не должен иметь отметки времени `deletedAt`.

**Ответ (200):**

```json
{
  "success": true,
  "data": {
    "id": "comment_123abc",
    "content": "This is an updated comment with more details.",
    "rating": 5,
    "userId": "user_456def",
    "itemId": "item_789ghi",
    "createdAt": "2024-01-20T10:30:00.000Z",
    "updatedAt": "2024-01-20T16:15:00.000Z",
    "user": { "id": "user_456def", "name": "John Doe", "email": "john.doe@example.com", "image": null }
  },
  "message": "Comment updated successfully"
}
```

### УДАЛИТЬ `/api/admin/comments/{id}`

Выполняет обратимое удаление комментария, устанавливая временную метку `deletedAt`. Комментарий должен существовать и еще не быть удален. Обратно удаляемые комментарии исключаются из всех запросов к спискам.

**Параметры пути:**

|Параметр|Тип|Описание|
|-----------|------|-------------|
|`id`|строка|Уникальный идентификатор комментария|

**Ответ (200):**

```json
{
  "success": true,
  "message": "Comment deleted successfully"
}
```

## Модель данных комментариев

|Поле|Тип|Обнуляемый|Описание|
|-------|------|----------|-------------|
|`id`|строка|Нет|Уникальный идентификатор комментария|
|`content`|строка|Нет|Текст комментария|
|`rating`|целое число|Да|Значение рейтинга (1--5)|
|`userId`|строка|Нет|Идентификатор пользователя автора|
|`itemId`|строка|Нет|Идентификатор связанного элемента|
|`createdAt`|дата и время|Да|Временная метка создания|
|`updatedAt`|дата и время|Да|Временная метка последнего обновления|
|`deletedAt`|дата и время|Да|Временная метка мягкого удаления (нуль, если активно)|

## Коды ошибок

|Статус|Ошибка|Причина|
|--------|-------|-------|
| `400` |Требуется контент|Пустой или отсутствующий контент при обновлении|
| `403` |Запрещено|Пользователь без прав администратора пытается получить доступ|
| `404` |Комментарий не найден|Неверный идентификатор или уже удален|
| `500` |Внутренняя ошибка сервера|Сбой базы данных или сервера|

## Замечания по реализации

- В комментариях используется **мягкое удаление**: вместо удаления строки устанавливается поле `deletedAt`. Это сохраняет целостность данных и обеспечивает возможность их потенциального восстановления.
- Все запросы к спискам фильтруются с помощью `isNull(comments.deletedAt)`, чтобы исключить удаленные комментарии.
- Пользовательские данные извлекаются через `LEFT JOIN` на `clientProfiles`, что обеспечивает возможность получения комментариев удаленных пользователей.
- Для `runtime` установлено значение `"nodejs"` для этих маршрутов (не Edge).
