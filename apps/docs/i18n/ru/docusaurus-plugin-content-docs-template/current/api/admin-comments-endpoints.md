# Конечные точки API комментариев администратора

API комментариев администратора предоставляет конечные точки модерации пользовательских комментариев. Администраторы могут просматривать, обновлять и мягко удалять комментарии. Все конечные точки требуют доступности к базе данных. Неадминистративные запросы получают `403 Forbidden`.

## Сводка маршрутов

| Метод | Путь | Аутентификация | Описание |
|--------|------|------|-------------|
| `GET` | `/api/admin/comments` | Администратор | Список комментариев (с пагинацией, с поиском) |
| `GET` | `/api/admin/comments/{id}` | Администратор | Получить комментарий с инфурмацией о пользователе |
| `PUT` | `/api/admin/comments/{id}` | Администратор | Обновить содержимое комментария |
| `DELETE` | `/api/admin/comments/{id}` | Администратор | Мягкое удаление комментария |

## Аутентификация

Конечные точки модерации проверяют права администратора и возвращают `403 Forbidden` (не `401`) для неадминистративных пользователей.

## Конечные точки

### GET `/api/admin/comments`

Возвращает список комментариев с информацией о пользователях. Поддерживает полнотекстовый поиск по содержимому, имени и email пользователя.

**Параметры запроса:**

| Параметр | Тип | По умолчанию | Описание |
|-----------|------|---------|-------------|
| `page` | integer | `1` | Номер страницы |
| `limit` | integer | `10` | Комментариев на странице (1–100) |
| `search` | string | `""` | Поиск по содержимому, имени или email |

**Ответ (200):**

```json
{
  "success": true,
  "data": {
    "comments": [
      {
        "id": "comment_123abc",
        "content": "This is a great product!",
        "rating": 5,
        "userId": "user_456def",
        "itemId": "item_789ghi",
        "createdAt": "2024-01-20T10:30:00.000Z",
        "user": { "id": "user_456def", "name": "John Doe", "email": "john.doe@example.com" }
      }
    ],
    "pagination": { "total": 156, "page": 1, "limit": 10, "totalPages": 16 }
  }
}
```

### GET `/api/admin/comments/{id}`

Извлекает один комментарий по ID с полной информацией о пользователе.

**Параметры пути:**

| Параметр | Тип | Описание |
|-----------|------|-------------|
| `id` | string | Уникальный идентификатор комментария |

### PUT `/api/admin/comments/{id}`

Обновляет содержимое комментария. Только поле `content` можно изменить.

**Тело запроса:**

```json
{ "content": "Обновлённый текст комментария." }
```

### DELETE `/api/admin/comments/{id}`

Выполняет мягкое удаление комментария путём установки метки `deletedAt`.

## Модель данных комментария

| Поле | Тип | Описание |
|-------|------|-------------|
| `id` | string | Уникальный идентификатор |
| `content` | string | Текст комментария |
| `rating` | integer | Оценка (1–5), может быть null |
| `userId` | string | ID автора |
| `itemId` | string | ID связанного элемента |
| `deletedAt` | datetime | Метка мягкого удаления (null — активен) |

## Коды ошибок

| Статус | Причина |
|--------|-------|
| `400` | Отсутствует или пустое содержимое |
| `403` | Нет прав администратора |
| `404` | Комментарий не найден или уже удалён |
| `500` | Внутренняя ошибка |
