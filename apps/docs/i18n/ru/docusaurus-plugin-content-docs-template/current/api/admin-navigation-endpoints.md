# Конечные точки административной навигации и индекса местоположения

Эти конечные точки администратора управляют пользовательскими ссылками навигации по сайту и индексом географического местоположения. Конечные точки навигации позволяют настраивать собственные ссылки верхнего и нижнего колонтитула, хранящиеся в `config.yml`. Конечные точки индекса местоположения управляют пространственным индексом, используемым для географической аналитики и объектов карты.

## Обзор

|Конечная точка|Метод|Авторизация|Описание|
|---|---|---|---|
|`/api/admin/navigation`|ПОЛУЧИТЬ|Админ|Получить пользовательскую конфигурацию навигации|
|`/api/admin/navigation`|ПАТЧ|Админ|Обновить пользовательские элементы навигации|
|`/api/admin/location-index`|ПОЛУЧИТЬ|Админ|Получить статистику индекса местоположения|
|`/api/admin/location-index`|ПОСТ|Админ|Перестроить или очистить индекс местоположения|

## Конечные точки навигации

### Получить конфигурацию навигации

```
GET /api/admin/navigation
```

Извлекает элементы навигации `custom_header` и `custom_footer` из файла `config.yml` сайта. Возвращает пустые массивы, если пользовательская навигация не настроена.

**Аутентификация:** Требуется администратор (через `getCachedApiSession`)

**Успешный ответ (200):**

```json
{
  "custom_header": [
    {
      "label": "About",
      "path": "/about"
    },
    {
      "label": "Documentation",
      "path": "/pages/docs"
    }
  ],
  "custom_footer": [
    {
      "label": "GitHub",
      "path": "https://github.com/example"
    },
    {
      "label": "footer.PRIVACY_POLICY",
      "path": "/pages/privacy-policy"
    }
  ]
}
```

Каждый элемент навигации имеет два поля:

|Поле|Тип|Описание|
|---|---|---|
|`label`|строка|Отображаемый текст (обычный текст или ключ перевода i18n, например `"footer.PRIVACY_POLICY"`)|
|`path`|строка|URL-путь (внутренний маршрут, начинающийся с `/`, или внешний URL-адрес с `http://`/`https://`)|

|Статус|Состояние|
|---|---|
| 401 |Не авторизован как администратор|
| 500 |Не удалось прочитать конфигурацию|

**Источник:** `template/app/api/admin/navigation/route.ts`

### Обновить конфигурацию навигации

```
PATCH /api/admin/navigation
```

Обновляет настраиваемые элементы навигации верхнего или нижнего колонтитула в `config.yml`. Проверяет формат пути каждого элемента для предотвращения XSS-атак через опасные схемы URL-адресов.

**Аутентификация:** Требуется администратор

**Тело запроса:**

```json
{
  "type": "header",
  "items": [
    {
      "label": "About",
      "path": "/about"
    },
    {
      "label": "Blog",
      "path": "https://blog.example.com"
    }
  ]
}
```

|Поле|Тип|Требуется|Описание|
|---|---|---|---|
|`type`|строка|Да|`"header"` или `"footer"`|
|`items`|массив|Да|Массив элементов навигации|
|`items[].label`|строка|Да|Непустая отображаемая метка|
|`items[].path`|строка|Да|Действительный URL-путь|

**Проверка пути:**

Функция `isValidNavigationPath()` обеспечивает соблюдение строгих правил формата пути:

|Формат пути|Разрешено|Пример|
|---|---|---|
|Внутренние маршруты|Да|`/about`, `/pages/docs`|
|URL-адреса HTTPS|Да|`https://example.com`|
|HTTP-URL-адреса|Да|`http://example.com`|
|URL-адреса, относящиеся к протоколу|Нет|`//evil.com`|
|URL-адреса JavaScript|Нет|`javascript:alert(1)`|
|URL-адреса данных|Нет|`data:text/html,...`|
|Другие схемы|Нет|`vbscript:`, `file:`|

**Успешный ответ (200):**

```json
{
  "success": true,
  "type": "header",
  "items": [
    {
      "label": "About",
      "path": "/about"
    }
  ]
}
```

**Ответы об ошибках:**

|Статус|Состояние|
|---|---|
| 400 |`type` не является `"header"` или `"footer"`|
| 400 |`items` не является массивом|
| 400 |В элементе отсутствуют строковые поля `label` или `path`.|
| 400 |Неверный формат пути (предотвращение XSS)|
| 401 |Не авторизован как администратор|
| 500 |Не удалось записать конфигурацию|

Передайте пустой массив `items`, чтобы очистить всю пользовательскую навигацию для указанного типа.

**Источник:** `template/app/api/admin/navigation/route.ts`

## Конечные точки индекса местоположения

### Получить статистику индекса местоположения

```
GET /api/admin/location-index
```

Возвращает статистику об индексе географического местоположения, включая общее количество проиндексированных элементов, количество городов и стран, а также перестраивает метаданные. Использует службу индексирования местоположения для получения данных.

**Аутентификация:** Требуется администратор (через `checkAdminAuth()`)

**Кэширование:** отключено — используются `force-dynamic`, `revalidate: 0` и `force-no-store`.

**Успешный ответ (200):**

```json
{
  "success": true,
  "data": {
    "totalIndexed": 450,
    "citiesCount": 85,
    "countriesCount": 25,
    "remoteCount": 30,
    "lastIndexedAt": "2024-01-20T10:30:00.000Z",
    "lastRebuildAt": "2024-01-15T08:00:00.000Z"
  }
}
```

|Статус|Состояние|
|---|---|
| 401 |Не авторизован как администратор|
| 500 |Внутренняя ошибка сервера|

**Источник:** `template/app/api/admin/location-index/route.ts`

### Управление индексом местоположения

```
POST /api/admin/location-index
```

Выполняет действия по управлению индексом местоположения. Поддерживает восстановление индекса с нуля или очистку всех записей.

**Аутентификация:** Требуется администратор

**Тело запроса:**

```json
{
  "action": "rebuild"
}
```

|Поле|Тип|Требуется|Описание|
|---|---|---|---|
|`action`|строка|Да|`"rebuild"` или `"clear"`|

**Действия:**

|Действие|Описание|
|---|---|
|`rebuild`|Извлекает все элементы из репозитория и повторно индексирует данные об их местоположении. Возвращает статистику восстановления.|
|`clear`|Удаляет все записи из индекса местоположения. Возвращает количество удаленных записей.|

**Успешный ответ (200) – Перестройка:**

```json
{
  "success": true,
  "data": {
    "indexed": 420,
    "skipped": 80,
    "errors": 0
  }
}
```

**Ответ об успешном завершении (200) – Очистить:**

```json
{
  "success": true,
  "data": {
    "cleared": 450
  }
}
```

**Ответы об ошибках:**

|Статус|Состояние|
|---|---|
| 400 |Недопустимое действие (не `"rebuild"` или `"clear"`)|
| 401 |Не авторизован как администратор|
| 500 |Внутренняя ошибка сервера|

**Источник:** `template/app/api/admin/location-index/route.ts`

## Ключевые детали реализации

- **Предотвращение XSS:** При проверке пути навигации отклоняются все схемы URL-адресов, кроме `/`, `http://` и `https://`. Это блокирует `javascript:`, `data:`, `vbscript:` и URL-адреса, относящиеся к протоколу (`//evil.com`), которые могут использоваться для межсайтового скриптинга.
- **Хранилище конфигурации:** элементы навигации хранятся в `config.yml` под ключами `custom_header` и `custom_footer` и сохраняются через `configManager.updateNestedKey()`.
- **i18n Ярлыки:** Навигационные метки могут представлять собой обычный текст или клавиши перевода (например, `"footer.PRIVACY_POLICY"`). Интерфейс отвечает за разрешение ключей перевода.
- **Перестроение индекса местоположения.** Операция восстановления загружает все элементы из `ItemRepository` и передает их в службу индексирования местоположений. Для больших наборов данных это может оказаться ресурсоемкой операцией.
- **Очистка кеша.** Конечные точки индекса местоположения явно отключают все кэширование, чтобы на панели администратора всегда отображались текущие данные.
