# Конечные точки навигации и индекса местоположений

Эти конечные точки управляют пользовательскими ссылками навигации сайта и географическим индексом. Навигационные конечные точки позволяют настраивать ссылки шапки и подвала, хранящиеся в `config.yml`.

## Обзор

| Конечная точка | Метод | Аутентификация | Описание |
|---|---|---|---|
| `/api/admin/navigation` | GET | Администратор | Получить конфигурацию навигации |
| `/api/admin/navigation` | PATCH | Администратор | Обновить элементы навигации |
| `/api/admin/location-index` | GET | Администратор | Статистика индекса местоположений |
| `/api/admin/location-index` | POST | Администратор | Перестроить или очистить индекс |

## Конечные точки навигации

### Получить конфигурацию навигации

```
GET /api/admin/navigation
```

Извлекает элементы `custom_header` и `custom_footer` из `config.yml`.

**Успешный ответ (200):**

```json
{
  "custom_header": [
    { "label": "About", "path": "/about" }
  ],
  "custom_footer": [
    { "label": "GitHub", "path": "https://github.com/example" }
  ]
}
```

| Поле | Тип | Описание |
|---|---|---|
| `label` | string | Отображаемый текст (план текст или ключ i18n) |
| `path` | string | URL-путь |

### Обновить конфигурацию навигации

```
PATCH /api/admin/navigation
```

**Тело запроса:**

```json
{
  "type": "header",
  "items": [
    { "label": "About", "path": "/about" }
  ]
}
```

| Поле | Тип | Обязательный | Описание |
|---|---|---|---|
| `type` | string | Да | `"header"` или `"footer"` |
| `items` | array | Да | Массив элементов навигации |

**Валидация путей:**

| Формат пути | Разрешён | Пример |
|---|---|---|
| Внутренние маршруты | Да | `/about` |
| HTTPS URL | Да | `https://example.com` |
| HTTP URL | Да | `http://example.com` |
| URL без протокола | Нет | `//evil.com` |
| JavaScript URL | Нет | `javascript:alert(1)` |
| Data URL | Нет | `data:text/html,...` |

## Конечные точки индекса местоположений

### Получить статистику индекса

```
GET /api/admin/location-index
```

Возвращает статистику индекса: общее число индексированных элементов, количество городов/стран, метаданные перестройки.

### Управление индексом

```
POST /api/admin/location-index
```

**Тело запроса:**

```json
{ "action": "rebuild" }
```

| Поле | Тип | Обязательный | Описание |
|---|---|---|---|
| `action` | string | Да | `"rebuild"` или `"clear"` |

| Действие | Описание |
|---|---|
| `rebuild` | Переиндексирует все элементы с нуля |
| `clear` | Удаляет все записи |

## Детали реализации

- **Защита от XSS:** Валидация пути навигации запрещает `javascript:`, `data:`, `vbscript:` и `//evil.com`.
- **Хранение конфигурации:** Элементы навигации хранятся в `config.yml`.
- **Метки i18n:** Метки могут быть либо обычным текстом, либо ключами перевода.
