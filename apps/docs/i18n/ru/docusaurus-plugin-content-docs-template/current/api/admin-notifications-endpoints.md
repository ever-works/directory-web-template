# Конечные точки API уведомлений администратора

API уведомлений администратора управляет встроенными уведомлениями для пользователей-администраторов: получение списка с количеством непрочитанных, создание уведомлений для пользователей и отметка прочитанным — по одному или всем сразу.

## Сводка маршрутов

| Метод | Путь | Аутентификация | Описание |
|--------|------|------|-------------|
| `GET` | `/api/admin/notifications` | Администратор | Список уведомлений |
| `POST` | `/api/admin/notifications` | Аутентифицирован |Создать уведомление |
| `PATCH` | `/api/admin/notifications/{id}/read` | Аутентифицирован | Отметить одно прочитанным |
| `PATCH` | `/api/admin/notifications/mark-all-read` | Аутентифицирован | Отметить все прочитанными |

## Конечные точки

### GET `/api/admin/notifications`

Извлекает последние 50 уведомлений аутентифицированного админа (от новых к старым). Также возвращает количество непрочитанных.

### POST `/api/admin/notifications`

Создаёт новое уведомление для указанного пользователя. Не требует прав администратора.

| Поле | Тип | Обязательный | Описание |
|-------|------|----------|-------------|
| `type` | string | Да | Тип уведомления |
| `title` | string | Да | Краткий заголовок |
| `message` | string | Да | Текст сообщения |
| `userId` | string | Да | ID получателя |
| `data` | object | Нет | Метаданные (JSON) |

### PATCH `/api/admin/notifications/{id}/read`

Отмечает одно уведомление прочитанным. Устанавливает `isRead = true` и `readAt`. Только владелец уведомления может его отметить.

### PATCH `/api/admin/notifications/mark-all-read`

Отмечает все непрочитанные уведомления пользователя за один запрос.

## Модель данных уведомления

| Поле | Тип | Описание |
|-------|------|-------------|
| `id` | string | Уникальный идентификатор |
| `userId` | string | ID пользователя |
| `type` | string | Тип уведомления |
| `title` | string | Заголовок |
| `message` | string | Текст сообщения |
| `isRead` | boolean | Прочитано ли |
| `readAt` | datetime | Время прочтения |

## Обычные типы уведомлений

| Тип | Описание |
|------|-------------|
| `item_approved` | Элемент одобрен |
| `item_rejected` | Элемент отклонён |
| `comment_received` | Новый комментарий |
| `submission_received` | Получена заявка |

## Коды ошибок

| Статус | Причина |
|--------|-------|
| `401` | Нет сессии |
| `403` | Нет прав администратора |
| `404` | Уведомление не найдено или чужое |
| `500` | Внутренняя ошибка |
