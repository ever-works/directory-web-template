# Конечные точки API ролей администратора

API ролей предоставляет конечные точки для управления ролями пользователей и их правами доступа. Роли контролируют уровни доступа во всем приложении.

## Базовый путь

```
/api/admin/roles
```

## Сводка маршрутов

| Метод | Путь | Аутентификация | Описание |
|--------|-----|-------|-------------|
| `GET` | `/api/admin/roles` | Администратор | Список ролей |
| `POST` | `/api/admin/roles` | Администратор | Создать роль |
| `GET` | `/api/admin/roles/active` | Публичный | Активные роли |
| `GET` | `/api/admin/roles/stats` | Администратор | Статистика |
| `GET` | `/api/admin/roles/{id}` | Администратор | Получить по ID |
| `PUT` | `/api/admin/roles/{id}` | Администратор | Обновить |
| `DELETE` | `/api/admin/roles/{id}` | Администратор | Удалить |
| `GET` | `/api/admin/roles/{id}/permissions` | Администратор | Права роли |
| `PUT` | `/api/admin/roles/{id}/permissions` | Администратор | Обновить права |

---

## Список ролей

```
GET /api/admin/roles
```

**Параметры запроса:**

| Параметр | Тип | По умолчанию | Описание |
|-----------|------|---------|-------------|
| `page` | integer | `1` | Номер страницы |
| `limit` | integer | `10` | Элементов на странице (1–100) |
| `status` | string | — | `active` или `inactive` |
| `sortBy` | string | `name` | `name`, `id`, `created_at` |
| `sortOrder` | string | `asc` | `asc` или `desc` |

---

## Создать роль

```
POST /api/admin/roles
```

| Поле | Тип | Обязательный | Описание |
|-------|------|----------|-------------|
| `name` | string | Да | Название роли (3–100 симв.) |
| `description` | string | Да | Описание (макс. 500 симв.) |
| `status` | string | Нет | `active` (default) / `inactive` |
| `isAdmin` | boolean | Нет | Флаг администратора (default: `false`) |

---

## Активные роли

```
GET /api/admin/roles/active
```

Возвращает все роли со статусом `active`. Аутентификация не требуется.

---

## Права роли

### Получить права

```
GET /api/admin/roles/{id}/permissions
```

### Обновить права

```
PUT /api/admin/roles/{id}/permissions
```

Заменяет весь массив прав. Каждая строка проверяется на соответствие системным определениям прав.

## Правила валидации

| Поле | Правило |
|-------|-------|
| `name` | 3–100 симв.; служит для формирования уникального ID |
| `description` | Макс. 500 симв. |
| `permissions` | Массив строк; каждая — валидное системное разрешение |

## Коды ошибок

| Статус | Значение |
|--------|----------|
| `400` | Ошибка валидации |
| `401` | Требуется аутентификация |
| `403` | Требуются права администратора |
| `404` | Роль не найдена |
| `409` | Дублирующееся имя |
| `500` | Внутренняя ошибка |
