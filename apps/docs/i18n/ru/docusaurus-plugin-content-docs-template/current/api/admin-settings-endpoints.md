# Конечные точки настроек администратора

API настроек администратора предоставляет конечные точки для чтения и изменения конфигурации сайта, хранящейся в `config.yml`. Сюда входят общие настройки и статус поставщика карт. Все конечные точки требуют аутентификации администратора.

## Обзор

|Конечная точка|Метод|Авторизация|Описание|
|---|---|---|---|
|`/api/admin/settings`|ПОЛУЧИТЬ|Админ|Получить все настройки|
|`/api/admin/settings`|ПАТЧ|Админ|Обновить конкретную настройку|
|`/api/admin/settings/map-status`|ПОЛУЧИТЬ|Админ|Получить статус конфигурации поставщика карт|

## Получить настройки

```
GET /api/admin/settings
```

Извлекает полный раздел `settings` из файла `config.yml` сайта.

**Аутентификация:** Требуется администратор (через `getCachedApiSession`)

**Успешный ответ (200):**

```json
{
  "settings": {
    "theme": "light",
    "itemsPerPage": 20,
    "enableComments": true,
    "enableVoting": true,
    "enableNewsletter": true,
    "mapProvider": "mapbox",
    "defaultLanguage": "en"
  }
}
```

Точная форма объекта `settings` зависит от конфигурации `config.yml` сайта. Конечная точка возвращает все, что хранится под ключом `settings`.

|Статус|Состояние|
|---|---|
| 401 |Не авторизован как администратор|
| 500 |Не удалось прочитать конфигурацию|

**Источник:** `template/app/api/admin/settings/route.ts`

## Обновить настройку

```
PATCH /api/admin/settings
```

Обновляет одно значение параметра в разделе `settings` `config.yml`. Ключ автоматически ограничивается пространством имен `settings` (например, обеспечивая обновления ключа `"theme"` `settings.theme` в файле конфигурации).

**Аутентификация:** Требуется администратор

**Тело запроса:**

```json
{
  "key": "itemsPerPage",
  "value": 30
}
```

|Поле|Тип|Требуется|Описание|
|---|---|---|---|
|`key`|строка|Да|Ключ настройки для обновления (относительно `settings.`)|
|`value`|любой|Да|Новое значение настройки|

**Успешный ответ (200):**

```json
{
  "success": true,
  "key": "itemsPerPage",
  "value": 30
}
```

Обновление сохраняется через `configManager.updateNestedKey()`, который изменяет базовый файл `config.yml`. Ключу автоматически присваивается префикс `settings.` перед передачей менеджеру конфигурации.

**Ответы об ошибках:**

|Статус|Состояние|
|---|---|
| 400 |В теле запроса отсутствует поле `key`.|
| 401 |Не авторизован как администратор|
| 500 |Не удалось записать конфигурацию|

**Источник:** `template/app/api/admin/settings/route.ts`

## Статус поставщика карт

### Получить статус карты

```
GET /api/admin/settings/map-status
```

Возвращает статус конфигурации поддерживаемых поставщиков карт, не раскрывая фактические ключи API. Это позволяет панели администратора показывать, какие поставщики карт доступны для использования.

**Аутентификация:** Требуется администратор

**Успешный ответ (200):**

```json
{
  "status": {
    "mapbox": {
      "isConfigured": true,
      "isPreviewAvailable": true,
      "name": "Mapbox"
    },
    "google": {
      "isConfigured": false,
      "isPreviewAvailable": false,
      "name": "Google Maps"
    }
  }
}
```

|Поле|Тип|Описание|
|---|---|---|
|`mapbox.isConfigured`|логическое значение|Установлен ли `NEXT_PUBLIC_MAPBOX_ACCESS_TOKEN`|
|`mapbox.isPreviewAvailable`|логическое значение|То же, что `isConfigured` — для предварительного просмотра требуется токен.|
|`google.isConfigured`|логическое значение|Установлен ли `NEXT_PUBLIC_GOOGLE_MAPS_API_KEY`|
|`google.isPreviewAvailable`|логическое значение|То же, что `isConfigured`|

Конечная точка проверяет наличие переменных среды:

- `NEXT_PUBLIC_MAPBOX_ACCESS_TOKEN` для Mapbox
- `NEXT_PUBLIC_GOOGLE_MAPS_API_KEY` для Карт Google

В ответе не отображаются фактические ключевые значения.

|Статус|Состояние|
|---|---|
| 401 |Не авторизован как администратор|
| 500 |Внутренняя ошибка сервера|

**Источник:** `template/app/api/admin/settings/map-status/route.ts`

## Архитектура конфигурации

Система настроек построена на синглтоне `configManager` от `lib/config-manager`:

- **Хранение:** Настройки хранятся в файле конфигурации YAML (`config.yml`)
- **Доступ:** Метод `configManager.getConfig()` считывает полную конфигурацию.
- **Обновления:** Метод `configManager.updateNestedKey()` изменяет определенные вложенные ключи.
- **Кэширование.** Сеансы кэшируются через `getCachedApiSession()` для повышения производительности.

Все обновления настроек относятся к пространству имен `settings` в файле конфигурации. Это предотвращает случайное изменение ключей конфигурации верхнего уровня через API настроек.
