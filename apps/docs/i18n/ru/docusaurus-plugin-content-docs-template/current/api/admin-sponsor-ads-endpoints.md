# Конечные точки API рекламных объявлений администратора

API рекламных объявлений предоставляет конечные точки для управления рекламными объявлениями: просмотр, одобрение, отклонение, отмена. Статусы: `pending_payment`, `pending`, `active`, `rejected`, `expired`, `cancelled`.

## Базовый путь

```
/api/admin/sponsor-ads
```

## Сводка маршрутов

| Метод | Путь | Аутентификация | Описание |
|--------|-----|-------|-------------|
| `GET` | `/api/admin/sponsor-ads` | Администратор | Список объявлений |
| `GET` | `/api/admin/sponsor-ads/{id}` | Администратор | Получить по ID |
| `DELETE` | `/api/admin/sponsor-ads/{id}` | Администратор | Удалить |
| `POST` | `/api/admin/sponsor-ads/{id}/approve` | Администратор | Одобрить и активировать |
| `POST` | `/api/admin/sponsor-ads/{id}/reject` | Администратор | Отклонить |
| `POST` | `/api/admin/sponsor-ads/{id}/cancel` | Администратор | Отменить |

---

## Список объявлений

```
GET /api/admin/sponsor-ads
```

**Параметры запроса:**

| Параметр | Тип | По умолчанию | Описание |
|-----------|------|---------|-------------|
| `page` | integer | `1` | Номер страницы |
| `limit` | integer | `10` | Элементов на странице |
| `status` | string | — | Фильтр по статусу |
| `interval` | string | — | `weekly` или `monthly` |
| `search` | string | — | Поиск по тексту |
| `sortBy` | string | `createdAt` | `createdAt`, `updatedAt` и др. |
| `sortOrder` | string | `desc` | `asc` или `desc` |

---

## Одобрить объявление

```
POST /api/admin/sponsor-ads/{id}/approve
```

Одобряет и активирует объявление. Для статуса `pending_payment` используйте `forceApprove: true`.

---

## Отклонить объявление

```
POST /api/admin/sponsor-ads/{id}/reject
```

| Поле | Тип | Обязательный | Описание |
|-------|------|----------|-------------|
| `rejectionReason` | string | Да | Причина отклонения (10–500 симв.) |

---

## Отменить объявление

```
POST /api/admin/sponsor-ads/{id}/cancel
```

| Поле | Тип | Обязательный | Описание |
|-------|------|----------|-------------|
| `cancelReason` | string | Нет | Причина отмены (макс. 500 симв.) |

---

## Жизненный цикл статусов

```
pending_payment --> pending --> active --> expired
                       |          |
                       v          v
                   rejected   cancelled
```

- **`pending_payment`** — Создано, ожидает оплаты.
- **`pending`** — Оплата получена, ожидает проверки.
- **`active`** — Одобрено, активно.
- **`rejected`** — Отклонено.
- **`expired`** — Истек срок.
- **`cancelled`** — Отменено.

## Коды ошибок

| Статус | Значение |
|--------|----------|
| `400` | Ошибка валидации / недопустимый переход статуса |
| `401` | Требуется аутентификация |
| `403` | Требуются права администратора |
| `404` | Объявление не найдено |
| `500` | Внутренняя ошибка |
