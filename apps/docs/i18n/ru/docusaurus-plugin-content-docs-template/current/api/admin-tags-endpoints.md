# Конечные точки API тегов администратора

API Admin Tags предоставляет полные операции CRUD для управления тегами контента. Теги используются для классификации и фильтрации элементов в каталоге. API поддерживает постраничный листинг, создание с активным/неактивным состоянием, обновления, удаление и извлечение с учетом локали из кэша контента. Все операции записи делают недействительными кэши контента для немедленной видимости.

## Сводка маршрута

|Метод|Путь|Авторизация|Описание|
|--------|------|------|-------------|
|`GET`|`/api/admin/tags`|Админ|Список тегов (постраничный)|
|`POST`|`/api/admin/tags`|Админ|Создать новый тег|
|`GET`|`/api/admin/tags/all`|Админ|Получить все теги (из кеша контента)|
|`GET`|`/api/admin/tags/{id}`|Админ|Получить единый тег по идентификатору|
|`PUT`|`/api/admin/tags/{id}`|Админ|Обновить тег|
|`DELETE`|`/api/admin/tags/{id}`|Админ|Удаление тега навсегда|

## Аутентификация

Для всех конечных точек управления тегами требуются права администратора:

```typescript
const session = await auth();
if (!session?.user?.isAdmin) {
  return NextResponse.json(
    { success: false, error: 'Unauthorized' },
    { status: 401 }
  );
}
```

## Конечные точки

### ПОЛУЧИТЬ `/api/admin/tags`

Возвращает постраничный список всех тегов в системе. Параметры пагинации проверяются с помощью общей утилиты `validatePaginationParams`.

**Параметры запроса:**

|Параметр|Тип|По умолчанию|Описание|
|-----------|------|---------|-------------|
|`page`|целое число| `1` |Номер страницы (минимум: 1)|
|`limit`|целое число| `10` |Элементов на странице (1–100)|

**Ответ (200):**

```json
{
  "success": true,
  "data": {
    "tags": [
      {
        "id": "productivity",
        "name": "Productivity",
        "isActive": true,
        "itemCount": 156,
        "created_at": "2024-01-20T10:30:00.000Z",
        "updated_at": "2024-01-20T10:30:00.000Z"
      },
      {
        "id": "design",
        "name": "Design",
        "isActive": true,
        "itemCount": 89,
        "created_at": "2024-01-19T15:20:00.000Z",
        "updated_at": "2024-01-19T15:20:00.000Z"
      }
    ],
    "total": 45,
    "page": 1,
    "limit": 10,
    "totalPages": 5
  }
}
```

### ПОСТ `/api/admin/tags`

Создает новый тег с указанным идентификатором, именем и необязательным активным статусом. В случае успеха делает недействительными кэши контента.

**Тело запроса:**

```json
{
  "id": "artificial-intelligence",
  "name": "Artificial Intelligence",
  "isActive": true
}
```

|Поле|Тип|Требуется|Описание|
|-------|------|----------|-------------|
|`id`|строка|Да|Удобный для URL-адреса идентификатор пула|
|`name`|строка|Да|Понятное для человека имя тега (2–50 символов)|
|`isActive`|логическое значение|Нет|Активен ли тег (по умолчанию: `true`)|

**Правила проверки:**
- Оба `id` и `name` обязательны.
- Имя тега должно содержать от 2 до 50 символов.
- Идентификатор тега должен быть уникальным для всех существующих тегов.
- Имя тега должно быть уникальным для всех существующих тегов.

**Ответ (201):**

```json
{
  "success": true,
  "tag": {
    "id": "artificial-intelligence",
    "name": "Artificial Intelligence",
    "isActive": true,
    "itemCount": 0,
    "created_at": "2024-01-20T10:30:00.000Z",
    "updated_at": "2024-01-20T10:30:00.000Z"
  }
}
```

### ПОЛУЧИТЬ `/api/admin/tags/all`

Возвращает все теги из кэша контента для заданной локали. Эта конечная точка считывает данные из слоя кэшированного контента, а не из базы данных, что делает ее подходящей для заполнения селекторов тегов в пользовательском интерфейсе администратора.

**Параметры запроса:**

|Параметр|Тип|По умолчанию|Описание|
|-----------|------|---------|-------------|
|`locale`|строка|`"en"`|Код локали для получения контента|

**Ответ (200):**

```json
{
  "success": true,
  "data": [
    {
      "id": "productivity",
      "name": "Productivity",
      "isActive": true,
      "itemCount": 156
    }
  ]
}
```

### ПОЛУЧИТЬ `/api/admin/tags/{id}`

Извлекает один тег по его уникальному идентификатору с полной информацией, включая статистику использования.

**Параметры пути:**

|Параметр|Тип|Описание|
|-----------|------|-------------|
|`id`|строка|Уникальный идентификатор тега|

**Ответ (200):**

```json
{
  "success": true,
  "data": {
    "id": "productivity",
    "name": "Productivity",
    "isActive": true,
    "itemCount": 156,
    "created_at": "2024-01-20T10:30:00.000Z",
    "updated_at": "2024-01-20T10:30:00.000Z"
  }
}
```

### ПОСТАВЬТЕ `/api/admin/tags/{id}`

Обновляет имя и/или активный статус тега. Идентификатор тега нельзя изменить после создания. В случае успеха делает недействительными кэши контента.

**Параметры пути:**

|Параметр|Тип|Описание|
|-----------|------|-------------|
|`id`|строка|Уникальный идентификатор тега|

**Тело запроса:**

```json
{
  "name": "Productivity & Efficiency",
  "isActive": true
}
```

|Поле|Тип|Требуется|Описание|
|-------|------|----------|-------------|
|`name`|строка|Да|Обновлено отображаемое имя тега.|
|`isActive`|логическое значение|Нет|Обновлен активный статус|

**Ответ (200):**

```json
{
  "success": true,
  "data": {
    "id": "productivity",
    "name": "Productivity & Efficiency",
    "isActive": true,
    "itemCount": 156,
    "created_at": "2024-01-20T10:30:00.000Z",
    "updated_at": "2024-01-20T16:45:00.000Z"
  },
  "message": "Tag updated successfully"
}
```

### УДАЛИТЬ `/api/admin/tags/{id}`

Безвозвратно удаляет тег из системы. При этом тег также будет удален со всех связанных элементов. В случае успеха делает недействительными кэши контента.

**Параметры пути:**

|Параметр|Тип|Описание|
|-----------|------|-------------|
|`id`|строка|Уникальный идентификатор тега|

**Ответ (200):**

```json
{
  "success": true,
  "message": "Tag deleted successfully"
}
```

:::осторожно
Удаление тега является окончательным и не может быть отменено. Все связи элемента и тега для удаленного тега будут удалены. Вместо этого рассмотрите возможность деактивации тега (установив `isActive` на `false` через PUT), если вы хотите сохранить целостность данных.
:::

## Модель данных тегов

|Поле|Тип|Обнуляемый|Описание|
|-------|------|----------|-------------|
|`id`|строка|Нет|Уникальный идентификатор, удобный для URL-адреса|
|`name`|строка|Нет|Понятное отображаемое имя|
|`isActive`|логическое значение|Нет|Можно ли присвоить тег элементам|
|`itemCount`|целое число|Нет|Количество элементов, использующих этот тег|
|`created_at`|дата и время|Нет|Временная метка создания|
|`updated_at`|дата и время|Нет|Временная метка последнего обновления|

## Коды ошибок

|Статус|Ошибка|Причина|
|--------|-------|-------|
| `400` |Укажите идентификатор и имя тега.|Отсутствуют обязательные поля при создании|
| `400` |Укажите имя тега.|Отсутствует имя при обновлении|
| `400` |Имя тега должно содержать от 2 до 50 символов.|Ошибка проверки длины имени|
| `400` |Неверный параметр страницы/лимита|Параметр пагинации вне диапазона|
| `401` |Несанкционированный|Сеанс отсутствует или не является администратором|
| `404` |Тег не найден|Нет тега с данным идентификатором|
| `409` |Тег с идентификатором уже существует|Повторяющийся идентификатор при создании|
| `409` |Тег с названием уже существует|Повторяющееся имя при создании/обновлении|
| `500` |Не удалось получить/создать/обновить/удалить тег.|Ошибка сервера или базы данных|

## Инвалидация кэша

Все операции записи (создание, обновление, удаление) вызывают `invalidateContentCaches()`, чтобы изменения тегов немедленно отражались в общедоступном контенте:

```typescript
await invalidateContentCaches();
```

При этом очищается как кэш содержимого в памяти, так и все кэши уровня CDN, которые могут быть активными.

## Источники данных

API тегов использует два разных источника данных в зависимости от конечной точки:

|Конечная точка|Источник данных|Вариант использования|
|----------|------------|----------|
|`GET /api/admin/tags`|`tagRepository` (база данных)|Административное управление с нумерацией страниц|
|`POST /api/admin/tags`|`tagRepository` (база данных)|Создание новых тегов|
|`GET /api/admin/tags/all`|`getCachedItems()` (кэш контента)|Выпадающие селекторы, быстрый поиск|
|`GET /api/admin/tags/{id}`|`tagRepository` (база данных)|Подробный просмотр тегов|
|`PUT /api/admin/tags/{id}`|`tagRepository` (база данных)|Обновление свойств тега|
|`DELETE /api/admin/tags/{id}`|`tagRepository` (база данных)|Удаление тегов|
