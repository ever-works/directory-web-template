# Конечные точки API пользователей администратора

API пользователей предоставляет конечные точки для управления учётными записями: создание, обновление, смена статуса, назначение ролей.

## Базовый путь

```
/api/admin/users
```

## Сводка маршрутов

| Метод | Путь | Аутентификация | Описание |
|--------|-----|-------|-------------|
| `GET` | `/api/admin/users` | Администратор | Список пользователей |
| `POST` | `/api/admin/users` | Администратор | Создать пользователя |
| `GET` | `/api/admin/users/stats` | Администратор | Статистика |
| `POST` | `/api/admin/users/check-email` | Администратор | Проверка email |
| `POST` | `/api/admin/users/check-username` | Администратор | Проверка имени |
| `GET` | `/api/admin/users/{id}` | Администратор | Получить по ID |
| `PUT` | `/api/admin/users/{id}` | Администратор | Обновить |
| `DELETE` | `/api/admin/users/{id}` | Администратор | Удалить |

---

## Список пользователей

```
GET /api/admin/users
```

**Параметры запроса:**

| Параметр | Тип | По умолчанию | Описание |
|-----------|------|---------|-------------|
| `page` | integer | `1` | Номер страницы |
| `limit` | integer | `10` | Элементов на странице (1–100) |
| `search` | string | — | Поиск по имени, email или логину |
| `role` | string | — | Фильтр по ID роли |
| `status` | string | — | `active` или `inactive` |
| `sortBy` | string | `name` | `name`, `email`, `role`, `created_at` |
| `sortOrder` | string | `asc` | `asc` или `desc` |

---

## Создать пользователя

```
POST /api/admin/users
```

| Поле | Тип | Обязательный | Описание |
|-------|------|----------|-------------|
| `username` | string | Да | 3–30 симв., алфанум. + `-` и `_` |
| `email` | string | Да | Действительный email |
| `name` | string | Да | Полное имя (2–100 симв.) |
| `password` | string | Да | Мин. 8 симв. |
| `role` | string | Да | Должна существовать в системе |
| `title` | string | Нет | Должность (max 100) |
| `avatar` | string | Нет | URL аватара (max 500) |

---

## Проверка доступности email

```
POST /api/admin/users/check-email
```

Проверяет, занят ли email. Поддерживает `excludeId` для сценариев обновления.

---

## Проверка доступности имени пользователя

```
POST /api/admin/users/check-username
```

Аналогично проверке email.

---

## Получить / Обновить / Удалить пользователя

Частичное обновление (PUT) — только указанные поля. Удаление (DELETE) необратимо; админ не может удалить себя.

## Правила валидации

| Поле | Правило |
|-------|-------|
| `username` | 3–30 симв. при создании; 3–50 симв. при обновлении |
| `email` | Действительный email |
| `name` | 2–100 симв. |
| `password` | Мин. 8 симв. |
| `role` | Должна существовать; валидируется по таблице ролей |
| `status` | `active` или `inactive` |

## Коды ошибок

| Статус | Значение |
|--------|----------|
| `400` | Ошибка валидации; дубликация данных; самоудаление |
| `401` | Требуется аутентификация |
| `403` | Требуются права администратора |
| `404` | Пользователь не найден |
| `500` | Внутренняя ошибка |
