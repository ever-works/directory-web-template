# Конечные точки клиентского API

Клиентский API предоставляет аутентифицированные конечные точки для управления элементами, просмотра статистики дашборда и географических данных. Все конечные точки требуют аутентификации через `requireClientAuth()`.

**Каталог источников:** `template/app/api/client/`

---

## Аутентификация

Каждая конечная точка в этой группе требует действительной сессии. Неаутентифицированные запросы получают ответ 401.

---

## Сводка маршрутов

| Метод | Путь | Описание |
|--------|------|-------------|
| `GET` | `/api/client/dashboard/stats` | Статистика дашборда |
| `GET` | `/api/client/geo-stats` | Географическая статистика |
| `GET` | `/api/client/items/coordinates` | Координаты элементов |
| `GET` | `/api/client/items` | Список элементов |
| `POST` | `/api/client/items` | Создать элемент |
| `GET` | `/api/client/items/stats` | Статистика отправок |
| `GET` | `/api/client/items/{id}` | Получить элемент |
| `PUT` | `/api/client/items/{id}` | Обновить элемент |
| `DELETE` | `/api/client/items/{id}` | Мягкое удаление |
| `POST` | `/api/client/items/{id}/restore` | Восстановить |

---

## Статистика дашборда

```
GET /api/client/dashboard/stats
```

Возвращает комплексную статистику дашборда аутентифицированного пользователя: общее число отправок, просмотры, голоса, комментарии, активность, графики и распределение по статусам.

---

## Географическая статистика

```
GET /api/client/geo-stats
```

Возвращает географический анализ элементов пользователя: общее число, элементы с локацией, удалённая работа, топ города и страны.

---

## Координаты элементов

```
GET /api/client/items/coordinates
```

Возвращает координаты всех элементов пользователя, имеющих геоданные, для отображения на карте.

---

## Управление элементами

### Список элементов

```
GET /api/client/items
```

**Параметры запроса:**

| Параметр | Тип | По умолч. | Описание |
|-----------|------|---------|-------------|
| `page` | integer | `1` | Номер страницы |
| `limit` | integer | `10` | Элементов на странице (1–100) |
| `status` | string | — | `all`, `pending`, `approved`, `rejected` |
| `search` | string | — | Поиск по названию / описанию |
| `deleted` | boolean | `false` | `true` — возвращает мягко удалённые элементы |

### Создать элемент

```
POST /api/client/items
```

Добавляет новый элемент со статусом `pending` для проверки.

| Поле | Тип | Обязательный | Описание |
|-------|------|----------|-------------|
| `name` | string | Да | 3–100 симв. |
| `description` | string | Да | 10–500 симв. |
| `source_url` | string (URI) | Да | Основная ссылка |
| `category` | string/string[] | Нет | Категория |
| `tags` | string[] | Нет | Теги |
| `icon_url` | string (URI) | Нет | URL иконки |

### Обновить элемент

```
PUT /api/client/items/{id}
```

Обновляет элемент, принадлежащий пользователю. Если элемент был `approved`, статус сбрасывается в `pending` для повторной проверки.

### Удалить (мягкое удаление)

```
DELETE /api/client/items/{id}
```

Устанавливает `deletedAt`; восстановление возможно позже.

### Восстановить элемент

```
POST /api/client/items/{id}/restore
```

Восстанавливает мягко удалённый элемент.

---

## Коды ошибок

| Статус | Значение |
|--------|----------|
| `401` | Требуется аутентификация |
| `403` | Не владелец элемента |
| `404` | Элемент не найден |
| `400` | Ошибка валидации |

Все конечные точки используют `serverErrorResponse()`, чтобы не раскрывать подробности внутренних ошибок клиенту.
