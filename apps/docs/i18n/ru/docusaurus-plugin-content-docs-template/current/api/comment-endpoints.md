# Конечные точки комментариев

Система комментариев поддерживает создание, чтение, обновление и удаление комментариев к элементам. Комментарий включает оценку от 1 до 5 звёзд.

## Обзор

### Публичные конечные точки

| Метод | Конечная точка | Описание |
|--------|----------|-------------|
| `GET` | `/api/items/[slug]/comments` | Список комментариев |
| `GET` | `/api/items/[slug]/comments/rating` | Статистика оценок |
| `GET` | `/api/items/[slug]/comments/rating/[commentId]` | Оценка комментария |

### Аутентифицированные конечные точки

| Метод | Конечная точка | Описание |
|--------|----------|-------------|
| `POST` | `/api/items/[slug]/comments` | Создать комментарий |
| `PUT` | `/api/items/[slug]/comments/[commentId]` | Изменить свой |
| `DELETE` | `/api/items/[slug]/comments/[commentId]` | Удалить свой |
| `PATCH` | `/api/items/[slug]/comments/rating/[commentId]` | Обновить оценку |

### Административные конечные точки

| Метод | Конечная точка | Описание |
|--------|----------|-------------|
| `GET` | `/api/admin/comments` | Список всех комментариев |
| `GET` | `/api/admin/comments/[id]` | Получить по ID |
| `PUT` | `/api/admin/comments/[id]` | Обновить содержимое |
| `DELETE` | `/api/admin/comments/[id]` | Мягкое удаление |

---

## Публичные конечные точки

### Список комментариев

```
GET /api/items/[slug]/comments
```

Возвращает все комментарии с данными профиля пользователя. Аутентификация не требуется.

### Статистика оценок

```
GET /api/items/[slug]/comments/rating
```

Возвращает среднюю оценку и общее число оценок. Учитываются только неудалённые комментарии.

---

## Аутентифицированные конечные точки

### Создать комментарий

```
POST /api/items/[slug]/comments
```

Требуется авторизация и действительный профиль. Заблокированные пользователи не могут оставлять комментарии.

**Тело запроса:**

```json
{
  "content": "Отличный инструмент!",
  "rating": 5
}
```

| Поле | Тип | Обязательный | Ограничения |
|-------|------|----------|----------|
| `content` | string | Да | Непустой |
| `rating` | integer | Да | От 1 до 5 |

| Статус | Условие |
|--------|----------|
| `400` | Пустой контент или неверная оценка |
| `401` | Не аутентифицирован |
| `403` | Пользователь заблокирован |
| `404` | Профиль не найден |

### Обновить комментарий

```
PUT /api/items/[slug]/comments/[commentId]
```

Только автор может редактировать свой комментарий. Требуется хотя бы одно из полей (`content` или `rating`).

| Поле | Ограничения |
|-------|----------|
| `content` | 1–1000 симв. |
| `rating` | От 1 до 5 |

### Удалить комментарий

```
DELETE /api/items/[slug]/comments/[commentId]
```

Мягкое удаление через `deletedAt`. Ответ: 204 No Content.

---

## Административные конечные точки

Требуют `session.user.isAdmin = true`.

### Список комментариев

```
GET /api/admin/comments
```

**Параметры:** `page` (1), `limit` (10), `search`.

### Админ: удалить комментарий

```
DELETE /api/admin/comments/[id]
```

Мягкое удаление. Комментарий должен существовать и не быть удалён ранее.

| Статус | Условие |
|--------|----------|
| `403` | Не админ |
| `404` | Не найден или уже удалён |

---

## Ключевые особенности реализации

- **Мягкое удаление:** устанавливается `deletedAt`, запись не удаляется.
- **Проверка владельца:** `clientProfileId` сессии должен совпадать с `userId` комментария.
- **Блокировка:** `isUserBlocked()` препятствует оставлять комментарии зазаблокированным пользователям.
- **Поиск (admin):** ILIKE с экранированием `%` и `_`.
