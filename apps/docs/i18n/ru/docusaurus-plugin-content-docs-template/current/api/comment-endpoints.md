# Конечные точки комментариев

Система комментариев предоставляет конечные точки для создания, чтения, обновления и удаления комментариев к элементам. Комментарии имеют рейтинг от 1 до 5 звезд и поддерживают как публичный доступ (чтение), так и операции с аутентификацией (создание/редактирование/удаление). Конечные точки администратора предоставляют возможности модерации.

## Обзор

### Публичные конечные точки

|Конечная точка|Метод|Авторизация|Описание|
|---|---|---|---|
|`/api/items/[slug]/comments`|ПОЛУЧИТЬ|Общественный|Список комментариев к элементу|
|`/api/items/[slug]/comments/rating`|ПОЛУЧИТЬ|Общественный|Получить совокупную рейтинговую статистику|
|`/api/items/[slug]/comments/rating/[commentId]`|ПОЛУЧИТЬ|Общественный|Получить оценку одного комментария|

### Аутентифицированные конечные точки

|Конечная точка|Метод|Авторизация|Описание|
|---|---|---|---|
|`/api/items/[slug]/comments`|ПОСТ|Пользователь|Создать новый комментарий|
|`/api/items/[slug]/comments/[commentId]`|ПОСТАВИТЬ|Владелец|Обновить собственный комментарий|
|`/api/items/[slug]/comments/[commentId]`|УДАЛИТЬ|Владелец|Удалить собственный комментарий|
|`/api/items/[slug]/comments/rating/[commentId]`|ПАТЧ|Пользователь|Обновить рейтинг комментария|

### Конечные точки администратора

|Конечная точка|Метод|Авторизация|Описание|
|---|---|---|---|
|`/api/admin/comments`|ПОЛУЧИТЬ|Админ|Список всех комментариев с нумерацией страниц|
|`/api/admin/comments/[id]`|ПОЛУЧИТЬ|Админ|Получить комментарий по идентификатору|
|`/api/admin/comments/[id]`|ПОСТАВИТЬ|Админ|Обновить содержание комментариев|
|`/api/admin/comments/[id]`|УДАЛИТЬ|Админ|Мягкое удаление комментария|

## Публичные конечные точки

### Комментарии к элементу списка

```
GET /api/items/[slug]/comments
```

Возвращает все комментарии к определенному элементу, включая информацию профиля пользователя. Никакой аутентификации не требуется.

**Параметры пути:**

|Параметр|Тип|Описание|
|---|---|---|
|`slug`|строка|Пуля элемента|

**Успешный ответ (200):**

```json
{
  "success": true,
  "comments": [
    {
      "id": "comment_123abc",
      "content": "This is an amazing tool!",
      "rating": 5,
      "userId": "client_456def",
      "itemId": "item_123abc",
      "createdAt": "2024-01-20T10:30:00.000Z",
      "updatedAt": "2024-01-20T10:30:00.000Z",
      "deletedAt": null,
      "user": {
        "id": "client_456def",
        "name": "John Doe",
        "email": "john.doe@example.com",
        "avatar": "https://example.com/avatars/john.jpg"
      }
    }
  ]
}
```

**Источник:** `template/app/api/items/[slug]/comments/route.ts`

### Получить рейтинговую статистику

```
GET /api/items/[slug]/comments/rating
```

Возвращает средний рейтинг и общее количество оценок для элемента. Учитываются только неудаленные комментарии.

**Успешный ответ (200):**

```json
{
  "averageRating": 4.2,
  "totalRatings": 15
}
```

Возвращает `averageRating: 0` и `totalRatings: 0`, если рейтинги отсутствуют.

**Источник:** `template/app/api/items/[slug]/comments/rating/route.ts`

## Аутентифицированные конечные точки

### Создать комментарий

```
POST /api/items/[slug]/comments
```

Создает новый комментарий с оценкой элемента. Требуется аутентификация и действительный профиль клиента. Заблокированные пользователи не могут комментировать.

**Аутентификация:** Требуется

**Тело запроса:**

```json
{
  "content": "This is an amazing tool! Really helped boost my productivity.",
  "rating": 5
}
```

|Поле|Тип|Требуется|Ограничения|
|---|---|---|---|
|`content`|строка|Да|Должно быть непустым после обрезки|
|`rating`|целое число|Да|Должно быть от 1 до 5 включительно.|

**Успешный ответ (200):**

```json
{
  "success": true,
  "comment": {
    "id": "comment_123abc",
    "content": "This is an amazing tool!",
    "rating": 5,
    "userId": "client_456def",
    "itemId": "item_123abc",
    "createdAt": "2024-01-20T10:30:00.000Z",
    "user": {
      "id": "client_456def",
      "name": "John Doe",
      "email": "john.doe@example.com",
      "avatar": "https://example.com/avatars/john.jpg"
    }
  }
}
```

|Статус|Состояние|
|---|---|
| 400 |Пустой контент или неверный рейтинг.|
| 401 |Не аутентифицирован|
| 403 |Пользователь заблокирован или заблокирован|
| 404 |Профиль клиента не найден|

**Источник:** `template/app/api/items/[slug]/comments/route.ts`

### Обновить комментарий

```
PUT /api/items/[slug]/comments/[commentId]
```

Обновляет содержание и/или рейтинг существующего комментария. Только автор комментария может обновлять свой комментарий. Должен быть указан хотя бы один из `content` или `rating`.

**Аутентификация:** Требуется (должен быть владелец комментария)

**Тело запроса:**

```json
{
  "content": "Updated review text",
  "rating": 4
}
```

|Поле|Тип|Требуется|Ограничения|
|---|---|---|---|
|`content`|строка|Нет|1-1000 символов|
|`rating`|целое число|Нет|Должно быть от 1 до 5|

Ответ включает обновленный комментарий с отметкой времени `editedAt`.

|Статус|Состояние|
|---|---|
| 400 |Поля не предоставлены, содержание слишком длинное или недействительный рейтинг.|
| 401 |Не аутентифицирован|
| 404 |Комментарий не найден или пользователь не является автором|

**Источник:** `template/app/api/items/[slug]/comments/[commentId]/route.ts`

### Удалить комментарий

```
DELETE /api/items/[slug]/comments/[commentId]
```

Мягкое удаление комментария. Только автор комментария может удалить свой комментарий. Комментарий помечается меткой времени `deletedAt`, а не удаляется навсегда.

**Аутентификация:** Требуется (должен быть владелец комментария)

**Ответ об успешном завершении:** 204 Нет содержимого

|Статус|Состояние|
|---|---|
| 401 |Не аутентифицирован|
| 404 |Комментарий не найден, уже удален или не принадлежит пользователю|

**Источник:** `template/app/api/items/[slug]/comments/[commentId]/route.ts`

### Обновить рейтинг комментариев

```
PATCH /api/items/[slug]/comments/rating/[commentId]
```

Обновляет только рейтинг конкретного комментария.

**Тело запроса:**

```json
{
  "rating": 4
}
```

**Источник:** `template/app/api/items/[slug]/comments/rating/[commentId]/route.ts`

## Конечные точки администратора

Все конечные точки администратора требуют, чтобы `session.user.isAdmin` имело значение true.

### Список всех комментариев

```
GET /api/admin/comments
```

Возвращает постраничный список всех комментариев (за исключением обратимо удаленных) с информацией о пользователе. Поддерживает поиск по содержанию комментариев, имени пользователя и электронной почте пользователя.

**Параметры запроса:**

|Параметр|Тип|По умолчанию|Описание|
|---|---|---|---|
|`page`|целое число| 1 |Номер страницы|
|`limit`|целое число| 10 |Результатов на странице (1–100)|
|`search`|строка| - |Поиск по содержимому, имени пользователя или электронной почте|

**Успешный ответ (200):**

```json
{
  "success": true,
  "data": {
    "comments": [
      {
        "id": "comment_123abc",
        "content": "Great product!",
        "rating": 5,
        "userId": "user_456def",
        "itemId": "item_789ghi",
        "createdAt": "2024-01-20T10:30:00.000Z",
        "updatedAt": "2024-01-20T10:30:00.000Z",
        "user": {
          "id": "user_456def",
          "name": "John Doe",
          "email": "john.doe@example.com",
          "image": "https://example.com/avatar.jpg"
        }
      }
    ],
    "pagination": {
      "total": 156,
      "page": 1,
      "limit": 10,
      "totalPages": 16
    }
  }
}
```

**Источник:** `template/app/api/admin/comments/route.ts`

### Получить комментарий по идентификатору

```
GET /api/admin/comments/[id]
```

Получает конкретный комментарий с полной информацией о пользователе.

**Источник:** `template/app/api/admin/comments/[id]/route.ts`

### Комментарий администратора к обновлению

```
PUT /api/admin/comments/[id]
```

Позволяет администраторам обновлять содержимое любого комментария независимо от владельца.

**Тело запроса:**

```json
{
  "content": "This content has been moderated by an administrator."
}
```

**Источник:** `template/app/api/admin/comments/[id]/route.ts`

### Админ Удалить комментарий

```
DELETE /api/admin/comments/[id]
```

Мягкое удаление любого комментария. Комментарий должен существовать и еще не быть удален.

**Успешный ответ (200):**

```json
{
  "success": true,
  "message": "Comment deleted successfully"
}
```

|Статус|Состояние|
|---|---|
| 403 |Не администратор|
| 404 |Комментарий не найден или уже удален|

**Источник:** `template/app/api/admin/comments/[id]/route.ts`

## Ключевые детали реализации

- **Мягкое удаление:** при любом удалении устанавливается `deletedAt`, а не запись. Запросы отфильтровывают удаленные комментарии через `isNull(comments.deletedAt)`.
- **Подтверждение владения.** Конечные точки пользователя проверяют, что идентификатор профиля клиента аутентифицированного пользователя соответствует полю `userId` комментария.
- **Предотвращение блокировки пользователей:** Проверка `isUserBlocked()` не позволяет заблокированным или заблокированным пользователям создавать комментарии.
- **Поиск (Администратор):** Использует ILIKE для поиска без учета регистра с правильным экранированием подстановочных знаков SQL (`%` и `_`).
