# Справочник API: конфигурация и флаги функций

## Обзор

Конечная точка Config Features возвращает текущие флаги доступности функций приложения. Эти флаги указывают, какие зависящие от базы данных функции активны, позволяя фронтенду корректно деградировать при недоступности функций. Это публичная, кешируемая конечная точка, рассчитанная на частое обращение.

## Конечные точки

### GET /api/config/features

Возвращает текущую доступность функций на основе конфигурации системы и доступности базы данных.

**Запрос**

Параметры и тело запроса не требуются.

**Ответ**
```typescript
{
  ratings: boolean;         // Доступна ли функция рейтингов
  comments: boolean;        // Доступна ли функция комментариев
  favorites: boolean;       // Доступна ли функция избранного
  featuredItems: boolean;   // Доступна ли функция рекомендуемых элементов
  surveys: boolean;         // Доступна ли функция опросов
}
```

**Пример**
```typescript
const response = await fetch('/api/config/features');
const features = await response.json();

if (features.ratings) {
  // Отобразить компонент рейтинга
}

if (!features.surveys) {
  // Скрыть раздел опросов
}
```

## Аутентификация

Эта конечная точка **публична** — аутентификация не требуется. Она предназначена для использования фронтендом при начальной загрузке страницы для определения, какие функции UI должны отображаться.

## Коды ошибок

| Статус | Описание |
|--------|----------|
| 200 | Флаги функций успешно получены |
| 500 | Внутренняя ошибка — возвращает все флаги как `false` с заголовком `no-cache` |

При ошибке конечная точка возвращает все функции как `false`, чтобы приложение деградировало безопасно, не показывая сломанный функционал.

## Ограничения частоты запросов

Ответы кешируются с заголовками:
- `Cache-Control: public, s-maxage=300, stale-while-revalidate=600`
- Эффективно кешируется на 5 минут на уровне CDN с 10-минутным окном stale-while-revalidate.

Ответы с ошибками используют `Cache-Control: no-cache` для предотвращения кеширования деградированного состояния.
