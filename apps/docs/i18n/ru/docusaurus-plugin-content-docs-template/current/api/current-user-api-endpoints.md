# Конечные точки API текущего пользователя

API текущего пользователя предоставляет единственную конечную точку для получения информации аутентифицированного пользователя. Фронтенд использует её для определения состояния аутентификации и привилегий пользователя.

**Источник:** `template/app/api/current-user/route.ts`

---

## Получить текущего пользователя

Возвращает безопасную информацию профиля аутентифицированного пользователя. Возвращает `null`, если пользователь не аутентифицирован.

| Свойство | Значение |
|----------|-------|
| **Метод** | `GET` |
| **Путь** | `/api/current-user` |
| **Аутентификация** | Не требуется (возвращает `null` если не аутентифицирован) |

### Ответ

**Статус 200** — Аутентифицированный пользователь.

```json
{
  "id": "user_123abc",
  "name": "John Doe",
  "email": "john.doe@example.com",
  "image": "https://example.com/avatars/john.jpg",
  "provider": "google",
  "isAdmin": false
}
```

**Статус 200** — Пользователь не аутентифицирован.

```json
null
```

### Поля ответа

| Поле | Тип | Может быть null | Описание |
|-------|------|----------|----------|
| `id` | `string` | Нет | Уникальный идентификатор пользователя |
| `name` | `string` | Да | Полное имя пользователя |
| `email` | `string` | Да | Адрес электронной почты |
| `image` | `string` | Да | URL аватара профиля |
| `provider` | `string` | Да | Провайдер аутентификации (`google`, `github`, `credentials`) |
| `isAdmin` | `boolean` | Нет | Имеет ли пользователь права администратора (no умолчанию `false`) |

### Примеры запросов curl

```bash
# Получить текущего пользователя (аутентифицированный)
curl -s http://localhost:3000/api/current-user \
  -H "Cookie: next-auth.session-token=<session_token>"

# Получить текущего пользователя (не аутентифицированный — возвращает null)
curl -s http://localhost:3000/api/current-user
```

### Имплементационные примечания

- Конечная точка использует функцию `auth()` из `@/lib/auth` (NextAuth.js) для чтения серверной сессии.
- Ответ санитизирован — возвращаются только безопасные поля профиля. Чувствительные данные исключены.
- Конечная точка всегда возвращает HTTP 200, даже если пользователь не аутентифицирован. Отличие определяется тем, является ли ответ `null`.
- Поле `isAdmin` умолчанию `false`, что обеспечивает безопасное поведение для обычных пользователей.
