# Справочник API геокодирования

## Обзор

Конечные точки геокодирования предоставляют возможности прямого геокодирования (адрес → координаты) и обратного геокодирования (координаты → адрес). Результаты кешируются на 15 минут для снижения нагрузки на внешние API. Эти конечные точки требуют аутентификации администратора для предотвращения злоупотреблений платными сервисами геокодирования Mapbox/Google.

## Конечные точки

### POST /api/geocode

Преобразует адрес в координаты (прямое геокодирование) или координаты в адрес (обратное геокодирование). Операция определяется содержимым тела запроса — в зависимости от наличия поля `address` или полей `latitude`/`longitude`.

#### Прямое геокодирование (адрес → координаты)

**Запрос**
```typescript
{
  address: string;          // 1-500 символов, обязательное
  options?: {
    countryCodes?: string[];  // Коды ISO 3166-1 alpha-2, например ["US", "CA"]
    language?: string;        // Код языка ISO 639-1, например "en"
    proximity?: {
      latitude: number;       // от -90 до 90
      longitude: number;      // от -180 до 180
    };
  };
}
```

**Ответ**
```typescript
{
  success: true;
  data: {
    latitude: number;
    longitude: number;
    formattedAddress: string;
    city: string;
    state: string;
    country: string;
    countryCode: string;
    postalCode: string;
    confidence: number;       // от 0 до 1
  };
}
```

#### Обратное геокодирование (координаты → адрес)

**Запрос**
```typescript
{
  latitude: number;         // от -90 до 90, обязательное
  longitude: number;        // от -180 до 180, обязательное
  options?: {
    language?: string;        // Код языка ISO 639-1
  };
}
```

**Ответ**
```typescript
{
  success: true;
  data: {
    formattedAddress: string;
    streetAddress: string;
    city: string;
    state: string;
    country: string;
    countryCode: string;
    postalCode: string;
  };
}
```

### GET /api/geocode

Возвращает статус сервиса геокодирования, включая информацию о настроенных провайдерах и статистику кеша.

**Ответ**
```typescript
{
  success: true;
  data: {
    enabled: boolean;         // Включены ли функции местоположения
    configured: boolean;      // Настроен ли хотя бы один провайдер геокодирования
    providers: {
      mapbox: boolean;
      google: boolean;
    };
    cache: {
      size: number;           // Текущий размер кеша
      maxSize: number;        // Максимальный размер кеша (1000)
      ttlMs: number;          // TTL кеша в миллисекундах (900000 = 15 мин)
    };
  };
}
```

## Аутентификация

- **GET /api/geocode**: Требуется аутентифицированная сессия (любой пользователь).
- **POST /api/geocode**: Требуется аутентифицированная сессия с **ролью администратора**. Пользователи без прав администратора получают ответ `403 Forbidden`. Это ограничение предотвращает злоупотребление платными API.

## Коды ошибок

| Статус | Описание |
|--------|----------|
| 400 | Некорректные данные запроса — неверный адрес, координаты или ошибка валидации схемы |
| 401 | Не авторизован — нет аутентифицированной сессии |
| 403 | Доступ запрещён — требуются права администратора (только POST) |
| 404 | Результаты геокодирования не найдены для указанного адреса или координат |
| 503 | Функции местоположения отключены в настройках или сервис геокодирования не настроен |

## Ограничения частоты запросов

Результаты кешируются на 15 минут (TTL 900 000 мс) с максимальным размером кеша 1000 записей. Все запросы геокодирования журналируются для отслеживания затрат.
