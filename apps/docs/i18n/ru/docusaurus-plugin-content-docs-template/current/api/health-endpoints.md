# Справочник по API здоровья

## Обзор

Конечная точка работоспособности обеспечивает простую проверку подключения к базе данных для целей мониторинга и инфраструктуры. Он выполняет упрощенный запрос, чтобы убедиться, что соединение с базой данных активно и отвечает, возвращая информацию о состоянии с метками времени.

## Конечные точки

### ПОЛУЧИТЬ /api/здоровье/базу данных

Выполняет базовую проверку работоспособности базы данных, выполняя запрос `SELECT 1` для проверки соединения с базой данных.

**Запрос**

Никаких параметров или тела не требуется.

**Ответ**
```typescript
// Healthy response
{
  status: "healthy";
  database: "connected";
  timestamp: string;        // ISO 8601 format, e.g. "2024-01-15T10:30:00.000Z"
  result: object;           // Raw query result from SELECT 1
}

// Unhealthy response (status 500)
{
  status: "unhealthy";
  database: "disconnected";
  error: "Database connection check failed";
  timestamp: string;
}
```

**Пример**
```typescript
const response = await fetch('/api/health/database');
const health = await response.json();

if (health.status === 'healthy') {
  console.log('Database is connected at', health.timestamp);
} else {
  console.error('Database is disconnected:', health.error);
}
```

## Аутентификация

Эта конечная точка является **общедоступной** – аутентификация не требуется. Он предназначен для использования балансировщиками нагрузки, мониторами времени безотказной работы и проверками работоспособности развертываний.

## Реакции на ошибки

|Статус|Описание|
|--------|-------------|
| 200 |Соединение с базой данных исправно|
| 500 |Не удалось подключиться к базе данных — возвращает статус `"unhealthy"` со сведениями об ошибке.|

## Ограничение скорости

Явное ограничение скорости не применяется. Эта конечная точка является легкой и подходит для частого опроса системами мониторинга.
