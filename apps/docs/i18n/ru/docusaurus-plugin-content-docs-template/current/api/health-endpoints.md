# Справочник API состояния системы

## Обзор

Конечная точка Health обеспечивает простую проверку соединения с базой данных для целей мониторинга и инфраструктуры. Она выполняет лёгкий запрос, чтобы убедиться, что соединение с базой данных активно и отвечает, и возвращает статусную информацию с временными метками.

## Конечные точки

### GET /api/health/database

Выполняет базовую проверку состояния базы данных, выполняя запрос `SELECT 1` для проверки соединения.

**Запрос**

Параметры и тело запроса не требуются.

**Ответ**
```typescript
// Ответ при нормальном состоянии
{
  status: "healthy";
  database: "connected";
  timestamp: string;        // ISO 8601, например "2024-01-15T10:30:00.000Z"
  result: object;           // Сырой результат запроса SELECT 1
}

// Ответ при сбое (статус 500)
{
  status: "unhealthy";
  database: "disconnected";
  error: "Database connection check failed";
  timestamp: string;
}
```

**Пример**
```typescript
const response = await fetch('/api/health/database');
const health = await response.json();

if (health.status === 'healthy') {
  console.log('База данных подключена:', health.timestamp);
} else {
  console.error('База данных недоступна:', health.error);
}
```

## Аутентификация

Эта конечная точка **публична** — аутентификация не требуется. Она предназначена для использования балансировщиками нагрузки, мониторингом доступности и проверками состояния при развёртывании.

## Коды ошибок

| Статус | Описание |
|--------|----------|
| 200 | Соединение с базой данных в норме |
| 500 | Соединение с базой данных не установлено — возвращает статус `"unhealthy"` с деталями ошибки |

## Ограничения частоты запросов

Явных ограничений нет. Конечная точка лёгкая и подходит для частого опроса системами мониторинга.
