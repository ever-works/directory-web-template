# Конечные точки платежного API

Шаблон поддерживает четыре платежных системы: **Stripe**, **Lemon Squeezy**, **Polar** и **Solidgate**. Каждый провайдер имеет свой собственный набор маршрутов API для оформления заказа, управления подписками и обработки веб-перехватчиков. Общая группа `/api/payment` предоставляет запросы на подписку, не зависящие от поставщика.

## Полоса (`/api/stripe`)

Stripe — это наиболее полнофункциональная интеграция с 17 обработчиками маршрутов, охватывающими оформление заказа, подписки, способы оплаты, намерения настройки и продукты.

### Оформить заказ

|Метод|Путь|Описание|
|--------|------|-------------|
|`POST`|`/api/stripe/checkout`|Создайте сеанс проверки Stripe|

### Подписки

|Метод|Путь|Описание|
|--------|------|-------------|
|`GET`|`/api/stripe/subscription`|Получить активную подписку текущего пользователя|
|`POST`|`/api/stripe/subscription`|Создать новую подписку|
|`GET`|`/api/stripe/subscriptions`|Список всех подписок пользователей|
|`POST`|`/api/stripe/subscription/[subscriptionId]/cancel`|Отменить подписку|
|`POST`|`/api/stripe/subscription/[subscriptionId]/reactivate`|Повторно активировать отмененную подписку|
|`POST`|`/api/stripe/subscription/[subscriptionId]/update`|Обновить подписку (сменить план)|
|`POST`|`/api/stripe/subscription/portal`|Создайте сеанс клиентского портала Stripe.|

### Способы оплаты

|Метод|Путь|Описание|
|--------|------|-------------|
|`GET`|`/api/stripe/payment-methods/list`|Список сохраненных способов оплаты|
|`POST`|`/api/stripe/payment-methods/create`|Добавить новый способ оплаты|
|`PUT`|`/api/stripe/payment-methods/update`|Обновить способ оплаты по умолчанию|
|`DELETE`|`/api/stripe/payment-methods/delete`|Удаление способа оплаты|
|`GET`|`/api/stripe/payment-methods/[id]`|Получить информацию о способе оплаты|

### Намерения установки

|Метод|Путь|Описание|
|--------|------|-------------|
|`POST`|`/api/stripe/setup-intent`|Создайте намерение настройки для сохранения способа оплаты.|
|`GET`|`/api/stripe/setup-intent/[id]`|Получить статус намерения установки|

### Платежные намерения

|Метод|Путь|Описание|
|--------|------|-------------|
|`POST`|`/api/stripe/payment-intent`|Создайте намерение единовременного платежа|

### Продукты

|Метод|Путь|Описание|
|--------|------|-------------|
|`GET`|`/api/stripe/products`|Список доступных продуктов и цен Stripe|

### Вебхук

|Метод|Путь|Описание|
|--------|------|-------------|
|`POST`|`/api/stripe/webhook`|Обработчик событий веб-перехватчика Stripe|

Обработчик веб-перехватчика Stripe обрабатывает такие события, как:
- `checkout.session.completed` - Завершение оформления заказа
- `customer.subscription.created` - Новая подписка
- `customer.subscription.updated` - Изменения в подписке
- `customer.subscription.deleted` - Отмена подписки
- `invoice.payment_succeeded` - Успешная оплата
- `invoice.payment_failed` - Неудачная оплата

## Лимонная выжимка (`/api/lemonsqueezy`)

Lemon Squeezy предоставляет более простую модель подписки с 7 конечными точками.

|Метод|Путь|Описание|
|--------|------|-------------|
|`POST`|`/api/lemonsqueezy/checkout`|Создайте кассу с лимонной выжимкой|
|`GET`|`/api/lemonsqueezy/list`|Получение списка подписок пользователей|
|`POST`|`/api/lemonsqueezy/cancel`|Отменить подписку|
|`POST`|`/api/lemonsqueezy/reactivate`|Повторно активировать отмененную подписку|
|`POST`|`/api/lemonsqueezy/update`|Обновить сведения о подписке|
|`POST`|`/api/lemonsqueezy/update-plan`|Изменить план подписки|
|`POST`|`/api/lemonsqueezy/webhook`|Обработчик веб-перехватчика Lemon Squeezy|

### События вебхука

Вебхук Lemon Squeezy обрабатывает:
- `subscription_created` - Новая подписка
- `subscription_updated` - Изменения в плане
- `subscription_cancelled` - Отмена
- `subscription_payment_success` - Подтверждение оплаты
- `subscription_payment_failed` - Сбой платежа

## Полярный (`/api/polar`)

Polar предоставляет 5 конечных точек для оформления заказа и управления подписками.

|Метод|Путь|Описание|
|--------|------|-------------|
|`POST`|`/api/polar/checkout`|Создайте сеанс оформления заказа Polar|
|`POST`|`/api/polar/subscription/[subscriptionId]/cancel`|Отменить подписку|
|`POST`|`/api/polar/subscription/[subscriptionId]/reactivate`|Повторно активировать подписку|
|`POST`|`/api/polar/subscription/portal`|Доступ к порталу подписки|
|`POST`|`/api/polar/webhook`|Обработчик веб-перехватчика Polar|

## Солидгейт (`/api/solidgate`)

Solidgate — самая минимальная интеграция с двумя конечными точками.

|Метод|Путь|Описание|
|--------|------|-------------|
|`POST`|`/api/solidgate/checkout`|Создать кассу Solidgate|
|`POST`|`/api/solidgate/webhook`|Обработчик вебхука Solidgate|

## Общий платеж (`/api/payment`)

Независимые от поставщика конечные точки оплаты для управления подписками независимо от основного поставщика платежей.

|Метод|Путь|Описание|
|--------|------|-------------|
|`GET`|`/api/payment/[subscriptionId]`|Получить информацию о подписке по идентификатору|
|`GET`|`/api/payment/account`|Получить платежный счет для текущего пользователя|
|`GET`|`/api/payment/account/[userId]`|Получить платежный счет для конкретного пользователя (администратора)|

## Безопасность вебхука

Все конечные точки веб-перехватчиков реализуют проверку подписи для конкретного поставщика:

### Полоса

Веб-перехватчики Stripe проверяют заголовок `stripe-signature` с помощью переменной среды `STRIPE_WEBHOOK_SECRET` и метода `stripe.webhooks.constructEvent()`.

### Лимонный Скизи

Веб-перехватчики Lemon Squeezy проверяют заголовок `x-signature` с помощью HMAC-SHA256 с `LEMONSQUEEZY_WEBHOOK_SECRET`.

### Полярный

Веб-перехватчики Polar проверяют подписи запросов с помощью `POLAR_WEBHOOK_SECRET`.

### Солидгейт

Веб-перехватчики Solidgate используют встроенную в свой SDK проверку подписи с помощью `SOLIDGATE_SECRET_KEY`.

## Переменные среды

### Полоса

|Переменная|Описание|
|----------|-------------|
|`STRIPE_SECRET_KEY`|Секретный ключ Stripe API|
|`STRIPE_PUBLISHABLE_KEY`|Публикуемый ключ Stripe (на стороне клиента)|
|`STRIPE_WEBHOOK_SECRET`|Секрет подписи вебхука|

### Лимонный Скизи

|Переменная|Описание|
|----------|-------------|
|`LEMONSQUEEZY_API_KEY`|API-ключ Lemon Squeezy|
|`LEMONSQUEEZY_STORE_ID`|Идентификатор магазина|
|`LEMONSQUEEZY_WEBHOOK_SECRET`|Секрет подписи вебхука|

### Полярный

|Переменная|Описание|
|----------|-------------|
|`POLAR_ACCESS_TOKEN`|Токен доступа к API Polar|
|`POLAR_WEBHOOK_SECRET`|Секрет подписи вебхука|
|`POLAR_ORGANIZATION_ID`|Идентификатор организации|

### Солидгейт

|Переменная|Описание|
|----------|-------------|
|`SOLIDGATE_MERCHANT_ID`|Идентификатор продавца|
|`SOLIDGATE_SECRET_KEY`|Секретный ключ API|

## Требования аутентификации

|Тип конечной точки|Требуется авторизация|
|--------------|---------------|
|Создание оформления заказа|Да (аутентифицированный пользователь)|
|Управление подпиской|Да (владелец подписки)|
|Управление способами оплаты|Да (клиент Stripe)|
|Список продуктов|Общественный (продукты Stripe)|
|Обработчики вебхуков|Проверка подписи (без сеанса)|
|Общие платежные запросы|Да (владелец или администратор аккаунта)|
