# Пользовательские конечные точки

Пользовательский API предоставляет конечные точки для управления настройками прошедших проверку подлинности пользователей, сведениями о подписке, историей платежей и настройками местоположения профиля. Все конечные точки требуют аутентификации на основе сеанса.

## Обзор

|Конечная точка|Метод|Авторизация|Описание|
|---|---|---|---|
|`/api/user/currency`|ПОЛУЧИТЬ|Общественный|Определить валюту пользователя по заголовкам|
|`/api/user/currency`|ПОСТАВИТЬ|Пользователь|Обновить настройки валюты|
|`/api/user/payments`|ПОЛУЧИТЬ|Пользователь|Получить историю платежей от Stripe|
|`/api/user/plan-status`|ПОЛУЧИТЬ|Пользователь|Получить статус плана с информацией об истечении срока действия|
|`/api/user/subscription`|ПОЛУЧИТЬ|Пользователь|Получить подробную информацию о подписке|
|`/api/user/profile/location`|ПОЛУЧИТЬ|Пользователь|Получить сохраненные настройки местоположения|
|`/api/user/profile/location`|ПАТЧ|Пользователь|Обновить настройки местоположения|

## Обнаружение валюты и настройки

### Определить валюту

```
GET /api/user/currency
```

Определяет валюту пользователя на основе HTTP-заголовков от поставщиков CDN/прокси. Эта конечная точка использует плавную деградацию: она всегда возвращает 200 OK с действительным кодом валюты и возвращается к доллару США в случае неудачного обнаружения. Никакой аутентификации не требуется.

**Параметры запроса:**

|Параметр|Тип|По умолчанию|Описание|
|---|---|---|---|
|`provider`|строка|`"smart"`|Поставщик обнаружения: `"cloudflare"`, `"vercel"`, `"cloudfront"`, `"fastly"`, `"generic"`, `"auto"`, `"smart"`|

**Успешный ответ (200):**

```json
{
  "currency": "EUR",
  "country": "FR",
  "detected": true
}
```

|Поле|Тип|Описание|
|---|---|---|
|`currency`|строка|Код валюты ISO 4217 (3 символа), по умолчанию `"USD"`|
|`country`|строка или ноль|Код страны ISO 3166-1 альфа-2, ноль, если обнаружение не удалось.|
|`detected`|логическое значение|Удалось ли обнаружение или значение является запасным вариантом|

Если обнаружение не удалось, ответ по-прежнему возвращает 200 с `"USD"` и `detected: false`.

**Источник:** `template/app/api/user/currency/route.ts`

### Обновить настройки валюты

```
PUT /api/user/currency
```

Обновляет предпочитаемую валюту и страну аутентифицированного пользователя. Проверено с помощью Zod со списком `SUPPORTED_CURRENCIES` от `lib/config/billing`.

**Аутентификация:** Требуется

**Тело запроса:**

```json
{
  "currency": "EUR",
  "country": "FR"
}
```

|Поле|Тип|Требуется|Описание|
|---|---|---|---|
|`currency`|строка|Да|Код валюты ISO 4217 (ровно 3 символа, заглавные)|
|`country`|строка или ноль|Нет|Код страны ISO 3166-1 альфа-2 (ровно 2 символа)|

**Успешный ответ (200):**

```json
{
  "currency": "EUR",
  "country": "FR"
}
```

|Статус|Состояние|
|---|---|
| 400 |Неверный JSON, неподдерживаемый код валюты или неверный формат страны.|
| 401 |Пользователь не авторизован|
| 500 |Не удалось сохранить обновление.|

**Источник:** `template/app/api/user/currency/route.ts`

## История платежей

### Получить историю платежей

```
GET /api/user/payments
```

Получает полную историю платежей аутентифицированного пользователя из Stripe. Извлекает счета и подписки, дополняет их метаданными плана и возвращает отсортированный список записей о платежах.

**Аутентификация:** Требуется

**Успешный ответ (200):**

```json
[
  {
    "id": "in_1234567890abcdef",
    "date": "2024-01-15T10:30:00.000Z",
    "amount": 29.99,
    "currency": "USD",
    "plan": "Premium Plan",
    "planId": "pro",
    "status": "Paid",
    "billingInterval": "monthly",
    "paymentProvider": "stripe",
    "subscriptionId": "sub_1234567890abcdef",
    "description": "Premium Plan - monthly billing",
    "invoiceUrl": "https://invoice.stripe.com/i/acct_123/test_abc",
    "invoicePdf": "https://pay.stripe.com/invoice/acct_123/test_abc/pdf",
    "invoiceNumber": "INV-2024-001",
    "period_end": "2024-02-15T10:30:00.000Z",
    "period_start": "2024-01-15T10:30:00.000Z"
  }
]
```

Ключевые детали обработки:

- Фильтрует только счета `"paid"` и `"open"`.
- Преобразует суммы из центов в основные денежные единицы (делит на 100).
- Сортировка по дате, сначала самые новые
- Сопоставляет статус с удобочитаемыми значениями: `"Paid"`, `"Pending"`, `"Draft"`, `"Unknown"`
- Возвращает пустой массив `[]`, если клиент Stripe не существует.

**Источник:** `template/app/api/user/payments/route.ts`

## Статус плана

### Получить статус плана

```
GET /api/user/plan-status
```

Возвращает подробную информацию о состоянии плана, включая сведения об истечении срока действия. Используется внешним интерфейсом для отображения предупреждений плана и функций ворот после проверок плана.

**Аутентификация:** Требуется

**Успешный ответ (200):**

```json
{
  "success": true,
  "data": {
    "planId": "premium",
    "effectivePlan": "premium",
    "isExpired": false,
    "expiresAt": "2024-12-31T23:59:59.000Z",
    "daysUntilExpiration": 45,
    "isInWarningPeriod": false,
    "canAccessPlanFeatures": true,
    "warningMessage": null,
    "status": "active"
  }
}
```

|Поле|Тип|Описание|
|---|---|---|
|`planId`|строка|Подписной план пользователя: `"free"`, `"standard"`, `"premium"`|
|`effectivePlan`|строка|План, к которому фактически может получить доступ пользователь (может отличаться, если срок его действия истек).|
|`isExpired`|логическое значение|Истек ли срок подписки|
|`expiresAt`|строка или ноль|Срок годности в формате ISO|
|`daysUntilExpiration`|целое число или ноль|Количество дней до истечения срока действия (отрицательное значение, если срок действия уже истек)|
|`isInWarningPeriod`|логическое значение|Истинно, если срок действия подписки истекает в течение 7 дней.|
|`canAccessPlanFeatures`|логическое значение|Может ли пользователь получить доступ к функциям своего плана|
|`warningMessage`|строка или ноль|Предупреждающее сообщение для пользователя, если применимо|
|`status`|строка или ноль|Необработанный статус подписки|

Использует `subscriptionService.getUserPlanWithExpiration()` из `lib/services/subscription.service`.

**Источник:** `template/app/api/user/plan-status/route.ts`

## Подробности подписки

### Получить статус подписки

```
GET /api/user/subscription
```

Получает подробную информацию о подписке из Stripe, включая текущую активную подписку и полную историю подписок.

**Аутентификация:** Требуется

**Успешный ответ (200) – активная подписка:**

```json
{
  "hasActiveSubscription": true,
  "currentSubscription": {
    "id": "sub_1234567890abcdef",
    "planId": "price_1234567890abcdef",
    "planName": "Premium Plan",
    "status": "active",
    "startDate": "2024-01-15T10:30:00.000Z",
    "endDate": "2024-02-15T10:30:00.000Z",
    "nextBillingDate": "2024-02-15T10:30:00.000Z",
    "paymentProvider": "stripe",
    "subscriptionId": "sub_1234567890abcdef",
    "amount": 29.99,
    "currency": "USD",
    "billingInterval": "monthly"
  },
  "subscriptionHistory": [
    {
      "id": "sub_1234567890abcdef",
      "planId": "price_1234567890abcdef",
      "planName": "Premium Plan",
      "status": "active",
      "startDate": "2024-01-15T10:30:00.000Z",
      "endDate": "2024-02-15T10:30:00.000Z",
      "amount": 29.99,
      "currency": "USD",
      "billingInterval": "monthly"
    }
  ]
}
```

Активные подписки обозначаются `status === "active"` или `status === "trialing"`. Записи истории могут включать `cancelledAt` и `cancelReason` для отмененных подписок.

**Источник:** `template/app/api/user/subscription/route.ts`

## Расположение профиля

### Получить настройки местоположения

```
GET /api/user/profile/location
```

Возвращает сохраненное местоположение и настройки конфиденциальности прошедшего проверку подлинности пользователя по умолчанию.

**Аутентификация:** Требуется (профиль клиента)

**Успешный ответ (200):**

```json
{
  "defaultLatitude": 48.8566,
  "defaultLongitude": 2.3522,
  "defaultCity": "Paris",
  "defaultCountry": "FR",
  "locationPrivacy": "city"
}
```

**Источник:** `template/app/api/user/profile/location/route.ts`

### Обновить настройки местоположения

```
PATCH /api/user/profile/location
```

Обновляет местоположение и настройки конфиденциальности аутентифицированного пользователя по умолчанию. Подтверждено с помощью `updateLocationSchema` от `lib/validations/user-location`.

**Тело запроса:**

```json
{
  "defaultLatitude": 48.8566,
  "defaultLongitude": 2.3522,
  "defaultCity": "Paris",
  "defaultCountry": "FR",
  "locationPrivacy": "city"
}
```

|Поле|Тип|Требуется|Описание|
|---|---|---|---|
|`defaultLatitude`|число или ноль|Нет|Координата широты|
|`defaultLongitude`|число или ноль|Нет|Координата долготы|
|`defaultCity`|строка или ноль|Нет|Название города|
|`defaultCountry`|строка или ноль|Нет|Код страны|
|`locationPrivacy`|строка|Нет|Уровень конфиденциальности: `"private"`, `"city"`, `"exact"`|

Широта и долгота должны быть указаны вместе.

**Источник:** `template/app/api/user/profile/location/route.ts`
