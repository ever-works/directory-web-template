# Конечные точки API служебной программы

Конечные точки утилит предоставляют инфраструктурные услуги, включая проверки работоспособности, информацию о версии, конфигурацию функций, геокодирование, проверку reCAPTCHA, извлечение URL-адресов, данные о местоположении и внутренние операции.

## Проверка здоровья (`/api/health`)

### Состояние базы данных

|Метод|Путь|Описание|
|--------|------|-------------|
|`GET`|`/api/health/database`|Проверьте подключение к базе данных|

Возвращает статус подключения к базе данных. Используется системами мониторинга и проверками работоспособности развертывания.

**Ответ (здоровый):**

```json
{
  "status": "healthy",
  "database": "connected",
  "timestamp": "2025-01-15T10:30:00.000Z"
}
```

**Ответ (нездоровый):**

```json
{
  "status": "unhealthy",
  "database": "disconnected",
  "error": "Connection refused",
  "timestamp": "2025-01-15T10:30:00.000Z"
}
```

**Аутентификация:** общедоступная (авторизация не требуется). Эта конечная точка должна быть доступна балансировщикам нагрузки и службам мониторинга.

## Версия (`/api/version`)

|Метод|Путь|Описание|
|--------|------|-------------|
|`GET`|`/api/version`|Получить информацию о версии приложения|
|`GET`|`/api/version/sync`|Получить версию и статус синхронизации|

### Версия Ответа

Возвращает версию приложения, информацию о сборке и среду выполнения:

```json
{
  "version": "1.0.0",
  "environment": "production",
  "timestamp": "2025-01-15T10:30:00.000Z"
}
```

### Версия + Статус синхронизации

Конечная точка `/api/version/sync` дополняет информацию о версии статусом синхронизации репозитория контента, что полезно для отладки актуальности контента.

**Аутентификация:** общедоступная.

## Конфигурация функции (`/api/config`)

|Метод|Путь|Описание|
|--------|------|-------------|
|`GET`|`/api/config/features`|Получить флаги включенных функций|

Возвращает текущую конфигурацию флага функции для клиентского приложения. Это позволяет интерфейсу условно отображать функции на основе конфигурации на стороне сервера.

**Ответ:**

```json
{
  "features": {
    "payments": true,
    "sponsorAds": true,
    "surveys": false,
    "map": true,
    "newsletter": true
  }
}
```

**Аутентификация:** общедоступная. Флаги функций не являются конфиденциальными данными.

## Извлечение URL (`/api/extract`)

|Метод|Путь|Описание|
|--------|------|-------------|
|`POST`|`/api/extract`|Извлечение метаданных из URL-адреса|

Получает URL-адрес и извлекает метаданные, включая заголовок, описание, изображение и значок. Используется формой отправки элемента для автоматического заполнения полей из URL-адреса.

**Запрос:**

```json
{
  "url": "https://example.com/product"
}
```

**Ответ:**

```json
{
  "success": true,
  "data": {
    "title": "Product Name",
    "description": "Product description from meta tags",
    "image": "https://example.com/og-image.png",
    "favicon": "https://example.com/favicon.ico",
    "siteName": "Example.com"
  }
}
```

**Аутентификация:** Требуется. Предотвращает злоупотребление получением URL-адресов на стороне сервера.

## Геокодирование (`/api/geocode`)

|Метод|Путь|Описание|
|--------|------|-------------|
|`POST`|`/api/geocode`|Геокодирование адреса по координатам|

Преобразует текстовый адрес в географические координаты (широту/долготу) с помощью внешнего сервиса геокодирования.

**Запрос:**

```json
{
  "address": "1600 Amphitheatre Parkway, Mountain View, CA"
}
```

**Ответ:**

```json
{
  "success": true,
  "data": {
    "lat": 37.4224764,
    "lng": -122.0842499,
    "formattedAddress": "1600 Amphitheatre Pkwy, Mountain View, CA 94043, USA"
  }
}
```

**Аутентификация:** Требуется.

## Данные о местоположении (`/api/location`)

Конечные точки для поиска местоположения и справочных данных.

|Метод|Путь|Описание|
|--------|------|-------------|
|`GET`|`/api/location/countries`|Перечислить все страны|
|`GET`|`/api/location/cities`|Список городов (с фильтром страны)|
|`GET`|`/api/location/coordinates`|Получить координаты места|
|`GET`|`/api/location/search`|Поиск местоположений по строке запроса|

### Страны

Возвращает список стран с кодами ISO, названиями и дополнительными метаданными.

### Города

Поддерживает фильтрацию по коду страны:

```
GET /api/location/cities?country=US
```

### Поиск местоположения

Полнотекстовый поиск по местоположению:

```
GET /api/location/search?q=San Francisco
```

**Аутентификация:** общедоступная.

## Проверка reCAPTCHA (`/api/verify-recaptcha`)

|Метод|Путь|Описание|
|--------|------|-------------|
|`POST`|`/api/verify-recaptcha`|Проверьте токен reCAPTCHA|

Серверная проверка токенов Google reCAPTCHA. Используется формами, требующими защиты от ботов.

**Запрос:**

```json
{
  "token": "recaptcha-response-token"
}
```

**Ответ:**

```json
{
  "success": true,
  "score": 0.9,
  "action": "submit"
}
```

**Аутентификация:** общедоступная (reCAPTCHA обычно используется в общедоступных формах).

## Справочные данные (`/api/reference`)

|Метод|Путь|Описание|
|--------|------|-------------|
|`GET`|`/api/reference`|Получить справочные данные для раскрывающихся списков форм|

Возвращает справочные данные, используемые для заполнения раскрывающихся меню и полей выбора в приложении, например модели ценообразования, типы лицензий и категории платформ.

**Аутентификация:** общедоступная.

## Внутренние операции (`/api/internal`)

|Метод|Путь|Описание|
|--------|------|-------------|
|`POST`|`/api/internal/db-init`|Инициализируйте схему базы данных и исходные данные|

### Инициализация базы данных

Конечная точка `/api/internal/db-init` запускает миграцию базы данных и дополнительную вставку исходных данных. Обычно он вызывается один раз во время первоначального развертывания или при сбросе среды разработки.

**Аутентификация.** Эта конечная точка должна быть защищена с помощью средств управления доступом, зависящих от среды, или общего секрета. Он не предназначен для регулярного использования.

## Вопросы безопасности

### Публичные конечные точки

Следующие конечные точки утилиты намеренно являются общедоступными:
- Проверки работоспособности (необходимы для мониторинга/балансировщиков нагрузки)
- Информация о версии (неконфиденциально)
- Флаги функций (неконфиденциальная конфигурация)
- Данные о местоположении (справочные данные)
- Проверка reCAPTCHA (защита публичной формы)
- Справочные данные (выпадающие значения)

### Защищенные конечные точки

Эти конечные точки требуют аутентификации для предотвращения злоупотреблений:
- Извлечение URL-адресов (предотвращает злоупотребление подделкой запросов на стороне сервера)
- Геокодирование (вызовы внешних API с ограничением скорости)
- Инициализация базы данных (разрушающая операция)
