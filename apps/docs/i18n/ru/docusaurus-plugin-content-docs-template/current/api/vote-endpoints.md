# Конечные точки голосования

Система голосования предоставляет конечные точки для голосования за и против элементов. При голосовании используется модель чистого балла, в которой подсчет представляет собой количество голосов за минус количество голосов против. Публичные конечные точки возвращают подсчет голосов, а аутентифицированные конечные точки позволяют подавать, обновлять и удалять голоса. Заблокированные пользователи не могут голосовать.

## Обзор

|Конечная точка|Метод|Авторизация|Описание|
|---|---|---|---|
|`/api/items/[slug]/votes`|ПОЛУЧИТЬ|Общественный|Получите подсчет голосов и статус голосования пользователя|
|`/api/items/[slug]/votes`|ПОСТ|Пользователь|Отдать или обновить голос|
|`/api/items/[slug]/votes`|УДАЛИТЬ|Пользователь|Удалить голос|
|`/api/items/[slug]/votes/count`|ПОЛУЧИТЬ|Общественный|Получить только чистый подсчет голосов|
|`/api/items/[slug]/votes/status`|ПОЛУЧИТЬ|Пользователь|Получить полную запись голосования для пользователя|

## Конечная точка комбинированного голосования

### Получить информацию о голосовании

```
GET /api/items/[slug]/votes
```

Возвращает чистое количество голосов за элемент и статус голосования текущего пользователя, если он прошел проверку подлинности. Аутентификация не требуется, но прошедшие проверку пользователи получают статус своего голоса в ответе.

**Параметры пути:**

|Параметр|Тип|Описание|
|---|---|---|
|`slug`|строка|Пуля элемента|

**Успешный ответ (200):**

```json
{
  "success": true,
  "count": 15,
  "userVote": "up"
}
```

|Поле|Тип|Описание|
|---|---|---|
|`success`|логическое значение|Всегда `true` об успехе|
|`count`|целое число|Чистый подсчет голосов (голоса за минус голоса против)|
|`userVote`|строка или ноль|`"up"`, `"down"` или `null`, если не авторизован или нет права голоса|

Для неаутентифицированных пользователей `userVote` всегда будет `null`. `count` может быть отрицательным, если голосов «против» больше, чем «за».

**Источник:** `template/app/api/items/[slug]/votes/route.ts`

### Проголосовать или обновить голосование

```
POST /api/items/[slug]/votes
```

Отдает новый голос или заменяет существующий голос за элемент. Если у пользователя уже есть голос, предыдущий голос удаляется перед созданием нового. Это означает, что переход от голосования «за» к голосованию «против» (или наоборот) — это одна операция.

**Аутентификация:** Требуется

**Тело запроса:**

```json
{
  "type": "up"
}
```

|Поле|Тип|Требуется|Описание|
|---|---|---|---|
|`type`|строка|Да|`"up"` для голосования «за», `"down"` для голосования «против»|

**Успешный ответ (200):**

```json
{
  "success": true,
  "count": 16,
  "userVote": "up"
}
```

Ответ возвращает обновленное чистое количество голосов после применения голоса.

**Ответы об ошибках:**

|Статус|Состояние|
|---|---|
| 400 |Неверный тип голосования (должен быть `"up"` или `"down"`)|
| 401 |Не аутентифицирован|
| 403 |Пользователь заблокирован или заблокирован|
| 404 |Профиль клиента не найден|

**Источник:** `template/app/api/items/[slug]/votes/route.ts`

### Удалить голосование

```
DELETE /api/items/[slug]/votes
```

Удаляет голос текущего пользователя за элемент. Если голосование не существует, операция завершается успешно без ошибок (идемпотентная). После удаления `userVote` станет `null`.

**Аутентификация:** Требуется

**Успешный ответ (200):**

```json
{
  "success": true,
  "count": 14,
  "userVote": null
}
```

|Статус|Состояние|
|---|---|
| 401 |Не аутентифицирован|
| 404 |Профиль клиента не найден|

**Источник:** `template/app/api/items/[slug]/votes/route.ts`

## Конечная точка подсчета голосов

### Получить подсчет голосов

```
GET /api/items/[slug]/votes/count
```

Возвращает только чистое количество голосов за элемент. Это облегченная общедоступная конечная точка, оптимизированная для быстрого получения подсчета голосов без указания статуса голосования для конкретного пользователя.

**Успешный ответ (200):**

```json
{
  "success": true,
  "count": 15
}
```

Число может быть положительным, отрицательным или нулевым в зависимости от баланса голосов «за» и «против».

**Источник:** `template/app/api/items/[slug]/votes/count/route.ts`

## Конечная точка статуса голосования

### Получить статус голосования пользователя

```
GET /api/items/[slug]/votes/status
```

Возвращает полную запись голосования для аутентифицированного пользователя по определенному элементу. Возвращает `null`, если пользователь не голосовал за элемент.

**Аутентификация:** Требуется

**Успешный ответ (200) – пользователь проголосовал:**

```json
{
  "id": "vote_123abc",
  "userId": "client_456def",
  "itemId": "item_123abc",
  "voteType": "UPVOTE",
  "createdAt": "2024-01-20T10:30:00.000Z",
  "updatedAt": "2024-01-20T10:30:00.000Z"
}
```

**Успешный ответ (200) – нет голосования:**

```json
null
```

Обратите внимание, что эта конечная точка возвращает необработанные значения `voteType` базы данных (`"UPVOTE"` или `"DOWNVOTE"`), а не упрощенный формат `"up"` / `"down"`, используемый объединенной конечной точкой.

|Статус|Состояние|
|---|---|
| 401 |Не аутентифицирован|
| 404 |Профиль клиента не найден|

**Источник:** `template/app/api/items/[slug]/votes/status/route.ts`

## Ключевые детали реализации

- **Чистый рейтинг:** Подсчет голосов рассчитывается как количество голосов «за» минус «против». Отрицательное значение указывает на большее количество голосов против, чем за.
- **Замена голоса.** Когда пользователь меняет тип голосования, существующий голос удаляется и создается новый. Обновления на месте нет.
- **Предотвращение блокировки пользователей.** Проверка `isUserBlocked()` на конечной точке POST предотвращает голосование заблокированных или заблокированных пользователей. Проверка блокировки применяется только при создании голосования, а не при его удалении.
- **VoteType Enum:** В базе данных голоса хранятся как `VoteType.UPVOTE` и `VoteType.DOWNVOTE`. API преобразует их в `"up"` и `"down"` для внешних потребителей.
- **Идемпотентное удаление:** удаление несуществующего голоса по-прежнему возвращает ответ 200 с текущим количеством голосов и `userVote: null`.
