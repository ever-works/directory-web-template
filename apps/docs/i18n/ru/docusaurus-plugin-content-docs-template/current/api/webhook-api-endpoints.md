# Конечные точки API вебхука

Шаблон поддерживает обработчики платежных веб-перехватчиков для четырех поставщиков: Stripe, LemonSqueezy, Polar и Solidgate. Каждая конечная точка веб-перехватчика обрабатывает входящие события от соответствующего поставщика платежей, обеспечивая управление жизненным циклом подписки, уведомления о платежах и доставку электронной почты. Все конечные точки проверяют подписи запросов в целях безопасности.

## Обзор

|Конечная точка|Поставщик|Заголовок подписи|Описание|
|---|---|---|---|
|`/api/stripe/webhook`|Полоса|`stripe-signature`|Обработка событий оплаты и подписки Stripe|
|`/api/lemonsqueezy/webhook`|ЛимонСжатый|`x-signature`|Обработка событий оплаты LemonSqueezy|
|`/api/polar/webhook`|Полярный|`webhook-signature`|Обработка событий оплаты Polar|
|`/api/solidgate/webhook`|Солидгейт|`x-signature`|Обработка событий оплаты Solidgate|

Все конечные точки веб-перехватчиков принимают только запросы POST и в случае успеха возвращают `{"received": true}`.

## Общая архитектура

Все четыре обработчика веб-перехватчиков следуют одному и тому же общему шаблону:

1. Прочитайте необработанное тело запроса в виде текста (необходимо для проверки подписи).
2. Извлеките подпись из заголовков, специфичных для поставщика
3. Передайте тело и подпись методу `handleWebhook()` провайдера для проверки и анализа.
4. Направьте анализируемое событие соответствующему обработчику на основе `WebhookEventType`
5. Выполнение бизнес-логики (обновления баз данных, уведомления по электронной почте)
6. Верните `{"received": true}`, чтобы подтвердить вебхук.

### Распространенные типы событий

Перечисление `WebhookEventType` из `lib/payment/types/payment-types` стандартизирует события между поставщиками:

|Тип события|Описание|
|---|---|
|`SUBSCRIPTION_CREATED`|Новая подписка активирована|
|`SUBSCRIPTION_UPDATED`|План подписки или детали изменены|
|`SUBSCRIPTION_CANCELLED`|Подписка отменена|
|`PAYMENT_SUCCEEDED`|Разовый платеж завершен|
|`PAYMENT_FAILED`|Попытка оплаты не удалась|
|`SUBSCRIPTION_PAYMENT_SUCCEEDED`|Регулярный платеж по подписке завершен|
|`SUBSCRIPTION_PAYMENT_FAILED`|Не удалось выполнить регулярный платеж по подписке|
|`SUBSCRIPTION_TRIAL_ENDING`|Пробный период подходит к концу|
|`REFUND_SUCCEEDED`|Возврат обработан|
|`BILLING_PORTAL_SESSION_UPDATED`|Сеанс платежного портала изменен (только Stripe)|

## Полосатый вебхук

```
POST /api/stripe/webhook
```

Обрабатывает события веб-перехватчика Stripe с проверкой подписи через заголовок `stripe-signature`. Это наиболее полнофункциональный обработчик веб-перехватчиков, включая уведомления по электронной почте для всех типов событий и обработку подписки на спонсорскую рекламу.

**Обязательный заголовок:**

|Заголовок|Описание|
|---|---|
|`stripe-signature`|Подпись вебхука Stripe (`t=...,v1=...` в формате)|

**Поддерживаемые события:**

|Полосатое событие|Сопоставленный тип|Действия|
|---|---|---|
|`customer.subscription.created`|`SUBSCRIPTION_CREATED`|Обновление базы данных, приветственное письмо|
|`customer.subscription.updated`|`SUBSCRIPTION_UPDATED`|Обновление базы данных, обновление электронной почты|
|`customer.subscription.deleted`|`SUBSCRIPTION_CANCELLED`|Обновление базы данных, электронное письмо об отмене|
|`invoice.payment_succeeded`|`SUBSCRIPTION_PAYMENT_SUCCEEDED`|Обновление базы данных, электронная почта квитанции|
|`invoice.payment_failed`|`SUBSCRIPTION_PAYMENT_FAILED`|Обновление базы данных, повторите письмо.|
|`payment_intent.succeeded`|`PAYMENT_SUCCEEDED`|Электронное письмо с подтверждением|
|`payment_intent.payment_failed`|`PAYMENT_FAILED`|Электронное письмо с уведомлением о сбое|
|`customer.subscription.trial_will_end`|`SUBSCRIPTION_TRIAL_ENDING`|Письмо об окончании пробного периода|
|`billing_portal.session.updated`|`BILLING_PORTAL_SESSION_UPDATED`|Только ведение журнала|

**Обработка спонсорской рекламы:**

Веб-хуки Stripe обнаруживают подписки на спонсорскую рекламу через `metadata.type === "sponsor_ad"` в данных о подписке. При обнаружении специальные обработчики активируют, отменяют или возобновляют спонсорские объявления вместо обработки обычных подписок.

**Ответы об ошибках:**

|Статус|Состояние|
|---|---|
| 400 |Отсутствует заголовок `stripe-signature`|
| 400 |Вебхук не обработан (неверная подпись)|
| 400 |Обработка вебхука не удалась|

**Источник:** `template/app/api/stripe/webhook/route.ts`

## LemonSqueezy Webhook

```
POST /api/lemonsqueezy/webhook
```

Обрабатывает события веб-перехватчика LemonSqueezy с проверкой подписи через заголовок `x-signature`. Использует функцию сопоставления событий для преобразования имен событий, специфичных для LemonSqueezy, в общий `WebhookEventType`.

**Обязательный заголовок:**

|Заголовок|Описание|
|---|---|
|`x-signature`|Подпись вебхука LemonSqueezy|

**Сопоставление событий:**

|Событие LemonSqueezy|Сопоставленный тип|
|---|---|
|`subscription_created`|`SUBSCRIPTION_CREATED`|
|`subscription_updated`|`SUBSCRIPTION_UPDATED`|
|`subscription_cancelled`|`SUBSCRIPTION_CANCELLED`|
|`subscription_payment_success`|`SUBSCRIPTION_PAYMENT_SUCCEEDED`|
|`subscription_payment_failed`|`SUBSCRIPTION_PAYMENT_FAILED`|
|`subscription_trial_will_end`|`SUBSCRIPTION_TRIAL_ENDING`|
|`order_created`|`PAYMENT_SUCCEEDED`|
|`order_refunded`|`REFUND_SUCCEEDED`|

**Обработка спонсорской рекламы:**

LemonSqueezy использует `custom_data.type === "sponsor_ad"` или `meta.custom_data.type === "sponsor_ad"` для идентификации подписок на рекламу спонсоров.

**Источник:** `template/app/api/lemonsqueezy/webhook/route.ts`

## Полярный вебхук

```
POST /api/polar/webhook
```

Обрабатывает события веб-перехватчика Polar с проверкой подписи с несколькими заголовками. Polar использует три заголовка для проверки безопасности и делегирует маршрутизацию событий отдельному модулю маршрутизатора.

**Обязательные заголовки:**

|Заголовок|Описание|
|---|---|
|`webhook-signature`|Подпись HMAC SHA256 (`v1,<hex_signature>` формат)|
|`webhook-timestamp`|Временная метка Unix события веб-перехватчика|
|`webhook-id`|Уникальный идентификатор доставки вебхука|

**Поддерживаемые события:**

|Полярное событие|Описание|
|---|---|
|`checkout.succeeded`|Оформление заказа завершено|
|`checkout.failed`|Оформить заказ не удалось|
|`subscription.created`|Подписка создана|
|`subscription.updated`|Подписка обновлена|
|`subscription.canceled`|Подписка отменена|
|`invoice.paid`|Оплата счета завершена|
|`invoice.payment_failed`|Оплата счета не удалась|

**Обработка:**

В отличие от других поставщиков, обработчик веб-перехватчиков Polar использует отдельную функцию `routeWebhookEvent()` из модуля `router` и утилиту `validateWebhookPayload()` для проверки структуры полезной нагрузки перед проверкой подписи.

**Источник:** `template/app/api/polar/webhook/route.ts`

## Вебхук Solidgate

```
POST /api/solidgate/webhook
```

Обрабатывает события веб-перехватчика Solidgate с проверкой подписи. Включает защиту идемпотентности в памяти для предотвращения дублирующей обработки одного и того же события веб-перехватчика.

**Обязательный заголовок:**

|Заголовок|Описание|
|---|---|
|`x-signature` или `solidgate-signature`|Подпись вебхука Solidgate|

**Идемпотентность:**

Обработчик хранит в памяти `Set` обработанных идентификаторов веб-перехватчиков. Дубликаты веб-перехватчиков возвращают `{"received": true}` без повторной обработки. Срок действия идентификаторов веб-перехватчиков из кеша истекает через 24 часа.

**Примечание.** Кэш идемпотентности в памяти не сохраняется при вызове бессерверных функций. В производственных бессерверных средах его следует заменить Redis или решением на базе базы данных.

**Поддерживаемые события:**

Обработчик принимает как общие константы `WebhookEventType`, так и строковые имена событий (например, `WebhookEventType.PAYMENT_SUCCEEDED` и `"payment_succeeded"`).

|Событие|Действия|
|---|---|
|`payment_succeeded`|Зафиксировать платеж|
|`payment_failed`|Ошибка записи|
|`subscription_created`|Создать подписку|
|`subscription_updated`|Обновить подписку|
|`subscription_cancelled`|Отменить подписку|
|`subscription_payment_succeeded`|Зафиксировать оплату подписки|
|`subscription_payment_failed`|Зафиксировать сбой оплаты подписки|
|`subscription_trial_ending`|Обработать окончание пробной версии|
|`refund_processed`|Возврат журнала|

**Получить конечную точку:**

Solidgate также предоставляет обработчик GET, который возвращает информационное сообщение о конечной точке веб-перехватчика:

```json
{
  "message": "Solidgate webhook endpoint",
  "instructions": "This endpoint accepts POST requests from Solidgate webhooks",
  "method": "POST"
}
```

**Источник:** `template/app/api/solidgate/webhook/route.ts`

## Уведомления по электронной почте

Обработчик веб-перехватчика Stripe отправляет наиболее полные уведомления по электронной почте. Все поставщики делегируют `WebhookSubscriptionService` операции с базой данных, но шаблоны электронной почты различаются в зависимости от поставщика.

|Тип электронной почты|Триггер|
|---|---|
|Добро пожаловать / Новая подписка|Подписка создана|
|Обновление подписки|План подписки изменен|
|Подтверждение отмены|Подписка отменена|
|Квитанция об оплате|Подписка или разовый платеж прошел успешно|
|Оплата не удалась/Повторите попытку|Попытка оплаты не удалась|
|Пробная концовка|Пробный период подходит к концу|

Конфигурация электронной почты загружается из `lib/config/server-config` через `getEmailConfig()` и включает название компании, URL-адрес компании и адрес электронной почты службы поддержки.

## Ключевые детали реализации

- **Проверка подписи.** Все поставщики проверяют подписи веб-перехватчиков перед обработкой событий. Неверные подписи приводят к ответу 400.
- **Разбор необработанного тела.** Веб-перехватчики считывают тело запроса как текст, используя `request.text()`, а не `request.json()`, поскольку для проверки подписи требуются необработанные, неизмененные полезные данные.
- **WebhookSubscriptionService:** Общий класс `WebhookSubscriptionService` обрабатывает операции с базой данных для событий жизненного цикла подписки у всех поставщиков.
- **Обнаружение спонсорской рекламы.** Веб-перехватчики Stripe и LemonSqueezy обнаруживают подписки на спонсорскую рекламу с помощью метаданных и направляют их отдельным обработчикам для активации, отмены и продления рекламы.
- **Надежная обработка ошибок.** Ошибки отправки электронной почты фиксируются и протоколируются, но не приводят к тому, что веб-перехватчик возвращает ошибку. Веб-перехватчик всегда подтверждает получение, чтобы предотвратить повторные попытки поставщика.
