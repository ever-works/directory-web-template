# Обзор архитектуры

На этой странице представлена высокоуровневая карта архитектуры шаблона Ever Works. Используйте его в качестве отправной точки, прежде чем углубляться в последующие подробные страницы.

## Технологический фонд

Шаблон представляет собой приложение **Next.js 16**, использующее **App Router** с **React 19**. Он создает выходные данные `standalone` для контейнерных развертываний и применяет несколько оптимизаций уровня платформы в `next.config.ts`:

|Слой|Технология|Цель|
|---|---|---|
|**Рамка**|Next.js 16 (маршрутизатор приложений)|Рендеринг сервера и клиента, маршрутизация, маршруты API|
|**Интерфейс**|React 19, HeroUI, Radix UI, Tailwind CSS 4|Библиотека компонентов, примитивы, стили|
|**База данных**|Добавьте ORM + PostgreSQL (или SQLite локально)|Управление схемами, миграции, запросы|
|**Аутентификация**|NextAuth.js v5 (бета)|Мультипровайдерская аутентификация с кэшированием сеанса|
|**Интернационализация**|следующий-международный|Маршрутизация и пакеты сообщений с учетом локали|
|**Платежи**|Полоса, Полярный, LemonSqueezy, Solidgate|Подписка и потоки единовременных платежей|
|**Содержание**|CMS на базе Git (`.content/` каталог)|Содержимое Markdown/YAML, клонированное из репозитория данных.|
|**Мониторинг**|Sentry, PostHog, Vercel Analytics|Отслеживание ошибок, аналитика продукта, производительность|
|**Электронная почта**|Отправить повторно|Транзакционная доставка электронной почты|
|**Форматированный текст**|Типтап|Редактор WYSIWYG для административного контента|

## Структура проекта

Шаблон соответствует многоуровневой организации, основанной на функциях. Вот каталоги верхнего уровня и их обязанности:

```text
template/
  app/              # Next.js App Router -- routes and layouts
    [locale]/       # Locale-prefixed pages (i18n)
      admin/        # Admin dashboard pages
      auth/         # Authentication flows
      dashboard/    # Client dashboard
      items/        # Item detail pages
      categories/   # Category browsing
      ...
    api/            # API route handlers
  components/       # Shared React components (UI, layout, features)
  lib/              # Core logic -- the heart of the application
    auth/           # Authentication providers, guards, session caching
    db/             # Drizzle schema, migrations, seed, queries
    middleware/     # Permission checks and middleware utilities
    repositories/  # Data-access layer (database queries)
    services/      # Business logic services
    payment/       # Payment provider integrations
    mail/           # Email templates and sending
    analytics/     # Analytics tracking layer
    config/        # Centralized configuration service
    validations/   # Zod schemas for input validation
    utils/         # General utility functions
    ...
  hooks/            # Custom React hooks (React Query wrappers, UI logic)
  constants/        # Application-wide constants
  types/            # Shared TypeScript type definitions
  i18n/             # Internationalization setup and locale request config
  messages/         # Translation message files (JSON per locale)
  e2e/              # Playwright end-to-end tests
  scripts/          # Build, seed, migration, and utility scripts
  public/           # Static assets
```

Полное описание каталога см. на странице [Структура проекта](/architecture/project-structure).

## Многоуровневая архитектура

Кодовая база обеспечивает четкое разделение задач по трем уровням:

### Уровень представления

Компоненты React в `components/` и файлы страниц в `app/[locale]/` обрабатывают рендеринг и взаимодействие с пользователем. Серверные компоненты извлекают данные напрямую; Клиентские компоненты используют перехватчики React Query из `hooks/` для состояния на стороне клиента.

### Уровень бизнес-логики

Сервисы в `lib/services/` содержат основные бизнес-правила. Шаблон поставляется с более чем 30 служебными файлами, охватывающими аналитику, подписки, модерацию, синхронизацию CRM, геокодирование, уведомления и многое другое. Службы вызываются обработчиками маршрутов API и серверными компонентами, но никогда напрямую с помощью кода пользовательского интерфейса в браузере.

### Уровень доступа к данным

Репозитории в `lib/repositories/` инкапсулируют все запросы к базе данных с помощью Drizzle ORM. Каждый объект домена (элементы, категории, коллекции, пользователи, роли, теги, рекламные объявления спонсоров) имеет свой собственный файл репозитория. Благодаря этому детали уровня SQL не попадают на уровень обслуживания.

Более подробно о потоке данных между этими уровнями см. в разделе [Поток данных](/architecture/data-flow).

## Маршрутизатор приложений Next.js и маршрутизация

Все маршруты, ориентированные на пользователя, находятся под `app/[locale]/`, что позволяет использовать URL-адреса с префиксом локали прямо из коробки через `next-intl`. Приложение использует несколько функций App Router:

- **Макеты** – вложенные файлы `layout.tsx` для администратора, панели управления клиента и общедоступных областей.
- **Группы маршрутов** — группа `(listing)` обрабатывает список основных каталогов и просмотр тегов, не затрагивая структуру URL-адресов.
- **Динамические маршруты** — `[page]`, `[...tag]` и именованные сегменты для элементов, категорий и коллекций.
- **Перезапись** — определено в `next.config.ts` для перенаправления пустых путей к категориям в представление их разбивки на страницы.

Полную карту маршрутов см. в разделе [Маршрутизация](/architecture/routing).

## Система аутентификации

Аутентификация построена на **NextAuth.js v5** с системой конфигурации провайдера `lib/auth/`. Файл `auth.config.ts` в корне проекта управляет:

- **Поставщики OAuth** – Google и GitHub, настраиваются с помощью переменных среды и включаются/отключаются динамически.
- **Поставщик учетных данных** – аутентификация по электронной почте и паролю с хешированием bcrypt.
- **Адаптер Supabase** – дополнительное хранилище сеансов на базе Supabase.
- **Кэширование сеанса** – `lib/auth/cached-session.ts` сокращает количество избыточных поисков сеанса.
- **Система защиты** — `lib/auth/guards.ts` и `lib/guards/` обеспечивает доступ на основе ролей на уровне маршрута.

Подробную информацию о системе защиты и разрешениях на основе ролей см. в разделах [Система защиты](/architecture/guards-system) и [Система разрешений](/architecture/permissions-system).

## Морось ORM и база данных

Уровень базы данных использует **Drizzle ORM** со схемой, определенной в `lib/db/schema.ts`. Ключевые аспекты:

- **Миграции** создаются с помощью `drizzle-kit generate` и применяются с помощью `drizzle-kit migrate`.
- Скрипты **заполнения** в `lib/db/seed.ts` и `scripts/cli-seed.ts` заполняют исходные данные, включая роли.
- **Конфигурация** находится в `drizzle.config.ts` в корне проекта.
- PostgreSQL необходим для производства; SQLite поддерживается для локальной разработки.

См. [Шаблоны репозитория](/architecture/repository-patterns), чтобы узнать, как структурирован уровень доступа к данным.

## Цепочка промежуточного программного обеспечения

В шаблоне используется промежуточное программное обеспечение Next.js (через плагин `next-intl`, примененный в `next.config.ts`) в сочетании с настраиваемыми проверками разрешений в `lib/middleware/permission-check.ts`. Конвейер промежуточного программного обеспечения обрабатывает:

- Обнаружение локали и маршрутизация
- Проверка состояния аутентификации
- Ролевая защита маршрутов
- Заголовки безопасности (HSTS, CSP, X-Frame-Options и т. д. – настраиваются в `next.config.ts`)

Подробную информацию см. в разделах [Middleware](/architecture/middleware) и [Middleware Deep Dive](/architecture/middleware-deep-dive).

## Конфигурация и безопасность

Файл `next.config.ts` устанавливает несколько значений безопасности и производительности по умолчанию:

- **Автономный вывод** для развертываний с поддержкой Docker.
- **Заголовки безопасности**, включая Content-Security-Policy, HSTS, X-Content-Type-Options и X-Frame-Options.
- **Оптимизация изображений** с удаленной поддержкой шаблонов и политиками безопасности SVG.
- **Интеграция Sentry** применяется как внешняя оболочка конфигурации для отслеживания ошибок.
- **Оптимизация пакета** для HeroUI и Lucid React для уменьшения размера пакета.
