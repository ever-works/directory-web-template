# Архитектура компонентов и шаблоны

Шаблон Ever Works организует свои компоненты React с использованием структуры каталогов на основе функций с четким разделением между компонентами функций, общими компонентами и базовыми примитивами пользовательского интерфейса.

## Организация каталогов

Каталог `components/` соответствует организации, ориентированной на функции, где каждый основной домен имеет свой собственный подкаталог, а также общие компоненты и компоненты уровня пользовательского интерфейса.

```

├── admin/              # Admin panel feature components
├── auth/               # Authentication feature components

├── collections/        # Collection display components

├── dashboard/          # Dashboard feature components
├── directory/          # Directory listing components

├── featured-items/     # Featured items display
├── filters/            # Search and filter components
├── footer/             # Footer components
├── header/             # Header and navigation
├── home-two/           # Alternate homepage layout
├── icons/              # Custom icon components
├── item-detail/        # Item detail page components
├── layout/             # Layout wrapper components
├── layouts/            # Layout variant components
├── maps/               # Map integration components
├── newsletter/         # Newsletter components
├── payment/            # Payment flow components
├── pricing/            # Pricing display components
├── profile/            # User profile components
├── profile-button/     # Profile button dropdown
├── providers/          # Provider wrapper components
├── settings/           # Settings panel components
├── shared/             # Shared reusable components
```

## Компоненты на основе функций

Каждый каталог функций содержит все компоненты, относящиеся к этому домену. Это обеспечивает совместное расположение связанного кода и упрощает поиск компонентов для конкретной функции.

### админ/

Содержит все компоненты панели администратора, включая таблицы данных, формы, модальные окна и интерфейсы управления. Это клиентские компоненты, которые используют хуки администратора из `hooks/use-admin-*.ts`.

### авторизация/

Компоненты аутентификации, включая формы входа, формы регистрации, потоки сброса пароля, кнопки OAuth и экраны проверки электронной почты.

### выставление счетов/

Компоненты управления выставлением счетов и подписками, включая выбор плана, формы способов оплаты, отображение счетов и индикаторы состояния подписки.

### фильтры/

Компоненты поиска и фильтрации, используемые на страницах списков. Они взаимодействуют с параметрами поиска URL-адресов и состоянием фильтра Zustand, обеспечивая фильтрацию в реальном времени.

### цены/

Компоненты страницы цен, включая карты сравнения планов, матрицы функций и интеграцию оформления заказа.

## Общие компоненты

### общий/

Каталог `shared/` содержит повторно используемые компоненты, используемые в различных функциях. Это независимые от предметной области строительные блоки, которые объединяют примитивы пользовательского интерфейса в функциональные шаблоны.

### общая карта/

Общие компоненты карточек, используемые для отображения элементов, коллекций и другого контента в макетах карточек в приложении.

## Серверные и клиентские компоненты

Шаблон соответствует соглашениям Next.js по разделению серверных и клиентских компонентов:

### Компоненты сервера

Серверные компоненты используются в App Router по умолчанию. Они используются для:
- Макеты страниц и оболочки
- Получение данных на уровне страницы
- Статический рендеринг контента
- SEO-критичный контент

Серверные компоненты живут в основном в файлах страниц и макетов `app/[locale]/`. Они могут напрямую импортировать функции запросов к базе данных и методы репозитория.

### Клиентские компоненты

Клиентские компоненты отмечены `'use client'` и используются для:
- Интерактивные элементы пользовательского интерфейса (формы, кнопки, переключатели)
- Компоненты, использующие перехваты React (useState, useEffect, пользовательские перехватчики)
- Компоненты, использующие API браузера
- Компоненты, зависящие от React Query или Zustand

Большинство компонентов в каталоге `components/` являются клиентскими компонентами, поскольку они обрабатывают взаимодействие пользователя и его состояние.

## Поставщики контекста

### компоненты/контекст/

Поставщики контекста React для совместного использования состояния между деревьями компонентов:
- Контекст ошибки для граничного состояния ошибки
- Контекст флага функции для ограничения функций во время выполнения

### компоненты/провайдеры/

Компоненты оболочки поставщика, составляющие несколько поставщиков:
- Поставщик клиента запроса (TanStack Query)
- Поставщик тем
- Поставщик сеансов (NextAuth)
- Поставщик тостов

Оболочка корневых поставщиков по адресу `app/[locale]/providers.tsx` содержит всех необходимых поставщиков для приложения.

## Соглашения о компонентах

1. **Именование файлов**: компоненты используют имена файлов в стиле «кебаб» (например, `favorite-button.tsx`).
2. **Шаблон экспорта**: компоненты используют именованный экспорт, файлы стволов (`index.ts`) в каталогах функций.
3. **Совместное размещение перехватчиков**: перехватчики для конкретных функций находятся в каталоге верхнего уровня `hooks/`, а не внутри каталогов компонентов.
4. **Стилизация**: компоненты используют служебные классы CSS Tailwind; некоторые используют модули SCSS для сложной стилизации.
5. **Типы**: Типы свойств компонента определяются внутри или в соседних файлах типов в каталоге `types/`.
6. **Значки**: Пользовательские значки централизованно хранятся в `components/icons/`; стандартные значки используйте `lucide-react`
