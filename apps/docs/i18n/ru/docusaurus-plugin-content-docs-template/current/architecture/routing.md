# Архитектура маршрутизации

Шаблон Ever Works использует маршрутизатор приложений Next.js с интернационализацией через `next-intl`, предоставляя маршруты с префиксом локали, группы маршрутов для логической организации и комплексный уровень API.

## Маршрутизатор приложений с сегментом локали

Все страницы, ориентированные на пользователя, вложены в динамический сегмент `[locale]`, что обеспечивает многоязычную поддержку для 6 языков: `en`, `fr`, `es`, `de`, `ar` и `zh`.

```
app/
├── [locale]/           # Dynamic locale segment
│   ├── layout.tsx      # Locale layout (wraps all localized pages)
│   ├── providers.tsx   # Client providers for the locale subtree
│   ├── globals.css     # Global styles
│   └── ...pages        # All localized pages
├── api/                # API routes (not locale-prefixed)
├── layout.tsx          # Root layout (HTML, fonts, metadata)
└── not-found.tsx       # 404 page
```

URL-адреса следуют шаблону `/{locale}/path`, например:
- `/en/pricing` -- страница цен на английском языке
- `/fr/admin/items` -- страница элементов администрирования на французском языке.
- `/de/categories` -- страница категорий на немецком языке

## Конфигурация Next.js

`next.config.ts` настраивает несколько вариантов маршрутизации:

### Переписывает

```typescript
async rewrites() {
  return [
    {
      source: "/:path",
      destination: "/:path/discover/1",
    },
    {
      source: "/:path/discover",
      destination: "/:path/discover/1",
    },
  ];
}
```

Эти перезаписи перенаправляют путь к корневому языковому стандарту и `/discover` на первую страницу списка обнаружения (`/discover/1`), предоставляя чистый URL-адрес по умолчанию.

### Заголовки безопасности

Все маршруты получают заголовки безопасности, включая:
- `X-Content-Type-Options: nosniff`
- `X-Frame-Options: DENY`
- `Strict-Transport-Security` с максимальным возрастом 2 года
- `Content-Security-Policy` с ограничительными настройками по умолчанию
- `Referrer-Policy: strict-origin-when-cross-origin`

### Плагин next-intl

Плагин `next-intl` применяется к конфигурации Next.js, указывая на `./i18n/request.ts` для разрешения локали:

```typescript
const withNextIntl = createNextIntlPlugin('./i18n/request.ts');
const configWithIntl = withNextIntl(nextConfig);
```

## Группы маршрутов

Каталог `[locale]` использует несколько логических группировок для организации страниц:

### (объявление) -- Основные страницы листинга

Группа маршрутов `(listing)` представляет собой группу в скобках (без сегмента URL-адреса), которая обертывает страницы со списком основного каталога общим макетом.

### admin/ -- Панель администратора

Раздел администратора предоставляет полный интерфейс бэк-офиса:

```
[locale]/admin/
├── auth/               # Admin sign-in
├── categories/         # Category CRUD
├── clients/            # Client management
├── collections/        # Collection CRUD
├── comments/           # Comment moderation
├── companies/          # Company management
├── featured-items/     # Featured item management
├── items/              # Item review and management
├── reports/            # Report review
├── roles/              # Role and permission management
├── settings/           # Site settings
├── sponsorships/       # Sponsorship management
├── surveys/            # Survey builder
├── tags/               # Tag management
├── users/              # User management
├── layout.tsx          # Admin layout (sidebar, navigation)
├── layout-client.tsx   # Client-side admin layout logic
└── page.tsx            # Admin dashboard
```

### auth/ -- Страницы аутентификации

```
[locale]/auth/
├── signin/             # Sign in page
├── signup/             # Sign up page
├── forgot-password/    # Password reset request
├── reset-password/     # Password reset form
├── verify-email/       # Email verification
└── error/              # Authentication error page
```

### client/ -- Панель управления клиента

Клиентский раздел предоставляет аутентифицированным пользователям функции для управления своими собственными материалами и учетной записью.

### приборная панель/ -- Панель управления пользователя

Общая панель пользователя с обзором учетной записи, активностью и настройками.

## Промежуточное ПО

Приложение использует промежуточное программное обеспечение `next-intl` для определения локали и маршрутизации. Промежуточное программное обеспечение обрабатывает:

1. **Определение локали**: определяет локаль пользователя по URL-адресу, файлам cookie или заголовку `Accept-Language`.
2. **Перенаправление локали**: перенаправляет запросы без префикса локали на соответствующий локаль.
3. **Язык по умолчанию**: возвращается к английскому (`en`), если не обнаружено языковых стандартов.

Промежуточное программное обеспечение настраивается в каталоге `i18n/` с языковыми правилами маршрутизации, определенными в `i18n/routing.ts`, и обработкой запросов в `i18n/request.ts`.

## Статическая генерация и динамические маршруты

В шаблоне используется несколько стратегий получения данных:

- **Статическое создание**. Такие страницы, как политика конфиденциальности, условия обслуживания и информация, генерируются статически.
- **Динамическая отрисовка**: страницы администрирования, информационные панели и страницы, прошедшие проверку подлинности, отображаются динамически.
- **ISR (инкрементная статическая регенерация)**: страницы со списком категорий и тегов используют ISR с повторной проверкой.
- **Создание карты сайта**: `app/sitemap.ts` динамически генерирует карту сайта на основе данных контента.

`staticPageGenerationTimeout` установлено на 180 секунд в `next.config.ts` для размещения больших репозиториев контента во время сборок.
