# Управление Доступом на Основе Ролей

## Обзор

Шаблон реализует RBAC с ролями, хранящимися в базе данных.

## Роли по Умолчанию

| Роль | Описание |
|------|----------|
| admin | Полный доступ к системе |
| moderator | Доступ к модерации контента |
| user | Стандартный аутентифицированный доступ |
| guest | Ограниченный публичный доступ |

## Назначение Ролей

Роли назначаются в базе данных. Пользователи-администраторы могут управлять ролями через панель администратора по адресу /admin/users.

## Проверка Разрешений

```typescript
// В API-маршрутах
const session = await auth();
if (!session?.user?.role || session.user.role !== 'admin') {
  return NextResponse.json({ error: 'Forbidden' }, { status: 403 });
}
```

## Защита Маршрутов

Используйте промежуточное ПО для защиты маршрутов на основе ролей. Промежуточное ПО auth проверяет сессию и роль перед предоставлением доступа.
