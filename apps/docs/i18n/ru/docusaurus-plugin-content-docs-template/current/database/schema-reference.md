# Справочник по схеме

Все таблицы базы данных определены в `lib/db/schema.ts`. В этом документе каталогизирована каждая таблица, ее ключевые столбцы, связи и назначение.

## Пользователи и аутентификация

### пользователи

Основная таблица пользователей, используемая NextAuth.js для аутентификации.

|Столбец|Тип|Примечания|
|--------|------|-------|
|`id`|текст (ПК)|UUID, генерируется автоматически|
|`email`|текст|Уникальный|
|`image`|текст|URL изображения профиля|
|`emailVerified`|временная метка|Дата подтверждения электронной почты|
|`passwordHash`|текст|Хэш Bcrypt для аутентификации учетных данных|
|`createdAt`|временная метка|Автоустановка|
|`updatedAt`|временная метка|Автоустановка|
|`deletedAt`|временная метка|Мягкое удаление|

**Индексы**: `users_created_at_idx`

### счета

Ссылки на учетные записи OAuth и учетные данные, соответствующие схеме адаптера NextAuth.js.

|Столбец|Тип|Примечания|
|--------|------|-------|
|`userId`|текст (ФК)|Ссылки `users.id` (каскадное удаление)|
|`type`|текст|Тип учетной записи (oauth, учетные данные и т. д.)|
|`provider`|текст|Имя провайдера (google, github, учетные данные)|
|`providerAccountId`|текст|Идентификатор учетной записи конкретного поставщика|
|`email`|текст|Адрес электронной почты аккаунта|
|`passwordHash`|текст|Для аутентификации учетных данных клиента|
|`refresh_token`|текст|Токен обновления OAuth|
|`access_token`|текст|Токен доступа OAuth|
|`expires_at`|целое число|Срок действия токена|

**Первичный ключ**: Составной на (`provider`, `providerAccountId`)
**Индексы**: `accounts_email_idx`, `accounts_provider_idx`

### сессии

Активные пользовательские сессии.

|Столбец|Тип|Примечания|
|--------|------|-------|
|`sessionToken`|текст (ПК)|Идентификатор сеанса|
|`userId`|текст (ФК)|Рекомендации `users.id`|
|`expires`|временная метка|Срок действия сеанса|

### токены проверки

Токены проверки электронной почты.

|Столбец|Тип|Примечания|
|--------|------|-------|
|`identifier`|текст|Идентификатор пользователя|
|`email`|текст|Адрес электронной почты|
|`token`|текст|Токен подтверждения|
|`expires`|временная метка|Срок действия токена|

**Первичный ключ**: Составной на (`identifier`, `token`)

### аутентификаторы

Хранилище учетных данных WebAuthn/FIDO2.

|Столбец|Тип|Примечания|
|--------|------|-------|
|`credentialID`|текст|Уникальный идентификатор учетных данных|
|`userId`|текст (ФК)|Рекомендации `users.id`|
|`providerAccountId`|текст|Ссылка на аккаунт провайдера|
|`credentialPublicKey`|текст|Открытый ключ для проверки|
|`counter`|целое число|Счетчик аутентификации|

### парольResetTokens

Токены сброса пароля для потока забытых паролей.

|Столбец|Тип|Примечания|
|--------|------|-------|
|`id`|текст (ПК)|UUID|
|`email`|текст|Целевой адрес электронной почты|
|`token`|текст|Уникальный токен сброса|
|`expires`|временная метка|Срок действия токена|

### журналы активности

Отслеживает действия пользователей и клиентов в целях аудита.

|Столбец|Тип|Примечания|
|--------|------|-------|
|`id`|серийный (ПК)|Автоинкремент|
|`userId`|текст (ФК)|Ссылки `users.id` (обнуляемое)|
|`clientId`|текст (ФК)|Ссылки `clientProfiles.id` (обнуляемое)|
|`action`|текст|Тип активности (SIGN_UP, SIGN_IN и т. д.)|
|`timestamp`|временная метка|Когда произошло действие|
|`ipAddress`|варчар(45)|IP-адрес клиента|

**Индексы**: `activity_logs_user_idx`, `activity_logs_timestamp_idx`, `activity_logs_action_idx`

## Роли и разрешения

### роли

Определения ролей для RBAC.

|Столбец|Тип|Примечания|
|--------|------|-------|
|`id`|текст (ПК)|Идентификатор роли (например, «администратор», «клиент»)|
|`name`|текст|Уникальное имя роли|
|`description`|текст|Читабельное описание|
|`isAdmin`|логическое значение|Является ли это ролью администратора|
|`status`|текст|«активный» или «неактивный»|
|`created_by`|текст|Кто создал роль|

### разрешения

Детальные определения разрешений.

|Столбец|Тип|Примечания|
|--------|------|-------|
|`id`|текст (ПК)|UUID|
|`key`|текст|Уникальный ключ разрешения (например, «items:create»)|
|`description`|текст|Читабельное описание|

### РольРазрешения

Таблица соединений «многие ко многим», связывающая роли с разрешениями.

|Столбец|Тип|Примечания|
|--------|------|-------|
|`roleId`|текст (ФК)|Референции `roles.id` (каскад)|
|`permissionId`|текст (ФК)|Референции `permissions.id` (каскад)|

**Первичный ключ**: Составной на (`roleId`, `permissionId`)

### роли пользователя

Таблица соединений «многие ко многим», связывающая пользователей с ролями.

|Столбец|Тип|Примечания|
|--------|------|-------|
|`userId`|текст (ФК)|Референции `users.id` (каскад)|
|`roleId`|текст (ФК)|Референции `roles.id` (каскад)|

**Первичный ключ**: Составной на (`userId`, `roleId`)

## Другие таблицы

### спонсорская реклама

Реклама спонсируемых товаров с полным жизненным циклом платежей.

Ключевые столбцы: `userId`, `itemSlug`, `status` (ожидающий_платеж, ожидающий, отклоненный, активный, истекший, отмененный), `interval` (еженедельно, ежемесячно), `amount`, `paymentProvider`, `subscriptionId`.

### компании/предметыКомпании

Записи компаний и ассоциации товаров и компаний для списков каталогов.

### опросы / опросыОтветы

Конструктор опросов с определениями вопросов на основе JSON и хранилищем ответов.

### двадцатьCrmConfig/интеграцияMappings

Таблицы интеграции CRM для функции синхронизации Twenty CRM. Таблица конфигурации применяет одноэлементный шаблон (разрешена только одна строка).

### информационный бюллетеньПодписки

Отслеживание подписки на рассылку новостей по электронной почте с указанием времени подписки/отписки.

### семенной статус

Таблица Singleton отслеживает состояние заполнения базы данных (заполнение, завершение, сбой) для предотвращения одновременных операций заполнения.

## Тип экспорта

Файл схемы экспортирует типы TypeScript для каждой таблицы, используя вывод Drizzle:

```typescript
export type User = typeof users.$inferSelect;
export type NewUser = typeof users.$inferInsert;
export type Subscription = typeof subscriptions.$inferSelect;
export type NewSubscription = typeof subscriptions.$inferInsert;
// ... and so on for all tables
```

Эти типы используются во всем приложении для типобезопасных операций с базой данных.
