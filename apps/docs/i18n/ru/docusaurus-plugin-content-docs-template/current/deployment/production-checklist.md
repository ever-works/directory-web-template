# Контрольный Список для Продакшена

Комплексный контрольный список для обеспечения готовности развёртывания Ever Works к продакшену.

## Контрольный Список Перед Развёртыванием

### 1. Конфигурация Окружения

#### Обязательные Переменные Окружения

- [ ] **База Данных**
  - `DATABASE_URL` настроен с продакшен PostgreSQL
  - Включён connection pooling базы данных
  - Включён режим SSL для продакшена

- [ ] **Аутентификация**
  - `NEXTAUTH_URL` установлен на продакшен-домен
  - `NEXTAUTH_SECRET` сгенерирован (минимум 32 символа)
  - Настроены OAuth-провайдеры (Google, GitHub и др.)
  - Учётные данные Supabase Auth (если используется)

- [ ] **Платёжные Провайдеры**
  - Продакшен-ключи Stripe (`STRIPE_SECRET_KEY`, `NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY`)
  - Продакшен-ключи LemonSqueezy (если используется)
  - Настроены секреты webhook
  - Тестовый режим отключён

- [ ] **Email-сервисы**
  - Настроен API-ключ Resend
  - Заданы учётные данные Novu (если используется)
  - Протестированы шаблоны писем
  - Верифицирован домен отправителя

- [ ] **Аналитика и Мониторинг**
  - Продакшен-ключ PostHog
  - Настроен DSN Sentry
  - Определён провайдер отслеживания исключений
  - Включена Vercel Analytics (если на Vercel)

- [ ] **Интеграция CRM**
  - Учётные данные Twenty CRM (если используется)
  - Настроены конечные точки webhook

- [ ] **Безопасность**
  - `NODE_ENV=production`
  - Настроено ограничение частоты запросов
  - Проверены настройки CORS
  - Настроены заголовки CSP

### 2. База Данных

- [ ] **Схема и Миграции**
  - Все миграции применены
  - Схема базы данных соответствует коду
  - Созданы индексы для производительности
  - Проверены ограничения внешних ключей

- [ ] **Целостность Данных**
  - Загружены seed-данные (при необходимости)
  - Удалены тестовые данные
  - Внедрены правила валидации данных

- [ ] **Резервное Копирование и Восстановление**
  - Настроено автоматическое резервное копирование
  - Протестировано восстановление из резервной копии
  - Включено восстановление на момент времени
  - Определена политика хранения резервных копий

- [ ] **Производительность**
  - Настроен connection pooling
  - Оптимизирована производительность запросов
  - Включено логирование медленных запросов
  - Активен мониторинг базы данных

### 3. Безопасность

- [ ] **Аутентификация и Авторизация**
  - Проверено хеширование паролей (bcrypt)
  - Безопасное управление сессиями
  - Корректно подписаны JWT-токены
  - Протестировано управление доступом на основе ролей

- [ ] **Защита Данных**
  - PII-данные зашифрованы в покое
  - Настроена очистка конфиденциальных данных
  - Принудительно используется HTTPS
  - Включены безопасные cookie

- [ ] **Безопасность API**
  - Активно ограничение частоты запросов
  - Обязательна аутентификация API
  - Валидация входных данных на всех конечных точках
  - Проверена защита от SQL-инъекций

- [ ] **Зависимости**
  - Все зависимости обновлены
  - Проверены уязвимости безопасности (`npm audit`)
  - Отсутствуют критические уязвимости
  - Зафиксирован lock-файл зависимостей

### 4. Производительность

- [ ] **Оптимизация Фронтенда**
  - Оптимизированы изображения (компонент Image Next.js)
  - Реализовано разделение кода (code splitting)
  - Ленивая загрузка для тяжёлых компонентов
  - Проанализирован размер бандла

- [ ] **Кеширование**

  - Ответы API кешированы (где применимо)
  - Настроен CDN
  - Внедрена стратегия инвалидации кеша

- [ ] **Core Web Vitals**
  - LCP < 2,5с
  - FID < 100мс
  - CLS < 0,1
  - Активен мониторинг производительности

- [ ] **Запросы к Базе Данных**
  - Устранены запросы N+1
  - Созданы необходимые индексы
  - Включено кеширование запросов
  - Оптимизирован connection pooling

## Команды Проверки

Выполните следующие команды для проверки готовности к продакшену:

### Аудит Безопасности

```bash
# Проверить уязвимости безопасности
npm audit --production

# Исправить уязвимости
npm audit fix

# Проверить устаревшие зависимости
npm outdated
```

### Проверка Сборки

```bash
# Продакшен-сборка
npm run build

# Проверить вывод сборки
ls -lh .next/

# Анализ размера бандла
npm run analyze
```

### Проверка Базы Данных

```bash
# Проверить статус миграций
npx drizzle-kit check

# Сгенерировать миграцию при необходимости
npx drizzle-kit generate

# Применить миграции
npx drizzle-kit push
```

### Документация API

```bash
# Сгенерировать спецификацию OpenAPI
yarn generate-docs

# Валидировать документацию
yarn docs:validate

# Проверить актуальность документации
git diff --exit-code public/openapi.json
```

### Переменные Окружения

```bash
# Проверить наличие всех обязательных переменных
node scripts/check-env.js

# Протестировать конфигурацию окружения
npm run test:env
```

## Рабочий Процесс Развёртывания

### Перед Развёртыванием

1. **Ревью Кода**
   - Все PR проверены и одобрены
   - Отсутствуют конфликты слияния
   - CI/CD-конвейер прошёл

2. **Тестирование**
   - Все тесты прошли успешно
   - Завершено ручное QA-тестирование
   - Протестирована staging-среда

3. **Документация**
   - Обновлён changelog
   - Перегенерирована документация API
   - Подготовлены заметки о развёртывании

### Шаги Развёртывания

1. **Резервное Копирование**

   ```bash
   # Создать резервную копию базы данных
   pg_dump $DATABASE_URL > backup-$(date +%Y%m%d).sql
   ```

2. **Развёртывание**

   ```bash
   # Развернуть в продакшен
   git push production main

   # Или с Vercel
   vercel --prod
   ```

3. **Проверка**

   ```bash
   # Проверить конечную точку health
   curl https://your-domain.com/api/health

   # Проверить логи ошибок
   tail -f logs/error.log
   ```

4. **Мониторинг**
   - Мониторинг частоты ошибок в Sentry
   - Мониторинг производительности в PostHog
   - Проверка мониторинга доступности

### После Развёртывания

1. **Дымовые Тесты**
   - Главная страница загружается
   - Пользователь может войти в систему
   - Процесс оплаты работает
   - Панель администратора доступна

2. **Мониторинг**
   - Нормальная частота ошибок
   - Приемлемое время ответа
   - Отсутствие утечек памяти
   - Стабильная производительность базы данных

3. **Коммуникация**
   - Уведомить команду о развёртывании
   - Обновить страницу статуса
   - Анонсировать новые функции (если применимо)

## План Отката

При обнаружении проблем после развёртывания:

### Быстрый Откат

```bash
# Откатиться к предыдущему развёртыванию
git revert HEAD
git push production main

# Или с Vercel
vercel rollback
```

### Откат Базы Данных

```bash
# Восстановить из резервной копии
psql $DATABASE_URL < backup-YYYYMMDD.sql

# Или использовать восстановление на момент времени
# (если поддерживается вашим провайдером хостинга)
```

### Коммуникация

1. Немедленно уведомить команду
2. Обновить страницу статуса
3. Связаться с затронутыми пользователями
4. Задокументировать инцидент для post-mortem

## Метрики Успеха

Отслеживайте эти метрики для обеспечения здоровья продакшена:

### Производительность

- **Время Ответа**: < 200мс (p95)
- **Доступность**: > 99,9%
- **Частота Ошибок**: < 0,1%
- **Core Web Vitals**: Все в зелёной зоне

### Бизнес

- **Регистрации Пользователей**: Работающее отслеживание
- **Успешность Платежей**: > 95%
- **Доставляемость Email**: > 98%
- **Доступность API**: > 99,9%

### Безопасность

- **Неудачные Попытки Входа**: Отслеживаются
- **Превышения Лимита Запросов API**: < 1%
- **Уязвимости Безопасности**: 0 критических
- **SSL-сертификат**: Действителен и с автообновлением
