# Руководство по Адаптивному Тестированию

Это руководство охватывает лучшие практики тестирования адаптивного дизайна на различных устройствах и размерах экрана.

## Контрольный Список Тестирования

### 1. Навигация

- [ ] **Мобильные**: Кнопка-гамбургер видна и функциональна
- [ ] **ПК**: Горизонтальная панель навигации отображается корректно
- [ ] **Все устройства**: Все навигационные ссылки доступны
- [ ] **Области касания**: Минимум 44x44px на мобильных
- [ ] **Клавиатурная навигация**: Порядок переключения логичен

### 2. Контент

- [ ] **Читаемость текста**: Нет необходимости увеличивать масштаб для чтения
- [ ] **Изображения**: Адаптивны и правильно масштабированы для каждой точки останова
- [ ] **Нет горизонтальной прокрутки**: Контент умещается в области просмотра
- [ ] **Длина строки**: Оптимальная ширина чтения (45-75 символов)
- [ ] **Размеры шрифтов**: Соответствующие для каждого размера устройства

### 3. Взаимодействия

- [ ] **Области касания**: Минимум 44x44px для мобильных
- [ ] **Отступы**: Достаточное пространство между кликабельными элементами
- [ ] **Состояния hover**: Только на устройствах с поддержкой hover
- [ ] **Состояния фокуса**: Видимые индикаторы фокуса клавиатуры
- [ ] **Формы**: Удобны для заполнения на мобильных устройствах

### 4. Производительность

- [ ] **Время загрузки**: < 3 секунд при 3G-подключении
- [ ] **Изображения**: Оптимизированы для каждого размера экрана
- [ ] **Анимации**: Плавная производительность 60 FPS
- [ ] **Core Web Vitals**: Соответствуют порогам Google
- [ ] **Размер бандла**: Оптимизированные JavaScript и CSS

### 5. Макет

- [ ] **Сетка**: Корректно адаптируется в точках останова
- [ ] **Flexbox/Grid**: Нет нарушений макета
- [ ] **Отступы**: Согласованные padding и margin
- [ ] **Выравнивание**: Правильное выравнивание при всех размерах
- [ ] **Overflow**: Нет проблем с выходом контента за границы

## Инструменты Тестирования

### DevTools Браузера

#### Chrome DevTools
1. Откройте DevTools (F12 или Cmd+Option+I)
2. Нажмите иконку панели устройств (Cmd+Shift+M)
3. Выберите устройство или введите пользовательские размеры
4. Протестируйте разные скорости сети

#### Firefox Developer Tools
1. Откройте DevTools (F12)
2. Нажмите «Режим адаптивного дизайна» (Cmd+Option+M)
3. Выберите предустановки устройств
4. Протестируйте события касания

### Онлайн-сервисы Тестирования

- **[BrowserStack](https://www.browserstack.com/)** - Тестирование на реальных устройствах
- **[LambdaTest](https://www.lambdatest.com/)** - Кросс-браузерное тестирование
- **[Sauce Labs](https://saucelabs.com/)** - Платформа автоматизированного тестирования

### Тестирование Производительности

- **[Lighthouse](https://developers.google.com/web/tools/lighthouse)** - Аудит производительности
- **[WebPageTest](https://www.webpagetest.org/)** - Детальный анализ производительности
- **[PageSpeed Insights](https://pagespeed.web.dev/)** - Инструмент производительности Google

## Целевые Метрики

### Оценки Lighthouse

| Метрика            | Цель  | Критично |
|--------------------|-------|----------|
| Производительность | > 90  | > 50     |
| Доступность        | > 95  | > 80     |
| Лучшие Практики    | > 95  | > 80     |
| SEO                | > 95  | > 80     |

### Core Web Vitals

| Метрика | Хорошо | Требует Улучшения | Плохо |
|---------|--------|-------------------|-------|
| **LCP** (Largest Contentful Paint) | < 2.5s | 2.5s - 4.0s | > 4.0s |
| **FID** (First Input Delay) | < 100ms | 100ms - 300ms | > 300ms |
| **CLS** (Cumulative Layout Shift) | < 0.1 | 0.1 - 0.25 | > 0.25 |

### Мобильные Метрики

- **First Contentful Paint**: < 1.8s
- **Time to Interactive**: < 3.8s
- **Размер области касания**: >= 48x48px
- **Задержка касания**: < 300ms

## Рабочий Процесс Тестирования

### 1. Локальное Тестирование

```bash
# Запустить сервер разработки
npm run dev

# Открыть в браузере
http://localhost:3000
```

### 2. Тестирование на Устройствах

1. **Использовать адаптивный режим DevTools**
2. **Тестировать на физических устройствах** при возможности
3. **Использовать удалённую отладку** для мобильных устройств
4. **Тестировать разные ориентации** (портрет/альбом)

### 3. Автоматизированное Тестирование

```bash
# Запустить аудит Lighthouse
npm run lighthouse

# Запустить тесты доступности
npm run test:a11y

# Запустить тесты визуальной регрессии
npm run test:visual
```

## Распространённые Проблемы и Решения

### Проблема: Горизонтальная Прокрутка на Мобильных

**Решение**: Проверьте элементы с фиксированной шириной

```css
/* ❌ Плохо */
.container {
  width: 1200px;
}

/* ✅ Хорошо */
.container {
  max-width: 1200px;
  width: 100%;
  padding: 0 1rem;
}
```

### Проблема: Слишком Маленький Текст на Мобильных

**Решение**: Используйте адаптивные размеры шрифтов

```css
/* ❌ Плохо */
body {
  font-size: 12px;
}

/* ✅ Хорошо */
body {
  font-size: 16px; /* Базовый размер для мобильных */
}

@media (min-width: 768px) {
  body {
    font-size: 18px;
  }
}
```

### Проблема: Слишком Маленькие Области Касания

**Решение**: Обеспечьте минимальный размер 44x44px

```css
/* ✅ Хорошо */
.button {
  min-height: 44px;
  min-width: 44px;
  padding: 12px 24px;
}
```

### Проблема: Неадаптивные Изображения

**Решение**: Используйте техники адаптивных изображений

```jsx
// ✅ Хорошо - компонент Image Next.js
import Image from 'next/image';

<Image
  src="/image.jpg"
  alt="Description"
  width={800}
  height={600}
  sizes="(max-width: 768px) 100vw, (max-width: 1200px) 50vw, 33vw"
  priority
/>
```

## Лучшие Практики

### 1. Подход Mobile-First
- Разрабатывайте сначала для мобильных, затем улучшайте для больших экранов
- Используйте медиа-запросы `min-width` вместо `max-width`

### 2. Прогрессивное Улучшение
- Убедитесь, что основная функциональность работает без JavaScript
- Добавляйте улучшения для поддерживающих браузеров

### 3. Дизайн, Удобный для Касания
- Минимальные области касания 44x44px
- Достаточные отступы между интерактивными элементами
- Избегайте взаимодействий только через hover

### 4. Оптимизация Производительности
- Ленивая загрузка изображений и компонентов
- Минимизируйте размер JavaScript-бандла
- Используйте разделение кода
- Оптимизируйте шрифты и ресурсы

### 5. Доступность
- Тестируйте со скринридерами
- Убедитесь в работе клавиатурной навигации
- Поддерживайте достаточный контраст цветов
- Предоставляйте текстовые альтернативы для изображений
