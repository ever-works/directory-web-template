# Интернационализация (i18n)

Ever Works Template использует `next-intl` для интернационализации, поддерживая 21 языковой стандарт с маршрутизацией на основе URL, автоматическим определением браузера, резервными сообщениями, макетами справа налevo и SEO-тегами hreflang. Эта страница документирует архитектуру i18n, конфигурацию и паттерны использования.

## Архитектура

| Файл | Назначение |
|------|---------|
| `i18n/routing.ts` | Конфигурация маршрутизации (поддерживаемые языки, стратегия префиксов) |
| `i18n/request.ts` | Определение языка и загрузка сообщений для каждого запроса |
| `i18n/navigation.ts` | Link, redirect, usePathname, useRouter с поддержкой языка |
| `lib/constants.ts` | Список языков, язык по умолчанию, определения RTL |
| `lib/seo/hreflang.ts` | Генерация SEO-тегов hreflang для поисковых систем |
| `components/language-switcher.tsx` | Переключатель языка в интерфейсе |
| `messages/*.json` | Файлы переводов (один на язык) |
| `app/[locale]/layout.tsx` | Корневой макет с NextIntlClientProvider |

## Поддерживаемые языки

Шаблон определяет 21 языковой стандарт в `lib/constants.ts`:

```typescript
export const DEFAULT_LOCALE = 'en';
export const LOCALES = [
  'en', 'fr', 'es', 'de', 'zh', 'ar', 'he',
  'ru', 'uk', 'pt', 'it', 'ja', 'ko', 'nl',
  'pl', 'tr', 'vi', 'th', 'hi', 'id', 'bg'
] as const;

export type Locale = (typeof LOCALES)[number];

/** Языковые стандарты, использующие направление текста справа налево */
export const RTL_LOCALES: readonly Locale[] = ['ar', 'he'] as const;
```

| `fr` | Французский | LTR |

| `de` | Немецкий | LTR |
| `zh` | Китайский упрощённый | LTR |
| `ar` | Арабский | **RTL** |

## Конфигурация маршрутизации

Модуль маршрутизации в `i18n/routing.ts` определяет соответствие языков URL-адресам:

```typescript
import { defineRouting } from "next-intl/routing";
import { DEFAULT_LOCALE, LOCALES } from "@/lib/constants";

export const routing = defineRouting({
  locales: LOCALES,
  defaultLocale: DEFAULT_LOCALE,
  localeDetection: true,
  localePrefix: "as-needed",
});
```

### Стратегия префиксов

Стратегия `"as-needed"` сохраняет чистые URL для языка по умолчанию:

| URL | Определённый язык | Пояснение |
|-----|-----------------|-------------|
| `/about` | `en` (по умолчанию) | Префикс не нужен для языка по умолчанию |
| `/fr/about` | `fr` | Префикс обязателен для других языков |
| `/ar/about` | `ar` | Префикс обязателен для других языков |

### Определение языка

При `localeDetection: true` система автоматически проверяет:

1. Cookie языка (устанавливается компонентом LanguageSwitcher)
2. HTTP-заголовок `Accept-Language` браузера
3. Возвращает к `DEFAULT_LOCALE` (`'en'`) если совпадение не найдено

## Загрузка сообщений и резервный вариант

Модуль `i18n/request.ts` выполняется при каждом запросе сервера:

```typescript
export default getRequestConfig(async ({ requestLocale }) => {
  let locale = await requestLocale;

  if (!locale || !routing.locales.includes(locale as any)) {
    locale = routing.defaultLocale;
  }

  const userMessages = (await import(`../messages/${locale}.json`)).default;
  const defaultMessages = (await import(`../messages/en.json`)).default;
  const messages = deepmerge(defaultMessages, userMessages);

  return { locale, messages };
});
```

Стратегия `deepmerge` гарантирует, что непереведённые ключи показывают английский текст.

## Файлы переводов

Сообщения переводов хранятся как JSON-файлы в директории `messages/`:

```
messages/
  en.json    # Английский (источник, все ключи присутствуют)
  fr.json    # Французский
  ru.json    # Русский
  ... (21 файл всего)
```

### Использование переводов в компонентах

**Клиентские компоненты** используют хук `useTranslations`:

```tsx
import { useTranslations } from "next-intl";

function LoginButton() {
  const t = useTranslations("auth");
  return <button>{t("SIGN_IN")}</button>;
}
```

**Серверные компоненты** используют асинхронный `getTranslations`:

```tsx
import { getTranslations } from "next-intl/server";

export default async function Page() {
  const t = await getTranslations("common");
  return <h1>{t("DISCOVER")}</h1>;
}
```

## Добавление нового языка

1. Добавить код языка в массив `LOCALES` в `lib/constants.ts`
2. Если язык RTL — добавить его в `RTL_LOCALES`
3. Создать `messages/<код>.json` с переведёнными ключами
4. Добавить SVG флага в `public/flags/<код>.svg`
5. Добавить запись языка в `languageMap` в `components/language-switcher.tsx`
6. Добавить сопоставление hreflang в `lib/seo/hreflang.ts`

## Лучшие практики

1. **Использовать `Link` из `@/i18n/navigation`** вместо `next/link`
2. **Добавлять новые ключи сначала в `en.json`** так как это языку-источнику
3. **Никогда не хардкодить английские строки** в компонентах
4. **Полагаться на резервный `deepmerge`** — частично переведённые файлы работают корректно
5. **Тестировать RTL-макеты** переключаясь на арабский или иврит
