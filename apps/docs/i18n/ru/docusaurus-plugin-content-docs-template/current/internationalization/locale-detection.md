# Определение Локали и Маршрутизация

Шаблон использует `next-intl` для определения языка с автоматическим сопоставлением языка браузера, маршрутизацией на основе URL, сохранением в cookies и системой резервных сообщений.

## Процесс определения

При поступлении запроса язык определяется следующим образом:

1. **Префикс URL** — Если URL содержит префикс языка (например `/fr/about`), используется этот язык напрямую
2. **Cookie** — Если префикс URL отсутствует, система проверяет cookie языка, установленную компонентом LanguageSwitcher
3. **Заголовок Accept-Language** — Если cookie нет, считывается заголовок языковых предпочтений браузера
4. **Резервный вариант** — Если совпадение не найдено, используется язык по умолчанию (`en`)

## Исходные файлы

| Файл | Роль в определении |
|------|-------------------|
| `i18n/routing.ts` | Определяет поддерживаемые языки, стратегию префиксов |
| `i18n/request.ts` | Валидирует определённый язык, загружает сообщения |
| `i18n/navigation.ts` | Предоставляет Link, router, redirect с поддержкой языка |
| `lib/constants.ts` | Источник истины для массива LOCALES и RTL_LOCALES |
| `components/language-switcher.tsx` | Устанавливает cookie языка через router.replace |
| `app/[locale]/layout.tsx` | Валидирует язык, отклоняет недопустимые через notFound() |

## Конфигурация маршрутизации

```typescript
import { defineRouting } from "next-intl/routing";
import { DEFAULT_LOCALE, LOCALES } from "@/lib/constants";

export const routing = defineRouting({
  locales: LOCALES,
  defaultLocale: DEFAULT_LOCALE,
  localeDetection: true,
  localePrefix: "as-needed",
});
```

### Стратегия префиксов

| Запрос | Определённый язык | Отображаемый URL |
|---------|-----------------|-----------|
| `/about` | `en` | `/about` (без префикса для языка по умолчанию) |
| `/fr/about` | `fr` | `/fr/about` (префикс для других языков) |
| `/en/about` | `en` | Перенаправлен на `/about` |

## Логика резервных сообщений

- Английские сообщения служат базовым слоем со всеми ключами
- Сообщения конкретного языка перезаписывают только те ключи, которые определены
- Отсутствующие ключи в файле языка сохраняют английское значение
- Вложенные объекты рекурсивно объединяются

## Сохранение в cookie

Когда пользователь выбирает язык через LanguageSwitcher, `next-intl` устанавливает cookie:

```typescript
const changeLanguage = useCallback(
  (locale: string) => {
    if (locale === currentLocale || isPending) return;

    startTransition(() => {
      router.replace(pathname, { locale });
    });
    setIsOpen(false);
  },
  [currentLocale, isPending, router, pathname]
);
```

## Определение Accept-Language

```
Accept-Language: fr-FR,fr;q=0.9,en-US;q=0.8,en;q=0.7
```

Система ищет совпадение с поддерживаемым массивом `LOCALES`. Побеждает первый совпавший язык.

## Решение проблем с локалью

| Проблема | Вероятная причина | Решение |
|---------|-------------|----------|
| Ключи переводов вместо текста | Отсутствует ключ в файле языка | Добавить ключ в `messages/en.json` (резерв) |
| Отображается неправильный язык | Cookie перекрывает URL | Очистить cookies или использовать режим инкогнито |
| 404 на URL-ах языка | Язык не в массиве LOCALES | Добавить код в `lib/constants.ts` |
| RTL-макет не применяется | Язык не в RTL_LOCALES | Добавить в `RTL_LOCALES` в `lib/constants.ts` |

## Лучшие практики

1. **Всегда использовать `Link` из `@/i18n/navigation`** вместо `next/link`
2. **Добавлять все новые ключи сначала в `en.json`** так как это резервный язык
3. **Тестировать определение** настраивая языковые предпочтения браузера
4. **Полагаться на резервный `deepmerge`** — частично переведённые файлы ожидаемы и обрабатываются
