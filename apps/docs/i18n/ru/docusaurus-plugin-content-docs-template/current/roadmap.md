# Дорожная Карта

На этой странице описывается текущее направление развития Directory Web Template и то, как сообщество может участвовать в формировании его будущего.

## Видение Продукта

Directory Web Template стремится стать наиболее комплексным решением с открытым исходным кодом для создания профессиональных сайтов-каталогов. Долгосрочная концепция охватывает:

- **Готовые к производству сайты-каталоги**, которые красивы, производительны и полностью настраиваемы
- **Простое управление контентом** через CMS на основе Git с опциональной генерацией контента на базе ИИ через [платформу Ever Works](https://docs.ever.works)
- **Расширяемые платежи и аутентификацию** с поддержкой нескольких провайдеров из коробки
- **Интернационализацию первого класса** с полной поддержкой RTL и растущим языковым покрытием

## Области Активной Разработки

### Производительность и Core Web Vitals

- Оптимизация Largest Contentful Paint (LCP) для страниц списков и деталей элементов
- Уменьшение размера JavaScript-бандла путём более эффективного разделения кода и tree shaking
- Улучшение конвейера оптимизации изображений для скриншотов и логотипов элементов каталога
- Реализация частичного пре-рендеринга для более быстрой начальной загрузки страниц

### Улучшения Функциональности

- Добавление дополнительных возможностей фильтрации и поиска (фасетный поиск, расширенные фильтры)
- Реализация функций пользовательского контента (отзывы, рейтинги, комментарии)
- Добавление интеграций с новыми платёжными провайдерами и функций управления подпиской
- Расширение системы тем с новыми встроенными темами и упрощённой настройкой

### Опыт Разработчиков

- Улучшение настройки локальной среды разработки с более понятной документацией и сообщениями об ошибках
- Добавление более полного покрытия E2E-тестами с помощью Playwright
- Создание стартовых шаблонов для распространённых типов каталогов (SaaS, локальный бизнес, ресурсы)
- Улучшение безопасности типов TypeScript по всей кодовой базе

### Интернационализация

- Добавление новых встроенных языковых переводов
- Улучшение поддержки RTL-разметки для арабского и иврита
- Поддержка конфигурации языка для каждого каталога отдельно
- Добавление автоматизированных рабочих процессов перевода

### Документация

- Расширение справочной документации API с большим количеством примеров
- Добавление видеоуроков по распространённым задачам
- Создание архивов решений по архитектуре (ADR) для основных проектных решений
- Создание интерактивных руководств и тестовых сред

## Как Предлагать Функции

### GitHub Issues

Основной способ предлагать функции — через GitHub Issues по адресу [github.com/ever-works/directory-web-template/issues](https://github.com/ever-works/directory-web-template/issues).

При создании запроса функции:

1. **Проверьте существующие issues** сначала, чтобы избежать дублирования.
2. **Опишите проблему**, которую вы пытаетесь решить, а не только желаемое решение.
3. **Предоставьте контекст** о вашем сценарии использования, типе каталога и масштабе.
4. **Включите примеры** (макеты, схемы API, примеры конфигурации).

### GitHub Discussions

Для более широких идей, требующих участия сообщества: [github.com/ever-works/directory-web-template/discussions](https://github.com/ever-works/directory-web-template/discussions)

### Discord

Присоединяйтесь к [Discord Ever Works](https://discord.gg/ever) для обсуждений функций и направления проекта в режиме реального времени.

## Как Определяются Приоритеты

| Фактор                          | Вес      | Описание                                                                 |
| ------------------------------- | -------- | ------------------------------------------------------------------------ |
| **Спрос пользователей**         | Высокий  | Количество запросов, голосов «за» и интереса сообщества                  |
| **Стратегическое соответствие** | Высокий  | Насколько хорошо функция соответствует видению продукта                  |
| **Усилия по реализации**        | Средний  | Сложность, временные затраты и нагрузка на поддержку                     |
| **Риск критических изменений**  | Средний  | Потенциал нарушить работу существующих пользователей                     |
| **Доступность участников**      | Средний  | Могут ли сопровождающие или члены сообщества взяться за это              |

### Уровни Приоритетов

- **P0 (Критический):** Уязвимости безопасности, ошибки потери данных или блокирующие проблемы. Устраняются немедленно.
- **P1 (Высокий):** Функции или исправления, над которыми активно работают для следующего релиза.
- **P2 (Средний):** Утверждённые функции, запланированные, но ещё не включённые в расписание.
- **P3 (Низкий):** Желательные улучшения. Хорошие кандидаты для вклада сообщества.

## Участие в Формировании Дорожной Карты

1. **Отправляйте хорошо написанные запросы функций** с чёткими описаниями проблем и сценариями использования.
2. **Вносите код.** Pull request'ы — самый быстрый путь от идеи к реализации. Смотрите [Руководство по Участию](/contributing).
3. **Участвуйте в обсуждениях.** Давайте обратную связь по предложениям и делитесь опытом.
4. **Сообщайте об ошибках.** Надёжные отчёты об ошибках помогают расставлять приоритеты исправлений и улучшать стабильность.

## Частота Релизов

Релизы выходят, когда готов значимый набор функций и исправлений:

- **Patch-релизы** (исправления ошибок) публикуются по мере необходимости, часто еженедельно во время активной разработки.
- **Minor-релизы** (новые функции) публикуются примерно ежемесячно.
- **Major-релизы** (критические изменения) редки и сопровождаются руководствами по миграции.

Подробности смотрите на странице [Журнал Изменений](/changelog).
