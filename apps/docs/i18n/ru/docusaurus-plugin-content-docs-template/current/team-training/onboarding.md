# Руководство по Адаптации

Добро пожаловать в Ever Works! Это руководство поможет вам настроить рабочую среду и внести первый вклад.

## 🎯 Цели

По завершении этого модуля:

- ✅ У вас будет полностью настроенная среда разработки
- ✅ Вы поймёте структуру проекта
- ✅ Вы сможете запустить приложение локально
- ✅ Вы внесёте первое изменение в код
- ✅ Вы поймёте рабочий процесс разработки

**Ориентировочное время**: 1–2 дня

---

## Шаг 1: Настройка среды

### 1.1 Установить необходимые инструменты

Следуйте подробному [Руководству по установке](/getting-started/installation) для установки:

- Node.js 20.19.0+
- pnpm ([установка](https://pnpm.io/installation))
- PostgreSQL 14+
- Git
- VS Code (рекомендуется)

### 1.2 Клонировать репозиторий

```bash
git clone https://github.com/ever-co/ever-works.git
cd ever-works
pnpm install
```

### 1.3 Настроить переменные среды

**Быстрый контрольный список**:

- [ ] Подключение к базе данных настроено
- [ ] Секреты аутентификации установлены
- [ ] Ключи платёжного провайдера добавлены (необязательно для разработки)

---

## Шаг 2: Настройка базы данных

### 2.1 Запустить PostgreSQL

```bash
docker run --name everworks-postgres \
  -e POSTGRES_PASSWORD=password \
  -e POSTGRES_DB=everworks \
  -p 5432:5432 \
  -d postgres:14
```

### 2.2 Выполнить миграции

```bash
cd apps/web
pnpm exec drizzle-kit push
pnpm run db:seed
```

---

## Шаг 3: Запустить сервер разработки

```bash
pnpm run dev
```

Проверьте в браузере:

- [ ] Главная страница загружается по адресу `http://localhost:3000`
- [ ] Можно создать аккаунт
- [ ] Можно войти/выйти
- [ ] Документация API доступна по адресу `http://localhost:3000/api/reference`

---

## Шаг 4: Понять структуру проекта

```
directory-web-template/
├── apps/
│   ├── web/
│   │   ├── app/
│   │   ├── components/
│   │   ├── lib/
│   │   ├── public/
│   │   └── messages/
│   └── web-e2e/
├── packages/
└── turbo.json
```

---

## Шаг 5: Рабочий процесс разработки

### 5.1 Создать ветку функции

```bash
git checkout main
git pull origin main
git checkout -b feature/название-функции
```

### 5.2 Зафиксировать и отправить

```bash
git add .
git commit -m "feat: добавить систему уведомлений пользователя"
git push origin feature/название-функции
```
