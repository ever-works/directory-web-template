## 验证命令

运行以下命令验证生产就绪状态：

### 安全审计

```bash
# 检查安全漏洞
npm audit --production

# 修复漏洞
npm audit fix

# 检查过时的依赖项
npm outdated
```

### 构建验证

```bash
# 生产构建
npm run build

# 检查构建输出
ls -lh .next/

# 包大小分析
npm run analyze
```

### 数据库验证

```bash
# 检查迁移状态
npx drizzle-kit check

# 如需生成迁移
npx drizzle-kit generate

# 应用迁移
npx drizzle-kit push
```

### API 文档

```bash
# 生成 OpenAPI 规范
yarn generate-docs

# 验证文档
yarn docs:validate

# 检查文档是否最新
git diff --exit-code public/openapi.json
```

### 环境变量

```bash
# 检查所有必需的变量是否已设置
node scripts/check-env.js

# 测试环境配置
npm run test:env
```

## 部署工作流程

### 部署前

1. **代码审查**
   - 所有 PR 已审查并批准
   - 无合并冲突
   - CI/CD 流水线已通过

2. **测试**
   - 所有测试已通过
   - 手动 QA 测试已完成
   - Staging 环境已测试

3. **文档**
   - Changelog 已更新
   - API 文档已重新生成
   - 部署说明已准备

### 部署步骤

1. **备份**

   ```bash
   # 数据库备份
   pg_dump $DATABASE_URL > backup-$(date +%Y%m%d).sql
   ```

2. **部署**

   ```bash
   # 部署到生产环境
   git push production main

   # 或使用 Vercel
   vercel --prod
   ```

3. **验证**

   ```bash
   # 检查健康状态端点
   curl https://your-domain.com/api/health

   # 检查错误日志
   tail -f logs/error.log
   ```

4. **监控**
   - 在 Sentry 监控错误率
   - 在 PostHog 监控性能
   - 检查正常运行时间监控

### 部署后

1. **冒烟测试**
   - 首页正常加载
   - 用户可以登录
   - 支付流程正常工作
   - 管理员面板可访问

2. **监控**
   - 错误率正常
   - 响应时间可接受
   - 无内存泄漏
   - 数据库性能稳定

3. **沟通**
   - 通知团队部署情况
   - 更新状态页面
   - 公告新功能（如适用）

## 回滚计划

如果部署后发现问题：

### 快速回滚

```bash
# 恢复到上一次部署
git revert HEAD
git push production main

# 或使用 Vercel
vercel rollback
```

### 数据库回滚

```bash
# 从备份恢复
psql $DATABASE_URL < backup-YYYYMMDD.sql

# 或使用时间点恢复
# （如果托管提供商支持）
```

### 沟通

1. 立即通知团队
2. 更新状态页面
3. 与受影响的用户沟通
4. 记录事件以供事后分析
