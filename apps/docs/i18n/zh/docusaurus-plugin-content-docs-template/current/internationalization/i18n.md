# 国际化 (i18n)

Ever Works Template 使用 `next-intl` 实现国际化，支持 21 种语言环境，具备基于 URL 的路由、浏览器自动检测、备用消息、从右到左布局和 SEO hreflang 标签等功能。本页面记录了 i18n 架构、配置和使用模式。

## 架构

| 文件 | 用途 |
|------|---------|
| `i18n/routing.ts` | 路由配置（支持的语言环境、前缀策略） |
| `i18n/request.ts` | 每次请求的语言检测与消息加载 |
| `i18n/navigation.ts` | 支持语言环境的 Link、redirect、usePathname、useRouter |
| `lib/constants.ts` | 语言列表、默认语言、RTL 定义 |
| `lib/seo/hreflang.ts` | 为搜索引擎生成 SEO hreflang 标签 |
| `components/language-switcher.tsx` | 界面中的语言切换组件 |
| `messages/*.json` | 翻译文件（每种语言一个） |
| `app/[locale]/layout.tsx` | 带 NextIntlClientProvider 的根布局 |

## 支持的语言

模板在 `lib/constants.ts` 中定义了 21 种语言环境：

```typescript
export const DEFAULT_LOCALE = 'en';
export const LOCALES = [
  'en', 'fr', 'es', 'de', 'zh', 'ar', 'he',
  'ru', 'uk', 'pt', 'it', 'ja', 'ko', 'nl',
  'pl', 'tr', 'vi', 'th', 'hi', 'id', 'bg'
] as const;

export type Locale = (typeof LOCALES)[number];

/** 使用从右到左文字方向的语言环境 */
export const RTL_LOCALES: readonly Locale[] = ['ar', 'he'] as const;
```

| 代码 | 语言 | 方向 |
|------|----------|-----------|
| `en` | 英语 | LTR |
| `fr` | 法语 | LTR |
| `es` | 西班牙语 | LTR |
| `de` | 德语 | LTR |
| `zh` | 简体中文 | LTR |
| `ar` | 阿拉伯语 | **RTL** |
| `he` | 希伯来语 | **RTL** |
| `it` | 意大利语 | LTR |
| `pt` | 葡萄牙语 | LTR |
| `nl` | 荷兰语 | LTR |
| `pl` | 波兰语 | LTR |
| `ru` | 俄语 | LTR |

## 路由配置

`i18n/routing.ts` 中的路由模块定义语言环境到 URL 的映射：

```typescript
import { defineRouting } from "next-intl/routing";
import { DEFAULT_LOCALE, LOCALES } from "@/lib/constants";

export const routing = defineRouting({
  locales: LOCALES,
  defaultLocale: DEFAULT_LOCALE,
  localeDetection: true,
  localePrefix: "as-needed",
});
```

### 前缀策略

`"as-needed"` 策略为默认语言环境保持简洁的 URL：

| URL | 检测到的语言 | 说明 |
|-----|-----------------|-------------|
| `/about` | `en`（默认） | 默认语言无需前缀 |
| `/fr/about` | `fr` | 其他语言需要前缀 |
| `/ar/about` | `ar` | 其他语言需要前缀 |

### 语言检测

当 `localeDetection: true` 时，系统自动检查：

1. 语言 Cookie（由 LanguageSwitcher 组件设置）
2. 浏览器的 HTTP `Accept-Language` 请求头
3. 未找到匹配时回退到 `DEFAULT_LOCALE`（`'en'`）

## 消息加载与回退

`i18n/request.ts` 模块在每次服务器请求时执行：

```typescript
export default getRequestConfig(async ({ requestLocale }) => {
  let locale = await requestLocale;

  if (!locale || !routing.locales.includes(locale as any)) {
    locale = routing.defaultLocale;
  }

  const userMessages = (await import(`../messages/${locale}.json`)).default;
  const defaultMessages = (await import(`../messages/en.json`)).default;
  const messages = deepmerge(defaultMessages, userMessages);

  return { locale, messages };
});
```

`deepmerge` 策略确保未翻译的键显示英文文本。

## 翻译文件

翻译消息作为 JSON 文件存储在 `messages/` 目录下：

```
messages/
  en.json    # 英语（源文件，包含所有键）
  fr.json    # 法语
  zh.json    # 中文
  ... （共 21 个文件）
```

### 在组件中使用翻译

**客户端组件**使用 `useTranslations` Hook：

```tsx
import { useTranslations } from "next-intl";

function LoginButton() {
  const t = useTranslations("auth");
  return <button>{t("SIGN_IN")}</button>;
}
```

**服务器组件**使用异步 `getTranslations`：

```tsx
import { getTranslations } from "next-intl/server";

export default async function Page() {
  const t = await getTranslations("common");
  return <h1>{t("DISCOVER")}</h1>;
}
```

## 添加新语言

1. 将语言代码添加到 `lib/constants.ts` 中的 `LOCALES` 数组
2. 如果是 RTL 语言，将其添加到 `RTL_LOCALES`
3. 创建包含翻译键的 `messages/<代码>.json`
4. 在 `public/flags/<代码>.svg` 中添加国旗 SVG
5. 在 `components/language-switcher.tsx` 的 `languageMap` 中添加语言条目
6. 在 `lib/seo/hreflang.ts` 中添加 hreflang 映射

## 最佳实践

1. **使用 `@/i18n/navigation` 中的 `Link`** 而非 `next/link`
2. **先将新键添加到 `en.json`** 因为这是源语言
3. **不要在组件中硬编码英语字符串**
4. **依赖 `deepmerge` 回退机制** — 部分翻译的文件可以正常工作
5. **测试 RTL 布局** — 切换到阿拉伯语或希伯来语进行测试
